# Albedo 캘리브레이션 결과 R2 — radial R² 문제 분석

## 요약
radial 경계면에서 J_net = C × φ 선형 모델의 R²가 매우 낮은 원인을 조사.
직각/대각 분리, Z평면별 분리 모두 시도했으나 R²는 개선되지 않음.
**radial에서 선형 모델 자체가 구조적으로 부적합**한 것으로 판단.

## 배경
R1에서 방향별 행렬 C 캘리브레이션 수행 시:
- bottom/top: R² = 0.91~0.99 (우수)
- **radial: R² = 0.001** (매우 낮음)

## 수행한 테스트와 결과

### 테스트 1: radial을 직각/대각으로 분리 (R2)

MAS_XSL의 반사체 배치도에서:
- 직각(ortho): R1, R2 인접면 (24개/Z평면) — 높은 SS 비율
- 대각(diag): R3~R6 인접면 (48개/Z평면) — 높은 물 비율

| 방향 | N | R² g1 | R² g2 |
|------|--:|:---:|:---:|
| ortho (R1,R2) | 9,600 | **0.001** | 0.116 |
| diag (R3~R6) | 19,200 | **0.002** | 0.102 |
| bottom | 4,560 | 0.999 | 0.990 |
| top | 4,560 | 0.995 | 0.914 |

**결과**: 직각/대각 분리 후에도 R² 개선 없음.
반사체 조성 차이는 R² 저하의 원인이 아님.

### 테스트 2: Z 평면별 독립 C 역산

각 Z 평면(K=2~21)에서 72개 radial 경계면으로 독립 회귀:

| Z (K) | N | R² g1 | R² g2 |
|------|--:|:---:|:---:|
| 0 (K=2) | 72 | 0.001 | 0.031 |
| 5 (K=7) | 72 | 0.001 | 0.028 |
| 9 (K=11) | 72 | 0.002 | 0.028 |
| 14 (K=16) | 72 | 0.002 | 0.024 |
| 19 (K=21) | 72 | 0.002 | 0.024 |

**결과**: Z 평면별 분리 후에도 R² 개선 없음.
축방향 위치 혼합은 R² 저하의 원인이 아님.

### 테스트 요약

| 테스트 | 분리 기준 | R² 변화 | 원인 여부 |
|--------|----------|:---:|:---:|
| R1 (baseline) | 없음 | 0.001 | — |
| R2 직각/대각 분리 | 반사체 조성 (R1,R2 vs R3~R6) | 0.001~0.002 | **아님** |
| Z 평면별 분리 | 축방향 위치 | 0.001~0.002 | **아님** |

## 분석

### bottom/top은 R²가 높은 이유
- 동일 Z 평면의 모든 연료 노드가 **동일 방향(Z축)**으로 반사체 인접
- 반사체 조건 균일 (물 반사체, 온도/밀도 비교적 균일)
- φ₁, φ₂ 값 범위가 넓고 다양 → 회귀에 충분한 정보

### radial에서 R²가 낮은 가능한 원인

1. **동일 Z 평면 내 radial 경계면 위치별 조건 차이**:
   각 Z 평면에서 72개 경계면은 노심 둘레를 따라 분포.
   각 위치의 flux 수준, 인접 연료 조성, 반사체 두께가 다름.
   **하나의 C 행렬로 72개 다양한 조건을 대표 불가**.

2. **radial 방향 flux 변이가 적음**:
   같은 Z 평면 내에서 경계 노드의 φ₁, φ₂ 값이 비교적 유사할 수 있음.
   → least squares에서 설명변수(φ)의 변이가 작아 R² 저하.

3. **2변수(φ₁, φ₂) 선형 모델의 한계**:
   radial 누설은 φ 외에도 인접 노드의 flux 구배, 반사체 위치별 물성 등
   추가 변수에 의존할 수 있음.

## 잠정 결론 (전문가 검토 요청)

- **radial 경계면에서 J_net = C × φ 선형 모델은 구조적으로 부적합**
- 축방향(bottom/top)과 달리, radial은 경계면 위치별 조건이 매우 다양
- **권장 처리**: radial은 trainable scalar α로 처리하고, 훈련 중 데이터가 최적값으로 조정
- 축방향은 행렬 C(2×2) 사용 (R²>0.91 검증 완료)

### 전문가 검토 요청 사항
1. radial에서 J_net ≈ C×φ가 성립하지 않는 물리적 이유는?
2. 위치별 독립 α(또는 C)를 부여하는 것이 적절한가?
3. 반사체 두께 / SS-water 비율의 위치별 차이가 원인인가?

### 테스트 3: 구배 항 추가 (φ + ∇φ 4변수 모델)

전문가 제안: NEM flux 전개 계수 a₁을 인접 노드 구배로 복원.
```
J_net,g ≈ C₀×φ̄_g + C₁×(φ̄_g,boundary - φ̄_g,inner) / h
```
- φ̄_g,boundary: 반사체 인접 연료 노드의 flux
- φ̄_g,inner: 반사체 **반대 방향** 이웃 연료 노드의 flux
- h = 10.804 cm (노드 피치)

회귀: `Phi = [φ₁, φ₂, ∇φ₁, ∇φ₂]` (4변수) → least squares

| Z (K) | 모델 A [φ₁,φ₂] R² g1 | 모델 B [φ₁,φ₂,∇φ₁,∇φ₂] R² g1 |
|------|:---:|:---:|
| 0 (K=2) | 0.001 | 0.006 |
| 9 (K=11) | 0.002 | 0.005 |
| 19 (K=21) | 0.002 | 0.007 |

**결과**: 구배 항 추가 후에도 R² ≈ 0.005 → **여전히 ~0**.

**개선 안 된 원인 (전문가 피드백으로 확인)**:
- NEM 면 전류 = 노드 내부 전개 계수의 **면(face) 미분값**: `J = -(D/h)(a₁ + a₃/4 + ⋯)`
- 노드 중심 간 구배 `(φ̄_boundary - φ̄_inner)/h`는 면이 아닌 **노드 중심 부근의 구배** → 위치 불일치
- NEM의 a₁은 양쪽 **면 전류(JNET0)**로 결정됨 → 노드 평균 flux 차이로는 복원 불가

### 테스트 4: 반대면 JNET0 추가 (전문가 제안 검증)

모델 C: `Phi = [φ₁, φ₂, JNET0_opposite_g1, JNET0_opposite_g2]`
- 면 N이 반사체 → S면 JNET0 (내부면 전류)을 독립변수에 추가
- a₁ 정보를 직접 인코딩

| Z (K) | 모델 A [φ] R² | 모델 B [φ,∇φ] R² | **모델 C [φ,JNET_opp] R²** |
|------|:---:|:---:|:---:|
| 0 (K=2) | g1=0.001, g2=0.031 | g1=0.006, g2=0.034 | **g1=0.977, g2=0.988** |
| 5 (K=7) | g1=0.001, g2=0.028 | g1=0.005, g2=0.030 | **g1=0.985, g2=0.992** |
| 9 (K=11) | g1=0.002, g2=0.028 | g1=0.005, g2=0.030 | **g1=0.985, g2=0.993** |
| 14 (K=16) | g1=0.002, g2=0.024 | g1=0.005, g2=0.025 | **g1=0.988, g2=0.994** |
| 19 (K=21) | g1=0.002, g2=0.024 | g1=0.007, g2=0.026 | **g1=0.980, g2=0.989** |

**결과**: R² = 0.002 → **0.985** (500배 개선). 전문가 예측 정확.
반대면 JNET0이 NEM a₁을 완전히 복원.

**PINN 적용 한계**: 추론 시 JNET0_interior를 알 수 없음 (NN이 예측해야 하는 값) → 순환 의존.

## 미해결 / 후속 과제

### 노드 vs 집합체 레벨 불일치 (중요)
현재 캘리브레이션은 **노드 단위(h=10.804 cm, MAS_NXS)**로 수행됨.
실제 L_diffusion은 **집합체 단위(h=21.608 cm, xs_fuel + MAS_OUT)**에서 작동.

Marshak FD: `J = αD/(αh/2 + D) × φ̄` — **h가 다르면 α도 달라짐**.
→ 집합체 단위에서 재캘리브레이션 필요 (JNET0 집계 + MAS_OUT flux + h=21.6cm).

### 최종 결론 및 L_diffusion 적용 방침

L_diffusion에서 반사체 인접면 누설항 = albedo BC (모델 예측 φ̂ 사용):

| 방향 | BC 형태 | 초기값 출처 | trainable | R² (캘리브레이션) |
|------|---------|-----------|:---------:|:---:|
| **bottom** | 행렬 C (2×2) | R1 캘리브레이션 | ✓ | 0.99 |
| **top** | 행렬 C (2×2) | R1 캘리브레이션 | ✓ | 0.91~0.99 |
| **radial** | scalar α (g1, g2) | R0 median | ✓ | 0.002 (부적합) |

Radial R²가 낮은 이유 (테스트 4로 확인):
- NEM a₁ 계수가 면 전류(JNET0)에 의존 → φ̄만으로 J_net 예측 불가
- **반대면 JNET0 추가 시 R²=0.99로 회복** → 원인 확인 완료
- PINN에서 JNET0 사용 불가 (순환 의존) → **trainable scalar α가 유일한 현실적 방안**

PINN에서의 동작:
```
내부면: leak = D̃(φ̂_nb - φ̂_center)/h × A/V    (CMFD)
axial 경계면: leak = C_matrix × φ̂ × A/V        (행렬 BC, trainable)
radial 경계면: leak = α×D/(αh/2+D) × φ̂ × A/V  (scalar BC, trainable)
```
- φ̂ = 모델 예측값 (GT 아님), JNET0 미사용
- α, C는 캘리브레이션 초기값에서 시작, 훈련 중 L_diffusion + L_data가 자동 조정

### 테스트 5: Per-face Marshak α 역산 (ortho/diag별 분포)

**방법** (이미 수행 완료):
1. 모델 C로 각 면의 J_refl을 정확히 예측 (R²>0.97)
2. 각 면에서 **그 노드의 D(확산계수)를 개별 사용**하여 α 역산:
   `α = J_refl × D / (D × φ - J_refl × h/2)`
3. 회전대칭을 활용하여 ortho/diag별 분포 확인

**J = C×φ 모델(테스트 1,2)에서 R²가 낮았던 이유**:
`J = C×φ`는 모든 면에 동일한 C 사용. 하지만 실제 Marshak FD:
`J = α×D/(α×h/2+D)×φ` 에서 **D가 노드마다 다르면 J/φ도 달라짐**.
Per-face Marshak은 각 면의 D를 개별 사용하므로 이 문제 해결.

**결과**:

| 그룹 | 군 | 양수 α (median ± std) | β = (1-α)/(1+α) | 음수 α 비율 |
|------|:--:|:---:|:---:|:---:|
| ortho | g1 | 0.100 ± 0.015 | 0.818 | 50% |
| ortho | g2 | 0.400 ± 0.148 | 0.429 | 50% |
| diag | g1 | 0.081 ± 0.021 | 0.850 | 50% |
| diag | g2 | 0.549 ± 0.295 | 0.291 | 50% |

**g1 α의 std가 매우 작음** (15~26%) → ortho/diag 내에서 albedo 일정 ✓

**용어**:
- α: Marshak FD 공식의 추출 파라미터 (`J = α×D/(α×h/2+D)×φ̄`)
- β: 반사율(albedo), 나간 중성자 중 돌아오는 비율 (`β = (1-α)/(1+α)`)

## 최종 초기 Albedo 값 (모델 C + Per-face Marshak 기반)

```
radial ortho: α_g1 = 0.100 (β=0.818), α_g2 = 0.400 (β=0.429)
radial diag:  α_g1 = 0.081 (β=0.850), α_g2 = 0.549 (β=0.291)
bottom:       C행렬 [[+0.156, -0.142], [-0.025, +0.079]] (R²=0.99)
top:          C행렬 [[+0.144, +0.027], [-0.035, +0.076]] (R²=0.91~0.99)
```
