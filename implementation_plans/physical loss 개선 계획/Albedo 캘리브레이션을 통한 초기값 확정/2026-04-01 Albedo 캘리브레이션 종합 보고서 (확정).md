## 1. 왜 Albedo가 필요한가

### L_diffusion의 구조

L_diffusion은 **단일 시점**에서 모델이 예측한 flux 분포가 확산방정식의 공간 밸런스를
얼마나 만족하는지 검사하는 physical loss:

```
R_g = [6면 누설] + [제거 × V] - [생성 × V] = 0 (이상적)
```

각 면의 누설은 **인접 노드의 flux 차이**로 계산(CMFD):
```
J_face = D̃ × (φ̂_이웃 - φ̂_중심) / h
```

### 문제: 반사체 인접면

- 이웃이 **연료**이면 → 위 CMFD 공식으로 계산 가능
- 이웃이 **반사체**이면 → 반사체의 φ를 모름 (모델이 연료만 예측)
- → **Albedo BC**로 반사체면 누설을 대체해야 함

### 영향 범위

집합체 단위 (20, 5, 5) quarter-core 기준:
- 전체 연료 어셈블리: 57개
- 반사체 인접 어셈블리: **20개 (35%)**
- → 35%의 연료 노드에서 L_diffusion을 적용하려면 Albedo 필수

## 2. Albedo의 물리적 의미

### 기본 개념

반사체 경계에서 나간 중성자의 일부가 반사되어 돌아옴:

```
연료 노드          │ 반사체
                   │
  j⁺ ──────→      │  (outgoing: 나가는 중성자)
  j⁻ ←──────      │  (incoming: 돌아오는 중성자)
                   │
```

### β (Albedo, 반사율)

```
β = j⁻ / j⁺ = 돌아오는 비율
```
- β = 0: 진공 (나간 중성자 전부 흡수, 돌아오지 않음)
- β = 1: 완전 반사체 (나간 중성자 전부 돌아옴)
- 0 < β < 1: 실제 반사체 (일부 반사, 일부 흡수)

### α (Marshak 추출 파라미터)

β를 사용하기 편한 형태로 변환한 BC 파라미터:
```
α = (1 - β) / (1 + β)
```
- β 큼 (반사 강함) → α 작음
- β 작음 (반사 약함) → α 큼

### Marshak BC FD 공식 (D-Marshak) 상세

L_diffusion에서 반사체 인접면의 누설을 계산하는 공식.

**D (확산계수)란?**:
- 정의: `D = 1 / (3 × Σ_tr)` [cm]
- Σ_tr: 수송 단면적 [/cm], xs_fuel의 채널 [3](g1), [8](g2)에서 유도
- 물리적 의미: 중성자가 매질 내에서 확산하는 "용이도"
  - D 큼 → 중성자가 잘 퍼짐 (투과성 높음)
  - D 작음 → 중성자가 잘 퍼지지 않음 (흡수/산란 강함)
- **노드 단위**: 각 연료 노드(어셈블리)마다 연료 조성이 달라 D가 다름

**공식 유도**:

Step 1. 반사체 경계에서의 Marshak BC 기본형:
```
J_net = (α/2) × φ_surface
```

Step 2. 면 flux를 노드 중심 flux로 대체 (half-node Fick 근사):
```
J_net = D × (φ̄ - φ_surface) / (h/2)
→ φ_surface = φ̄ - J_net × h / (2D)
```

Step 3. Step 1과 2를 결합하여 φ_surface 소거:
```
J_net = (α/2) × [φ̄ - J_net × h / (2D)]
J_net × [1 + αh/(4D)] = (α/2) × φ̄
```

Step 4. **최종 Marshak FD 공식**:
```
J_boundary = αD / (αh/2 + D) × φ̂
```
- α: Marshak 파라미터 (캘리브레이션 대상, trainable)
- D: 해당 연료 노드의 확산계수 [cm] (xs_fuel에서 유도, 노드마다 다름)
- h: 노드 피치 [cm] (집합체 21.608 cm)
- φ̂: 모델이 예측한 경계 연료 노드의 flux

**α가 크면 누설이 크고, 작으면 누설이 적음.**

### 이론값

Marshak 이론 (순수 진공 경계): **α = 0.4692, β = 0.3613**
→ 실제 반사체는 이와 다름 (SS+H₂O 반사 효과)

## 3. 노심 형상과 좌표계

### MAS_NXS 좌표계 (I, J, K)

```
I: 열 (X방향, 동→서 증가)
J: 행 (Y방향, 북→남 증가)  ← 주의! J↑ = 남쪽
K: 축방향 (Z방향, 하→상 증가)
```

### JNET0 방향 매핑 (확정)

```
JNET0-E = +I 방향 (동쪽)    ← (I,J).E + (I+1,J).W = 0 으로 검증 완료
JNET0-W = -I 방향 (서쪽)
JNET0-S = +J 방향 (남쪽)    ← (I,J).S + (I,J+1).N = 0 으로 검증 완료
JNET0-N = -J 방향 (북쪽)
JNET0-T = +K 방향 (상부)
JNET0-B = -K 방향 (하부)
```

### JNET0 저장 규약 (확정)

```
JNET0 = J_net / 2  (half net current)
→ 밸런스 시 ×2 필요: leak = 2 × Σ(JNET0_face × A_face)
```
이유: partial current 정의에서 j⁺ = φ/4 + J_net/2 → J_net/2가 자연스러운 변수.
원자로 물리 코드(MASTER, PARCS 등)의 프로그래밍 관습.

### 반사체 인접면 분류

```
직각(ortho): 연료 노드가 R1 또는 R2와 직접 인접하는 면

대각(diag): 연료 노드가 R3~R6와 인접하는 면
  → 노심 모서리 방향
  → H₂O(물) 비율 높음
```

### R1: 행렬 C (2×2), J = C × φ (두 번째 시도)

**방법**: 2군 행렬 Albedo BC: `J_net = C × [φ₁, φ₂]`
least squares로 방향별 2×2 C 행렬 역산.

**결과**:
- bottom/top (축방향): **R² > 0.91** ✓ → C 행렬 사용 가능
- radial (방사방향): **R² ≈ 0.001** ✗ → C×φ 모델 부적합

**R²의 의미**: J_net = C×φ 선형 모델이 데이터를 얼마나 잘 설명하는지.
R²=1이면 완벽, R²=0이면 전혀 설명 못함.
→ bottom/top의 C 행렬은 신뢰 가능, radial의 C 행렬은 사용 불가.

**오류**: N/S 매핑 오류 포함 (R3에서 발견)

### R3: JNET0 부호 규약 확정 + N/S 매핑 오류 발견 (네 번째 시도)

**발견 1**: JNET0은 **외향(outward) 방향** (절대 좌표 아님)
- 밸런스: SUM × 2 = 잔차 0.000% ✓, DIFF = 잔차 31~73% ✗
- 대칭 노드 비교: 동일 부호·크기 → 외향 확정

**발견 2**: **J 인덱스가 남쪽(South)으로 증가!**
```
이전 (잘못됨): N = J 증가 방향, S = J 감소 방향
올바름 (확정): N = J 감소 방향, S = J 증가 방향
```
검증: 내부 연료-연료 면에서 `(I,J).S + (I,J+1).N = 0` 완벽 성립

**이 오류의 영향 — "50:50 양수/음수 분할" 현상 설명**:
R0~R2에서 radial 경계면의 per-face α(Marshak 파라미터)를 역산했을 때,

### Axial (축방향) — 행렬 C (2×2) + α, β 변환

| 방향 | C 행렬 | R² (g1, g2) |
|------|--------|:---:|
| **bottom** | `[[+0.156, -0.142], [-0.025, +0.079]]` | (0.999, 0.990) |
| **top** | `[[+0.144, +0.027], [-0.035, +0.076]]` | (0.995, 0.914) |

**C 행렬이란?**: `J_net = C × [φ₁, φ₂]` 형태의 경계 조건.
2군 확산에서 반사체 경계의 누설을 **2×2 행렬**로 표현:
```
[J_net_g1]   [C₁₁  C₁₂] [φ₁]
[J_net_g2] = [C₂₁  C₂₂] [φ₂]
```
- C₁₁: 고속 flux → 고속 누설 기여 (대각: 스칼라 α와 동일 역할)
- C₁₂: 열 flux → 고속 누설 기여 (비대각: 군간 결합)
- C₂₁: 고속 flux → 열 누설 기여 (**감속 반환 효과**, 핵심!)
- C₂₂: 열 flux → 열 누설 기여 (대각: 스칼라 α와 동일 역할)

**C 대각 성분 → α, β 변환** (참고):
- α ≈ C₁₁ × h (근사적 대응, 정확한 변환은 D 의존)
- bottom: C₁₁=0.156 → α_g1≈0.21, C₂₂=0.079 → α_g2≈0.11
- top: C₁₁=0.144 → α_g1≈0.32, C₂₂=0.076 → α_g2≈0.16

**왜 Axial은 행렬, Radial은 스칼라인가?**:
- Axial: 비대각 C₂₁이 C₂₂의 **32~46%** → 고속→열 감속 반환 효과 무시 불가 → 행렬 필수
- Radial: 비대각 C₂₁이 C₂₂의 **0.7~7%** → 무시 가능 → 스칼라 α 충분
- (axial 반사체는 물 비율 높아 감속 효과 큼, radial은 SS 비율 높아 감속 효과 작음)

### R²의 의미와 Albedo 신뢰도 — 상세 설명

#### α 캘리브레이션의 전체 흐름

```
[Step 1] 데이터 수집 (MAS_NXS GT값, 오프라인)
  9600개 ortho 경계면에서 각각:
  - φ₁, φ₂: 해당 연료 노드의 체적 평균 flux (FLX g1, g2)
  - J_opp₁, J_opp₂: 같은 노드의 반사체 반대편 면 JNET0 × 2
      예) E면이 반사체 → W면(연료쪽) JNET0 × 2
  - J_refl (정답): 같은 노드의 반사체면 JNET0 × 2
  ※ 모든 데이터는 MAS_NXS(GT값). 반사체 노드 데이터는 사용하지 않음.

[Step 2] 모델 C fitting (최소제곱법, 군별 독립)
  g1: J_refl_g1_예측 = CC₀×φ₁ + CC₁×J_opp_g1
      CC = lstsq([φ₁, J_opp_g1], J_refl_g1)  ← 2개 계수, 같은 군만
  g2: J_refl_g2_예측 = CC₀×φ₂ + CC₁×J_opp_g2
      CC = lstsq([φ₂, J_opp_g2], J_refl_g2)  ← 2개 계수, 같은 군만
  ※ 각 에너지군은 독립적으로 fitting. 군 간 혼합 없음.

  왜 J_opp를 포함하는가?
  - φ만으로는 R²≈0 (모델 A). NEM에서 면 전류는 노드 평균 flux만으로 결정 안 됨.
  - 반대편 면 전류(J_opp)는 같은 노드 내 flux 곡선(4차 다항식)에서 나오므로,
    반사체면 전류(J_refl)와 공통의 전개 계수(a₁)를 공유.
  - J_opp를 추가하면 R²=0.99 (이 공유 관계를 데이터가 포착).

[Step 3] R² 계산
  R² = 1 - Σ(J_정답 - J_예측)² / Σ(J_정답 - J_평균)²
  의미: 모델 C가 J_refl을 얼마나 잘 예측하는가 (1.0 = 완벽)

[Step 4] per-face α 역산
  각 면에서: α = J_예측 × D / (D×φ - J_예측×h/2)
  (Marshak FD 역산, 각 면의 D를 개별 사용)

[Step 5] α 분포 확인 → 대표값 확정
  ortho g1: 9600개 면에서 α median=0.109, std=0.002
  → std가 매우 작음 → 0.109를 대표값으로 사용 가능
```

#### J_opp를 독립변수로 사용하는 타당성

J_opp는 MAS_NXS GT값이지만, **캘리브레이션 목적**에서는 문제 없음:
- 캘리브레이션은 GT 데이터를 사용하여 α의 "진짜 값"을 측정하는 **오프라인 과정**
- 실제 L_diffusion에서는 모델 C를 사용하지 않음 → J_opp 불필요
- 모델 C는 "정밀 측정 도구"일 뿐, α 측정 후 폐기

참고: 모델 C 없이 J_refl GT값을 직접 사용해도 α 역산 가능:
`α = J_refl_GT × D / (D×φ - J_refl_GT×h/2)`
모델 C의 장점은 출력 노이즈의 **평활화(smoothing)** 효과.

#### R²와 α 신뢰도의 연결

**R²는 α의 정확도 지표가 아닙니다.** 논리 구조:

```
R² 높음 → J_예측이 정확 → per-face α 역산이 정확
→ 같은 유형(ortho) 내 α들의 std가 작음
→ 하나의 대표값(median)이 해당 유형 전체를 잘 대표
```

**α의 신뢰도는 최종적으로 std(표준편차)로 판단**:

| 방향/군 | α median | α std | std/median | 판정 |
|---------|:---:|:---:|:---:|:---:|
| ortho g1 | 0.109 | 0.002 | 1.8% | ✅ 매우 안정 |
| ortho g2 | 0.482 | 0.063 | 13% | ✅ 안정 |
| diag g1 | 0.082 | 0.008 | 9.8% | ✅ 안정 |
| diag g2 | 0.565 | 0.072 | 12.7% | ✅ 안정 |

**Radial**: 모델 C R²>0.99 → α std 작음 → 대표값 신뢰 가능
**Axial**: J=C×φ 모델 R²>0.91 → C 행렬 직접 사용 가능 (per-face 역산 불필요)

### Full-core vs Quarter-core 적용 전략

**Full-core (20, 9, 9)** 사용 시:
- 4중 대칭 활용: 반사체 인접면이 대칭적으로 분포
- 경계면 위치를 fuel map에서 자동 탐지 (NFS > 0인 노드의 이웃이 NFS = 0)
- 대칭면(x=0, y=0 라인): Mirror CMFD (REFLECT, ghost=mirror neighbor)

**Quarter-core (20, 5, 5)** 사용 시:
- 대칭면(x=0, y=0): Mirror CMFD (REFLECT, Albedo 아님)
- 반사체면(외곽): Albedo BC 적용
- 구현: boundary_mask를 미리 생성하여 경계면 위치와 방향(ortho/diag) 저장
