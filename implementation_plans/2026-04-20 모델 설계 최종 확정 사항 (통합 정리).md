## 1. 전체 파이프라인 개요

```
[데이터셋 (HDF5)]
   ├ state(t): 10ch (Xe, I, flux, 온도 등 정규화)
   ├ xs_fuel: 10ch (시나리오 고정 거시단면적)
   ├ rod_map: 1ch (제어봉 삽입 깊이 분포)
   ├ p_load(t): 스칼라 (출력 수준 0~1)
   └ sym_type: 메타데이터 (mirror or rotation)
        ↓
[halo_expand] (5,5) → (6,6)
        ↓
[Spatial Encoder] (구현 완료)
   ├ CellEmbedder Conv3D 1×1×1 (21→128)
   ├ ConditionalLAPE3D (sym_type 분기 add)
   └ FullAttention3D × 3 stages (STRING 상대 PE)
        ↓ z(t) ∈ (B, 720, 128)
[Spatiotemporal Processor] ★미착수★
   ├ N=2 Blocks: [Mamba-3 ×2 + Attention + 소형 MLP]
   ├ p_load 주입: 공유 임베더 → Mamba concat + Attention AdaLN-Zero
   └ 절대위치 재주입: encoder_lape skip connection (Raw α + WD)
        ↓ d(t) ∈ (B, 720, 128)
[Decoder] ★미착수★
   └ Conv3D 1×1×1 (128→10), cell-wise 독립
        ↓ delta_pred ∈ (B, 720, 10)
[잔차 결합] state(t) + delta_pred = X_next_pred
        ↓
[Loss 계산] ★미착수★
   ├ L_data (정규화 공간 MSE)
   └ L_physics (역정규화 후 물리 단위)
       ├ L_Bateman (Xe/I ODE 잔차) — Branch 단일 step + CRS 시퀀스 모두 가능
       ├ L_sigma_a_Xe (Taylor 단면적 정합성) — 시간 비의존
       ├ L_diffusion (PDE 확산 잔차) — 시간 비의존
       └ L_keff (Rayleigh 몫) — **후순위/검토중**
```

**물리적 비유 (전체)**:
- **인코더**: 노심 격자에 "지도(좌표) + 대칭 정체성"을 부여
- **시공간 프로세서**: Mamba가 시간 동역학 진행, Attention이 매 시점 공간 결합 정리
- **디코더**: 잠재 표현을 물리량으로 번역 (투명한 유리)
- **Physical Loss**: 물리 법칙으로부터의 이탈 페널티

#### p_load 주입 — 이중 경로 (방안 1)

**핵심 결정**:
- **Mamba 측**: 공유 임베더(MLP 1→8) → 720 cell 균일 broadcast → 입력에 concat
- **Attention 측**: 같은 공유 임베더 → AdaLN-Zero (γ, β 0 초기화) → LN 변조
- **Detach 없음**: 양방향 gradient 허용 (시너지)

##### AdaLN-Zero의 LN 레이어 정확한 위치 — Attention Pre-LN

표준 Transformer 블록은 **Pre-LN 패턴**으로 Attention 이전과 FFN 이전에 각각 별도의 LN이 있음
(인코더 [04_normalization_omitted_options.md §1.1](공간인코더 구현 계획/인코더 컴포넌트별 채용 이유/04_normalization_omitted_options.md)의 "LN → Attention → residual add / LN → FFN → residual add" 패턴과 동일).

**3가지 LN 구분**:

| LN 위치 | 역할 | p_load 주입? | 구현 |
|--------|------|----------|------|
| **Mamba 내부 LN** | Mamba block 자체 정규화 | ✗ (concat 방식으로 이미 주입) | Mamba layer가 자체 관리 (변형 없음) |
| **Attention Pre-LN** | Attention 입력 정규화 | **✓ (AdaLN-Zero)** | `LayerNormalization(center=False, scale=False)` + γ/β 동적 생성 |
| **FFN Pre-LN** | FFN 입력 정규화 | ✗ (일반 LN) | `LayerNormalization(axis=-1)` 표준 |

→ **AdaLN-Zero는 Attention Pre-LN만** 대체. Mamba 내부 LN과 FFN Pre-LN은 변형 없음.

##### STBlock 전체 구조 다이어그램

```
[입력 z_t]  (B, T, 720, 128)
   ↓
[Mamba-3 #1]      ← 내부 LN 자체 관리 (p_load concat 입력)
   ↓
[Mamba-3 #2]      ← 동일
   ↓
d_mamba
   ↓
[+ α · lape_cache]   ← LAPE skip connection (Raw α, ReZero)
   ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[LN_adaln (affine 제거)] · (1 + γ) + β   ← ★ AdaLN-Zero (Attention Pre-LN) ★
                              ↑↑↑↑↑
                    p_load_emb → adaln_proj (0 초기화)
   ↓
[Self-Attention]
   ↓
[+ residual]      (d_mamba + attn_out)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ↓
[LN_ffn (일반)]   ← FFN Pre-LN (p_load 무관, 표준 Pre-LN)
   ↓
[소형 MLP/FFN]
   ↓
[+ residual]
   ↓
[출력]
```

**TF2/Keras 구현**:
```python
class STBlock(tf.keras.layers.Layer):
    def __init__(self, dim=128, p_load_embed_dim=8, num_cells=720):
        super().__init__()
        self.num_cells = num_cells
        # 공유 p_load 임베더 (양방향 gradient)
        self.p_load_encoder = tf.keras.Sequential([
            tf.keras.layers.Dense(p_load_embed_dim, activation='gelu'),
            tf.keras.layers.Dense(p_load_embed_dim),
        ], name='p_load_encoder')

        # AdaLN-Zero projection (γ, β 0 초기화)
        self.adaln_proj = tf.keras.layers.Dense(
            2 * dim, kernel_initializer='zeros', name='adaln_proj'
        )

        # ★ Attention Pre-LN (AdaLN-Zero용, affine 제거 — γ/β는 adaln_proj로 동적 생성)
        self.ln_adaln = tf.keras.layers.LayerNormalization(
            axis=-1, center=False, scale=False, name='attn_pre_adaln_ln'
        )

        # FFN Pre-LN (일반 LN, p_load 무관, affine 유지)
        self.ln_ffn = tf.keras.layers.LayerNormalization(
            axis=-1, name='ffn_pre_ln'
        )

        # 절대위치 skip connection α (Raw scalar, 0 초기화)
        self.alpha = self.add_weight(
            name='lape_alpha', shape=(1,),
            initializer='zeros', trainable=True
        )

        self.mamba = MambaLayer(...)          # 내부 LN 자체 관리
        self.attention = Attention(...)
        self.ffn = FFN(...)                    # 소형 MLP

    def call(self, z_t, p_load, sym_type, encoder_lape):
        # [1] p_load 공유 임베딩
        p_load_emb = self.p_load_encoder(p_load)  # (B, T, 8)

        # [2] Mamba 측 — p_load concat (720 cell 균일)
        p_load_b = tf.tile(
            p_load_emb[:, :, None, :], [1, 1, self.num_cells, 1]
        )  # (B, T, 720, 8)
        mamba_in = tf.concat([z_t, p_load_b], axis=-1)
        d_mamba = self.mamba(mamba_in)            # Mamba 내부 LN 자체 적용

        # [3] 절대위치 skip connection (encoder_lape stop_gradient)
        lape_cache = tf.stop_gradient(encoder_lape(sym_type))
        d_mamba = d_mamba + self.alpha * lape_cache  # ReZero α

        # [4] Attention 측 — AdaLN-Zero (★ Attention Pre-LN 위치 ★)
        gamma_beta = self.adaln_proj(p_load_emb)  # (B, T, 2*dim), detach 없음
        gamma, beta = tf.split(gamma_beta, 2, axis=-1)
        gamma = gamma[:, :, None, :]
        beta = beta[:, :, None, :]
        x_norm = self.ln_adaln(d_mamba)            # Attention Pre-LN (affine 제거)
        x_modulated = x_norm * (1 + gamma) + beta  # AdaLN-Zero 변조
        attn_out = d_mamba + self.attention(x_modulated)  # residual

        # [5] FFN — 일반 Pre-LN + residual (p_load 무관, 표준 Transformer 패턴)
        ffn_out = attn_out + self.ffn(self.ln_ffn(attn_out))

        return ffn_out
```

**Optimizer**: `tf.keras.optimizers.AdamW(learning_rate=1e-3, weight_decay=1e-4)` — 모든 가중치 (alpha 포함)에 자연 정규화

#### 차선책 (1순위 미작동 시)

| 우선순위 | 증상 | 차선책 |
|--------|------|------|
| 1 | (c) p_load 구간 오차 plateau, 표현 용량 부족 | embed_dim 확장 (8→16→32) |
| 2 | AdaLN 효과 없음 (γ,β ≈ 0 유지) | AdaLN 제거 (concat 단독) |
| 3 | 학습 발산, gradient 불안정 | AdaLN 측 detach (방안 2) |
| 4 | 양 경로 plateau (표현 충돌) | 별도 임베더 (방안 3) |

#### 출력 활성화 및 정규화 맥락 (Normalization-aware Activation)

> **맥락**: 초기 L_physics 검토 시 "원자로 물리량은 음수 불가이므로 Softplus/ReLU 등 활성화로 하한선 강제" 제안이 있었으나, 본 프로젝트 정규화 방식 하에서는 **잘못된 설계** 임이 확인됨. 정규화 공간 맥락을 명시하여 재발 방지.

**결정**: **디코더 마지막 출력 = linear (활성화 없음) 확정.** Softplus / ReLU 등 비음수 강제 활성화 **금지**.

**근거 — 정규화 공간은 음수가 정상**:

- 본 프로젝트 정규화: **z-score** 또는 **log-z-score** (채널별 선택)
- z-score 정규화: `Z = (X − μ) / σ` → 평균보다 작은 X는 **정규화 공간에서 Z < 0 정상**
- log-z-score 정규화: `log(X)` 또는 `log(1+X)` 후 표준화 → 정규화 공간은 부호 자유
- 정규화 공간에 Softplus / ReLU 적용 시 → **음의 z-score 구간 표현 불가** → 평균 이하 값 전부 왜곡, 모델 표현력 손상

**물리적 양수성 보장 위치 = 역정규화 단계** (activation 아님):

| 채널 유형 | 역정규화 공식 | 양수 보장 메커니즘 |
|---------|------------|------------------|
| **log-z-score 채널** (농도 N_Xe/N_I, flux φ_g1/g2 등) | `X = exp(σ·z + μ) − 1` 또는 `X = exp(σ·z + μ)` | **exp가 자동으로 양수 보장** — 별도 처리 불필요 |
| **z-score 채널** (온도 T_fuel/T_mod 등) | `X = σ·z + μ` | 학습 정상 수렴 시 valid range 내. 극단 상황 대비 `max(X, ε)` 클리핑 선택 적용 (대부분 불필요) |
| **음수 허용 채널** (Axial Offset 등) | 위 중 하나 | 역변환 결과의 음수 **그대로 유지** (AO는 음수 정상) |

**본 프로젝트 state(t) 10ch 음수 허용 현황**:

- **Axial Offset (AO) 1개 채널만 음수 허용** — 나머지 9채널은 물리적으로 양수이나, 이는 역정규화 단계에서 자연스럽게 보장됨 (Softplus 불필요)

**L_physics 계산 시 유의사항**:

- **L_data**: 정규화 공간 MSE — 음수 영향 없음
- **L_Bateman / L_sigma_a_Xe / L_diffusion**: 역정규화 후 **물리 단위** 에서 계산
  - log-z-score 채널 (농도, flux): exp 역변환으로 자동 양수 → `log`, `sqrt`, 분모 연산 시 NaN 발생 없음
  - z-score 채널 (온도): 정상 학습 시 안전. 학습 초기 발산 방어 위해 역정규화 직후 `max(X, ε)` 클리핑 선택 적용 가능 (gradient는 통과)
- **정규화 공간에 Softplus 등을 적용하지 말 것** — 물리적 양수성은 역정규화 단계가 담당

**연계 검증 (TR-V-DC-1)**:

- 디코더 출력 물리 범위 확인은 **역정규화 후 물리 단위**에서 측정 (정규화 공간에서는 음수 정상이라 체크 의미 없음)
- 학습 초기 수렴 확인: log-z-score 채널의 역정규화 결과 분포가 학습 데이터 분포 내 유지되는지 히스토그램 모니터링
- AO 채널: 음수 값이 자연스럽게 예측되는지 별도 확인 (값이 0 근처로 수렴 시 AO 학습 실패 의심)

### 5.1 핵심 결정

> **용어 안내**:
> - **자기회귀 전개 (Autoregressive Rollout)**: 자기 예측을 다음 step 입력으로 사용
> - **K-step Pushforward (K-step 자기회귀 오차 학습)**: K step 자기회귀 후 GT와 비교
> - **상호 적응 (Co-adaptation)**: 두 모듈이 서로에 맞춰 함께 미세조정
> - **모듈별 차등 학습률 (Differential LR)**: 모듈마다 다른 학습률 적용
> - **커리큘럼 학습 (Curriculum Learning)**: 쉬운 샘플 → 어려운 샘플 순서 학습

| 항목 | 결정 |
|------|------|
| **학습 전략** | **3-Phase**: 공간 사전학습 → 시공간 학습 → 자기회귀 정렬 |
| **시퀀스 처리** | Phase 1, 2: Mamba parallel scan (575 step 한번에). Phase 3: 1-step recurrence × K (자기회귀 전개) |
| **TBPTT** | 미사용 (Mamba parallel scan으로 대체) |
| **h(t) 관리** | Drama 패턴 — detach 주입, snapshot/restore |
| **데이터 비율 통제** | **CRS batch : Branch batch sampling 비율**로 통제 (λ_branch 작게 할 필요 없음 — Phase 분리로 freeze 보호 자동) |
| **상호 적응 (Co-adaptation)** | Phase 2 후반에 Encoder/Attention/Decoder freeze 해제 + 모듈별 차등 학습률 (Mamba의 1/10) → Mamba 출력 분포에 공간 모듈이 맞춰 미세조정 |
| **롤아웃 안정화 (Pushforward)** | Phase 3 본격 적용: K-step 자기회귀 + 커리큘럼 (K=2→4→8→16→...→575) |

### 5.1.1 데이터 규모 (round-robin LP-profile 매칭)

| 단위 | 수량 | 비고 |
|------|------|------|
| 시퀀스 (LP, profile 1개씩 round-robin) | **100** | LP 수와 동일 |
| 각 시퀀스 길이 (CRS step) | **575 step** | 5분 간격 × 48시간 |
| 각 CRS step의 Branch | **30개** (b=0=critical, b=1~29=분기 29개) | rod_offset 변경 + 5분 진화 |
| **CRS 시점 합계** | **57,500** (= 100 × 575) | |
| **Branch 시점 합계** | **~1,667,500** (= 57,500 × 29) | CRS의 ~29배 |

→ Branch 데이터가 CRS보다 ~29배 많음. **Phase 분리로 학습 비율 자연스럽게 통제** (§5.1.2 참조).

### 5.1.2 LLM 3단계와의 대응 (Phase 본질)

| LLM | 우리 프로젝트 | 학습 대상 | 데이터 | 학습 신호 |
|-----|---------|--------|------|---------|
| Pretraining | **Phase 1** 공간 사전학습 | Encoder + Attention + Decoder | Branch (~167만) | 1-step (시점별 GT 입력) |
| SFT (Instruction) | **Phase 2** 시공간 학습 | **Mamba / h(t)** (Phase 2a freeze, Phase 2b 상호 적응) | CRS 시퀀스 + Branch sampling | 575-step parallel scan (GT 입력) |
| RLHF (Alignment) | **Phase 3** 자기회귀 물리 정렬 | 전체 unfreeze (낮은 lr) | CRS + Branch (커리큘럼) | **K-step 자기회귀 (자기 예측 입력)** + Physical Loss 강조 |

**우리 도메인의 강점**:
- LLM RLHF는 인간 피드백 (노이즈 있음) 사용
- 우리는 **원자로 노심 해석 코드의 Ground Truth** + **물리 제약 loss** 사용
- → 결정론적 GT가 존재하므로 RLHF보다 강한 정렬 가능

**핵심**: 모델 구조와 입출력은 **3 Phase 모두 동일**. 차이는:
1. **학습 활성 모듈** (Phase 1=공간, Phase 2=시간(+상호 적응), Phase 3=전체)
2. **사용 데이터** (Branch 위주 → CRS 위주 + Branch 보조 → 혼합 + 커리큘럼)
3. **Mamba 학습 신호** (1-step → 575-step parallel scan → **K-step 자기회귀 전개**)
4. **Loss 가중치** (균형 → 균형 → **Physical Loss 강조**)

### 5.2 Phase 1 (공간 사전학습) — Pretraining 단계

```
학습 대상: Encoder + Attention + Decoder
Mamba: 1-step만 (h(0)=0, 누적 없음, 학습 가능하지만 메인 아님)
데이터: Branch (~167만 시점, 단일 step, 5분 진화 반영)
       (참고: 100 LP × 575 step × 29 branch ≈ 1.67M)
목적: 공간 응답 연산자 학습 (rod_map 변화 → 노심 응답)

Loss: L_data + L_data_halo + L_diff_rel + L_Bateman + L_sigma_a_Xe
       (모든 시간 비의존 항목 + L_Bateman 단일 step 잔차)
       (L_keff는 후순위/검토중)

Learning rate: 표준 (예: 1e-3)
```

> **L_Bateman Phase 1 적용 가능 근거**: Branch는 frozen Xenon 아님. 5분 진화 반영 ([2026-04-20 검증 결과](디코더 구현 및 Physical Loss 적용/Physical loss 검토 및 개선 계획/L_Bateman/2026-04-20 Branch Xenon 진화 검증 — L_Bateman Phase 1 적용 가능성.md)). 단일 step Bateman 잔차 (`(N(t+5min) - N(t))/300s ≈ ODE 우변`) 학습 가능.

### 5.3 Phase 2 (시공간 학습) — SFT 단계, 2단계 분리

> **2026-04-20 정정**: 자문 검토 결과 "완전 freeze의 함정" (Mamba 출력 분포 변화 → freeze된 Attention이 적응 못 함) 회피를 위해 **Phase 2a + 2b 단계 분리** 도입.

#### Phase 2a (초반 ~30% epoch) — Mamba 단독 학습

```
학습 대상: Mamba / h(t)만
Freeze: Encoder + Attention + Decoder
데이터: CRS 시퀀스 + Branch sampling
목적: Mamba가 시간 동역학에 빠르게 수렴 (안정적 시작점 확보)

Mamba LR: 표준 (예: 1e-3)
Encoder/Attn/Decoder LR: 0 (freeze)
```

#### Phase 2b (중후반 ~70% epoch) — 상호 적응 (Co-adaptation)

```
학습 대상: 전체 (Mamba + Encoder + Attention + Decoder)
Freeze 해제: Encoder/Attention/Decoder의 freeze 해제
데이터: CRS 시퀀스 + Branch sampling
목적: Mamba 출력 분포에 공간 모듈이 맞춰 미세조정 (상호 적응)
```

##### 모듈별 차등 학습률 (Differential Learning Rate) 상세

**개념**: 네트워크의 각 부분 (모듈, 블록, 레이어 그룹) 마다 **서로 다른 학습률을 적용**.
- 이미 수렴한 모듈은 낮은 lr로 보호 (큰 변화 시 학습 결과 파괴)
- 새로 학습하는 모듈은 높은 lr로 빠르게 적응

**Phase 2b 권장 설정**:

| 모듈 | 학습률 | 이유 |
|------|------|------|
| **인코더** (Phase 1 학습 완료) | 매우 낮음 (예: 1e-5) | 이미 수렴, 흔들리면 학습 결과 파괴 |
| **Attention** (Phase 1 학습 완료) | 낮음 (예: 5e-5) | 인코더와 같은 이유. Mamba 출력 분포 적응 위해 약간 학습 허용 |
| **Mamba** (Phase 2a부터 새로 학습) | 표준 (예: 1e-3) | 시간 동역학 학습 진행 중, 빠른 적응 필요 |
| **디코더** (Phase 1 학습 완료) | 매우 낮음 또는 freeze | 출력 head, 크게 바뀔 필요 없음 |

**TF2/Keras 구현**:
```python
# 가중치 그룹별 학습률 분리 (Optimizer 여러 개)
opt_mamba = tf.keras.optimizers.AdamW(learning_rate=1e-3, weight_decay=1e-4)
opt_attn  = tf.keras.optimizers.AdamW(learning_rate=5e-5, weight_decay=1e-4)
opt_enc   = tf.keras.optimizers.AdamW(learning_rate=1e-5, weight_decay=1e-4)
opt_dec   = tf.keras.optimizers.AdamW(learning_rate=1e-5, weight_decay=1e-4)

with tf.GradientTape(persistent=True) as tape:
    pred = model(inputs)
    loss = compute_total_loss(pred, gt)

grads_mamba = tape.gradient(loss, mamba.trainable_variables)
grads_attn  = tape.gradient(loss, attention.trainable_variables)
grads_enc   = tape.gradient(loss, encoder.trainable_variables)
grads_dec   = tape.gradient(loss, decoder.trainable_variables)

opt_mamba.apply_gradients(zip(grads_mamba, mamba.trainable_variables))
opt_attn.apply_gradients(zip(grads_attn, attention.trainable_variables))
opt_enc.apply_gradients(zip(grads_enc, encoder.trainable_variables))
opt_dec.apply_gradients(zip(grads_dec, decoder.trainable_variables))
```

#### 단계 분리 근거 — 완전 freeze의 함정

**위험 (자문 검토)**:
- Phase 1에서 Attention은 "Encoder의 깨끗한 직접 출력"에 최적화됨
- Phase 2에서 Mamba 활성화 → Mamba 통과 후 잠재 벡터 분포 (스케일, 위상) 변화
- **Attention 완전 freeze 시 Mamba가 "구식 분포"로 끼워맞춰야 함** → 학습 병목

**완화**:
- Phase 2a: Mamba가 빠르게 안정점 확보 (전체 흔들리지 않게 보호)
- Phase 2b: Attention/Decoder 풀어서 Mamba 출력 분포에 적응 (상호 적응)
- 차등 학습률로 Phase 1 학습 결과 보존

**우리 ConditionalLAPE skip의 부분 완화 효과**:
- Attention 직전 LAPE skip 재주입 → 위치 정보는 Phase 1 분포 일부 복원
- 그러나 Mamba 출력의 동역학적 잠재 표현 변화는 보호 못 함
- → Phase 2b 상호 적응 여전히 필요

#### 공통 사항 (Phase 2a, 2b 모두)

```
Loss: L_data + L_data_halo + L_diff_rel + L_Bateman + L_sigma_a_Xe + λ_branch · L_branch
      (Phase 1과 동일 + L_branch 보조)
      (L_keff는 후순위/검토중)
      (λ_branch: 작게 할 필요 없음 — Phase 분리로 freeze 보호 자동)

Mamba 학습 단위: 575-step parallel scan (GT 입력, Teacher Forcing)
```

#### Branch 데이터의 Phase 2 역할

- Phase 1에서 Encoder/Decoder/Attention은 이미 충분히 학습됨
- Phase 2 Branch 사용 시: **"동일 (state, p_load) 조건에서 다른 rod_map → 다른 시간 진화 응답"을 Mamba가 학습**
- h(t) 공급: **CRS parallel scan에서 추출 → detach 후 Branch에 주입**
  - detach 필수 (Branch gradient가 CRS 시퀀스 역방향 역전파 차단)

#### Phase 2의 freeze 효과 — λ_branch 자유 (Phase 2a 한정)

- Phase 2a: Encoder/Attention/Decoder freeze → Branch gradient가 그 부분 변경 불가
- Branch gradient는 **Mamba/h(t)에만** 영향 → λ_branch "보호 목적" 사라짐
- → 작게 둘 필요 없음, 데이터 비율과 물리적 중요도에 따라 결정
- Phase 2b에서는 차등 학습률로 보호 (lr 자체가 1/10)

### 5.4 Phase 3 (자기회귀 물리 정렬) — Alignment 단계 (신설)

> **2026-04-20 자문 검토 반영**: Phase 3의 핵심은 **Mamba parallel scan을 끄고 K-step 자기회귀 전개로 전환**. 자기 예측 입력 → 누적 오차 학습 → 모델이 오차 damping 학습. Pushforward trick 본격 적용 단계.

#### 핵심 변경: Parallel Scan OFF, K-step Autoregressive ON

```
Phase 1, 2: parallel scan (GT 입력) → 1-step transition 학습
Phase 3: 1-step recurrence × K (자기 예측 입력) → K-step 누적 오차 학습
```

#### 학습 흐름

```
시점 t: GT_state(t) → predict → pred(t+1)
                                  ↓ (자기 예측을 다음 step 입력으로)
                               predict → pred(t+2)
                                           ↓
                                          ...
                                       predict → pred(t+K)
                                                   ↓
                                               vs GT_state(t+K) → loss
                                               (K-step 누적)
```

- **첫 입력만 GT**, 이후 K-1 step은 **자기 예측 사용** (자기회귀 전개)
- K step 동안의 누적 오차 학습
- 모델이 "**자기 오차를 다음 step에서 damping**" 하는 방법 학습

#### 커리큘럼 학습 (K 점진 확장)

```
K=2  → K=4 → K=8 → K=16 → K=32 → K=64 → K=128 → K=256 → K=575
       (각 K에서 안정화 후 다음 K로 진행)
```

- **K가 작을수록**: 누적 오차 작음, Data Loss 의미 있음
- **K가 클수록**: 누적 오차 큼, Data Loss 약해짐 → **Physical Loss로 정렬**
- 최종 K=575 (전체 시퀀스 자기회귀)

#### 학습 설정

```
학습 대상: 전체 unfreeze (Encoder + Mamba + Attention + Decoder)
Mamba 동작: 1-step recurrence + InferenceParams cache (parallel scan OFF)
데이터: CRS 시퀀스 + Branch (커리큘럼 K 적용)
목적:
  1. Phase 1, 2에서 분리 학습된 모듈 간 gap 조정 (상호 적응 마무리)
  2. 자기회귀 누적 오차 damping 학습 (장기 안정성)
  3. 어려운 구간 (감출력/저출력/상승) 물리 정확도 향상
```

#### 모듈별 차등 학습률 (Phase 3)

Phase 2와 동일 원칙. 단, **Phase 2 대비 1/10 ~ 1/100** 추가 감소:

| 모듈 | Phase 3 학습률 | Phase 2b 대비 |
|------|------------|------------|
| **Mamba** | 5e-5 ~ 1e-4 | × 1/10 ~ 1/20 |
| **Attention** | 1e-5 ~ 5e-5 | × 1/5 ~ 1/10 |
| **Encoder** | 1e-6 ~ 1e-5 | × 1/10 |
| **Decoder** | 1e-6 ~ 1e-5 | × 1/10 |

→ 전체적으로 작은 lr로 미세조정 (Phase 1, 2 학습 결과 보존)

#### Loss 가중치 변화 (K에 따라)

| K | Data Loss 의미 | Physical Loss 가중치 |
|---|------------|------------------|
| K=2~4 | 의미 있음 (궤적 큰 차이 X) | 표준 (Phase 2와 동일) |
| K=8~16 | 약해짐 (궤적 어긋남 시작) | ↑ (예: × 1.5) |
| K=32~128 | 더 약해짐 (궤적 발산 가능) | ↑↑ (예: × 2~3) |
| K=575 | 거의 의미 없음 | ↑↑↑ (Physical Loss가 주, 예: × 5) |

#### Phase 3가 필요한 이유 (Phase 1과 중복 아님)

| 측면 | Phase 1 | Phase 3 |
|------|--------|--------|
| 학습 신호 | Teacher Forcing (1-step GT 매핑) | **자기회귀 전개 (K-step 자기 예측)** |
| 가중치 업데이트 방향 | "1-step 공간 응답 정확도" | "**누적 오차 damping**" (장기 안정성) |
| 모델이 본 입력 분포 | 항상 GT | 자기 오차 포함된 입력 (Phase 1, 2에서 못 봄) |

→ **동일 가중치 + 동일 Loss여도 학습 신호 자체가 다름**. 학습 궤적의 성질 완전히 다름.

#### 우리 도메인 강점 — Ground Truth 기반 정렬

| 항목 | LLM RLHF | 우리 Phase 3 |
|------|---------|------------|
| 정렬 신호 | 인간 피드백 (노이즈 있음) | **원자로 노심 해석 코드의 Ground Truth** (결정론적) |
| 정렬 강도 | 약함 (제한된 사람 평가) | **강함** (전 시점 GT + 물리 잔차) |
| Reward model | 별도 학습 필요 | **불필요** (물리 잔차 = 직접 정렬 loss) |

→ **결정론적 GT + 물리 제약** = LLM RLHF보다 강하고 노이즈 없는 정렬

#### 메모리 부담 고려

- K-step 동안 활성값 보존 (gradient용) → K가 커질수록 메모리 ↑
- **Gradient checkpointing** 적용으로 완화 (forward 활성값 재계산)
- K=575 시점에는 메모리 부담 큼 → checkpointing 필수

#### 평가 지표 (Phase 3 핵심)

- **장기 rollout 안정성**: 64/128/575-step 누적 오차 추이
- **Physical Loss 잔차**: L_diff_rel, L_Bateman 추이 (감소해야 정렬 성공)
- **운영 구간별 정확도**: 정/감/저/상승 별 오차
- **AO/keff 정확도** (구간별)

### 5.5 h(t) 관리 (Drama 패턴, 모든 Phase 공통)

- **InferenceParams snapshot/restore**: Mamba SSM state + conv_state deepcopy
- **Detached h(t) 주입**: CRS parallel scan → h(t) 추출 → `tf.stop_gradient` → Branch 초기 상태
- **Prefix burn-in**: 실제 데이터 K step → parallel scan → SSM state 워밍업
- **Attention은 stateless**: snapshot 시 Mamba state만 저장

### 5.6 추론 프로토콜

모델은 **단순한 1-step 예측 함수**:
```
f(state(t), rod_map_query, p_load_query, h(t)) → state(t+1)_pred, h(t+1)
```

**4가지 운영 패턴** (사용자 측 활용 방식):
- **패턴 A**: 연속 자기회귀 (시뮬레이터 재현)
- **패턴 B**: 동일 시점 다중 query (What-if 평가, h(t) snapshot)
- **패턴 C**: 자기회귀 + 분기 평가 혼합
- **패턴 D**: Iterative Critical Search (수동 CRS 탐색)

## 7. 향후 작업 계획

### 7.1 미구현 영역 (코드 작성 필요)

1. **시공간 프로세서 (Mamba-3 + Attention 하이브리드)**
   - Mamba-3 layer (TF2/Keras 구현)
   - STBlock (Mamba ×2 + Attention + MLP, p_load 주입, LAPE skip)
   - Block N=2 적층
2. **디코더 (Conv3D 1×1×1)**
   - Conv3D layer + 잔차 결합 함수
3. **Physical Loss**
   - L_data, L_data_halo
   - L_Bateman (Euler/RK4 적분)
   - L_sigma_a_Xe (Taylor 보간)
   - L_diff_rel (상대 잔차 형식)
   - L_keff (Rayleigh 몫)
4. **학습 코드 (3-Phase)**
   - Phase 1 학습 루프: Branch (단일 step), Encoder/Attention/Decoder 학습 (Teacher Forcing)
   - **Phase 2a 학습 루프**: CRS parallel scan + Branch sampling, **Mamba/h(t) 단독 학습** (나머지 freeze)
   - **Phase 2b 학습 루프**: 전체 학습 (모듈별 차등 학습률 — Mamba 1e-3, 나머지 1e-5~5e-5)
   - **Phase 3 학습 루프**: parallel scan OFF, **K-step 자기회귀 + 커리큘럼** (K=2→4→8→16→...→575), Physical Loss 강조
   - 공통: h(t) snapshot/restore, Prefix burn-in
   - Phase 3 전용: gradient checkpointing (메모리 절약)

### 7.2 Phase별 후 재검토 항목 / 다음 Phase 트리거

| 항목 | 트리거 | 재검토 / 후속 |
|------|------|----------|
| **sym_type 별도 주입** | (b) mirror/rotation 분리 오차 plateau | FiLM 등 별도 추가 주입 검토 |
| **AdaLN-Zero 효과** | γ, β가 0 근처 유지 시 | AdaLN 제거 (concat 단독) |
| **p_load_encoder embed_dim** | (c) p_load 구간 오차 plateau | 8 → 16 → 32 확장 |
| **lape_alpha 폭주** | α 값 > 10 | WD 강화 (1e-4 → 1e-3) 또는 LayerNorm(LAPE) |
| **Mamba:Attention 비율** | 학습 정체 시 | 1:1, 3:1 비교 실험 |
| **프로세서 layer 수** | 학습 정체 시 | N=2 → 3, 4 확장 |
| **Phase 2a → 2b 전환** | Mamba 학습 안정 (~30% epoch) | freeze 해제 + 모듈별 차등 학습률 |
| **Phase 3 시작** | Phase 2b 학습 안정 | parallel scan OFF + K-step 자기회귀 (K=2부터) |
| **Phase 3 K 확장** | 현재 K에서 안정화 | K=2→4→8→16→32→64→128→256→575 점진 |
| **Phase 3 종료** | 장기 rollout (575 step) 물리 잔차 plateau, 운영 구간별 정확도 목표 도달 | 학습 종료 또는 추가 커리큘럼 |

### 7.3 후순위 도입 옵션

- ReLoBRaLo (Loss 가중치 자동 균형)
- JAWS Jacobian 정규화 (메모리 부족 시)
- PhysicsCorrect (추론 안정화)
- Equation-Aware conditioning (Branch 정확도 강화 시)
- Predict Change Not States (Xe/I 채널만 시간 미분)
- Differentiable NEM Layer
- L_diffusion Cross-Attention

### 7.4 학습 계획 폴더 신설 (별도)

본 정리 문서 §5의 큰 틀을 기반으로 다음 항목 상세화 필요:
- Loss 가중치 수치 (Phase별 + K별, 수동 → ReLoBRaLo 검토)

- Phase 2b 모듈별 차등 학습률 정확한 수치
- Phase 3 K 커리큘럼 스케줄 (K=2,4,8,16,...,575 각 단계 epoch 수)
- Phase 3 모듈별 차등 학습률 정확한 수치
- Branch sampling 비율 (시퀀스당 step 수, branch 수)
- Optimizer/LR/Batch size (각 Phase)
- Gradient checkpointing 적용 시점
- Scheduled sampling 사용 여부 (Phase 3)
- Prefix burn-in K step
