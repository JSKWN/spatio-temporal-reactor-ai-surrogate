## 1. 연속 상태공간 방정식 (Continuous SSM)

> 원본: `2026-03-23 구현 계획 초안.md` §2.1

SSM의 기본 개념은 제어이론의 상태공간 표현:

$$h'(t) = A\,h(t) + B\,x(t)$$
$$y(t) = C^\top h(t)$$

| 기호 | 의미 | 차원 |
|------|------|------|
| x(t) | 입력 시퀀스 (각 시점) | ℝ^D |
| h(t) | 숨겨진 상태 — 시스템의 "기억" | ℝ^N |
| y(t) | 출력 | ℝ^D |
| A | 상태 전이 행렬 | ℝ^{N×N} |
| B | 입력→상태 매핑 | ℝ^{N×1} |
| C | 상태→출력 매핑 | ℝ^{1×N} |

**원자로 맥락**: h(t) = "노심 메모리". Xe-135/Sm-149 축적·소멸 이력, 출력 분포 변화 패턴이 압축 저장됨.

## 2. ZOH 이산화

> 원본: `2026-03-23 구현 계획 초안.md` §2.2

원자로 데이터는 Δt=5분 간격 이산 시계열. 연속 방정식을 Zero-Order Hold(ZOH)로 이산화:

$$h_t = \bar{A}\,h_{t-1} + \bar{B}\,x_t, \quad y_t = C^\top h_t$$

이산화된 행렬:

$$\bar{A} = e^{\Delta A}, \quad \bar{B} = A^{-1}(e^{\Delta A} - I)\,B$$

**step size Δ**: 입력 해상도를 나타내는 학습 가능 파라미터.
- Δ 크면 → 현재 입력 x_t에 집중
- Δ 작으면 → 이전 hidden state h_{t-1}의 문맥을 더 많이 사용

## 3. S4 → Mamba: 선택적 SSM (S6)

> 원본: `2026-03-23 구현 계획 초안.md` §2.3~2.4

### 3.1 S4의 한계: LTI (선형 시불변)

S4 이전의 SSM은 A, B, C가 모든 시점에서 **고정**. 원자로에서의 문제:

- 고출력/저출력 구간을 구분 불가
- 제논 독화 진행 중 vs 평형 상태 구분 불가
- 제어봉 패턴 변경에 무반응

### 3.2 Mamba의 핵심 — 입력 의존 파라미터

Mamba는 B, C, Δ를 **입력 x_t에 의존하도록** 변경:

$$B_t = f_B(x_t), \quad C_t = f_C(x_t), \quad \Delta_t = f_\Delta(x_t)$$

결과:

$$h_t = e^{\Delta_t A_t}\,h_{t-1} + A_t^{-1}(e^{\Delta_t A_t} - I)\,B_t\,x_t$$

→ 모델이 현재 입력에 따라 어떤 정보를 hidden state에 유지할지 **선택적으로** 결정.

## 4. 선택적 스캔 알고리즘

> 원본: `vit3d_mamba_architecture_report.md` §4.2

### 4.1 Mamba 블록 구조

| 구성 요소 | 설명 | 역할 |
|-----------|------|------|
| Linear Projection | D_model → D_inner (expansion) | 차원 확장 |
| 1D Conv | kernel=4, causal padding | 지역 시간 패턴 포착 |
| SiLU Activation | Swish 게이팅 | 비선형성 도입 |
| SSM (Selective Scan) | A, B, C, delta가 입력 의존 | 시간 동역학 추적 핵심 |
| Gate + Output Proj | D_inner → D_model | 잔차 연결과 결합 |

### 4.2 하드웨어 효율

Mamba는 B, C, Δ가 입력 의존이므로 S4처럼 FFT 합성곱 트릭 사용 불가.
대신 **하드웨어 효율 선택적 스캔**: GPU SRAM을 활용한 커널 퓨전으로 O(T) 시간, O(1) 추가 메모리.

## 5. 원자로 물리 대응 (물리적 해석)

> 원본: `2026-03-23 구현 계획 초안.md` §2.5

| SSM 변수 | 원자로 물리 대응 |
|----------|-----------------|
| h_t ∈ ℝ^{N×D} | 노심 "메모리": 이전 Xe/Sm 이력, 출력 분포 변화 패턴 |
| A (HiPPO 초기화) | 시간에 따른 상태 감쇠: Xe-135 붕괴(~9.17h), Sm-149 축적 반영 |
| B_t(x_t) | 현재 노심 상태(xsfuel, rodmap, Tcool)가 hidden state를 갱신하는 방식 |
| C_t(x_t) | hidden state에서 출력(Qabs, keff, Xe 분포) 추출 방식 |
| Δ_t(x_t) | 해당 시점의 "중요도": 제논 트랜지언트 시 크게, 정상 운전 시 작게 |

### 물리 현상과 Mamba 반응

| 물리 현상 | Mamba 반응 | 정합성 |
|-----------|-----------|--------|
| Xe 과도현상 (pload 급변) | Δ_t 증가 → h_t 급격 갱신 | 높음: Xe 시간상수(~9h) 비선형 과도 추적 |
| 정상 운전 (pload 일정) | Δ_t 감소 → h_t 느리게 변화 | 적합: 평형 접근 시 이전 상태 장기 기억 |
| 봉 이동 이벤트 | B_t, C_t 입력 의존 변화 | 적합: 중성자속 재분포를 h_t에 반영 |

## 6. 훈련 vs 추론 이중성

> 원본: `2026-03-23 구현 계획 초안.md` §2.6

| 모드 | 표현 | 복잡도 | 원자로 적용 |
|------|------|--------|------------|
| **훈련** | 합성곱(Convolution) | 병렬화 가능, O(T log T) | 전체 시나리오 한번에 학습 |
| **추론** | 순환(Recurrent) | O(1) per step | 5분 간격 실시간 예측 |

→ Transformer 대비 추론 속도 압도적 우위 (온라인 모니터링 적합).

## 7. Mamba-2 (SSD) 및 Mamba-3

> 원본: `2026-03-23 구현 계획 초안.md` §2.7

### 7.1 Mamba-2 (SSD)

- A를 scalar×identity로 단순화 → 행렬곱 하드웨어(Tensor Core) 활용
- **2~8배 속도 향상**
- 상태 차원(N)을 16 → 64~256으로 확대 가능
- 500개 노드의 다양한 공간 패턴이 z(t) 벡터에 압축되어 있으므로, 더 큰 상태 공간이 유리

### 7.2 Mamba-3

- **Exponential-Trapezoidal 이산화** 도입 → 수치 정확도 향상
- **Complex-valued SSM** → state-tracking 능력 강화
- 원자로 Xe-135 진동(주기적 특성)에 **복소수 회전(rotation) 표현**이 유리할 수 있음

## 8. TF2 구현 고려사항

> (2026-03-30 수정) 현재 아키텍처는 Conv3D Encoder → Mamba → Conv3D Decoder로 확정 (`모델 구현 계획.md` §3 참조). ViT3D는 폐기됨.

- 공식 Mamba 구현은 PyTorch + Triton 기반
- TF2에서는 selective scan의 **커스텀 구현** 필요 (가장 큰 기술적 도전)
- 대안: `tf.while_loop` 기반 SSM 구현
- **순수 TF ops만 사용** (custom CUDA kernel 금지 — ONNX 변환 호환성)
- 현재 모델 설계: cell 축을 batch에 합침 (B×500, T, D_latent) → **하나의 Mamba가 모든 cell의 시계열 처리 (가중치 공유)**
