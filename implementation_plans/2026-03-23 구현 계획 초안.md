# Spatio-Temporal Reactor AI Surrogate 구현 계획 및 Mamba SSM 작동원리 정리

## 요약

> SMR 노심의 부하추종 운전 해석을 위한 Spatio-Temporal AI Surrogate 모델은 두 개의 핵심 서브모델로 구성됨.
> - (1) Spatial Encoder: 각 시점의 3D 노심 상태(채널) `(20, 5, 5, C_in)` 텐서를 latent vector `z_t`로 압축하는 역할이며, 기존 `CustomMaxViT3D`를 개선해 재설계 할 예정.
> - (2) Temporal Model: 시계열 latent vector `z_1, z_2, ..., z_T`를 처리하여 hidden state `h_t`를 유지·갱신하고, 다음 스텝 출력 분포(Xe, Sm, flux, T 등)를 예측한다. Mamba(SSM)는 이 Temporal Model에 자연스럽게 적합하며, 제논 독화 등 물리계의 장기 메모리 의존성을 선택적으로 압축하는 핵심 메커니즘을 제공한다.[^1][^2]

>- ⚠️점검 필요:
>  1. 만약, Mamba가 학습 시에는 시점 1~T 까지 모두 받는다고 하더라도, 실제 추론시에도 전체 예측할 profile을 한번에 받아야 하는지..? 시계열 latent vector란..  
 - z_t란 단일 시점 t에서의 노심 3D 상태인 (20, 5, 5, C_in)을 ViT3D Encoder가 압축한 벡터 하나임.
 ```
  z_t = ViT3D( state_t )
        입력: (B, 20, 5, 5, C_in)
        출력: (B, D_latent)  ← 예: D_latent = 256 또는 512
 ```
   - D_latent는 공간 압축 차원으로, 512라고 설정하면 노심의 3D 정보가 512 차원 벡터로 요약된다는 의미임. 이는 시간과 무관한 단일 시점에서의 정보임.
   - T는 시간축이며 시퀀스의 길이임. z_1, z_2, ..., z_T에서 T는 몇개의 시점을 묶어 하나의 시퀀스로 처리할지를 뜻함. 5분 간격으로 T=512라면 512 x 5분 = 2,560분 ≈ 42.7시간을 의미함.
  - 초기 시점에서의 시계열 문제? Mamba는 학습 시에는 전체 시퀀스(z_1, z_2, ...)를 한번에 병렬로 학습함. 추론시에는 한 스텝씩 진행함(recurrence)
    - 특정 시점에서 시작하려면 MASTER 코드 구동 -> 해석데이터 수집 -> 특정 시점에서의 XS 및 해석데이터를 spatio-encoder로 축약하여 z_t 생산. 그 후에 그 z_t, z_t+1 등의 일부를 묶어서 h(t)로 제공하며 이건 슬라이딩 윈도우와 비슷한 역할을 하고, mamba의 warm_up 이라 하는건가. -> 슬라이딩 윈도우는 아니라고 함.

## Part 1: 기존 모델 문제점 분석 (CustomMaxViT3D)

첨부된 `custom_voxel_attention...ipynb`를 분석한 결과, 아래와 같은 구조적 문제점이 확인된다.[^3]

### 1.1 입출력 텐서 구조

| 항목 | 현재 모델 | 목표 모델 |
|------|-----------|-----------|
| 주 입력 | `xsfuel (B, 24, 5, 5, 10)` — XS 4D | `xsfuel (B, 20, 5, 5, 10)` + 상태채널 `C_in` |
| 보조 입력 | `rodvoxel (B, 24, 5, 5)` — rod position | `rodmap3d (20, 5, 5)` + 이전 스텝 Xe/Sm/T 등 |
| 출력 | `P3D (B, 24, 5, 5)` — 출력분포 1채널 | `(Qabs, Tcool, Tfuel, rhocool, Xe, Sm), keff, AO` |
| 시간 모델링 | ❌ 없음 (단일 스텝 예측) | ✅ Mamba hidden state `h_t` |

### 1.2 주요 문제점

**문제 1: `Gradients do not exist for relativebiastable`**
- `RelativeAttention3DLayer`의 `relativebiastable` weight가 `addweight(trainable=True)`로 선언되었으나, `build()` 시 `getreindexedbias3d(D, H, W)` 내부에서 `self.reindexedbias = self.addweight(...)` 방식으로 추가됨
- `call()` 에서 `self.reindexedbias`가 직접 attention logit에 더해지지 않고 `build()` 시점에만 생성되어 backprop에서 누락 → **gradient flow 단절**
- 수정 방향: `call()` 내에서 매번 `self.reindexedbias`를 `scaledattentionlogits`에 더하도록 명시적으로 연결하거나, PyTorch 기반으로 재작성하여 autograd 의존성을 명확히 할 것

**문제 2: D=24 vs. 실제 연료 평면 20개**
- `BoundaryPad`가 `(B, 24, 5, 5, C)` → `(B, 26, 7, 7, C)`로 패딩하도록 하드코딩되어 있으나, 실제 HDF5 데이터는 `nfuelplanes=20`
- 전처리 패키지(md 파일 기준)에서 `spatialshape = (20, 5, 5)`로 명시됨: D 차원을 24 → 20으로 수정 필요[^4]

**문제 3: 단일 스텝 예측, 시간 정보 전무**
- 노심은 Xe-135(반감기 ~9.17h), Sm-149 축적 등 **고유한 시간 지연 특성**을 가짐
- 현재 모델은 이전 시점 정보 없이 단일 스냅샷만 처리하여 제논 진동 예측 불가

**문제 4: `SuperpositioningNeighborCells` + `BoundaryPad` 순서 불일치**
- AlphaV2 변경로그에서 superposition → boundarypad 순서가 뒤바뀐 것이 기록되어 있으나, 실제 `call()`에서 순서가 혼재

**문제 5: 출력 단일 채널 고정**
- 현재 `finalconv`가 `filters=1`로 고정되어 있어 Qabs 하나만 예측 가능
- 목표 아키텍처에서는 멀티채널 출력(Qabs, Tcool, Tfuel, rhocool, Xe, Sm, keff, AO)이 필요

## Part 2: Mamba SSM 작동원리 — 핵심 수학

### 2.1 연속 상태공간 방정식 (Continuous SSM)

SSM의 기본 개념은 제어이론의 상태공간 표현에서 출발한다:[^5]

\[
h'(t) = A\,h(t) + B\,x(t)
\]
\[
y(t) = C^\top h(t)
\]

여기서:
- \(x(t) \in \mathbb{R}^D\): 입력 시퀀스 (각 시점의 입력)
- \(h(t) \in \mathbb{R}^N\): **숨겨진 상태(hidden state)** — 시스템의 "기억"
- \(y(t) \in \mathbb{R}^D\): 출력
- \(A \in \mathbb{R}^{N \times N}\): 상태 전이 행렬 (이전 상태가 다음 상태에 미치는 영향)
- \(B \in \mathbb{R}^{N \times 1}\): 입력이 상태에 미치는 영향
- \(C \in \mathbb{R}^{1 \times N}\): 상태가 출력에 미치는 영향

**원자로 맥락 해석**: \(h(t)\)가 바로 "노심 메모리"다. Xe-135, Sm-149의 축적/소멸 이력, 이전 출력 분포의 변화 패턴이 \(h(t)\) 안에 압축되어 저장된다.

### 2.2 이산화 (Discretization) — ZOH 방법

실제 원자로 데이터는 `deltatimemin=5`분 간격의 이산 시계열이므로, 연속 방정식을 이산화해야 한다. ZOH(Zero-Order Hold) 방법 적용:[^6]

\[
h_t = \bar{A}\,h_{t-1} + \bar{B}\,x_t, \quad y_t = C^\top h_t
\]

여기서 이산화된 행렬은:
\[
\bar{A} = e^{\Delta A}, \quad \bar{B} = A^{-1}(e^{\Delta A} - I)\,B
\]

이 때 **step size** \(\Delta\)는 입력 해상도를 나타내는 학습 가능한 파라미터다. 원자로 응용에서 \(\Delta\)가 크면 현재 입력 \(x_t\)에 집중하고, 작으면 이전 hidden state \(h_{t-1}\)의 문맥을 더 많이 사용하는 효과를 낸다.[^7][^1]

### 2.3 S4의 한계: LTI(선형 시불변)

S4 이전의 SSM은 \(A, B, C\)가 모든 시점에서 **고정**(Linear Time Invariant)이었다. 이는 원자로에 치명적인 문제를 일으킨다:[^2]

- 노심이 고출력 구간인지 저출력 구간인지에 관계없이 동일한 \(B, C\) 적용
- 제논 독화가 빠르게 진행 중인 시점과 평형 상태를 구분 불가
- 제어봉 패턴(rodmap)이 바뀌어도 입력 처리 방식이 동일

### 2.4 Mamba의 핵심 — 선택적 SSM (S6)

Mamba는 **B, C, 그리고 Δ를 입력 \(x_t\)에 의존하도록** 만들어 LTI 문제를 해결한다:[^1][^2]

\[
B_t = f_B(x_t), \quad C_t = f_C(x_t), \quad \Delta_t = f_\Delta(x_t)
\]

이를 통해 모델은 현재 입력 내용에 따라 어떤 정보를 hidden state에 유지할지 **선택적으로** 결정한다. 결과적으로:

\[
h_t = e^{\Delta_t A_t}\,h_{t-1} + A_t^{-1}(e^{\Delta_t A_t} - I)\,B_t\,x_t
\]

이것이 Mamba의 **selective scan algorithm**의 핵심 점화식이다.[^6]

### 2.5 원자로에 대한 물리적 해석

| SSM 변수 | 원자로 물리 대응 |
|----------|-----------------|
| \(h_t \in \mathbb{R}^{N \times D}\) | 노심의 "메모리": 이전 Xe/Sm 이력, 출력 분포 변화 패턴 |
| \(A\) (HiPPO 초기화) | 시간에 따른 상태 감쇠: Xe-135 붕괴 (~9.17h), Sm-149 축적 반영 |
| \(B_t(x_t)\) | 현재 노심 상태(xsfuel, rodmap, Tcool)가 hidden state를 갱신하는 방식 |
| \(C_t(x_t)\) | hidden state에서 실제 출력 (Qabs, keff, Xe 분포)을 추출하는 방식 |
| \(\Delta_t(x_t)\) | 해당 시점을 얼마나 "중요하게" 볼지: 제논 트랜지언트 중 크게, 정상 운전 시 작게 |

**핵심 답변**: Mamba는 각 시점의 노심 상태(Xe, 아이오딘, flux, 온도)를 hidden state \(h_t\)에 **선택적으로** 저장하고, 이를 다음 스텝 예측에 활용할 수 있다. 이는 Mamba의 설계 목적과 정확히 일치한다.[^5][^1]

### 2.6 훈련 vs 추론 이중성

Mamba는 훈련/추론 시 서로 다른 표현을 사용한다:[^2][^1]

| 모드 | 표현 | 복잡도 |
|------|------|--------|
| **훈련** | 합성곱(Convolution) 표현 | 병렬화 가능, \(O(T \log T)\) |
| **추론** | 순환(Recurrent) 표현 | \(O(1)\) per step — 실시간 예측에 최적 |

원자로 온라인 모니터링에서 5분 간격 실시간 예측 시 추론 속도는 Transformer 대비 압도적으로 빠르다.

### 2.7 Mamba-2와 Mamba-3 개선사항

**Mamba-2 (SSD)**: `A`를 scalar×identity로 더 단순화하여 행렬곱 하드웨어(Tensor Core)를 활용, 2–8배 속도 향상.[^8][^9][^10]

**Mamba-3**: Exponential-Trapezoidal 이산화 도입으로 수치 정확도 향상, Complex-valued SSM으로 state-tracking 능력 강화. 원자로의 Xe-135 진동(주기적 특성)에 복소수 회전(rotation) 표현이 유리할 수 있다.[^11]

## Part 3: 전체 Spatio-Temporal 모델 아키텍처

### 3.1 설계 원칙

기존 md 파일들의 데이터 구조를 바탕으로, 두 서브모델의 인터페이스를 다음과 같이 정의한다:[^4]

```
[ViT3D Spatial Encoder]
  입력: state_t = (xsfuel, Xe_t, Sm_t, Qabs_{t-1}, Tcool_{t-1}, rodmap3d_t)
        shape: (B, 20, 5, 5, C_in)
  출력: z_t = latent vector, shape: (B, D_spatial)

[Mamba Temporal Model]
  입력: z_1, z_2, ..., z_T  (시퀀스)
  상태: h_t (hidden state — 노심 메모리)
  출력: pred_t+1 = (Qabs, Tcool, Tfuel, rhocool, Xe, Sm), keff, AO
```

### 3.2 개선된 Spatial Encoder (ViT3D v2)

현재 모델의 문제를 해결한 재설계 방향:

**프레임워크 전환**: TensorFlow → **PyTorch**
- `relativebiastable` gradient 문제는 TF의 `add_weight` 관리 방식에서 기인
- PyTorch `nn.Parameter`로 전환 시 autograd 자동 처리

**아키텍처 변경 사항**:
- 입력: `(B, 20, 5, 5, C_in)` (D=24 → 20으로 수정)
- `BoundaryPad` → `SuperpositioningNeighborCells` 순서 고정
- `Stem3D`: `Conv3D(strides=(1,1,1))` × 2 유지
- `MaxViTBlock3D` × 3 스테이지: `blocksize=(4,2,2)`, `gridsize=(4,2,2)`
- 출력: `z_t` latent vector `(B, D_spatial)` — `GlobalAvgPool3D` 후 `Dense`
- 다중 헤드 디코더:
  - `head_3d`: `(B, 20, 5, 5, C_out)` — Qabs, Tcool, Tfuel, rhocool, Xe, Sm
  - `head_scalar`: `(B, 2)` — keff, AO

**relative position bias 수정**:
```python
# 기존: build()에서만 생성, call()에서 누락
# 수정: call()에서 명시적 사용
def forward(self, x):
    ...
    attn_logits = (q @ k.transpose(-2,-1)) / math.sqrt(self.head_size)
    attn_logits = attn_logits + self.reindexed_bias  # 명시적 연결
```

### 3.3 Mamba Temporal Model 설계

**입력 시퀀스 구성** (HDF5 데이터 구조 기준):[^4]
- 각 시점 `t`: `z_t = ViT3D(state_t)`, shape `(D_spatial,)`
- 시퀀스 길이: `T = 576` (총 스텝 수, `deltatimemin=5`, `cycletimehour=24×2`)

**Mamba 레이어 구성** (권장):
```python
MambaBlock(
    d_model=D_spatial,     # spatial latent 차원
    d_state=64,            # hidden state 크기 N (Xe/Sm 이력 등)
    d_conv=4,              # local convolution
    expand=2,              # inner 확장비
    dt_rank="auto",        # Δ 파라미터 rank
)
```

**31-way Branch 처리**: md 파일의 설계에 따라 Critical(index 0) + 30 Branch를 처리할 때, Mamba의 hidden state `h_t`는 Critical 경로로 전파하고, Branch는 `h_t`에 `queryrodmap3d[:, p, ...]`를 조건으로 병렬 forward pass:[^4]

```python
# Critical path: Mamba로 h_t 갱신
z_t = spatial_encoder(state_t_critical)
h_t, y_critical = mamba(z_t, h_{t-1})

# Branch path: h_t 공유, rodmap만 다름
for p in range(30):
    z_branch = spatial_encoder(state_t_branch_p)  # 다른 rodmap
    _, y_branch_p = mamba_branch_head(z_branch, h_t)  # h_t frozen
```

### 3.4 학습 전략

**Warm-up Phase**: 처음 `N_warmup` 스텝은 ViT3D만 학습 (static spatial 예측), 이후 Mamba 포함 end-to-end 학습.[^4]

**Iterative (auto-regressive) 추론**:
```
t=1: state_1 → ViT3D → z_1 → Mamba → h_1, pred_2
t=2: pred_2 as state_2 → ViT3D → z_2 → Mamba(h_1) → h_2, pred_3
...
```

**손실 함수**: 
- 3D 출력: Masked MSPE (기존 `maskedmspeloss` 계승, ROI 마스크 적용)
- Scalar 출력: MSE (keff ±0.001 이내 목표)

## Part 6: 기술적 고려사항

### 6.1 Spatial Encoder vs. 시간 정보

현재 ViT3D는 순수 spatial 처리만 한다. Spatio-Temporal 결합 방식에는 두 가지 전략이 있다:

| 전략 | 설명 | 장단점 |
|------|------|--------|
| **분리형** (권장) | ViT3D가 z_t 생성, Mamba가 시간 처리 | 각 모듈 독립 학습·검증 가능, 모듈성 우수 |
| **통합형** | VideoMamba처럼 3D+시간을 하나의 SSM으로[^12] | 구현 복잡, 대용량 학습 데이터 필요 |

분리형이 현재 데이터 규모(~576 스텝 × 50 LP)에서 실용적이다.

### 6.2 Mamba의 Hidden State 크기 선택

hidden state \(N\) 크기는 정보 압축 용량을 결정한다:[^1]
- **소형** (\(N=16\)): 빠르지만 장기 의존성 손실 위험
- **권장** (\(N=64\)): Xe-135 반감기(~9.17h = ~110 스텝) 커버 가능
- **대형** (\(N=128\)): Sm-149 장기 이력 포함, 메모리 증가

원자로 응용에서는 Xe-135의 시정수를 고려할 때 \(N \geq 64\) 권장.

### 6.3 관련 연구 — 핵 시스템 Surrogate

- KAIST의 원전 중대사고 Deep Learning surrogate는 rolling-window forecast(3-step)로 이전 스텝 TH 변수를 활용하여 단일 스텝 대비 MAE 64% 감소를 달성[^13]
- INL의 SMR IES quasi-static surrogate는 시스템 통합 surrogate의 타당성을 입증[^14]
- 핵융합 MHD 분야의 SHRED 모델은 SVD+LSTM 기반으로 희소 센서에서 전체 spatio-temporal 상태를 재구성하며, Mamba 대체 가능성 제시[^15]

이들 연구는 모두 **시간 이력을 명시적으로 활용**하는 구조가 단일 스냅샷 방식보다 우수함을 공통적으로 보인다.

### 6.4 Δt-Mamba3D 참고 아키텍처

최근 Δt-Mamba3D (2025)는 의료 영상 종단 분석에서 불규칙 시간 간격의 3D 이미지 시퀀스를 처리하며, Mamba의 \(\Delta_t\)에 실제 시간 차이를 직접 주입하는 아이디어를 제안한다. 원자로 응용에서도 이 개념을 활용하면:[^16]
- 5분 간격(정상) vs. 2분 간격(트랜지언트) 운전 데이터를 자연스럽게 처리 가능
- `deltatimemin` 값을 \(\Delta_t\)의 prior로 활용

## 결론

Mamba SSM은 원자로 부하추종 시뮬레이션의 시간 모델링에 이상적으로 적합하다. hidden state \(h_t\)는 Xe-135, Sm-149 이력을 포함한 노심의 "메모리"를 압축하며, 선택적 메커니즘(\(B_t, C_t, \Delta_t\)의 입력 의존성)은 트랜지언트 구간과 정상 운전 구간을 자동으로 구분한다. 구현 순서는 **S(Spatial 재작성) → M(Mamba 공부) → T(통합) → D(데이터 연동)** 순서를 따르며, 각 Phase를 독립적으로 검증하면서 진행하는 것이 위험을 낮추는 접근이다.[^2][^1]
