## 답변 A

추가 정보를 반영하면, 제 권고는 "단일 벡터 압축"도 아니고 "cell-wise 독립 Mamba"도 아닌, 셀 토큰을 유지하되 시간모델 내부에 공간 결합을 넣는 C안 계열을 기본선으로 잡는 것입니다. [arxiv](https://arxiv.org/html/2407.14129v1)
DreamerV3와 Drama는 compact latent와 단일 시퀀스 dynamics에서 강점을 보였지만, 장기 격자 예측에서는 GraphCast류처럼 깊은 processor가 상호작용을 맡고 decoder는 가볍게 두는 구조가 더 직접적인 선례를 제공합니다. [niftep.snu.ac](http://niftep.snu.ac.kr/site/download.asp?bid=9&file=05_%EC%9D%B8%EA%B3%B5%EC%A7%80%EB%8A%A5+%EA%B8%B0%EC%88%A0%EC%9D%84+%ED%99%9C%EC%9A%A9%ED%95%9C+%EC%B0%A8%EC%84%B8%EB%8C%80+%EC%9B%90%EC%9E%90%EB%A0%A5+%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%9D%98+%EC%95%88%EC%A0%84%EC%84%B1+%EB%B0%8F+%EA%B2%BD%EC%A0%9C%EC%84%B1+%EA%B0%95%ED%99%94+%EC%A0%84%EB%9E%B5_%EC%9C%A0%EC%9A%A9%EA%B7%A0%280%29.pdf)

### 상태 표현

Q1에 대해서는, 720 cell을 하나의 벡터로 강하게 압축하는 Dreamer형 상태는 전역 정보를 시간모델이 다루기 쉽게 만들지만, DreamerV3 자체도 compact representation과 짧은 imagination horizon을 중심으로 설계되어 있어 7200차원 물리장의 세부 공간 구조를 장기적으로 복원하는 문제와는 목적이 다릅니다. [blog.naver](https://blog.naver.com/racecarve/223164085553)
반대로 토큰 표현은 공간 해상도를 보존하지만, 현재 A안처럼 시간모델이 토큰 간 직접 상호작용을 하지 못하면 그 장점이 전이 연산 안에서 사라집니다. [carbonsaurus](https://www.carbonsaurus.com/news_clip/ai%EC%99%80-%EA%B0%80%EC%83%81%EC%9B%90%EC%9E%90%EB%A1%9C%EC%9D%98-%EB%A7%8C%EB%82%A8-smr-%EC%84%A4%EA%B3%84%EC%9D%98-%EC%83%88-%EC%A7%80%ED%8F%89-%EC%97%B0%EB%8B%A4)
그래서 이 문제에는 "cell token 유지 + 소수의 global token + 축방향 summary token + 주기적 spatial attention" 같은 계층형 상태 표현이 가장 균형이 좋습니다. [youtube](https://www.youtube.com/watch?v=LnLqS5VVf4c)

### 독립 전이 한계

Q2와 Q3의 핵심은, PDE 쪽에서 Mamba 자체는 충분히 유망하지만 그 성공 사례가 곧바로 "cell-wise 완전 독립 시간전이"의 정당화로 이어지지는 않는다는 점입니다. [arxiv](https://arxiv.org/pdf/2505.12020.pdf)
MNO, GeoMaNO, DeepOMamba 계열은 Mamba/SSM이 PDE operator 학습에서 정확도와 효율을 높일 수 있음을 보여주지만, 제가 확인한 근거들에서는 강결합 다물리 3D 격자를 575-step 자기회귀하면서 공간 결합을 사실상 decoder gradient에만 맡긴 구조의 직접 선례는 충분하지 않았습니다. [ui.adsabs.harvard](https://ui.adsabs.harvard.edu/abs/2025JCoPh.54014272H/abstract)
장기 rollout 안정성에 관한 기상 분야 비교에서도 직접적인 spatial interaction을 processor 내부에 둔 구조가 더 안정적이라는 방향의 근거가 있으므로, 결합 위치는 decoder만이 아니라 최소한 시간모델 내부여야 하고, 가장 안전한 선택은 "processor는 무겁게, decoder는 얕게"입니다. [arxiv](https://arxiv.org/html/2407.14129v1)

### Branch와 hidden state

Q4에 대해, Branch는 같은 시점의 상태에서 rod_map만 바꾸어 29개 미래를 fan-out하는 문제이므로 h(t)에 이미 전역 반응도 문맥과 축방향 모드 정보가 들어 있어야 합니다. [carbonsaurus](https://www.carbonsaurus.com/news_clip/ai%EC%99%80-%EA%B0%80%EC%83%81%EC%9B%90%EC%9E%90%EB%A1%9C%EC%9D%98-%EB%A7%8C%EB%82%A8-smr-%EC%84%A4%EA%B3%84%EC%9D%98-%EC%83%88-%EC%A7%80%ED%8F%89-%EC%97%B0%EB%8B%A4)
Drama도 deterministic state를 과거 latent와 action의 함수로 만들고, real transitions로부터 더 informative한 hidden state를 추정한 뒤 imagination을 시작하는 점을 강조하는데, 이는 분기 정확도가 hidden state 품질에 직접 좌우된다는 뜻입니다. [carbonsaurus](https://www.carbonsaurus.com/news_clip/ai%EC%99%80-%EA%B0%80%EC%83%81%EC%9B%90%EC%9E%90%EB%A1%9C%EC%9D%98-%EB%A7%8C%EB%82%A8-smr-%EC%84%A4%EA%B3%84%EC%9D%98-%EC%83%88-%EC%A7%80%ED%8F%89-%EC%97%B0%EB%8B%A4)
따라서 720개 cell별 h_i(t)만 snapshot하기보다, branch 시점에 별도의 global state g(t)를 함께 저장하고 각 branch에서 local state와 global state를 같이 조건화하는 구조가 훨씬 안전합니다. [niftep.snu.ac](http://niftep.snu.ac.kr/site/download.asp?bid=9&file=05_%EC%9D%B8%EA%B3%B5%EC%A7%80%EB%8A%A5+%EA%B8%B0%EC%88%A0%EC%9D%84+%ED%99%9C%EC%9A%A9%ED%95%9C+%EC%B0%A8%EC%84%B8%EB%8C%80+%EC%9B%90%EC%9E%90%EB%A0%A5+%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%9D%98+%EC%95%88%EC%A0%84%EC%84%B1+%EB%B0%8F+%EA%B2%BD%EC%A0%9C%EC%84%B1+%EA%B0%95%ED%99%94+%EC%A0%84%EB%9E%B5_%EC%9C%A0%EC%9A%A9%EA%B7%A0%280%29.pdf)

### Decoder와 물리 loss

Q5에 대해서는, 경량 decoder가 physical loss gradient를 자동으로 약화시킨다고 볼 필요는 없습니다. [blog.naver](https://blog.naver.com/racecarve/223164085553)
DreamerV3에서도 세계모델 표현은 reconstruction과 prediction loss로 end-to-end로 형성되고, GraphCast류도 decoder는 주로 latent를 출력장으로 사상하는 역할이므로, 실제로 더 중요한 것은 decoder의 깊이보다 processor가 물리적으로 의미 있는 coupling을 표현하느냐입니다. [niftep.snu.ac](http://niftep.snu.ac.kr/site/download.asp?bid=9&file=05_%EC%9D%B8%EA%B3%B5%EC%A7%80%EB%8A%A5+%EA%B8%B0%EC%88%A0%EC%9D%84+%ED%99%9C%EC%9A%A9%ED%95%9C+%EC%B0%A8%EC%84%B8%EB%8C%80+%EC%9B%90%EC%9E%90%EB%A0%A5+%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%9D%98+%EC%95%88%EC%A0%84%EC%84%B1+%EB%B0%8F+%EA%B2%BD%EC%A0%9C%EC%84%B1+%EA%B0%95%ED%99%94+%EC%A0%84%EB%9E%B5_%EC%9C%A0%EC%9A%A9%EA%B7%A0%280%29.pdf)
오히려 FullAttention decoder가 꼭 필요하다면 그것은 time processor가 못한 공간 결합을 출력단에서 보정하고 있다는 신호일 수 있으므로, C안에서는 경량 decoder에 physical loss를 유지하되 short-horizon unroll loss와 latent prediction loss를 추가해 gradient를 processor에 더 직접 거는 편이 낫습니다. [blog.naver](https://blog.naver.com/racecarve/223164085553)

### 시작 전략

Q6의 제 결론은 "A안으로 먼저 시작하고 실패하면 C안으로 간다"보다, 처음부터 **C-lite**로 시작하는 편이 총비용이 더 낮다는 것입니다. [youtube](https://www.youtube.com/watch?v=LnLqS5VVf4c)
Jamba는 Mamba와 attention의 교차 배치가 장시퀀스에서 실용적임을 보여주고, Drama는 긴 sequence 활용의 이점을 보여주며, GraphCast 계열은 heavy processor 패턴이 장기 격자 rollout에 유리하다는 비교적 강한 구조적 힌트를 줍니다. [youtube](https://www.youtube.com/watch?v=LnLqS5VVf4c)
구체적으로는 "Mamba 3~4개마다 spatial attention 1개, global token 4~8개, delta prediction, branch용 global snapshot, decoder는 1~2층 MLP"를 첫 버전으로 두고, 평가는 one-step 오차보다 64/128/575-step rollout의 축방향 모드 drift, 공간 비일관성, 물리 잔차 폭주 시점, 29-branch ranking 정확도를 기준으로 보시는 것이 맞습니다. [arxiv](https://arxiv.org/html/2407.14129v1)

### 추가 질의: Physical Loss 역전파와 경량 구조에 관한 문헌

가장 가까운 선례는 **Spectral PINNSformer (S-Pformer)** ([neurips](https://neurips.cc/virtual/2025/125800), [arxiv](https://arxiv.org/abs/2510.05385)):
- 기존 encoder-decoder PINNSformer의 "encoder redundancy"를 명시적으로 비판
- Self-attention만으로 시공간 상관을 포착할 때 encoder가 불필요하다고 주장
- Decoder-only 구조로 단순화하여 파라미터 감소 + 동등 이상 성능

관련 논문:
- **PhysicsFormer** ([arxiv](https://arxiv.org/html/2601.03613v1)): physics-informed transformer의 계산 병목 비판, dynamics-weighted loss로 수렴 개선
- **OrientationNN** ([pmc](https://pmc.ncbi.nlm.nih.gov/articles/PMC12833216/)): compact MLP에 물리 제약을 넣은 경량 physics-informed network

해석: "가벼운 인코더/디코더 + 무거운 프로세서"는 단순한 취향이 아니라, physics-informed 최적화에서 불필요한 표현 병목을 줄이려는 설계로 정당화할 수 있다.

## 답변 B

핵심부터 말씀드리면, **추가된 Branch 학습 구조로 인해 A안(Cell-wise 독립 Mamba)은 구조적으로 작동하기 불가능에 가깝습니다.** 기상/물리 대리모델의 표준인 **C안(Jamba 패턴 기반의 시공간 통합 프로세서 + 경량 디코더)으로 즉시 직행하시는 것을 강력히 권고**합니다.

### (Q1) 상태 표현의 선택: 토큰 유지(TokenWM) vs 단일 벡터 압축(DreamerV3)

**결론: 720개 독립 토큰 유지가 압도적으로 유리합니다.**

- **해상도 보존의 필수성**: 원자로 노심은 국부적인 출력 피킹(Peaking factor)이 안전 해석에 치명적. 720개 cell × 10개 물리량을 단일 벡터로 압축하면 '공간적 블러링'이 발생. DreamerV3는 시각적으로 관대한 도메인이라 압축이 통하지만, PDE 도메인에서는 오차 누적의 주범.
- **결정론적 동역학**: RL Dreamer 계열은 확률적 환경 전이를 다루기 위해 RSSM을 쓰지만, 원자로 시뮬레이터는 결정론적 PDE. 토큰을 유지하여 공간 좌표계를 보존하는 것이 확산방정식 학습에 유리.

### (Q2) Cell-wise 독립 시간 전이의 한계

**결론: MNO의 성공은 원자로 노심 문제에 그대로 적용될 수 없습니다.**

- MNO가 다룬 단일 PDE(Navier-Stokes 등)와 달리, 원자로에서는 제어봉 삽입이라는 **강한 외부 개입이 공간적으로 발생**.
- 중성자 확산 방정식의 **타원형 특성**: 한 cell에 제어봉이 삽입되면, 라플라시안(∇²) 항으로 인해 노심 전체의 중성자속이 즉각적으로 재배열됨. A안에서 cell i의 Mamba는 cell j에 제어봉이 꽂혔다는 사실을 다음 시점 예측 시 전혀 알 수 없음.

**타원형 방정식의 물리적 의미 (트램펄린 비유)**:
- 팽팽한 트램펄린의 한 점을 누르면, 누르는 그 즉시 가장자리의 기울기와 장력도 동시에 바뀜
- 시간 지연 없이 전체 공간이 동시에 새로운 평형 상태로 재배열
- 이것이 중성자 확산 방정식의 본질: 매 시점 전체 노심이 동시에 평형을 맞춤
- 따라서 시간 모델 내부에 전체 공간을 한 번에 묶어주는 연산(Attention)이 교차로 존재해야 함

### (Q3) 공간 결합의 위치

**결론: 시간 모델 내부(Processor)에서 시공간 결합을 수행해야 합니다. (C안)**

- PDE 시간 적분에서 매 스텝 공간 미분항(∇²φ)을 계산해야 하듯, **시간 전이와 공간 결합은 분리될 수 없고 매 스텝 교차로 일어나야 함**.
- 디코더에서만 공간을 섞으면, 시간 모델은 "공간적으로 고립된 채 미래를 상상"하고 디코더가 억지로 물리적 정합성을 짜맞추는 구조. 575 스텝에서 발산 위험.

### (Q4) Branch 학습과 hidden state 품질

**결론: A안의 가장 치명적인 약점. Branch를 위해서라도 C안이 필수.**

- Branch 예측 시, h(t)와 새로운 rod_map으로 다음 스텝 예측. A안에서 cell i의 h_i(t)에는 노심 전체 공간 정보가 없으므로, cell j 위치의 rod_map 변경 효과를 cell i가 예측하는 것은 **구조적으로 불가능**.
- C안(Jamba)에서는 Attention 층이 지속적으로 cell 간 정보를 교환하므로, h_i(t)에 전역 맥락이 인코딩됨. 새 rod_map이 들어오면 Attention을 통해 제어봉 영향이 각 cell로 즉각 전파.

### (Q5) 경량 디코더에서의 Physical Loss 유효성

**결론: 디코더가 경량화될수록 Physical Loss의 Gradient 품질은 오히려 향상됩니다.**

- A안의 FullAttn 디코더는 너무 강력해서, Mamba가 엉망으로 예측해도 디코더 선에서 Loss가 작아지게 포장해 버릴 위험. 이 경우 Mamba로 흘러가는 Gradient는 노이즈.
- 경량 디코더(MLP 1~2층)면 Physical Loss 오차 Gradient가 **프로세서(Mamba+Attention)로 왜곡 없이 직접 전달**. 프로세서 내 Attention이 "공간 확산"을 모사하도록 강제하는 구조.

### (Q6) 시작점 전략

**결론: A안을 건너뛰고 처음부터 C안으로 설계하십시오.**

- Q2(제어봉 개입)와 Q4(Branch 예측)의 이유로 A안은 성공할 수 없는 구조.
- 720 token은 LLM/Vision 대비 극도로 작은 시퀀스 길이. 720×720 Attention map은 GPU에 거의 부하를 주지 않음.
- C안 도입 10~20% 비용 증가는 575 스텝 안정적 롤아웃 성능에 비하면 무시 가능.

### 실전 구현 제안

1. **C안(Jamba) 블록 구성**: `[Mamba Block (시간 전이)] → [Self-Attention Block (공간 결합)]`을 하나의 Layer로 묶어 N개 적층. 비율 1:1 또는 2:1.
2. **제어 입력 처리**: rod_map, p_load는 매 시점 Processor의 각 Layer에 지속적으로 주입 (Cross-Attention 또는 AdaLN).

|------|----------|
| **상태 표현** | 720 cell token 유지. 단일 벡터 압축(DreamerV3)은 공간 해상도 손실로 부적합 |
| **A안의 한계** | Cell-wise 독립 Mamba는 강결합 물리 시스템(중성자 확산)에서 공간 정보 교환 부재가 치명적. MNO의 성공이 곧 A안의 정당화가 되지 않음 |
| **C안 권장** | Mamba + Attention 하이브리드 프로세서 + 경량 디코더가 표준 패턴. 처음부터 C안(또는 C-lite)으로 시작 권장 |
| **경량 디코더** | 디코더 경량화가 Physical Loss gradient를 약화시키는 것이 아니라, 프로세서로의 gradient 전달을 오히려 직접적으로 만듦. FullAttn 디코더가 필요하다면 그것은 프로세서가 공간 결합을 못하고 있다는 신호 |
| **Branch 학습** | h(t)에 전역 공간 맥락이 필수. Cell-wise 독립 h(t)로는 rod_map 변경의 전역 효과를 예측 불가능. Global state 메커니즘 필요 |
| **시작 전략** | A안 → C안 점진적 전환보다, 처음부터 C안으로 설계하는 것이 총비용이 낮음 |
| **비용** | 720 token에 대한 Attention은 연산 부담이 매우 작음. C안의 10~20% 추가 비용은 무시 가능 |

### 두 답변의 차이

| 관점 | 답변 A | 답변 B |
|------|--------|--------|
| **A안 평가 강도** | "직접 선례가 충분하지 않다" (유보적) | "구조적으로 작동 불가능" (단정적) |
| **물리적 근거** | 기상 모델 비교, PDE 문헌 참조 | 타원형 PDE 특성(라플라시안)에 기반한 수학적 논증 |
| **상태 표현 세부** | "cell token + global token + 축방향 summary token + 주기적 spatial attention"의 **계층형** 구조 제안 | cell token + Attention으로 전역 정보 암묵 처리. 명시적 global token 미언급 |
| **Mamba:Attention 비율** | 3~4:1 권장 | 1:1 ~ 2:1 권장 (더 많은 Attention) |
| **추가 기법** | delta prediction, latent prediction loss, short-horizon unroll loss 등 보조 기법 구체 제안 | 제어 입력의 지속적 주입(매 Layer) 강조 |
| **문헌 참조** | S-Pformer, PhysicsFormer 등 physics-informed 경량 구조 논문 제시 | 중성자 확산 방정식의 타원형 특성에 대한 물리적 설명 |

### 우리 프로젝트에의 시사점

1. **A안 단독 채택은 재고 필요**: 두 답변 모두 A안의 구조적 한계를 지적. 특히 Branch 학습(Q4)에서 cell-wise 독립 h(t)의 전역 정보 부재가 가장 치명적인 약점으로 식별됨. 다만 답변 A는 "충분하지 않다"(유보적), 답변 B는 "불가능하다"(단정적)로 강도가 다름.

2. **C안(Jamba 패턴)으로의 방향 전환 검토**: 두 답변 모두 C안을 권장하되, 구체적인 Mamba:Attention 비율은 답변 A(3~4:1)와 답변 B(1:1~2:1)에서 차이. 우리 720 cell 규모에서의 최적 비율은 실험으로 확인 필요.

3. **디코더 설계 재검토**: 두 답변 모두 경량 디코더(MLP 1~2층)를 권장. 현재 계획의 FullAttn 디코더는 프로세서에 공간 결합이 들어가면 불필요한 중복. S-Pformer 논문이 encoder redundancy를 비판한 것과 같은 맥락.

4. **Global token / 계층형 상태 표현**: 답변 A가 제안한 "cell token + global token + 축방향 summary token"은 Xe 축방향 진동 추적에 유용할 수 있음. 답변 B는 Attention이 이를 암묵적으로 처리한다고 보지만, 명시적 global token의 저비용 + 잠재적 효과를 고려하면 포함하는 것이 안전.

5. **추가 기법 (답변 A)**: delta prediction(잔차 예측), latent prediction loss(CDP), short-horizon unroll loss는 C안에서도 유효한 보조 기법. C안 채택 시에도 함께 적용 검토.

6. **제어 입력 주입 (답변 B)**: rod_map/p_load를 초기 입력뿐 아니라 매 Layer에 지속 주입하라는 제안은, Branch 예측 정확도에 직접적으로 영향. 현재 설계(인코더 입력 + Mamba concat + 디코더 AdaLN)에서 프로세서 내 Attention에도 AdaLN 또는 Cross-Attention으로 주입을 확장하는 방안 검토.

7. **미해결 질문**: 
   - C안에서 인코더(현재 FullAttn ×3)도 경량화해야 하는가? S-Pformer의 "encoder redundancy" 비판이 우리 인코더에도 적용되는가?
   - Mamba-3 / GDN과 C안의 Attention이 결합될 때, 최적 조합은 무엇인가?
   - 2-Phase 학습 전략은 C안에서도 유지되는가? (Phase 1: Branch pretrain, Phase 2: CRS 학습)
