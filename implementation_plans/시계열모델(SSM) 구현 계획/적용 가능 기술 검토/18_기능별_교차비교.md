# 기능별 교차비교 — 전 소스에서 추출한 기법의 기능 축별 분석

> **작성일**: 2026-04-15
> **목적**: 01~17, 20~21번 문서에서 추출한 기법을 6개 기능 축으로 교차 비교하여 우리 아키텍처(A안)에 대한 최적 기법 도출
> **아키텍처 맥락**: Encoder(FullAttn ×3) → Mamba(cell-wise) → Decoder(FullAttn + AdaLN + Physical Loss)

---

## 1. 시간 모델링 (Temporal Dynamics)

### 우리 문제의 시간 모델링 요구사항

- 입력: z(t) ∈ (720, 128) + p_load 스칼라 → 출력: d(t) ∈ (720, 128)
- 575-step 자기회귀 (오차 누적이 치명적)
- 디코더 FullAttn + Physical Loss가 공간 결합을 담당 (A안)
- Branch-in-CRS: 특정 시점에서 hidden state snapshot/restore 필요
- Xe/온도의 지역 동역학 (Bateman ODE) + 중성자 확산 (전역)

### 후보 기법 비교

| 기법 | 소스 | 시간 복잡도 | 시점당 공간 복잡도 | 병렬 학습 | 자기회귀 안정성 | 선택 |
|------|------|-----------|------------------|----------|---------------|------|
| Block GRU | DreamerV3 | O(T) | O(N) | X (TBPTT) | 중간 (15 step 검증) | 후보 |
| **Mamba-2 (SSD)** | Drama, MNO | O(T) | O(N) | **O (chunked matmul)** | **높음 (MNO 실증)** | **1순위** |
| Mamba-3 | Mamba-3 논문 | O(T) | O(N) | O | 미검증 | 향후 |
| Gated DeltaNet | OLMo Hybrid | O(T) | O(N) | O | LLM에서만 검증 | 2순위 |
| Self-Attention | TokenWM, ClimaX | O(T) | **O(N²)** | 시간축 X, 공간축 O | 중간 (Post-Norm 필요) | 재검토 필요 |
| Transformer Predictor | V-JEPA 2 | O(T²) | O(N) | O | block-causal 보완 | 비용 과다 |
| Neural ODE | Neural ODE | 가변 | O(N) | X | Stiff 문제 | 부적합 |

### 후보별 상세 평가

#### Block GRU (DreamerV3)

**작동 방식**: 이전 상태 h(t-1)과 현재 입력에서 reset/update 게이트로 상태 갱신. DreamerV3에서는 8개 독립 블록으로 분할 + RMSNorm + SiLU.

**우리 문제에서의 강점**:
- TF2에서 즉시 사용 가능. 구현 부담 최소
- h(t) snapshot이 단순 텐서 clone (Branch-in-CRS에 가장 간단)
- DreamerV3에서 Atari, DMC, Minecraft 등 광범위 검증

**우리 문제에서의 약점**:
- **학습 시 순차 처리 필수**: 575 step을 순서대로 통과. TBPTT로 32~64 step 윈도우로 끊을 수 있지만, 장기 의존성(수백 step Xe 축적 추세)을 놓칠 수 있음
- **고정 게이트 구조**: sigmoid(Wx + Uh + b)가 고정. 입력 성격에 따른 적응적 동작이 Mamba 대비 제한적
- DreamerV3 imagination은 15 step 규모. 575 step 안정성은 미검증

**총평**: 안전한 baseline 후보. "잘 모르겠으면 GRU부터 시작"도 합리적 전략이나, 병렬 학습 불가가 실질적 병목.

#### Mamba-2 / SSD (현재 1순위)

**작동 방식**: 매 시점 입력에서 B_t, C_t, Δ_t를 생성하여 상태 전이 수행. A를 스칼라로 단순화하여 recurrence를 구조화된 행렬 연산으로 변환. 학습 시 청크 단위 행렬곱 병렬 처리, 추론 시 1-step 순차.

**우리 문제에서의 강점**:
- **병렬 학습**: 575 step을 청크(32 step 등)로 나누어 병렬 처리. GRU 대비 2-8배 학습 속도 향상 기대
- **입력 의존 선택성**: p_load 급변 시 Δ_t 증가 → 빠른 상태 전환, 안정 운전 시 Δ_t 감소 → 천천히 갱신. 물리적으로 자연스러운 적응
- **중간 상태 접근**: 청크 경계에서 h(t) 저장 → Phase 2 Branch 학습에 detached h_t 제공 가능
- **Drama MBRL 검증**: ~7M params 세계모델에서 DreamerV3 대등. InferenceParams deepcopy로 hidden state 관리 확인
- **MNO PDE 검증**: 격자점별 독립 Mamba로 자기회귀 롤아웃에서 Transformer 대비 ~90% 오차 감소

**우리 문제에서의 약점**:
- **TF2 직접 구현 필요**: selective scan CUDA 커널을 TF2로 이식해야 함. 상당한 구현 부담
- **A 행렬 스칼라 단순화**: 대각 행렬(Mamba-1) 대비 표현력 감소. 복잡한 물리 동역학에서 병목 가능성을 완전히 배제할 수 없음
- **575 step 규모 미검증**: Drama 15 step, MNO 수십~수백 step. 575 step은 새로운 영역
- **OLMo Hybrid 경고**: LLM 통제 실험에서 Mamba-2 하이브리드 < 순수 Transformer. PDE 도메인으로의 일반화 여부 불확실

**총평**: 병렬 학습 + 입력 의존성 + 이중 도메인(MBRL+PDE) 검증이 강력한 근거. TF2 구현 부담과 575 step 미검증이 리스크.

#### Mamba-3 (향후 검토)

**작동 방식**: Mamba-2에 (1) Trapezoidal 이산화(O(Δ³) 정밀도), (2) 복소수 상태(위상 추적), (3) MIMO(다중 입출력)를 추가.

**우리 문제에서의 강점**:
- **Trapezoidal 이산화**: 현재+이전 입력을 모두 사용. Xe 축적처럼 부드러운 변화에 Euler(Mamba-2)보다 정확
- **복소 상태 위상 추적**: Xe 진동이 "올라가는 중/내려가는 중"을 상태에 인코딩 가능
- **Conv1D 대체**: Trapezoidal이 인접 시점 정보를 자동 포함 → Conv1D 제거로 구조 단순화

**우리 문제에서의 약점**:
- 2026년 3월 공개. MBRL/PDE 검증 **전무**
- 복소수 연산 TF2 구현 부담이 Mamba-2보다 더 큼
- MIMO의 이점이 cell-wise SISO 구조에서 제한적일 수 있음

**총평**: 이론적으로 가장 정교. Mamba-2 성능 한계 확인 시 전환 검토. 특히 이산화 오차 누적이 575 step에서 문제가 될 경우 Trapezoidal이 해법이 될 수 있음.

#### Gated DeltaNet (2순위 후보)

**작동 방식**: Mamba-2의 게이팅에 delta rule(연관 기억 갱신)을 결합. "기존 기억과 새 입력의 차이"를 기반으로 상태 갱신.

**우리 문제에서의 강점**:
- **OLMo Hybrid 통제 실험**: GDN hybrid > 순수 GDN > Transformer > Mamba-2 hybrid > 순수 Mamba-2. **7B 규모 통제 조건에서 Mamba-2를 명확히 상회**
- **Qwen3-Next(80B), Qwen3.5에서 채택**: 대규모 생산 모델이 Mamba-2 대신 GDN 선택
- **상태 추적 개선**: delta rule이 Xe 변화 추세 추적에 유리할 수 있음

**우리 문제에서의 약점**:
- **물리/PDE 도메인 검증 부재**: LLM 전용. "연관 기억"과 "물리 상태 전이"가 같은 과제인지 불확실
- TF2 구현 미확인. Mamba-2보다 참고 자료 부족

**총평**: LLM에서 Mamba-2를 상회하는 것은 무시할 수 없는 결과. 물리 도메인 검증이 선행되어야 채택 가능. **Mamba-2 다음의 2순위 후보**.

#### Self-Attention (TokenWM, ClimaX) — 재검토 필요

**작동 방식**: TokenWM은 토큰 집합 h_t에 대해 매 시점 Self-Attention + FiLM(행동) → 다음 시점 h_{t+1} 생성. ClimaX는 8-layer ViT Full Attention으로 공간 토큰 처리.

**이전 평가에서 간과한 점**: 복잡도 O(T²)로 기재했으나, 이것은 **오해**. TokenWM의 Self-Attention은 시간 축이 아닌 **공간 축(토큰 간)**에서 수행. 시점당 O(N²) = O(720²) = 518K ops이며, 시간 축 복잡도는 O(T)(자기회귀). 기상 모델(GraphCast 40K+ 격자, ClimaX)에서 이 규모는 문제 없이 처리됨. 720은 이보다 훨씬 작음.

**우리 문제에서의 강점**:
- **시간 전이 중 공간 결합을 직접 수행**: 디코더의 간접 gradient 전달에 의존하지 않고, 시간 전이 자체에서 cell 간 상호작용
- **결정적 잠재 상태**: TokenWM도 KL 없는 결정적 토큰 사용
- **기상/물리 ML에서 광범위 검증**: GraphCast, GenCast, ClimaX 등에서 Self-Attention이 물리장 예측의 핵심
- **Post-Norm 장기 안정성**: 장기 rollout에서 Pre-Norm보다 안정적이라는 보고

**우리 문제에서의 약점**:
- **시간 축 병렬 학습 불가**: 자기회귀이므로 GRU와 동일한 순차 처리 제약. Mamba-2의 parallel scan에 비해 학습 속도 열세
- **Memory Bank 필요**: 장기 기억 유지를 위한 추가 메커니즘. 구현 복잡도 증가
- **A안과의 역할 중복**: A안에서 디코더 FullAttn이 이미 공간 결합 수행. 시간 모델에서도 Self-Attention으로 공간 결합을 하면 중복. 단, 이 "중복"이 오히려 더 강한 공간 결합을 제공할 수 있음

**총평**: 이전에 O(T²) 비용으로 너무 빨리 탈락시킴. 시점당 O(720²)는 실용적 범위 내이며, 시간 전이 중 공간 결합을 직접 수행하는 것은 원리적으로 A안(디코더 간접 전달)보다 더 직접적. 그러나 시간 축 병렬 학습 불가가 실용적 제약. **C안 전환 시 가장 유력한 대안**.

#### Transformer Predictor (V-JEPA 2)

**작동 방식**: 24-layer Transformer에 block-causal attention. 과거 시점의 (관측+행동) 토큰에만 attend. 1-step + multi-step loss 혼합.

**우리 문제에서의 강점**:
- block-causal 인과성 보존
- 1-step + multi-step 혼합 loss로 부드러운 자기회귀 전환
- 62시간 데이터로 zero-shot planning 성공

**우리 문제에서의 약점**:
- **720 × 575 = 414K 토큰**: 전체 과거를 보는 causal attention의 KV 캐시가 선형 증가. 메모리와 계산 모두 비현실적
- V-JEPA 2는 1B 인코더 위에 predictor를 올림. 우리 인코더(780K)와 규모 차이

**총평**: 규모 문제로 직접 적용 부적합. 1-step + multi-step loss 혼합 아이디어만 차용.

#### Neural ODE

**작동 방식**: 연속 시간 ODE dh/dt = f_θ(h, t)를 신경망으로 모델링. ODE solver로 적분.

**우리 문제에서의 강점**:
- Bateman 방정식이 실제 ODE이므로 물리적으로 가장 자연스러운 접근
- 가변 시간 간격 처리, adjoint method 메모리 효율

**우리 문제에서의 약점**:
- **Stiff ODE 치명적**: Xe 반감기(~9h) vs 중성자 수명(~10⁻⁵s)의 시간 스케일 차이 ~10⁹. ODE solver가 극소 시간 스텝을 요구하여 계산 비용 폭발
- 병렬화 불가, 이산 제어 입력 결합 복잡

**총평**: Stiff 문제로 직접 적용 부적합. SSM(Mamba)이 Neural ODE의 구조화된 이산 근사라는 이론적 연결로 간접 활용.

### 현 시점 판단

**우선 후보군 (물리적 근거가 강한 2개)**:

- **Mamba-3** — 복소 상태의 회전 동역학이 Xe 진동의 위상 추적에 직접 대응. 이전 SSM은 상태를 "줄이거나 유지"만 할 수 있었지만, Mamba-3는 상태를 "회전시키며 구조적으로 기억"할 수 있다. Xe 농도가 올라가는 중인지, 내려가는 중인지, 피크 근처인지를 상태 벡터의 회전 각도로 자연스럽게 인코딩한다. 또한 Trapezoidal 이산화(O(Δ³))가 Xe 축적처럼 부드러운 변화에 대해 Euler(Mamba-2)보다 정밀. **리스크**: 2026년 3월 공개로 MBRL/PDE 검증 부재, TF2 복소수 연산 구현 부담.

- **Gated DeltaNet** — delta rule(차분 기반 갱신)이 수치해석의 차분법과 원리적으로 유사. "현재 상태와 목표의 차이(delta)"를 기반으로 상태를 갱신하는 방식이, 물리량의 시간 변화율(dN_Xe/dt, dT/dt 등)을 학습하는 데 자연스럽다. OLMo Hybrid 통제 실험에서 Mamba-2를 명확히 상회. **리스크**: 물리/PDE 도메인 검증 부재, TF2 구현 미확인.

**실용 후보 (검증 근거가 강한 1개)**:

- **Mamba-2 (SSD)** — 병렬 학습(chunked matmul) + Drama MBRL 검증 + MNO PDE 검증의 이중 도메인 검증이 다른 후보에 없는 고유 강점. 구현 참고 자료가 가장 풍부. 우선 후보군의 검증이 완료되기 전까지 **실용적 출발점**으로 유효.

**대안 후보**:

- **Self-Attention (TokenWM 패턴)** — C안 전환 시 가장 유력. 시간 전이 중 공간 결합을 직접 수행. 시점당 O(720²)는 실용적 범위 내
- **Block GRU** — baseline 대조군. 가장 안전한 구현, 성능 비교 기준선

**향후 결정 방향**: Mamba-3와 GDN의 TF2 구현 가능성 및 물리 도메인 적합성을 추가 조사한 후, 최종 선택을 결정한다. 현 시점에서는 세 후보(Mamba-3, GDN, Mamba-2)를 열어두되, 각각의 물리적 적합성 근거를 구체화하는 것이 우선이다.

## 2. 상태 표현 (State Representation)

### 우리 문제의 상태 표현 요구사항

- 노심 3차원 구조 (20 axial layers × 6×6 halo grid) = 720 cell의 공간 위치가 물리적으로 고정
- 각 cell의 상태: 10ch (power, Xe-135, I-135, flux, 온도 등) → 인코더가 128D로 변환
- 시간 모델(Mamba)에서 각 cell의 h(t)로 사용되므로, 상태 표현이 곧 Mamba의 입력 단위를 결정
- 결정론적 시뮬레이터 → 확률적 잠재 변수 불필요
- 공간 구조 보존 필수: cell 간 물리 관계(중성자 확산, 열전달)를 디코더/Physical Loss가 처리

### 후보 기법 비교

| 기법 | 소스 | 구조 | 확률/결정 | 우리 채택 |
|------|------|------|----------|----------|
| 단일 벡터 h_t | DreamerV3, Drama | 1×D | 결정적(h) + 확률적(z) | X |
| **토큰 집합 N×D** | TokenWM | N_h×D | **결정적** | **O (720×128)** |
| 복셀 격자 | Tesla Occupancy | 3D 격자 | 결정적 | O (20×6×6 = 유사) |
| 이산 토큰 (VQ-VAE) | Cosmos, GR00T | 코드북 인덱스 | 이산 | X |
| Cat(32×32) 이산 잠재 | DreamerV3 | 32×32 | 확률적 | X |

### 후보별 상세 평가

#### 단일 벡터 h_t (DreamerV3, Drama)

**작동 방식**: 전체 관측(이미지 또는 상태 벡터)을 CNN/MLP로 인코딩한 뒤, 하나의 벡터 h_t ∈ R^D로 압축한다. DreamerV3는 h_t(결정적, Block GRU) + z_t(확률적, Cat(32×32))의 이중 구조, Drama는 Mamba-2의 내부 SSM state가 h_t 역할.

**우리 문제에서의 강점**:
- 시간 모델이 단일 벡터 하나만 갱신하면 되므로 계산이 단순
- h_t가 전체 시스템의 "요약"이므로, 전역 상태(keff 추세, AO 등)가 자연스럽게 포함

**우리 문제에서의 약점**:
- 720 cell × 10ch = 7200차원의 공간 분포를 하나의 벡터로 압축하면, "어느 cell이 어떤 상태인지"라는 공간 세부 정보가 소실된다. 노심 상단과 하단의 Xe 분포 차이, 제어봉 삽입 위치의 국소 효과 등이 벡터 하나에 녹아 구별이 어려워진다.
- DreamerV3/Drama의 관측 규모(64×64 이미지 또는 수십 차원 벡터)와 우리 720 cell은 정보 밀도가 본질적으로 다르다. Atari 화면은 배경 대부분이 중복이지만, 노심의 720 cell은 각각이 물리적으로 구별되는 독립 위치다.

**총평**: 우리 문제에는 **부적합**. 공간 세부 정보를 보존해야 하므로 단일 벡터 압축은 과도한 정보 손실.

#### 토큰 집합 N×D (TokenWM → 우리 720×128) — 채택

**작동 방식**: 상태를 하나의 벡터가 아니라 **N개 토큰의 집합** h_t ∈ R^(N×D)로 표현한다. TokenWM에서는 N_h개의 학습된 토큰이 관측의 서로 다른 측면을 담당한다. 우리의 경우 인코더 출력 (720, 128)이 이것에 직접 대응: 각 cell이 하나의 토큰이 되고, 128차원이 그 cell의 잠재 표현이 된다.

**우리 문제에서의 강점**:
- **공간 해상도 완전 보존**: 720 cell 각각이 독립 토큰이므로, "cell (5, 3, 2)의 상태"를 개별적으로 추적 가능. 단일 벡터처럼 공간 정보가 뭉개지지 않음
- **물리적 의미 고정**: TokenWM의 토큰은 의미가 학습에 의해 자유롭게 결정되지만, 우리 토큰은 LAPE + STRING에 의해 물리적 3D 위치에 고정됨. 해석 가능성이 높고, 디코더가 공간 재구성에 활용 가능
- **cell-wise Mamba와 자연스러운 결합**: 720 토큰 각각이 독립적으로 Mamba를 통과하며 시간 전이. 가중치 공유로 효율적
- **결정론적**: TokenWM도 KL 없이 결정적 토큰 사용. 우리 문제(결정론적 시뮬레이터)에 정확히 부합
- **학술적 정당화**: TokenWM이 "토큰화된 잠재 상태"의 우월성을 Atari/DMC에서 입증. 우리가 의식하지 못한 채 이 패턴을 이미 구현하고 있었음

**우리 문제에서의 약점 — 핵심 우려 3가지**:

**(a) Gradient는 교정 신호이지 정보 신호가 아니다**

Cell-wise 독립 Mamba에서 각 cell의 h(t)는 자기 과거만 기억한다. 이웃 cell의 상태를 직접 참조하지 않으므로, "cell B의 Xe가 올라가고 있으니 cell A의 중성자속이 영향받을 것"이라는 추론이 Mamba 내부에서는 불가능하다.

이에 대한 현재 대응(A안)은: 디코더 FullAttn + Physical Loss의 gradient가 역전파를 통해 "cell A의 출력이 cell B의 물리 정합성에 미치는 영향"을 Mamba에 간접 전달한다는 것이다. 원리적으로 맞지만, 이 gradient는 **"네 출력이 틀렸다, 조정해라"**(교정)라고 말할 뿐, **"이웃 cell이 이런 상태이니 이 방향으로 예측해라"**(정보)라고 말하지는 못한다. 충분한 학습 데이터와 반복을 거치면 Mamba가 이 간접 신호만으로 올바른 패턴을 내재화할 수 있지만, 학습 효율과 수렴 속도에서 직접 공간 결합(C안)보다 불리할 수 있다.

**(b) Cell 간 물리적 연동**

원자로 노심에서 cell의 상태 변화는 독립적이지 않다. 한 cell의 Xe 농도가 변하면 해당 cell의 거시단면적이 바뀌고, 이는 중성자 확산 방정식을 통해 **인접 cell의 중성자속에 즉각 영향**을 준다. 열전달도 마찬가지로 인접 cell 간에 결합되어 있다. 대표적인 예가 Xe 축방향 진동인데, 상단 Xe 증가 → 상단 중성자속 감소 → 하단으로 출력 이동 → 하단 Xe 증가의 연쇄가 cell 간 확산 결합 때문에 발생한다.

Cell-wise 독립 Mamba에서 각 cell은 자기 h(t)만 보고 다음 상태를 예측한다. "옆 cell에서 Xe가 올라가고 있으니, 내 중성자속이 영향받을 것"이라는 추론이 Mamba 내부에서는 불가능하다. 인접 cell의 시간적 추세를 직접 참조할 수 없으므로, 이러한 cell 간 연동 관계를 각 cell이 독립적으로 맞출 수 있는지가 불확실하다.

완화 요인: 인코더 FullAttn이 매 시점 z(t)에 전역 공간 정보를 인코딩하므로, Mamba 입력에 "현재 노심 전체의 상태 분포"가 반영된다. Mamba가 이 입력에서 인접 cell과의 관계를 읽어내는 법을 학습할 수 있다. 다만 이것은 "현재 시점의 스냅샷"이며, "인접 cell의 시간적 변화 추세"는 포함하지 않는다는 한계가 있다.

**(c) 학습 → 추론 격차 (exposure bias)**

학습 중 teacher forcing 구간에서는 z(t)가 GT에서 인코딩되므로 공간 정보가 정확하다. 그러나 추론 시에는 이전 step의 예측값 → 인코더 재입력으로 z(t)가 생성되므로, 공간 정보 자체에 오차가 누적된다.

Cell-wise 독립 Mamba의 각 cell이 독립적으로 미세하게 다른 방향으로 예측 오차를 만들면, 인코더에 재입력되는 상태는 "이미 공간적으로 어긋난 상태"가 된다. 인코더 FullAttn이 매 step 공간 관계를 재정렬하지만, 575 step에 걸쳐 이 재정렬 능력이 오차 누적을 상쇄하기에 충분한지는 실험으로만 확인 가능하다.

이 우려에 대한 기존 대응: scheduled sampling(학습 중 점진적으로 자기회귀 비율 증가)과 pushforward trick(k-step unroll gradient)이 학습-추론 격차를 줄이는 역할을 한다. 5번 항목(롤아웃 안정화)에서 상세히 다룬다.

**세 우려의 공통 근원**: cell-wise 독립 Mamba가 "이웃을 직접 보지 못하는" 구조적 한계를, gradient 교정과 인코더 재입력이라는 간접 경로만으로 보상할 수 있는가? "원리적으로 가능"과 "575 step에서 실제로 충분"은 다른 문제이며, 이 간극이 A안 → C안 전환 트리거를 정의해둔 이유이다 (아키텍처 유효성 조사 §7.3 참조).

**총평**: 720×128 토큰 집합은 우리 문제에 **가장 적합한 상태 표현**. 공간 해상도 보존, 결정론적 구조, cell-wise Mamba와의 자연스러운 결합이 강점. 위 세 가지 우려는 상태 표현 자체의 문제라기보다 **cell-wise 독립 처리의 한계**이며, 이는 시공간 결합(4번 항목)과 롤아웃 안정화(5번 항목)에서 다루어야 할 과제이다.

#### 복셀 격자 (Tesla Occupancy Network)

**작동 방식**: 차량 주변 3D 공간을 복셀로 나누고, 각 복셀의 점유 여부/의미 클래스/속도를 예측한다. Triplanar feature로 3D 정보를 XY/XZ/YZ 세 평면으로 분해하여 메모리를 절약한다.

**우리 문제에서의 강점**:
- 우리 (20, 6, 6) 격자와 **구조적으로 동일**: 3D 공간을 이산 셀로 나누고, 셀마다 물리량을 부여한 구조
- Tesla가 대규모 3D 복셀을 실시간으로 처리한다는 사실은, 우리 720 cell 규모가 충분히 다룰 만한 크기임을 간접 확인

**우리 문제에서의 약점**:
- Tesla의 복셀은 "점유 여부"라는 이산적 판단이 중심이고, 우리 cell은 "연속 물리량 10ch"의 정밀 예측이 필요. 표현의 목적이 다름
- Triplanar 분해는 해상도가 높은 격자에서 메모리 절약 효과가 크지만, 20×6×6 규모에서는 불필요

**총평**: 3D 격자 기반 상태 표현이라는 점에서 **구조적 유사성을 확인하는 참고 사례**. 직접 차용할 기법은 없으나, "3D 복셀 = 토큰 집합"이라는 등가 관계가 우리 설계를 뒷받침.

#### 이산 토큰 VQ-VAE (Cosmos, GR00T) — 부적합

**작동 방식**: 연속 잠재 벡터를 고정 크기 코드북에서 가장 가까운 벡터로 양자화한다. 잠재 상태가 코드북 인덱스(정수)로 표현되므로, 이산 Transformer의 next-token prediction으로 시간 모델링이 가능하다.

**부적합 사유**:
- 우리 물리량(중성자속, Xe 농도, 온도 등)은 **연속값의 미세한 차이**가 물리적으로 중요하다. 1% 출력 차이, 0.5ppm 붕소 농도 차이가 의미 있는 수준에서 작동하는데, 코드북 양자화는 이 정밀도를 보장하지 못한다.
- 코드북 크기를 극단적으로 키우면 정밀도가 올라가지만, 학습과 메모리 비용이 급증하며, 이산 토큰화의 본래 이점(단순한 next-token 학습)이 사라진다.
- Cosmos/GR00T는 비디오/이미지를 다루므로 VQ-VAE의 약간의 정보 손실이 인간 시각에는 무시되지만, 물리 시뮬레이션에서는 허용 불가.

#### Cat(32×32) 이산 잠재 (DreamerV3) — 부적합

**작동 방식**: 32개 카테고리 분포 × 32 클래스 = 1024차원의 확률적 잠재 상태. Straight-through gradient로 이산 샘플링의 역전파 문제를 해결. KL balancing(0.5 free nats)으로 Prior-Posterior 균형 유지.

**부적합 사유**:
- **확률적 잠재가 불필요**: 우리 문제는 결정론적 시뮬레이터. 동일한 입력 조건(초기 상태 + 제어 입력)에서 출력이 유일하게 결정됨. 관측에 노이즈가 있거나 환경이 확률적인 RL 문제에서는 Prior-Posterior 분리가 유용하지만, 우리에게는 불필요한 복잡성.
- **KL balancing 비용**: 확률적 잠재를 사용하면 KL 발산 손실을 추가로 관리해야 하고, 이 균형이 깨지면 posterior collapse(Prior만 사용하게 되는 현상)가 발생. DreamerV3가 이를 해결하기 위해 사용한 free nats, KL balancing 비율 조절 등은 우리 문제에서 불필요한 튜닝 부담.
- 1024차원 잠재가 720 cell × 10ch = 7200차원의 공간 분포를 충분히 표현하기에 **차원이 부족**할 수 있음.

### Global token 검토

720 cell token으로 상태를 표현할 때, keff 추세, 전체 Xe 축적량, Axial Offset(AO) 같은 **시스템 수준 요약 정보**를 어떻게 유지할 것인가의 문제가 있다. Cell-wise 독립 Mamba에서는 이러한 전역 정보가 720개 cell의 h(t)에 **분산 저장**되며, 명시적인 전역 상태 토큰은 없다.

#### 접근 A: 디코더가 암묵적으로 처리 (baseline)

디코더 FullAttn이 720 cell을 전부 보므로, 전역 통계량은 디코더 내부에서 자연스럽게 집계된다. 별도 전역 토큰 없이도 keff나 AO를 예측할 수 있다.

- 장점: 추가 구현 없음. 기존 A안 그대로
- 단점: Mamba 시간 전이 중에는 전역 정보가 부재. "전체 노심 출력이 올라가는 추세"를 개별 cell이 자기 h(t)에서 각각 독립적으로 추적해야 하며, cell 간에 이 추세 인식이 일치하리라는 보장이 없음

#### 접근 B: Global token 추가 (TokenWM 근거)

인코더에서 720 cell token 외에 4~8개의 학습 가능한 Global token을 추가한다. FullAttn에서 Global token이 전체 cell 정보를 집약하고, Mamba에서도 cell token과 함께 시간 전이한다.

```
인코더: [720 cell token + 8 global token] → FullAttn → [728 token × 128D]
Mamba:  728 token 각각 독립 시간 전이 (가중치 공유)
디코더: global token → 전역 정보 공급, cell token → 공간 재구성
```

- 장점: 최소 비용(8 token 추가, 1% 미만 연산 증가)으로 시스템 수준 시간 추세를 명시적으로 유지. TokenWM의 학습된 토큰 개념과 유사
- 단점: Global token이 어떤 전역 정보를 학습할지 보장 없음 (인코더 FullAttn의 학습에 의존). 또한 cell-wise Mamba에서 global token도 독립 처리되므로, global token과 cell token 간 시간 전이 중 정보 교환은 불가

#### 접근 C: 명시적 통계량 주입

Mamba 출력 또는 인코더 출력에서 mean/std/max 등 통계량을 계산하여, 각 cell의 Mamba 입력에 concat한다.

- 장점: 전역 정보가 "계산된 사실"이므로 학습 의존성 없음
- 단점: 어떤 통계량이 유용한지 사전에 결정해야 함 (도메인 지식 필요). Gradient가 통계량 계산을 통과하므로 미분 가능성 확인 필요

#### 권장

접근 A(baseline, 디코더에 위임)로 시작하되, keff 또는 AO 예측 정밀도가 부족할 경우 접근 B(Global token)를 추가한다. 접근 B는 구현 비용이 매우 낮으므로(인코더에 `nn.Parameter(8, 128)` 추가 + Mamba 입력 차원 변경 없음), 실험적으로 효과를 확인하기 용이하다.

### 현 시점 판단

**채택**: 720 cell token × 128D (결정적, 물리적 위치 고정)

이 표현은 세계모델 문헌에서 TokenWM이 제안한 "토큰 집합 잠재 상태"와 구조적으로 동일하며, Tesla의 3D 복셀 격자와도 대응된다. 확률적 잠재(DreamerV3)와 이산 토큰(Cosmos)은 결정론적 물리 시뮬레이션이라는 우리 문제의 성격에 부적합하다.

**핵심 과제는 상태 표현 자체가 아니라, 이 표현에서 파생되는 cell-wise 독립 처리의 한계를 어떻게 보상할 것인가이다.** 세 가지 우려(gradient 교정의 한계, Xe 파동 위상 동기화, 학습-추론 격차)는 4번 항목(시공간 결합)과 5번 항목(롤아웃 안정화)에서 구조적 대응 방안과 함께 평가한다.

**Global token**: baseline에서는 디코더에 위임(접근 A). keff/AO 정밀도 부족 시 Global token 추가(접근 B, 저비용 실험).

## 3. 제어/조건 주입 (Action/Condition Injection)

### 후보 기법 비교

| FiLM | TokenWM | scale+shift 변조 | X | O | 보류 |

## 4. 시공간 결합 (Spatiotemporal Coupling)

### 후보 기법 비교

| 전략 | 소스 | 메커니즘 | 비용 | 우리 채택 |
|------|------|---------|------|----------|
| **완전 분리 (공간 → 시간 → 공간)** | DreamerV3, Drama, **우리 A안** | Encoder Attn + Mamba + Decoder Attn | 효율적 | **1순위** |
| 시간 중 공간 결합 | TokenWM, ClimaX | Self-Attention during temporal | O(N²T) | C안 (보류) |
| GNN message-passing | GraphCast, MeshGraphNets | 그래프 상 정보 전파 | O(E·T) | X |
| Fourier 전역 혼합 | FNO, FourCastNet | FFT 기반 | O(N log N · T) | X |
| Factorized Attention | Tesla, V-JEPA 2 | 공간 Attn + 시간 Attn 분리 | 중간 | 우리와 유사 |

**결정**: A안 (완전 분리). 인코더/디코더 FullAttn이 공간 결합을 담당하고, Physical Loss gradient가 Mamba에 공간 정보를 간접 전달.

**A안의 물리적 정당화**:
1. Xe/온도의 지역 동역학 → Mamba(cell-wise)가 적합
2. 중성자 확산 결합(cell 간) → 디코더 FullAttn + L_diff_rel이 담당
3. Gradient 역전파: Physical Loss → Decoder(FullAttn) → Mamba로 공간 결합 정보 전달
