# GR00T N1 — VLA Dual-System 로봇 제어

> **출처**: NVIDIA (2025). *GR00T N1: An Open Foundation Model for Generalist Humanoid Robots.* [arXiv:2503.14734](https://arxiv.org/abs/2503.14734)
> **공개 수준**: 부분공개 (논문 공개, 2.2B params)
> **우리 아키텍처 맥락**: **Flow matching** 기반 연속 행동 생성, **Dual-system 아키텍처** (느린 추론 + 빠른 반응)

---

## 1. 시스템 개요

### 직관

GR00T N1은 "생각하는 뇌(System 2) + 반사하는 척수(System 1)"의 이중 시스템이다:
- **System 2 (VLM, 10Hz)**: 이미지+언어를 이해하고, 고수준 의도를 파악
- **System 1 (DiT, 120Hz)**: 의도를 실시간 관절 명령으로 변환

### 아키텍처 (2.2B params)

```
[Camera + Language] → [System 2: Eagle-2 VLM (1.34B)]
                         ↓ 64 image token embeddings (중간층 추출)
                      [System 1: DiT (action generation)]
                         ↓ Cross-Attention(VLM tokens) + Self-Attention(action tokens)
                         ↓ Flow matching denoising (K=4 steps)
                      [Action chunk: 16 future actions]
```

## 2. 기능별 상세 분석

### 2.1 시간 모델링 — Flow Matching

**직관**: 확산(Diffusion)이 "노이즈에서 천천히 깨끗한 이미지를 복원"하는 것이라면, Flow matching은 "노이즈에서 목표까지의 **직선 경로**를 따라가는" 더 효율적인 방법이다.

```
Flow matching loss:
L = E[ ||V_θ(φ_t, A_t^τ, q_t) - (ε - A_t)||² ]

노이즈 혼합: A_t^τ = τ·A_t + (1-τ)·ε    (선형 보간)
추론: 4 step Forward Euler 적분
```

- **확산 대비 이점**: 4 step만에 생성 가능 (확산은 50-1000 step)
- 연속값 출력 (이산화 불필요)
- 다중 모드 분포 처리 가능

**우리 문제에서의 시사점**: 우리는 자기회귀 Mamba로 상태를 예측하므로 flow matching이 직접 필요하지는 않다. 그러나 "연속값 예측을 위한 효율적 생성 모델"로서, 디코더에서 물리량을 생성할 때의 대안으로 검토 가능. 단, 우리 디코더는 결정론적 매핑이므로 확률적 생성이 불필요.

### 2.2 상태 표현 — VQ-VAE 통합 잠재 행동 공간

**직관**: 다양한 로봇(인간형, 팔, 이동)의 행동을 하나의 잠재 공간으로 통합.

```
[현재 프레임 x_t] + [미래 프레임 x_{t+H}]
    → [VQ-VAE Encoder] → 잠재 행동 z_t
    → [VQ-VAE Decoder] → 복원된 행동
```

**우리 문제에서의 시사점**: 우리도 다양한 LP(100개)와 프로파일(20개)의 데이터를 통합 학습한다. VQ-VAE로 "운전 조건 잠재 공간"을 학습하는 것은 흥미롭지만, 현재는 LP별 xs_fuel을 입력에 직접 제공하는 것으로 충분.

### 2.3 제어/조건 주입 — Cross-Attention + 실체 특화 MLP

```
VLM 토큰 φ_t → Cross-Attention → DiT에 조건 주입
관절 상태 q_t → 실체별 MLP → 공유 임베딩 차원으로 투영
행동 + diffusion timestep → Action Encoder MLP → DiT 입력
```

- **Cross-Attention**: 고수준 의도(VLM 출력)를 행동 생성에 주입
- **실체별 MLP**: 로봇 종류에 따라 다른 관절 구조를 공유 차원으로 매핑
- **AdaLN**: diffusion timestep 조건 주입

**우리 문제 대응**:
| GR00T | 우리 |
|---|---|
| VLM 토큰 (고수준 의도) | - (불필요) |
| 관절 상태 q_t (실체별 MLP) | sym_type → ConditionalLAPE (대칭별 PE 선택) |
| Diffusion timestep (AdaLN) | p_load (AdaLN-Zero in decoder) |

### 2.4 학습 전략 — 데이터 피라미드

```
              [실제 로봇 데이터: 88h]
           [신경 궤적 확장: 827h (×10)]
        [시뮬레이션: 780K 궤적 (6,500h)]
     [Open X-Embodiment (다중 로봇)]
  [인간 비디오 (Ego4D, EPIC-KITCHENS)]
```

**우리 문제 대응**: 우리도 유사한 데이터 피라미드를 구성할 수 있다:
- 최상위: MASTER 해석 결과 (고정밀, 제한량)
- 중간: 데이터 증강 (노이즈 추가, 보간)
- 하위: 단순화된 물리 모델 시뮬레이션 (저정밀, 대량)

## 3. 우리 문제 적용성 평가

| 기법 | 적용 가능 | 근거 |
|------|----------|------|
| **Flow matching** | X | 결정론적 예측 문제. 확률적 생성 불필요 |
| VQ-VAE 통합 잠재 공간 | △ | 다중 LP 통합 학습에 참고. 현재 xs_fuel 직접 입력으로 충분 |
| **Dual-system (느린+빠른)** | △ | 개념적 참고: "공간 이해(인코더)" + "빠른 시간 전이(Mamba)"로 대응 |
| Cross-Attention 조건 주입 | △ | 디코더에서 인코더 feature를 cross-attention으로 참조하는 skip connection 검토 가능 |
| 데이터 피라미드 | △ | 데이터 증강 전략 참고 |
| Action chunking (H=16) | X | 우리는 1-step 자기회귀. 청크 예측 불필요 |

### 핵심 차용
- **Cross-Attention skip connection**: GR00T의 VLM→DiT cross-attention 패턴을 참고하여, 인코더 중간 feature → 디코더 cross-attention으로 전달하는 skip connection 검토. 이는 디코더가 인코더의 공간 정보를 직접 참조할 수 있게 함
