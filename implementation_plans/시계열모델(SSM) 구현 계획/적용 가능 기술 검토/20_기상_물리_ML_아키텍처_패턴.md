# 기상/물리 ML의 "인코더-프로세서-디코더" 아키텍처 패턴

> **작성일**: 2026-04-15
> **목적**: 기상/기후/물리 시뮬레이션 ML에서 지배적인 아키텍처 패턴을 조사하고, 우리 노심 대리모델에 대한 적용성을 평가
> **공개 수준**: 완전공개 (모든 소스가 학술 논문)
> **핵심 발견**: 7개 모델 중 6개가 "비대칭 설계 (잠재 공간에 연산 집중 + 경량 디코더)" 채택

---

## 1. 시스템 개요

### 직관

기상 예측은 우리 문제와 구조적으로 매우 유사하다:
- **격자 위의 물리장(physical field)** 을 다음 시점으로 전이
- **공간적으로 결합된 동역학** (기압 변화가 풍속에 영향 ↔ Xe 변화가 flux에 영향)
- **자기회귀 롤아웃** (6시간 단위 예측을 반복)
- **물리 법칙 존재** (대기 역학 ↔ 중성자 확산/Bateman ODE)

기상 ML은 이 문제를 **인코더-프로세서-디코더** 3단 구조로 해결하며, 핵심 연산(공간 결합 + 시간 전이)을 **프로세서(잠재 공간)**에 집중시키는 것이 지배적 표준이다.

## 2. 개별 모델 상세

### 2.1 GraphCast (DeepMind, Science 2023)

> Lam et al. *Learning skillful medium-range global weather forecasting.* [arXiv:2212.12794](https://arxiv.org/abs/2212.12794)

**아키텍처**:
```
[위도×경도 격자] → [Encoder: 1 GNN layer] → [다중해상도 정이십면체 메시]
    → [Processor: 16 GNN layers (message-passing)] → [Decoder: 1 GNN layer] → [예측]
```

| 구성 요소 | 구조 | 역할 |
|---|---|---|
| **인코더** | GNN 1 layer. Grid → 40,962 icosahedral nodes. MLP(512) | 물리 변수 → 잠재 표현 매핑 |
| **프로세서** | **GNN 16 layers (비공유)**. 다중해상도 메시 위 message-passing. 6단계 정이십면체 세분화로 장거리 연결 포함 | 공간 결합 수행. 각 노드가 이웃 노드의 정보를 수집·갱신. 16 round의 반복으로 전역 정보 전파 |
| **디코더** | GNN 1 layer. Mesh → Grid. **잔차(residual) 예측** | 잠재 표현 → 물리 변수 투영 |

**핵심 설계 결정**:
- **비율 1:16:1**: 전체 연산의 ~94%가 프로세서에 집중
- 디코더는 **잔차 예측**: `state(t+1) = state(t) + decoder_output`. 전체 상태가 아닌 변화량만 예측
- 입력: 2개 시점 (t-6h, t) → 출력: 1개 시점 (t+6h). 자기회귀로 10일 예측

**우리 문제와의 대응**:
- 40,962 icosahedral nodes ↔ 720 cell token (우리가 55배 작음 → Attention 비용 문제 없음)
- Message-passing ↔ Attention (둘 다 노드/토큰 간 정보 교환)
- 잔차 예측 ↔ 우리도 `state(t+1) = state(t) + delta` 형태 검토 가능

### 2.2 FourCastNet (NVIDIA, 2022)

> Pathak et al. *FourCastNet: A Global Data-driven High-resolution Weather Forecasting Model.* [arXiv:2202.11214](https://arxiv.org/abs/2202.11214)

**아키텍처**:
```
[720×1440 격자] → [Patch Embedding: Conv2d] → [90×180 토큰, 768D]
    → [AFNO Block × 12] → [Linear Projection] → [예측]
```

| 구성 요소 | 구조 | 역할 |
|---|---|---|
| **인코더** | Patch embedding 1개 (patch_size=8, embed_dim=768). Conv2d 1회 | 격자 → 토큰 변환 |
| **프로세서** | **AFNO 12 layers**. 각 layer: FFT → 학습된 전역 필터 → IFFT → Channel MLP. O(N log N) 복잡도 | 주파수 공간에서 **전역 공간 혼합**. 모든 격자점이 한번에 상호작용 |
| **디코더** | **Linear projection 1개**. 768D → patch_size² × 변수 수 | 단일 행렬곱 |

**핵심 설계 결정**:
- **AFNO(Adaptive Fourier Neural Operator)**: FFT 기반 공간 혼합으로 O(N log N) 복잡도 달성. 720×1440 = 1,036,800 격자점에서도 실용적
- **비율 1:12:1**: 디코더가 문자 그대로 Linear 하나

**우리 문제에 대한 시사점**:
- AFNO의 Fourier 혼합은 우리 720 token에 대한 Attention(O(N²))보다 효율적이나, 720은 충분히 작아 직접 Attention이 가능
- 인코더가 극히 간단(Conv2d 1개)한 것은, **모든 공간 처리를 프로세서에 위임**하는 설계

### 2.3 GenCast (DeepMind, Nature 2024)

> Price et al. *GenCast: Diffusion-based ensemble forecasting for medium-range weather.* [arXiv:2312.15796](https://arxiv.org/abs/2312.15796)

**아키텍처**: GraphCast의 후속작. GNN을 **Sparse Transformer**로 교체.

| 구성 요소 | 구조 |
|---|---|
| **인코더** | 1 GNN layer (GraphCast와 동일) |
| **프로세서** | **16 sparse transformer blocks**, 512D, 4-head. 각 노드는 **16-hop 이웃**에만 attend (전역이 아닌 지역 attention) |
| **디코더** | 1 GNN layer |

**GraphCast 대비 차이**:
- GNN → Sparse Transformer: 더 유연한 정보 교환 패턴
- 확산(Diffusion) 기반: 앙상블 예측 가능 (불확실성 정량화)
- 비율은 동일 (1:16:1)

### 2.4 ClimaX (Microsoft, ICML 2023)

> Nguyen et al. *ClimaX: A foundation model for weather and climate.* [arXiv:2301.10343](https://arxiv.org/abs/2301.10343)

**아키텍처**: 기상 Foundation Model. 변수별 독립 토큰화 + 교차 어텐션 집약.

| 구성 요소 | 구조 |
|---|---|
| **인코더** | 변수별 Patch Embedding + **Cross-Attention 집약** (학습된 query가 V개 변수 임베딩에 attend → 통합) |
| **프로세서** | **8 ViT Self-Attention layers**, d=1024, FFN hidden=4096. 전체 공간 토큰에 대한 Full Attention |
| **디코더** | **2-layer MLP** (hidden 1024). 변수별 독립 head |

**핵심 설계 결정**:
- **변수별 토큰화**: 기온, 기압, 풍속 등 각 변수를 별도로 패치 임베딩. 이후 Cross-Attention으로 통합
- Lead time embedding: 예측 시간 간격을 임베딩으로 주입 (우리 p_load 주입과 유사)
- 비율 ~4:8:1

**우리 문제에 대한 시사점**:
- 변수별 토큰화: 우리 21ch 입력의 각 물리량(power, Xe, flux 등)을 별도로 임베딩 후 통합하는 방안 검토 가능
- Lead time embedding ↔ p_load / sym_type conditioning: 전역 조건을 임베딩으로 주입하는 동일 패턴

### 2.5 MeshGraphNets / GNS (DeepMind, ICLR 2021 / ICML 2020)

물리 시뮬레이션(유체, 천 시뮬레이션 등)의 학습 프레임워크. 기상 모델들의 기반 아키텍처.

| 구성 요소 | GNS | MeshGraphNets |
|---|---|---|
| **인코더** | 1 MLP (2-hidden, 128D) | 2-layer MLP (128D) |
| **프로세서** | **10 msg-passing rounds** | **15 msg-passing rounds** |
| **디코더** | 1 MLP | 2-layer MLP |

- **가장 원형적인 "인코더-프로세서-디코더" 구조**
- 프로세서의 각 message-passing round가 "한 번의 정보 전파"에 해당
- 10-15 round면 그래프 전체에 정보가 전파됨

### 2.6 Pangu-Weather (Huawei, Nature 2023) — 예외 사례

> Bi et al. *Accurate medium-range global weather forecasting with 3D neural networks.* [arXiv:2211.02556](https://arxiv.org/abs/2211.02556)

**유일한 대칭 설계 (U-Net 패턴)**:
```
인코더: 2 full-res layers + 6 downsampled layers
                                                    ← skip connection
디코더: 6 upsampled layers + 2 full-res layers
```

- Earth-Specific Transformer: Shifted window attention을 3D(고도×위도×경도)로 확장
- Skip connection: 인코더 2번째 layer → 디코더 7번째 layer로 직접 연결
- **이 설계가 나머지 6개와 다른 이유**: U-Net 구조는 **다중 해상도 처리**에 특화. 고해상도 정보(지표면)와 저해상도 정보(상층 대기)를 모두 활용해야 하는 기상 특성에 맞춤

**우리 문제와의 관련**: 우리 격자는 단일 해상도(20×6×6 halo)이므로 다중 해상도 U-Net의 이점이 적음. 비대칭 설계가 더 적합.

## 3. 종합 비교

### 3.1 아키텍처 패턴 비교표

| 모델 | 인코더 | 프로세서 | 디코더 | 공간 결합 방식 | 비율 |
|------|--------|---------|--------|-------------|------|
| GraphCast | 1 GNN | 16 GNN | 1 GNN | Message-passing | 1:16:1 |
| FourCastNet | 1 Conv | 12 AFNO | 1 Linear | Fourier 혼합 | 1:12:1 |
| GenCast | 1 GNN | 16 Sparse Attn | 1 GNN | 16-hop sparse attn | 1:16:1 |
| ClimaX | Cross-attn | 8 Full Attn | 2 MLP | Full self-attention | ~4:8:1 |
| MeshGraphNets | 2 MLP | 15 msg-passing | 2 MLP | Message-passing | 1:15:1 |
| GNS | 1 MLP | 10 msg-passing | 1 MLP | Message-passing | 1:10:1 |
| Pangu-Weather | 8 layers | U-Net | 8 layers | Window Attn | 1:1 (예외) |

### 3.2 우리 문제에 대한 시사점

**확인된 패턴**:
1. 핵심 연산(공간 결합 + 시간 전이)은 **프로세서(잠재 공간)**에 집중
2. 디코더는 잠재 표현 → 물리 변수 투영 (후보정 성격)
3. 프로세서:디코더 비율 = 8:1 ~ 16:1

**우리 아키텍처 적용**:
- 현재 설계: 인코더 3-stage Attn, Mamba (프로세서 해당), 디코더 미정
- C안 적용 시: Mamba+Attention이 프로세서 역할, 디코더를 1-2 layer로 경량화
- 기상 모델 패턴과의 정합성이 높음

## 4. 우리 문제 적용성 평가

| 기법 | 적용 가능 | 근거 | 구현 난이도 |
|------|----------|------|-----------|
| **비대칭 설계 (경량 디코더)** | O | 6/7 모델에서 채택. 프로세서에 연산 집중이 표준 | 하 |
| **잠재 공간 공간 결합** | O | 모든 모델에서 채택. 공간 결합 없이 시간 전이하는 사례 없음 | 중 |
| **잔차 예측 (GraphCast)** | △ | state(t+1) = state(t) + delta. 안정성 향상 가능. 실험 후 판단 | 하 |
| **변수별 토큰화 (ClimaX)** | △ | 21ch 입력을 변수별 독립 임베딩 후 통합. 현재 Conv3D(1,1,1)로 충분할 수 있음 | 중 |
| **Lead time embedding (ClimaX)** | O | p_load / sym_type 주입과 동일 패턴. 이미 설계에 반영 | - |
| **U-Net 대칭 (Pangu)** | X | 우리 격자는 단일 해상도. 다중 해상도 이점 없음 | - |
| **AFNO Fourier 혼합** | △ | 720 token에서 O(N log N) 이점 미미. 직접 Attention이 더 단순 | 중 |

## 5. 핵심 차용 후보

### 즉시 적용
- **비대칭 설계**: 프로세서(Mamba+Attention)에 연산 집중, 디코더 경량화. 기상 ML 표준과 일치
- **잠재 공간 공간 결합**: 프로세서 내에서 Spatial Attention 또는 Global token으로 공간 결합 수행

### 수정 후 적용
- **잔차 예측**: `state(t+1) = state(t) + model_output`. 학습 안정성 향상 가능. GraphCast 패턴
- **변수별 토큰화**: 각 물리량을 별도 임베딩 후 Cross-Attention 통합. 현재 21ch Conv3D(1,1,1) 대비 이점 불확실

### 부적합
- **U-Net 대칭 디코더**: 다중 해상도가 불필요한 우리 문제에는 과도
- **AFNO**: 720 token 규모에서 Fourier 혼합의 효율성 이점이 직접 Attention 대비 미미
