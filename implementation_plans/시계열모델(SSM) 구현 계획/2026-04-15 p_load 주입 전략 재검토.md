# p_load 주입 전략 재검토 — 디코더 단독 → Mamba + 디코더 확장

> **작성일**: 2026-04-15
> **배경**: 기존 결정 (`04_normalization_omitted_options.md` §1.4) 에서 p_load는 "디코더 AdaLN-Zero 전용"으로 결정되었으나, 시계열 동역학 모델 관점에서 재검토 필요성이 제기됨
> **결론**: p_load를 **Mamba (시계열 모델) + 디코더** 양쪽에 주입. 인코더 미주입은 유지

---

## 1. 기존 결정과 문제 제기

### 1.1. 기존 결정

`04_normalization_omitted_options.md` §1.4:
- 인코더: 일반 Pre-LN (p_load 미주입)
- 디코더: AdaLN-Zero로 p_load 주입
- 근거: "p_load는 다음 시점의 출력을 조건부로 만드는 조건이지, 현재 상태의 공간 압축에는 무관"

### 1.2. 문제: 시계열 모델(Mamba)의 "장님" 현상

현재 파이프라인:
```
state(t) + rod(t+1) → [인코더] → Z_t → [Mamba] → Z_{t+1} → [디코더 + p_load] → state(t+1)
```

Mamba가 Z_t에서 Z_{t+1}로 상태를 전이시킬 때 p_load를 모른다:
- p_load 100%: 높은 온도 상승률, 빠른 Xe-135 축적, 강한 반응도 피드백
- p_load 50%: 낮은 온도 상승률, 느린 Xe-135 축적, 약한 반응도 피드백
- **시간 변화율(dynamics)이 p_load에 직접 의존** → Mamba가 이를 모르면 정확한 상태 전이 불가

디코더에만 p_load를 주입하면, 디코더가 Z_{t+1}의 스케일/이동만으로 p_load 차이를 보정해야 함. 그러나 원자로 물리량은 p_load에 대해 **비선형 의존성** (Xe 동역학, Doppler 피드백 등) 이 있어 단순 affine 변조로는 부족.

### 1.3. DiT와의 차이 — 이미지 생성 vs 시간 전이

기존 결정이 DiT 패턴을 참조했으나, DiT와 본 프로젝트는 근본적으로 다른 문제를 푼다:
- **DiT**: "어떤 종류의 이미지를 생성할까" (class label → 정적 조건)
- **본 프로젝트**: "상태가 시간에 따라 어떻게 변하는가" (p_load → 동적 제어 변수)

DiT에서 class label이 매 블록 adaLN-Zero로 들어가는 것은, 이미지 생성에서 조건이 모든 레이어에 영향주기 때문. 본 프로젝트에서 p_load가 시간 전이에 영향준다면, **전이를 담당하는 Mamba에 p_load가 들어가야** 한다.

## 2. 학계 근거

### 2.1. DreamerV3 (World Model) — action은 시계열 모델에만 주입

> Hafner, D., Pasukonis, J., Ba, J., & Lillicrap, T. (2023). *Mastering Diverse Domains through World Models.* arXiv: 2301.04104.

RSSM (Recurrent State Space Model) 의 action 주입 위치:
```
h_t = GRU(h_{t-1}, [z_{t-1}, a_{t-1}])     ← action이 여기에만 들어감
z_t ~ q(z_t | h_t, e_t)                    ← 인코더: action 없음
p(o_t | h_t, z_t)                           ← 디코더: action 없음
```

action은 **GRU (시계열 모델) 입력에 concat**되어 들어가고, 인코더/디코더에는 들어가지 않음. DreamerV1/V2/V3 모두 동일한 패턴.

### 2.2. DreamerV3 vs 본 프로젝트 비교

| 구분 | DreamerV3 | 본 프로젝트 |
|---|---|---|
| 관측 | 게임 화면 이미지 | 3D 물리량 (20, 5, 5, 21ch) |
| 행동 (Action) | 조이스틱 조작 | **rod_map(t+1) + p_load(t+1)** |
| 시계열 모델 | GRU 기반 RSSM | Mamba (S6 selective scan) |
| **Action 주입 위치** | **GRU 입력에 concat** | 기존: 디코더만 → **수정: Mamba + 디코더** |
| 임계 경로 | 실제 궤적 (Replay Buffer) | 575 스텝 운전 궤적 데이터 |
| 분기 (Branch) | 정답 없음 (모델 상상) | **정답 있음** (MASTER 해석 결과 29개) |
| 물리적 제약 | 없음 (이미지 복원 Loss) | Physical Loss (L_diff_rel, L_Bateman 등) |

**핵심 차이**: 본 프로젝트는 DreamerV3과 달리 Branch 데이터에 정답이 있고, 물리적 제약 Loss가 존재. 그러나 **action(p_load)이 시계열 모델에 들어가야 한다는 구조적 요구는 동일**.

### 2.3. 물리 대리 모델 (SHRED) — 파라미터를 시계열 모델에 주입

> Lo Verso, L. et al. (2026). *Surrogate models for nuclear fusion with parametric Shallow Recurrent Decoder Networks.* Physics of Fluids. arXiv: 2603.10678.

핵융합 대리 모델에서 자기장 세기 파라미터를 **LSTM (시계열 모델) 입력에 concat**하여 주입. 디코더가 아닌 시계열 모델에 주입하는 것이 물리 대리 모델의 표준 패턴.

### 2.4. Navigation World Model — FiLM vs concat 비교

FiLM 방식으로 dynamics 모델에 action을 주입한 결과, 단순 concat 대비 **오차 약 50% 감소**. 조건부 affine 변조(FiLM/AdaLN)가 concat보다 우수하다는 실험적 근거.

### 2.5. Mamba 조건부 주입 방식

| 방식 | 논문 | 메커니즘 |
|---|---|---|
| **Hidden state 초기화** | DiMSUM (NeurIPS 2024, arXiv 2411.04168) | `h_{-1} = Linear(condition)`. 1회 주입, SSM 재귀로 전파 |
| **토큰 append** | DiM (arXiv 2405.14224) | 조건을 토큰으로 변환하여 시퀀스에 추가 |
| **adaLN per block** | AiM | DiT 패턴을 Mamba 블록에 적용. 매 블록 LN 변조 |

## 3. 수정된 p_load 주입 전략

### 3.1. 결정

| 컴포넌트 | p_load 주입 | 메커니즘 | 근거 |
|---|:---:|---|---|
| **인코더** | **X** (유지) | — | 현재 상태 공간 압축은 출력 수준과 무관. Dreamer 인코더도 action 미주입 |
| **Mamba** | **O** (신규) | 후보: concat / hidden state 초기화 / adaLN | 시간 변화율이 p_load에 직접 의존. Dreamer/SHRED 모두 이 패턴 |
| **디코더** | **O** (유지) | AdaLN-Zero (기존 결정 유지) | 출력 복원 시 추가 regime 정보 제공 |

### 3.2. Mamba 주입 방식 후보 비교

| 방식 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **Concat (Dreamer 패턴)** | p_load 임베딩을 Mamba 입력에 concat | 가장 단순. Dreamer에서 검증됨. rod_map도 이미 입력에 있음 | 입력 차원 증가 (D → D + d_emb) |
| **Hidden state 초기화** | `h_{-1} = Linear(p_load_emb)` | Mamba 고유 방식. 입력 차원 변경 없음 | 1회 주입 → 긴 시퀀스에서 신호 감쇠 가능 |
| **AdaLN per block** | Mamba 블록 내 LN을 p_load 조건부로 변조 | 매 블록에서 재적용되므로 신호 감쇠 없음 | 구현 복잡도 증가 |

### 3.3. 권장: Concat (Dreamer 패턴) 우선 적용

- Dreamer에서 가장 널리 검증된 방식
- rod_map(t+1)이 이미 인코더 입력 21ch에 포함되어 있으나, p_load는 포함되지 않음
- Mamba 입력 시점에서 p_load 임베딩을 concat하면 구현이 단순하고, Dreamer 패턴과 일치
- 성능이 부족하면 adaLN per block으로 업그레이드 가능

```
[수정된 파이프라인]

state(t) + rod(t+1) + xs_fuel
    → [인코더] → Z_t    (p_load 미주입)
    → [Mamba(Z_t, p_load_emb)] → Z_{t+1}    (p_load 주입 ★)
    → [디코더(Z_{t+1}, p_load)] → state(t+1)  (p_load AdaLN-Zero)
```

### 3.4. rod_map과 p_load의 주입 위치 차이

| 제어 변수 | 성격 | 주입 위치 | 이유 |
|---|---|---|---|
| **rod_map(t+1)** | 3D 공간 분포 (20, 5, 5, 1) | **인코더 입력** (21ch의 일부) | 공간 구조가 있으므로 인코더에서 공간 attention과 함께 처리 |
| **p_load(t+1)** | 스칼라 (전역 조건) | **Mamba 입력 + 디코더 AdaLN** | 공간 구조 없는 전역 조건. 시간 전이 동역학에 직접 영향 |

이 분리는 물리적으로 타당: rod_map은 "어디에서 반응도가 변하는가" (공간), p_load는 "전체적으로 얼마나 변하는가" (규모/속도).
