# 시계열 아키텍처 및 학습 방법론 재검증 (외부 조사 포함)

> **작성일**: 2026-04-16
> **배경**: 이전 분석에서 성능이 저하된 모델을 사용했을 가능성을 인지하고, 외부 조사(2025-2026 최신 문헌)까지 확장하여 시계열 아키텍처 및 학습 방법론을 재검증
> **고정 사항**: 공간 인코더 (FullAttn ×3 + ConditionalLAPE + STRING), 720 cell token (TokenWM 패턴), C안 (Mamba+Attention 하이브리드 + 경량 디코더)
> **재검증 대상**: 시계열 모델 세부 구조 (Mamba 변종, 비율, 순서), 학습 방법론 (Phase 구조, h(t) 관리, 안정화 기법, Loss 구성)
> **가정한 제약**: A100 80GB 단일 GPU, TF2.14, 평가 우선순위 = rollout 정확도 > Branch 정확도 > 물리 정합성 > 학습 효율

## 1. 외부 조사 핵심 발견 (2025-2026)

### 1.1 우리 설계를 직접 지지하는 새 증거

| 논문/소스 | 발견 | 우리 설계와의 관계 |
|----------|------|------------------|
| **DeepOMamba** (J. Comp. Phys. 2025) | DeepONet + Mamba 시간 모델이 4개 PDE 벤치마크에서 SOTA. FNO+시간 하이브리드 대비 ~10배 빠름 | "공간 인코더 + Mamba 시간 모델" 분리 패턴 검증 |
| **HMT-PF** (arXiv 2505.11578, 2025년 5월~2026년 2월) | Mamba+Transformer 하이브리드 + 물리 정보 미세조정 블록 (PDE 잔차를 잠재 공간에 인코딩) | 우리 Mamba+Attention + Physical Loss 설계의 가장 가까운 공개 사례 |
| **PGMNO** (Neural Networks, 2026) | 선형 다단계 수치 기법(linear multistep) + 구조화 SSM. 장기 PDE 예측에서 오차 누적 직접 해결 | 575-step 자기회귀 안정화에 직접 적용 가능한 기법 |
| **PIMRL** (arXiv 2503.10253, 2025년 3월) | 명시적 2단계 학습: 물리 사전학습(미시) → 데이터 기반 학습(거시). 5개 벤치마크에서 9~80% 개선 | **우리 2-Phase 학습의 직접 외부 검증** |
| **Beja-Battais et al.** (arXiv 2511.16148, 2025년 11월) | 원자로 노심 stiff ODE 부하추종 surrogate. ~1000배 가속 | 우리와 동일 도메인의 직접 선례 (필독) |

### 1.3 경고 신호 / 주의 사항

**REALM 벤치마크** (arXiv 2512.18595, 2025년 12월) — 11개 다중물리 데이터셋에서 핵심 결과:
- **FFNO가 장기 안정성에서 최고**
- **Transformer는 3D에서 메모리 한계** (우리 720 토큰은 작은 편이라 안전 범위)
- **GNN은 anisotropic 메시에서 over-smooth** (우리 GNN 미사용)
- **DeepONet은 비정형 메시에서만 우위** (우리 정형 메시)
- **R² 높음 ≠ 적분량 정확** (one-step 정확도가 long rollout 정확도와 anti-correlate 가능)
- 평가는 물리 인지 메트릭으로 해야 함

**우리 프로젝트의 진정한 novelty 영역** (외부 조사 결과 미존재):
- 3D 결합 elliptic Mamba (원자로 중성자 확산)
- Branch GT가 명시적으로 존재하는 세계 모델 학습 (CRS 본 + 29 분기 GT)
- 원자로 노심 시공간 surrogate에 SSM 적용

이 세 가지는 발표 시 강조할 기여점.

### 1.4 Mamba 변종 추가 정보

- **Mamba-4 미존재** (2026년 4월 현재). 검색 결과 0건. 확정.
- **RWKV-7 "Goose"** (ICLR 2026): 일반화된 delta rule + 벡터 게이팅. 모든 정규언어 인식 가능 + 병렬 학습 유지. **상태 추적 능력에서 Mamba-2 클래스를 이론적으로 능가**. 물리 적용 사례는 없음. → Mamba-3가 Xe 동역학 추적에 부족하면 차순위 후보
- **xLSTM, Hawk, Griffin**: 물리 적용 사례 없음. 후순위
- **Mamba-3**: ICLR 2026 채택. 1.5B 규모에서 GDN +1.8 pts (절반 state size)

## 2. 시계열 아키텍처 재검토

### 2.1 Mamba 변종 선택

| 변종 | 외부 검증 | 우리 도메인 적합성 | TF2 구현 | 종합 권장 |
|------|---------|----------------|---------|----------|
| **Mamba-2 (SSD)** | DeepOMamba, MNO, HMT-PF, Drama | 검증된 PDE 적합성 | 참고 자료 풍부 | **Phase 1 출발점** |
| **Mamba-3** | ICLR 2026 채택, GDN 상회 | Xe 진동 위상 추적에 이론적 우위 | 복소수 연산 부담 | **Phase 2 후 전환 검토** |
| **GDN** | OLMo Hybrid, Qwen3-Next | LLM 검증, 물리 미검증 | 미확인 | **3순위** |
| **RWKV-7** | ICLR 2026 채택, 상태 추적 우위 | 물리 미검증 | 미확인 | **장기 옵션** |

**권장 변경**: 기존 "Mamba-2 1순위, Mamba-3/GDN 후보"는 유효. 외부 조사로 새로운 변종 발견 없음. 단, **Mamba-3로 전환 트리거**를 명시:
- 575-step rollout에서 Xe 진동의 위상이 어긋남 (peak 시점 예측 오차 > 30분)
- L_Bateman 잔차가 정출력 구간에서 수렴하지 않음

### 2.2 Mamba:Attention 비율

**외부 조사 결과**: 물리 도메인에서 명시적 비율 권장 없음. LLM 관행(1:6~1:7 = Mamba 7개당 Attention 1개)이 유일한 참고점.

HMT-PF는 interleaved 구조를 사용하나 비율 sweep을 공개하지 않음.

**권장 시작점**: 우리 720 토큰 규모(LLM 대비 매우 작음)에서는 LLM 관행보다 Attention 비중을 높일 수 있음. 시작 비율 **2:1 또는 3:1** (Mamba 2~3개당 Attention 1개) 권장. 이유:
- 우리 720은 LLM의 수만 토큰 대비 매우 작음 → Attention 비용 부담 적음
- 물리 도메인은 매 시점 공간 결합이 LLM의 long-range dependency보다 자주 필요할 가능성

### 2.3 Mamba ↔ Attention 순서

**외부 조사 결과**: 명시적 권장 없음. HMT-PF는 interleaved하나 정확한 순서 sweep 미공개.

**물리적 직관 vs 실용적 고려**:
- **Attention → Mamba** (시뮬레이터 순서): 매 step "공간 미분 계산 → 시간 전진" 패턴에 대응. 물리적으로 자연스러움
- **Mamba → Attention** (현재 검토안): "시간 전이 후 공간 교정". 학습 안정성 측면에서 검증된 사례가 더 많음 (Jamba 패턴)

**권장**: 두 순서 모두 실험적으로 검증 필요. 시작은 **Mamba → Attention** (Jamba 패턴, 검증된 학습 안정성), Phase 2 후반에 Attention → Mamba 비교 실험.

### 2.4 신규 검토 항목: Equation-Aware Conditioning

기존 설계는 p_load를 Mamba에 concat, rod_map을 인코더 입력 채널로 주입. **Equation-Aware NO** 패턴은 이를 넘어:
- p_load + rod_map 위치 통계량(평균 깊이, AO 등) → 작은 metadata 벡터 → 모든 Mamba/Attention 블록의 conditioning으로 주입
- AdaLN으로 modulation

**효과**: 모델이 매 layer에서 "현재 운전 조건"을 명시적으로 인식. Branch 예측에서 rod_map 변경의 영향 전파 강화.

**도입 권장도**: 중간. 현재 설계가 작동하면 후순위 추가.

## 3. 학습 방법론 재검토

### 3.1 2-Phase 학습 — 외부 검증 강화

PIMRL (arXiv 2503.10253) 발견으로 우리 2-Phase 결정의 외부 근거가 명확해짐:

| 우리 2-Phase | PIMRL |
|-------------|-------|
| Phase 1: 공간 (Branch GT로 인코더+Attention+디코더) | Stage 1: 물리 사전학습 (미시 스케일) |
| Phase 2: 시간 추가 (CRS로 Mamba) | Stage 2: 데이터 기반 (거시 스케일) |
| Branch 보조 loss로 forgetting 방지 | (해당 메커니즘 없음, 우리 고유) |

**결론**: 2-Phase 유지. 이전 검토에서 "우리 고유 설계"라고 표현했으나, **PIMRL에 의해 외부 검증된 패턴**으로 격상.

### 3.2 잔차 예측 → "Predict Change Not States"로 정교화

**기존 설계 (답변 B 권장)**:
```
delta = Model(state_t)
state_t_next = state_t + delta
Loss = MSE(state_t_next, GT) + Physical Loss
```

**Zhou & Farimani 2025 패턴**:
```
dx/dt = Model(state_t)        ← 시간 미분 자체를 예측
state_t_next = ODE_integrate(state_t, dx/dt, Δt)  ← 추론 시 정밀 적분
Loss = MSE(dx/dt_pred, dx/dt_true) + Physical Loss
```

**우리 문제에서의 의미**:
- Bateman 방정식 (Xe/I)이 ODE-native이므로, 시간 미분 예측이 더 자연스러움
- 추론 시 5분 step 그대로 더하지 않고 RK4 등으로 정밀 적분 → 누적 오차 감소
- 단, 모든 변수에 적용할지 (Xe/I만 적용할지) 결정 필요

**권장**: 시작은 단순 잔차 예측 (현재 설계). Phase 2에서 Xe/I 채널만 시간 미분 예측 + ODE 적분 비교 실험.

### 3.3 Loss 가중치: ReLoBRaLo 도입

**기존 설계**: 수동 가중치
```
L = α₁·L_data + α₂·L_diff_rel + α₃·L_Bateman
```
α 값은 경험적/실험적으로 결정해야 함.

**ReLoBRaLo 패턴**: 학습 중 각 loss term의 진행 속도를 모니터링하여 가중치를 적응적으로 조정. GradNorm/SoftAdapt 능가.

**권장**: **즉시 도입**. 우리 3-term loss(데이터 + 확산 잔차 + Bateman ODE)는 ReLoBRaLo의 정확한 적용 사례. 수동 튜닝 부담 제거 + 더 안정적인 수렴 기대.

추가 권장: **per-quantity normalization**. 우리 10채널 (중성자속 ~10¹³, Xe 농도 ~10¹⁵, 온도 ~10², 출력 ~1) 스케일이 수십 자릿수 차이. ReLoBRaLo 전에 채널별 정규화 필수.

### 3.4 롤아웃 안정화 — 추가 옵션

기존 설계: pushforward + scheduled sampling + delta prediction + physical loss.

**JAWS 추가 검토** (arXiv 2603.05538):
- 공간 적응적 Jacobian 정규화. 모델 출력의 입력에 대한 민감도를 영역별로 다르게 제약
- 우리 문제: 정출력 구간(부드러움)은 강한 정규화, 변동 구간(급변)은 완화
- **짧은 학습 윈도우로 긴 pushforward 효과 근사** → A100 80GB 메모리 부담 완화 가능
- 도입 권장도: **중간**. Phase 2 후반 실험

**PhysicsCorrect** (arXiv 2507.02227):
- 학습 후 추론 시에만 적용. 매 step 출력을 물리 잔차 최소화 방향으로 작은 양만큼 투영
- 학습 무관 → 부담 없음
- 도입 권장도: **추론 단계 옵션으로 추가**

**Thermalizer** (arXiv 2503.18731):
- 추론 시 diffusion-score denoising으로 안정 horizon ~10배 확장
- 우리 5분 step에 적합한지 검증 필요
- 도입 권장도: **낮음** (복잡도 추가, 비교 실험으로만 검토)

### 3.5 h(t) 관리 — 외부 조사 결과 변화 없음

외부 조사: 2025-2026에 detach vs gradient pass-through에 대한 명확한 새 연구 없음. Drama 패턴이 여전히 best practice.

**LongSSM 경고** (2024, 여전히 인용됨): hidden state 재사용 시 데이터가 shuffle되면 noisy. 함의: CRS 시퀀스를 **연속적으로** 배치 구성, shuffling은 시퀀스 단위로만.

**우리 결정**: 이전 논의(1-step Branch 물리 분석 기반 detach 우선)가 여전히 유효.

### 3.6 Branch 학습 — Ctrl-World 패턴 비교

Ctrl-World (ICLR 2026):
- Action token + temporally-factored masking
- Policy-in-the-loop imagination

**우리 Branch와의 대응**:
- Action token = rod_map 변경 (이미 인코더 입력 채널)
- Temporally-factored masking = h(t) snapshot/restore (이미 적용)
- Policy-in-the-loop = (해당 없음, 우리는 GT 직접 비교)

**결론**: Ctrl-World 패턴은 우리와 구조적으로 동일. 추가 차용할 새 기법 없음. 단, 우리 Branch GT가 명시적으로 존재한다는 점이 Ctrl-World 대비 강점 (지도 학습 가능).

### 3.7 V-JEPA 2.1 Deep Self-Supervision

V-JEPA 2.1 신규 기법: 인코더 중간 레이어에서도 예측 loss 적용 (Deep Self-Supervision).

**우리 적용 가능성**:
- Phase 1 (인코더 사전학습) 시 중간 레이어에서도 1-step 예측 loss
- 인코더가 "물리적으로 의미 있는 중간 표현"을 학습하도록 압력
- 도입 권장도: **중간**. Phase 1 안정화 후 비교 실험

### 4.3 보류 (적용 어려움 또는 검증 부족)

| 기법 | 사유 |
|------|------|
| **FFNO 통합** | 인코더 고정으로 통합 어려움. REALM 결과는 인코더 변경 시 참고 가치 |
| **GDN, RWKV-7** | 물리 도메인 검증 없음. Mamba-3 실패 시 차순위 |
| **Diffusion-based rollout** | 575 step에서 autoregressive 우위 유지 |
| **Thermalizer** | 추가 복잡도 대비 검증 부족 |

### 4.4 유지 (재검토 후에도 기존 결정 유효)

| 결정 | 근거 |
|------|------|
| **C안 아키텍처 (Mamba+Attention 하이브리드 + 경량 디코더)** | DeepOMamba, HMT-PF로 외부 검증 강화 |
| **720 cell token 표현** | 외부 조사에서 더 나은 대안 미발견 |
| **2-Phase 학습** | PIMRL로 외부 검증 강화 |
| **h(t) detach 주입** | 1-step Branch 물리 분석 + LongSSM 경고로 detach가 여전히 안전 |
| **Pushforward trick** | 모든 외부 조사가 일관되게 권장 |
| **잔차 예측 + Physical Loss** | GraphCast, Predicting Change 패턴이 모두 지지 |

## 5. 우리 프로젝트의 Novelty (발표/논문 시 강조점)

외부 조사로 확인된 우리 프로젝트의 고유 기여:

1. **3D 결합 elliptic PDE에서 Mamba 적용**: 2026년 4월 현재 공개된 사례 없음. SSM의 PDE 적용은 1D~2D 단일 물리량이 대부분
2. **Deterministic Branch GT + 본 trajectory GT 동시 학습**: 기존 RL 세계 모델은 imagination 기반 (GT 없음). 우리처럼 명시적 분기 GT를 학습 신호로 쓰는 패러다임 미공개
3. **원자로 노심 시공간 surrogate에 Mamba+Attention 하이브리드**: Beja-Battais (2025년 11월)는 NN 기반이지만 Mamba 미사용. 우리가 첫 사례

## 7. 결론

### 7.1 이전 결정의 외부 검증

**모든 핵심 결정이 외부 조사에서 강화됨**:
- C안 아키텍처: DeepOMamba, HMT-PF가 직접 지지
- 2-Phase 학습: PIMRL이 명시적 외부 검증 제공
- 잔차 예측: Predicting Change Not States가 더 정교한 형태로 발전 가능
- Pushforward: 여전히 long-horizon 안정화의 표준

이전 분석이 "성능 저하된 모델"로 수행되었더라도, **결정 자체는 잘못되지 않음**. 다만 외부 검증 근거가 약하다고 표현했던 부분(특히 2-Phase)이 실제로는 검증된 패턴이었음.

### 7.2 새로 채택 권장하는 사항 4개

1. **ReLoBRaLo** — Loss 가중치 자동 균형 (즉시 도입)
2. **Per-quantity normalization** — ReLoBRaLo 전제 조건 (즉시 도입)
3. **PhysicsCorrect** — 추론 시 학습 무관 안정화 (저비용 보험)
4. **Predict Change Not States (Xe/I)** — 잔차 예측의 ODE-aware 정교화 (Phase 2 후반)

### 7.3 향후 검토 사항 (Phase 2 후반 ~ Phase 3)

- JAWS Jacobian 정규화 (메모리 절약 시)
- Equation-Aware conditioning (Branch 정확도 부족 시)
- V-JEPA 2.1 deep self-supervision (Phase 1 강화 시)
- Mamba-3 전환 (Xe 위상 추적 부족 시)

### 7.4 우리 프로젝트의 강조점

- 외부 조사 결과 우리 프로젝트는 3가지 축에서 **공개된 선례 없음** = novelty 명확
- 발표/논문 작성 시 이 점을 명시적으로 강조
