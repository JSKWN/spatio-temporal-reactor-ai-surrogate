# 시계열모델 적용 기법 검토

> **작성일**: 2026-04-15
> **목적**: 원자로 노심 대리 모델의 시계열 처리를 위한 기술 스택 조사, 우리 아키텍처에 대한 적용성 고찰
> **범위**: Latent recurrent planning (Dreamer/Drama/TokenWM), Predictive latent (V-JEPA 2, Dreamer-CDP), Generative simulation (Cosmos/Genie3), Industrial stack (Tesla/GR00T)

## 개별 기술 상세

### DreamerV3

> Hafner, D., Pasukonis, J., Ba, J., & Lillicrap, T. (2023/2025). *Mastering Diverse Domains through World Models.* ICLR 2024 / Nature 2025. [arXiv:2301.04104](https://arxiv.org/abs/2301.04104)

**RSSM 내부 구조**:

```
a_{t-1}, z_{t-1} → [concat + Linear] → [Block GRU] → h_t (deterministic)
                                                         ↓
                                              [Prior MLP] → z_hat_t ~ Cat(32×32)
x_t → [CNN Encoder] → [Posterior MLP(h_t, enc)] → z_t ~ Cat(32×32)
                                                         ↓
                                              [CNN Decoder] → x_hat_t
                                              [Reward MLP]  → r_hat_t
```

- **h_t**: Block GRU로 전파. 8개 블록 독립 업데이트, RMSNorm + SiLU
- **z_t**: 32개 categorical 분포 × 32 classes = 1024차원 이산 잠재 상태
- **Action 주입**: z_{t-1}과 a_{t-1}을 concat 후 Linear → GRU 입력
- **Imagination**: h_t에서 시작, GRU step으로 미래 전개 (관측 없이 prior만 사용)
- **Actor-Critic**: 잠재 공간에서 REINFORCE + entropy + lambda-return

**손실 함수**: L = L_pred (재구성+보상+종료) + L_dyn (KL, sg on posterior) + 0.1·L_rep (KL, sg on prior). Free bits = 1 nat.

**우리와의 대응**:
| DreamerV3 | 우리 |
|---|---|
| GRU h_t | Mamba hidden state |
| Cat(32×32) z_t | 없음 (확률적 잠재 미사용) |
| Action concat → GRU | p_load concat → Mamba |
| Imagination rollout | Branch 평가 (29개 GT 존재) |
| CNN encoder | Spatial Encoder (ViT + ConditionalLAPE) |
| Actor-Critic planning | 불필요 (Branch GT 직접 평가) |

### Drama

> Wang, W., Dusparic, I., Shi, Y., Zhang, K., & Cahill, V. (2025). *Drama: Mamba-Enabled Model-Based Reinforcement Learning Is Sample and Parameter Efficient.* ICLR 2025. [arXiv:2410.08893](https://arxiv.org/abs/2410.08893)

**핵심**: DreamerV3의 GRU를 Mamba-2로 교체. 전체 ~7M 파라미터.

```
[학습 - parallel]
  z_t, a_t → [concat + Linear] → x_t (B, T, 512) → [Mamba-2 ×2 blocks] → d_t
  d_t → [Prior MLP] → z_hat_{t+1}
  z_t → [CNN Decoder] → O_hat_t    ← 디코더는 z_t에서 재구성 (d_t가 아님!)

[Imagination - autoregressive]
  Real prefix 8 steps → [Mamba-2 parallel] → InferenceParams (SSM state) 구축
  From last step: z_hat_t, d_t → [Actor] → a_t
  → [concat + Linear] → [Mamba-2 step(InferenceParams)] → d_{t+1}  ← O(1)/step
  → [Prior] → z_hat_{t+1}, [Reward/Term] → r, c
```

**Mamba-2 dynamics**: hidden=512, 2 layers, state_dim=16, 4 heads, head_dim=128

**핵심 설계 결정**:
- 디코더가 predicted z_hat이 아닌 **encoded z_t에서 재구성** → 학습 시 순차 의존 제거 → 완전 병렬화
- Prefix burn-in 8 steps: 실제 궤적으로 SSM state를 채운 후 imagination 시작
- **InferenceParams**: ssm_state + conv_state로 구성. Python 객체 → deepcopy 가능
- DFS (Dynamic Frequency-based Sampling): 학습 불균형 방지

**우리와의 핵심 관련성**:
- **현재 설계와 가장 유사한 아키텍처**
- Prefix burn-in = 우리 CRS가 h_t를 구축하는 과정
- InferenceParams snapshot = Branch-in-CRS h(t) 관리 해법
- ~7M params = 우리 규모 (인코더 0.78M + Mamba + 디코더) 와 유사

### TokenWM

> Zhai, G., Zhang, X., & Navab, N. (2025). *Recurrent World Model with Tokenized Latent States.* ICLR 2025 Workshop. [OpenReview](https://openreview.net/forum?id=xmwcdUdcWz), [PDF](https://ymxlzgy.com/media/token_wm_init.pdf)

**핵심**: RSSM의 단일 벡터 h_t를 **토큰 집합 h_t ∈ R^(N_h × D)** 로 교체.

```
O_t → [ViT Encoder] → z_t ∈ R^(N_z × D_z)    (patch tokens)
z_t + h_{t-1} → [Cross-Attention] → h_t ∈ R^(N_h × D)    (posterior)

[Prior - imagination]:
h_t → [+ Positional Encoding]
    → [FiLM(a_t): scale, shift = MLP(a_t)]    ← action conditioning
    → [Post-Norm Self-Attention]                ← tokens 간 상호작용
    → [Memory Bank Cross-Attention]             ← 과거 맥락 조회
    → h_{t+1} ∈ R^(N_h × D)
```

**결정적 특징**:
- **확률적 잠재 상태 없음** (deterministic) — KL balancing 불필요
- **FiLM action conditioning** — 토큰별 scale/shift 변조
- **Memory bank** — 과거 토큰 표현 저장, cross-attention으로 조회
- **Post-Norm** (Pre-Norm이 아님) — 장기 rollout 안정성 위해

**우리와의 핵심 관련성**: 아래 "핵심 발견" 참조

### Dreamer-CDP

> Mohammadi, Halvagal, & Zenke. (2026). *Reconstruction-Free World Models.* [arXiv:2603.07083](https://arxiv.org/abs/2603.07083)

**핵심**: DreamerV3에서 **픽셀 디코더를 제거**하고, 잠재 공간에서의 예측 일관성 손실 (CDP) 로 대체.

```
RSSM 그대로 유지 (GRU h_t + Cat z_t)

변경: 디코더 제거, 대신 Predictor MLP 추가
  h_t → [Predictor MLP (8192→4096→4096)] → u_hat_t
  x_t → [Feature Extractor] → u_t (target)

  L_CDP = -cos(sg(u_t), u_hat_t)    ← cosine similarity, stop-gradient on target
```

- **EMA target network 없음** (V-JEPA 2와 달리 sg만 사용)
- KL 항 (L_dyn, L_rep) 은 유지 — 표현 붕괴 방지에 필수
- β_CDP = 500, encoder lr = 6e-6 (매우 낮음)

**우리와의 관련성**: 디코더 재구성 loss 외에 **잠재 예측 loss를 추가**하는 영감 제공

### V-JEPA 2

> Meta AI. (2025). *V-JEPA 2.* [arXiv:2506.09985](https://arxiv.org/abs/2506.09985)

**JEPA 원리**: 관측을 재구성하지 않고, 마스킹된 영역의 **표현(representation)** 을 예측.

```
O_t → [ViT-g Encoder (frozen)] → z_t ∈ R^(16×16×1408)    (spatial patch tokens)

[V-JEPA 2-AC - Action-Conditioned]:
  z_t, a_t (7-dim), s_t (end-effector state)
  → [Transformer 24L, 16H, d=1024, block-causal attention]
  → z_hat_{t+1}

  L = L_teacher_forcing (1-step) + L_rollout (multi-step, T=2)
  Loss function: L1 distance in latent space

[Planning]:
  E(a_{1:T}; z_k, z_goal) = ||P(a_{1:T}; z_k) - z_goal||_1
  Optimization: CEM (Cross-Entropy Method, derivative-free)
  Execution: MPC loop — execute first action, re-plan
```

**핵심 설계**:
- ViT 인코더 **완전 동결** → action-conditioned predictor만 학습 (62시간 로봇 데이터)
- 3D RoPE (temporal + spatial) — STRING과 동일 계열
- Block-causal attention — 과거 시점에만 attend

**우리와의 관련성**: Phase 2에서 인코더 동결 + Mamba 학습 패턴과 구조적으로 유사

### Mamba 버전 비교

| | **Mamba-1 (S6)** | **Mamba-2 (SSD)** | **Mamba-3** |
|---|---|---|---|
| **논문** | [arXiv:2312.00752](https://arxiv.org/abs/2312.00752) | [arXiv:2405.21060](https://arxiv.org/abs/2405.21060) (ICML 2024) | [arXiv:2603.15569](https://arxiv.org/abs/2603.15569) (ICLR 2026) |
| **A 행렬** | Diagonal R^(N×N) | Scalar (a_t ∈ R) | Scalar |
| **B, C** | 입력 의존 | 입력 의존 | 입력 의존 |
| **이산화** | ZOH | Exp-Euler O(Δ²) | **Exp-Trapezoidal O(Δ³)** |
| **상태 업데이트** | h = Ā·h + B̄·x | h = a·h + γ·B·x | h = a·h + β·B_{t-1}·x_{t-1} + γ·B_t·x_t |
| **상태 유형** | 실수 | 실수 | **복소수** (data-dependent RoPE) |
| **I/O 구조** | SISO | SISO | **MIMO rank R** |
| **병렬화** | Prefix scan | Chunked matmul (SSD) | Chunked matmul (2-semiseparable) |
| **중간 상태 접근** | 미저장 (재계산) | **Chunk 경계에서 저장** | Chunk 경계에서 저장 |
| **속도** | 기준 | 2-8× faster | Mamba-2와 유사 (N 절반으로 동일 성능) |

**Mamba-3 고유 특성**:
- **Exp-Trapezoidal**: 현재 + **이전** 입력 모두 사용 → 부드러운 동역학 (Xe 진화) 에 더 정확
- **Complex state + data-dependent RoPE**: 상태 벡터에 **위상 정보** 내장 → Xe 진동의 위상 추적에 잠재적 이점
- **MIMO**: rank R=2~4로 하드웨어 활용도 향상, state 크기 절반으로 동일 성능

### Cosmos / Genie 3 / GR00T / Tesla

**Cosmos Predict**: DiT 기반 잠재 확산 모델. 9000조 토큰 학습. Action conditioning은 post-training LoRA로 추가. [NVIDIA Cosmos](https://www.nvidia.com/en-us/ai/cosmos/)

**Genie 3**: Autoregressive transformer + Latent Action Model (8-dim). 720p 24fps 실시간 세계 생성. 병렬 다중 미래 분기는 미지원 (순차 단일 궤적). [DeepMind](https://deepmind.google/blog/genie-3-a-new-frontier-for-world-models/)

**GR00T N1**: VLA (Vision-Language-Action) 정책 모델. 세계 모델이 아님. Dual system: VLM (느린 추론) + DiT (빠른 행동, 120Hz). [arXiv:2503.14734](https://arxiv.org/abs/2503.14734)

**Tesla**: Occupancy Network (시공간 BEV) + MCTS trajectory planning + Neural World Simulator (카메라 8대 미래 예측). 매 시점 1000개 prediction tensor 출력. [Tesla AI](https://www.tesla.com/AI)

**이 계열의 시사점**: 세부 아키텍처보다 **설계 패턴**이 참고 가치:
- Cosmos: Predict(예측) / Transfer(sim-to-real) / Reason(물리 추론) 분리
- Tesla: Space-time fused representation + closed-loop planning/evaluation
- 공통: 단일 모델보다 **예측 → 검증 → 제어** 스택으로 시스템 설계

## 우리 아키텍처와의 대응 종합

| 우리 컴포넌트 | DreamerV3 | Drama | TokenWM | V-JEPA 2 |
|---|---|---|---|---|
| Spatial Encoder | CNN encoder | CNN encoder | **ViT encoder** | **ViT encoder (frozen)** |
| Temporal Model (Mamba) | Block GRU | **Mamba-2** | Self-Attention + FiLM | Transformer predictor |
| Spatial Decoder | CNN decoder | CNN decoder | Decoder | 없음 (reconstruction-free) |
| 720 cell tokens | 단일 벡터 h_t | 단일 벡터 d_t | **Token set h_t** | Patch tokens |
| rod_map (공간 제어) | action concat | action concat | **FiLM** | action vector |
| p_load (전역 제어) | action concat | action concat | FiLM | action vector |
| Branch 평가 | Imagination (모델 생성) | Imagination + snapshot | Imagination | CEM planning |
| Branch GT | **없음** (상상만) | **없음** | **없음** | **없음** |
| Physics loss | 없음 | 없음 | 없음 | 없음 |

**마지막 두 행이 결정적 차이**: 우리는 Branch에 GT가 있고, 물리 제약 loss가 있다. 이것이 RL 기반 월드모델과의 근본적 차이.

## 핵심 발견 (고찰 결과)

### 발견 1: 우리는 이미 TokenWM 아키텍처를 구현하고 있었다

우리 인코더 출력: 720개 cell token × 128차원. 이것은 TokenWM의 "토큰화된 잠재 상태"와 구조적으로 동일하다. 차이점:

| | TokenWM | 우리 (현재) |
|---|---|---|
| Dynamics model 내 토큰 상호작용 | Self-Attention | **없음** (cell-wise 독립 Mamba) |
| 전역 상태 | Memory bank + Cross-Attention | **없음** (720 cell h_i(t) 분산 저장) |

**결론**: 720 cell token에 **4~8개 global token** (시스템 상태: keff 추세, 총 Xe, AO 등) 을 추가하면, TokenWM의 전역 맥락 유지 장점을 최소 비용으로 흡수할 수 있다. Global token은 인코더 attention에서 720 cell과 함께 처리되고, Mamba에서 cell token과 함께 시간 전이를 거친다.

### 발견 2: Drama의 prefix burn-in이 Phase 1→2 전환의 핵심이다

현재 설계에서 Branch는 Mamba를 우회하여 시간 맥락이 없다. Drama의 패턴을 적용하면:

```
[Phase 2 학습]
  1. CRS 시퀀스 (B, T=575, D) → Mamba parallel scan → 중간 h_1...h_575 추출
  2. 각 시점 t의 h_t를 detach
  3. Branch 학습: h_t를 초기 상태로 주입 → 1-step Mamba → decoder → loss
     → Branch가 "이 Xe 축적 상태에서 rod를 움직이면?" 에 대한 시간적 맥락 하에 학습
```

Phase 1 (Branch pretrain, Mamba frozen) 은 유지하되, Phase 2에서 Branch도 CRS의 h_t 맥락을 받는 구조로 전환.

### 발견 3: Dreamer-CDP의 잠재 예측 loss가 Mamba 학습을 가속한다

현재: Mamba gradient는 decoder → physics loss → backprop 경로로만 전달.
추가 가능: **잠재 공간 직접 예측 loss**

```
z_t = Encoder(state_t)
d_t = Mamba(z_t)            → Mamba 출력
z_{t+1} = Encoder(state_{t+1}_GT)  → 다음 시점 인코더 출력 (stop-gradient)

L_latent = ||d_t - sg(z_{t+1})||   또는   -cos(sg(z_{t+1}), d_t)
```

이 loss는 Mamba에 **직접적인 gradient 신호**를 제공하여, decoder를 경유하지 않고도 "다음 잠재 상태를 정확히 예측하라"는 목표를 부여. Dreamer-CDP에서 KL 항이 표현 붕괴 방지에 필수라는 발견을 참고하여, 이 loss의 가중치는 신중히 조절해야 함.

### 발견 4: V-JEPA 2의 frozen-encoder 패턴이 우리 2-Phase 설계를 정당화한다

V-JEPA 2-AC: 62시간 로봇 데이터만으로, 동결된 ViT 위에 action-conditioned predictor 학습 → zero-shot planning 성공. 이것은 우리 Phase 2 (인코더 고정/미세조정 + Mamba 학습) 의 타당성을 뒷받침:

- 좋은 표현 (Phase 1에서 학습) 위에 시간 모델을 올리는 것이 효율적
- 전체를 end-to-end로 처음부터 학습하는 것보다 수렴이 빠름

### 발견 5: Branch GT는 "oracle imagination"으로, RL imagination보다 강력하다

Dreamer/Drama에서 imagination rollout은 **모델이 생성한 불완전한 미래**. 우리 Branch GT는 **MASTER가 계산한 거의 정확한 미래**.

| | RL Imagination | 우리 Branch GT |
|---|---|---|
| 정확도 | 모델 오차 누적 | MASTER 정밀도 (0.1% 이내) |
| 다양성 | actor가 생성하는 action에 제한 | 29개 rod offset 체계적 탐색 |
| 학습 신호 | reward prediction (간접) | **직접 지도 학습** (GT 존재) |
| planning | actor-critic 필요 | **불필요** (직접 평가) |

**결론**: actor-critic이 불필요하므로 모델 구조가 단순해짐. Mamba는 순수하게 "상태 전이 예측기" 역할만 수행.

### 발견 6: Mamba-2가 실용적 선택, Mamba-3는 향후 목표

| | Mamba-2 | Mamba-3 |
|---|---|---|
| MBRL 검증 | Drama에서 검증 | 미검증 |
| 구현 성숙도 | 높음 | 2026.03 공개, 초기 |
| 우리 문제 장점 | 충분한 선택성 + 빠른 병렬화 | Xe 위상 추적 (복소 상태), 더 정확한 이산화 |
| TF2 구현 | 직접 구현 필요 (관리 가능) | 복소수 연산 추가 부담 |

**결정**: Mamba-2 (SSD) 로 시작, 성능 부족 시 Mamba-3 전환 검토.

## 권장 하이브리드 설계 방향

| 요소 | 채택 원천 | 내용 |
|---|---|---|
| Temporal backbone | **Drama** | Mamba-2 기반, prefix burn-in, InferenceParams snapshot |
| State representation | **TokenWM** | 720 cell token + 4~8 global token |
| Action conditioning | **Drama + 우리 기존** | p_load → Mamba concat, rod_map → encoder input |
| 학습 목표 | **Dreamer-CDP + 우리 기존** | Physics losses + 잠재 예측 loss (CDP-inspired) |
| Phase 설계 | **V-JEPA 2 + 우리 기존** | Phase 1: encoder/decoder pretrain, Phase 2: Mamba + latent pred |
| Branch 처리 | **Drama** | 학습: detached h_t 주입. 추론: InferenceParams snapshot |
| Planning | **우리 고유** | 29 Branch 직접 평가 (actor-critic 불필요) |
