## 1. 문제 제기

### 현재 파이프라인

```
State(t) → [Spatial Encoder] → z(t) → [Mamba] → d(t) → [Spatial Decoder] → State_pred(t+1)
             (Attention 3단)    (720,128)  (cell별 독립)  (720,128)   (Attention + AdaLN)   (720,10)
```

### 구조적 우려

원자로 노심에서 시간 전이는 **공간적으로 결합된** 현상이다:

1. Cell A의 Xe-135가 변하면 → 해당 cell의 거시단면적 변화 → **전체 노심 중성자속 재분포** (확산 방정식이 모든 cell을 결합)
2. 이 결합은 시간 전이 과정 자체에 내재
3. Cell-wise 독립 Mamba는 이 결합을 보지 못함: 각 cell이 "옆 cell에서 무슨 일이 일어나는지 모르는 채" 독립적으로 다음 상태를 예측

**핵심 질문**: Mamba가 디코더 이전(잠재 공간)에서 시간 전이를 수행하는 것이 적합한가? 물리적 공간 상관관계가 디코더/물리 loss에 의해 사후에야 반영되는 구조가 맞는가?

## 2. 대안 아키텍처 비교

### A. 현재 설계: Encoder → Mamba(latent, cell-wise) → Decoder

```
Encoder(Attn) → z(t) → Mamba(독립) → d(t) → Decoder(Attn+AdaLN) → State(t+1)
```

- 장점: 역할 분리 깔끔, Mamba O(N) 효율
- 단점: Mamba가 공간 결합 미반영, 디코더가 사후 교정 부담
- 자기회귀 시: 비일관적 예측 → 인코더 재입력 → 오차 누적 가능

### B. Encoder → Decoder → Temporal(physical space)

```
Encoder → Decoder → State_recon(t) → Temporal(물리 공간) → State(t+1)
```

- 장점: 시간 전이가 물리적으로 해석 가능한 공간에서 수행
- 단점: 인코더-디코더 역할 축소, 10ch×720=7200차원에서 시간 모델 작동

### C. Encoder → [Mamba + 주기적 Spatial Attention] → 경량 Decoder

```
Encoder → z(t) → [Mamba Block × N + Attention × M] → d(t) → Lightweight Decoder → State(t+1)
```

- 장점: Mamba 효율성 + 공간 결합을 시간 전이에 포함. 기상/물리 ML의 지배적 패턴
- 단점: 매 시점 Attention 추가 비용 (단, 720 token은 기상 모델 대비 매우 소규모)

### D. Predictor-Corrector (Encoder-Decoder ⊂ 시간 루프)

```
z(t) → Mamba → d(t) → Decoder → pred(t+1) → Physical Loss → Encoder → z(t+1) → ...
```

- 장점: 매 step 물리 공간 왕복, 물리 제약 직접 반영
- 단점: 매 step Encoder+Decoder 통과, parallel scan 어려움
- 참고: 자기회귀 추론 시에는 이미 이 구조로 동작

## 3. 기상/물리 ML 사례 조사

### 3.1 조사 결과 요약

기상/물리 ML에서 "잠재 공간 공간결합 + 경량 디코더" 패턴의 채택 현황:

| 모델 | 기관 (게재) | 인코더 | 프로세서 (잠재 공간) | 디코더 | 비율 |
|------|-----------|--------|-------------------|--------|------|
| **GraphCast** | DeepMind (Science 2023) | 1 GNN layer | **16 GNN layers** (message-passing) | 1 GNN layer | **1:16:1** |
| **FourCastNet** | NVIDIA (2022) | 1 projection | **12 AFNO layers** (Fourier 공간 혼합) | 1 linear projection | **1:12:1** |
| **GenCast** | DeepMind (Nature 2024) | 1 GNN layer | **16 sparse transformer** | 1 GNN layer | **1:16:1** |
| **ClimaX** | Microsoft (ICML 2023) | Cross-attn 집약 | **8 ViT self-attention** | 2-layer MLP | **~4:8:1** |
| **MeshGraphNets** | DeepMind (ICLR 2021) | 2-layer MLP | **15 msg-passing** | 2-layer MLP | **1:15:1** |
| **GNS** | DeepMind (ICML 2020) | 1 MLP | **10 msg-passing** | 1 MLP | **1:10:1** |
| **Pangu-Weather** | Huawei (Nature 2023) | 8 layers | U-Net 통합 | 8 layers | 대칭 (**유일한 예외**) |

### 3.2 패턴 분석

**7개 중 6개가 비대칭 설계**:
- 프로세서(잠재 공간 처리)에 전체 연산의 80-95% 집중
- 디코더는 1-layer projection 또는 2-layer MLP 수준
- 프로세서:디코더 비율 = 8:1 ~ 16:1

**잠재 공간에서의 공간 결합이 보편적**:
- GraphCast/GenCast: GNN message-passing (이웃 노드 간 정보 교환)
- FourCastNet: Fourier 변환 기반 전역 혼합 (주파수 공간에서 모든 격자점 결합)
- ClimaX: Self-Attention (모든 토큰 간 직접 상호작용)
- 공통점: **시간 전이 과정에서 공간 토큰 간 상호작용 필수**

**디코더는 "물리 공간으로의 투영(projection)"**:
- 핵심 연산(공간 결합 + 시간 전이)은 프로세서가 수행
- 디코더는 잠재 표현을 물리 변수로 변환하는 역할만 담당
- 이는 **후보정(post-processing)에 가까움**

### 3.3 우리 아키텍처와의 대응

| 기상 모델 요소 | 우리 대응 | 비고 |
|---|---|---|
| 격자점 (수만 개) | 720 cell token | 기상 대비 매우 소규모 → Attention 비용 부담 적음 |
| 물리 변수 (기온, 기압, 풍속) | 10ch 상태 | 유사 규모 |
| GNN/Attention 공간 결합 | **현재 없음 → 추가 필요** | C안의 핵심 |
| 시간 전이 (autoregressive) | Mamba (cell-wise) | 기상 모델은 GNN/Transformer |
| 경량 디코더 | **현재 무거운 디코더 계획 → 경량화 검토** | C안 적용 시 |

## 4. Mamba + Attention 하이브리드 현황 (2024-2026)

C안 채택 시 "Mamba + 주기적 Attention" 교차 배치가 핵심. 이 패턴의 검증 현황:

### 4.1 도메인별 주요 사례

| 모델 | 기관 | 도메인 | SSM:Attention 비율 | 핵심 설계 |
|------|------|--------|-------------------|----------|
| Jamba | AI21 (ICLR 2025) | LLM | 7:1 | 교차 배치 + MoE |
| Nemotron-H | NVIDIA (2025) | LLM | ~6:1 (24 Mamba-2, 4 Attn) | 균등 분산 |
| Hymba | NVIDIA (ICLR 2025) | LLM | 5:1 | **병렬 융합** (동일 레이어 내) |
| Samba | Microsoft (ICLR 2025) | LLM | ~1:1 | Sliding Window Attention |
| Qwen3-Next | Alibaba (2025) | LLM | 3:1 | Gated DeltaNet + Attention |
| OLMo Hybrid | AI2 (2026) | LLM | 3:1 | GDN hybrid > pure GDN > Transformer |
| MambaVision | NVIDIA (CVPR 2025) | Vision | 계층적 (초기 Mamba, 후기 Attn) | 단계별 배치 |
| MNO | Cheng et al. (JCP 2025) | PDE | Mamba 단독 | Attention 없음 (비교 대상) |

### 4.2 합의 사항

- **최적 비율**: SSM:Attention = **3:1 ~ 10:1** (70-93% SSM, 7-25% Attention)
- **7-8% Attention만으로 충분** (NVIDIA Nemotron-H에서 실증)
- **Attention이 제공하는 대체 불가 기능**: 연관 기억(associative recall), 전역 문맥 파악. SSM을 아무리 확장해도 이 기능을 대체하지 못함 (2026년 3월 증명)
- **SSM이 제공하는 대체 불가 기능**: 선형 복잡도, 고정 메모리, 장기 시퀀스 효율

### 4.3 주의: 도메인 차이

위 사례는 대부분 **LLM 도메인**이다. 기상/물리 도메인에서 Mamba+Attention 하이브리드는 아직 초기 단계:
- MNO (Mamba Neural Operator): Mamba 단독으로 PDE에 적용, Attention 미사용
- 기상 모델(GraphCast, GenCast): GNN 또는 Transformer 단독, Mamba 미사용
- **Mamba + Attention 하이브리드를 물리 시뮬레이션에 적용한 주요 사례는 아직 없음**

따라서 C안은 **LLM의 검증된 패턴 + 기상 모델의 검증된 패턴을 결합하는 새로운 시도**로 볼 수 있다.

## 5. 위험도 평가

### 5.1 낮은 위험 (채택 근거 충분)

| 항목 | 근거 |
|------|------|
| "잠재 공간 공간결합" 패턴 | GraphCast(Science), GenCast(Nature), FourCastNet(NVIDIA), ClimaX(ICML). 최상위 기관/저널에서 6/7 채택 |
| "경량 디코더" 패턴 | 동일 사례에서 프로세서:디코더 = 8:1~16:1. 디코더 경량화는 검증된 표준 |
| 720 token 규모의 Attention 비용 | 기상 모델은 수만 격자점에서 Attention 사용. 720은 매우 소규모 → 비용 문제 없음 |
| 구현 복잡도 | 기존 인코더의 FullAttention3D 코드 재활용 가능 |

### 5.2 중간 위험 (모니터링 필요)

| 항목 | 근거 | 대응 |
|------|------|------|
| **Mamba+Attention 결합이 물리 도메인에서 미검증** | LLM에서 검증, 기상/PDE에서는 사례 없음 | A안 baseline 먼저 구축 → C안으로 전환 비교 |
| **디코더 경량화 시 p_load 조건 반영 약화** | 현재 AdaLN-Zero 디코더가 경량 projection으로 바뀌면 p_load 효과 감소 가능 | 경량 디코더에도 AdaLN 1-layer 유지, 또는 프로세서 내 Attention에서 p_load 처리 |
| **Mamba:Attention 최적 비율 탐색** | LLM의 3:1~10:1이 우리 문제에 그대로 적용되는지 불확실 | Mamba 2~4 blocks + Attention 1 block으로 시작, 실험으로 조절 |
| **잠재 공간에서의 물리적 해석성** | 잠재 공간에서 공간 결합이 일어나므로, 물리적 의미 파악이 어려울 수 있음 | Physical Loss가 최종 출력 교정. 잠재 예측 loss(CDP) 추가로 보강 |

### 5.3 높은 위험 (신중한 검토 필요)

| 항목 | 근거 | 대응 |
|------|------|------|
| **원자로 도메인 고유의 결합 강도** | 기상: 격자점 간 결합이 점진적 (풍속, 기압 전파). 원자로: 중성자 확산이 즉각적으로 전체 노심에 영향 (고유값 문제). 결합 강도가 본질적으로 다름 | 실험적으로만 확인 가능. L_diff_rel이 이 차이를 포착하는 "경보" 역할 |
| **IIET/Neural PC 미도입** | Predictor-Corrector 없이 단일 forward pass로 물리 일관성 달성 가능한지 불확실 | Phase 1에서 단일 pass 성능 확인 → 부족 시 D안(PC 루프) 검토. IIET 성숙도 개선 시 재도입 |

## 6. 재분석: 디코더 FullAttention + Physical Loss의 gradient 경로

### 6.1 추가 고찰 (2026-04-15)

§2의 초기 분석에서 "cell-wise 독립 Mamba가 공간 결합을 보지 못한다"는 우려를 제기했으나, **공간 디코더가 인코더와 동일하게 FullAttention을 수행한다면** 상황이 달라진다.

#### Gradient 역전파 추적

```
Forward:
  Encoder(FullAttn ×3) → z(t) → Mamba(cell-wise) → d(t) → Decoder(FullAttn + AdaLN) → pred(t+1)
                                                                                           ↓
                                                                                    Physical Loss
                                                                                (L_diff_rel, L_Bateman)

Backward:
  Physical Loss
      ↓  ∂L/∂pred
  Decoder(FullAttn)  ← 공간 결합 정보가 gradient에 내포됨
      ↓  ∂pred/∂d(t)  ← "cell A의 d(t)가 cell B의 물리 정합성에 미치는 영향" 이 전달됨
  Mamba              ← 간접적으로 공간 결합을 학습
      ↓
  Encoder
```

**핵심 발견**: 디코더의 FullAttention이 공간 결합을 수행하므로, Physical Loss의 gradient가 디코더를 통과할 때 **"cell A의 출력이 cell B의 물리 정합성에 미치는 영향"이 gradient에 자연스럽게 내포**된다. Mamba는 이 gradient를 받아서, 비록 자기 자신은 cell-wise 독립이지만, **결과적으로 공간 결합을 존중하는 출력을 내도록 학습**할 수 있다.

#### A안 재평가: 두 개의 공간 결합 모듈 사이의 Mamba

```
인코더(FullAttn ×3)  →  Mamba(cell-wise)  →  디코더(FullAttn + AdaLN)  →  Physical Loss
    ↑ 공간 결합 1차         ↑ 시간 전이             ↑ 공간 결합 2차 + 물리 교정      ↑ 물리 정합성 검증
```

Mamba는 **두 개의 공간 결합 모듈 사이에 끼여** 있다:
1. **인코더 FullAttn**: 입력 z(t)에 이미 inter-cell 공간 정보가 인코딩됨 → Mamba는 공간 맥락이 포함된 입력을 받음
2. **디코더 FullAttn**: 출력 d(t)를 공간적으로 재결합하여 물리 정합성 확보 → Physical Loss가 이를 검증
3. **Physical Loss gradient**: 공간 일관성 위반 정보가 디코더 FullAttn을 경유하여 Mamba에 도달 → Mamba가 공간 결합을 **간접적으로** 학습

#### C안과의 재비교

| 관점 | A안 (FullAttn 디코더 유지) | C안 (Mamba+Attn + 경량 디코더) |
|------|--------------------------|-------------------------------|
| 공간 결합 수행 위치 | 인코더 + **디코더** | 인코더 + **프로세서** |
| Physical Loss → Mamba gradient 경로 | Loss → Decoder(FullAttn) → Mamba (**짧음, 풍부한 gradient**) | Loss → Decoder(경량) → Processor(Mamba) (**경량 디코더가 gradient 정보를 충분히 전달하지 못할 수 있음**) |
| 물리 정합성 교정 주체 | **디코더 FullAttn** (Physical Loss 바로 앞에서 교정) | 프로세서 Attention (Physical Loss까지 디코더 하나 거침) |
| 구현 복잡도 | 현재 계획 그대로 | Mamba 수정 + 디코더 재설계 필요 |

**결론**: C안에서 디코더를 경량화하면 Physical Loss의 gradient가 Mamba에 도달할 때 **공간 결합 정보가 부족**해질 수 있다. 반면 A안에서 디코더를 FullAttention으로 충실히 구성하면, Physical Loss가 디코더의 풍부한 공간 결합을 경유하여 Mamba에 **강한 공간 결합 gradient**를 전달한다.

### 6.2 자기회귀 추론 시의 추가 안전장치

자기회귀 루프에서 매 step마다:
```
state_pred(t+1) → Encoder(FullAttn) → z(t+1) → Mamba → d(t+1) → Decoder(FullAttn) → state_pred(t+2)
```
인코더 FullAttn이 **매 step 공간 결합을 재수립**한다. 설령 Mamba가 부분적으로 비일관적인 d(t)를 출력하더라도, 다음 step에서 인코더가 공간 관계를 다시 잡아준다.

## 7. 결론 및 권장 사항

### 7.1 핵심 판단

1. **현재 설계(A안)의 cell-wise 독립 Mamba는 직접적으로는 공간 결합을 수행하지 않는다.** 그러나 이것이 곧 "물리적으로 불완전하다"는 의미는 아니다.

2. **디코더에 FullAttention이 포함되고 Physical Loss가 적용되면**, Mamba는 gradient 역전파를 통해 공간 결합을 **간접적으로 학습**할 수 있다. 디코더의 FullAttention이 gradient에 공간 결합 정보를 내포시키는 역할을 한다.

3. 따라서 **A안(현재 설계)을 우선 채택**한다:
   - 인코더: FullAttention ×3 (공간 결합 1차, 이미 완료)
   - 시계열: Mamba cell-wise (시간 전이, 코드 미착수)
   - 디코더: **FullAttention + AdaLN-Zero** (공간 결합 2차 + p_load 조건 + 물리 정합성)
   - Physical Loss: L_diff_rel, L_Bateman, L_σXe (물리 정합성 검증 + gradient 제공)

4. **C안(Mamba+Attention + 경량 디코더)은 대안으로 보류**한다:
   - A안의 실험 결과에서 Physical Loss 수렴이 불충분하거나, 장기 롤아웃에서 공간 비일관성이 누적될 경우에 전환 검토
   - 기상/물리 ML 사례는 C안의 타당성을 뒷받침하므로, 전환 시 참고

### 7.2 A안 성공의 전제 조건

A안이 작동하려면 디코더 설계가 핵심이다:

| 조건 | 이유 |
|------|------|
| 디코더에 **FullAttention** 필수 | 공간 결합을 수행하고 Physical Loss gradient에 공간 정보를 내포시키는 역할 |
| 디코더에 **AdaLN-Zero** 필수 | p_load 전역 조건을 공간 재구성에 반영 |
| **Physical Loss** 가중치 적절 설정 | L_diff_rel이 공간 결합 위반을 충분히 벌칙해야 gradient가 유의미 |
| 자기회귀 학습(pushforward) | 인코더 재입력을 통한 매 step 공간 결합 재수립이 실제로 작동하는지 확인 |

### 7.3 실패 판단 기준 (C안 전환 트리거)

다음 상황 발생 시 C안으로 전환을 검토한다:

| 지표 | 임계값 | 의미 |
|------|--------|------|
| L_diff_rel 수렴 정체 | 100 epoch 이후 감소 둔화 | 디코더 gradient만으로 Mamba가 공간 결합을 충분히 학습하지 못함 |
| 롤아웃 200 step 이후 오차 폭발 | 상대오차 > 10% | 인코더 재입력에 의한 공간 재수립이 오차 누적을 억제하지 못함 |
| 인접 cell 간 물리량 불연속 | GT 대비 불연속 패턴 발생 | cell-wise 독립 예측의 공간적 비일관성이 교정되지 않음 |

### 7.4 향후 검토 사항

- **Global token (4~8개)**: A안에서도 Mamba에 Global token을 추가하여 전역 상태(keff 추세, AO 등)를 유지하는 방안. C안 대비 훨씬 가벼운 개선. 독립적으로 검토 가능
- **Gated DeltaNet**: OLMo Hybrid(AI2, 2026)에서 Mamba-2보다 우수 성능 보고. Mamba-2 대신 GDN 적용 검토
- **IIET/Neural PC**: 성숙도 개선 시 D안의 Predictor-Corrector 반복 구조 재도입 검토
- **잠재 예측 loss (CDP)**: Mamba에 직접 gradient를 제공하여, 디코더 경유 gradient의 보조 수단으로 활용 (04_Dreamer-CDP.md 참조)
