## 1. 문제 정의

### 1.1. 예측 태스크

**1-step autoregressive prediction**:
```
state(t) + rod_command(t+1) → state(t+1)
```
- state: (20, 5, 5, 10) — power, T_cool, T_fuel, ρ_cool, N_Xe, N_I, φ_g1, φ_g2, σ_a^Xe_g1, σ_a^Xe_g2
- rod_command: rod_map_3d (20, 5, 5) + p_load (스칼라)
- 추론 시 autoregressive 반복: state(0) → state(1) → ... → state(575)

### 1.2. 데이터 이중 구조

| 데이터 | 구조 | 시간 연속 | 용도 | 규모 (per scenario) |
|---|---|:---:|---|---:|
| **CRS** | 575 timestep 시계열 | **✓** | temporal dynamics 학습 | 575 |
| **Branch** | 575 × 29 = 16,675 스냅샷 | **✗** | rod 위치 변화에 대한 공간 응답 학습 | 16,675 |

- **CRS (Critical Rod Search)**: 매 timestep t에서의 임계 제어봉 위치. t → t+1 상태 전이가 물리적으로 연속
- **Branch 29개**: 같은 시점 t에서 rod만 offset한 가상 분기. 시간 진행 없음, 연소도 반영 없음. "if rod가 이 위치였다면" 가상 상황

### 1.3. 핵심 도전

1. **575-step rollout 오차 누적**: autoregressive inference에서 1-step 오차가 누적되어 후반부 발산 위험
2. **CRS + Branch 혼합 학습**: 시계열(CRS)과 비시계열(Branch)을 같은 모델로 학습하는 방법
3. **Cell-wise 독립 처리의 한계**: 각 cell의 시간 시퀀스를 독립 처리하면 cell 간 시간적 동기 (예: 제어봉 삽입 시 전역 transient) 표현 한계

### 1.4. 데이터 규모

| 항목 | 규모 |
|---|---|
| LP 수 | 100 (40 + 60) |
| 시나리오/프로필 | 20 종 (train 16 / val 2 / test 2) |
| CRS 학습용 timestep | 100 LP × 16 profiles × 575 ≈ **920,000** |
| Branch 학습용 쌍 | 100 × 16 × 575 × 29 ≈ **26,680,000** |
| 각 timestep shape | (20, 5, 5, 10) = 5,000 값 |

## 2. 아키텍처 — Cell-wise Mamba SSM

### 2.1. 전체 파이프라인

```
[데이터 입력] (B, T, 20, 5, 5, 21)              ← quarter core
       │
[halo_expand] → (B, T, 20, 6, 6, 21)           ← 05_symmetry_mode
       │
[공간 인코더] per timestep
  CellEmbedder → LAPE → FullAttention3D × 3     ← 01~03 결정
       │
       ▼ (B, T, 20, 6, 6, D_latent)
[reshape] → (B×720, T, D_latent)                 ← cell 축을 batch에 합침
       │
[Mamba SSM] cell-wise 공유 가중치 ★ 본 문서 대상
       │
       ▼ (B×720, T, D_hidden)
[reshape] → (B, T, 20, 6, 6, D_hidden)
       │
[공간 디코더] per timestep
  → (B, T, 20, 6, 6, C_out)                     ← L_diff_rel, L_data_halo 적용 지점
       │
[final crop] → (B, T, 20, 5, 5, C_out)
```

### 2.2. Mamba 블록 구조

S6 Selective State Space Model (Gu & Dao, 2023):

```
[입력 x_t] (D_latent)
    │
    ├─ [Linear ×2] → (D_inner), (D_inner)     ← 입력 의존 게이팅
    │       │              │
    │  [Conv1D causal]    [SiLU]
    │       │              │
    │  [SSM (S6)]         gate
    │       │              │
    │  [SiLU]             │
    │       └──── × ──────┘                    ← element-wise 곱
    │              │
    └── [Linear] → (D_latent)                  ← output projection
              │
         + residual
```

**S6 Selective Scan** — 핵심 메커니즘:
```
연속: h'(t) = A·h(t) + B(x_t)·x(t),  y(t) = C(x_t)^T·h(t)
이산: h_t = Ā·h_{t-1} + B̄_t·x_t,     y_t = C_t^T·h_t

B_t = f_B(x_t),  C_t = f_C(x_t),  Δ_t = softplus(f_Δ(x_t))
```

- B, C, Δ가 **입력 의존** → 제논 과도현상 vs 정상 운전 구분, 제어봉 이벤트에 선택적 반응 가능
- A 행렬: diagonal (HiPPO 초기화) → 장기 기억
- 선형 시간 복잡도: O(T) scan (Transformer의 O(T²) 대비)

### 2.3. Cell-wise 공유 가중치 — 학계 선례

| 모델 | 공간 처리 | 시간 처리 | 공유 가중치 |
|---|---|---|:---:|
| **Pangu-Weather** (Bi et al., Nature 2023) | 2D attention | per-grid-point Transformer | ✓ |
| **FourCastNet v2** (Bonev et al., 2023) | Spherical Harmonics | per-grid-point operator | ✓ |
| **MNO** (Cheng et al., JCP 2025) | neural operator | per-point Mamba | ✓ |
| **본 프로젝트** | FullAttention3D (spatial encoder) | per-cell Mamba (720 cells) | ✓ |

**물리적 정당성**:
- 각 cell의 시간 변화 (Xe 축적, 온도 변화 등) 는 local 물리량에 주로 의존
- cell 간 공간 결합 (중성자 확산 누설) 은 **공간 인코더/디코더의 attention이 담당**
- Mamba는 시간 방향 동역학만 학습 → 역할 분리 (spatial ↔ temporal)

### 2.4. Mamba SSM 채택 근거 — Transformer 대비

**MNO (Cheng et al., JCP 2025)** — Mamba Neural Operator:
- 4개 2D PDE 벤치마크 (Darcy Flow, Shallow Water, Diffusion-Reaction, CFD2D) 에서 Transformer 대비 **~90% 오차 감소**
- **autoregressive rollout에서 오차 누적이 현저히 적음** ← 575-step rollout에 가장 중요
- 선형 시간 복잡도 O(T) vs Transformer O(T²) ← T=575에서 유의미한 차이

**PGMNO (Neural Networks 2026)** — Physics-Guided Mamba:
- implicit BDF 적분을 SSM에 내장 → stiff ODE (Xe 동역학) 안정성 향상

**결론**: Mamba가 본 문제의 temporal backbone으로 가장 적합

## 3. 데이터 경로 설계

### 3.1. 이중 경로 — CRS vs Branch

```
┌──────── CRS 경로 (autoregressive) ────────┐
│                                           │
│  state(t) ──┐                             │
│  rod(t+1) ──┤── [halo_expand] ──┐         │
│  xs_fuel ───┘                   │         │
│                                 ▼         │
│                          [Spatial Encoder] │
│                                 │         │
│                          (B×720, T, D)    │
│                                 │         │
│                          [Mamba SSM]  ★   │
│                                 │         │
│                          [Spatial Decoder] │
│                                 │         │
│                          state(t+1)_pred  │
│                                 │         │
│                    ┌──── autoregressive ──┘
│                    │     loop (inference)
│                    ▼
│               state(t+1) → state(t+2) → ...
└───────────────────────────────────────────┘

┌──────── Branch 경로 (single-step) ────────┐
│                                           │
│  state(t) ──┐                             │
│  rod(t,δ) ──┤── [halo_expand] ──┐         │
│  xs_fuel ───┘                   │         │
│                                 ▼         │
│                          [Spatial Encoder] │  ← 가중치 공유
│                                 │         │
│                          (B, 720, D)      │
│                                 │         │
│                          [Mamba 우회] ✗   │  ← Branch는 SSM 안 거침
│                                 │         │
│                          [Spatial Decoder] │  ← 가중치 공유
│                                 │         │
│                      perturbed_state(t,δ) │
└───────────────────────────────────────────┘
```

### 3.2. 핵심 설계 결정

| 항목 | CRS 경로 | Branch 경로 |
|---|---|---|
| **Spatial Encoder** | 공유 | 공유 |
| **Mamba SSM** | **사용** | **우회 (bypass)** |
| **Spatial Decoder** | 공유 | 공유 |
| **시계열 길이** | T = 575 (parallel scan으로 전체 처리) | T = 1 (single-step) |
| **rod 입력** | rod_map(t+1): 다음 시점 rod | rod_map(t, offset=δ): 현재 시점 + offset |
| **Loss** | L_data + L_data_halo + L_diff_rel + L_Bateman + L_σXe | L_data + L_data_halo (물리 loss는 선택) |

### 3.3. Rod Conditioning — FiLM

Branch 경로에서 rod_offset의 효과를 공간 feature에 주입하는 방식:

```python
# rod conditioning (FiLM)
rod_embed = Dense(d_model)(rod_command)     # (B, d_model) 또는 (B, T, d_model)
gamma = Dense(d_model)(rod_embed)           # 채널별 스케일
beta = Dense(d_model)(rod_embed)            # 채널별 이동
conditioned_feat = gamma * encoder_out + beta
```

- **CRS 경로**: rod_map(t+1)이 21ch 입력의 일부로 이미 concat됨 → 추가 FiLM 불필요 (기존 설계대로)
- **Branch 경로**: rod_offset(δ)을 별도 FiLM으로 conditioning → encoder output을 변조 후 decoder에 전달
- **디코더에서 p_load**: AdaLN-Zero로 별도 변조 (기존 결정 유지)

> **참고**: 현재 설계에서 rod_map은 이미 입력 21ch에 포함. FiLM 추가 도입은 Branch 경로에서 rod_offset의 *delta 효과*를 명시적으로 강조하기 위함. 필수는 아니며 ablation 대상

## 4. 학습 전략 — 2단계 + 보조 손실

### 4.1. 전체 학습 일정

```
Phase 1 — Branch 사전학습 (spatial encoder + decoder + rod conditioning)
  │  데이터: Branch 29개/timestep × 575 × (100 LP × 16 profiles)
  │  경로: encoder → [Mamba 우회] → decoder
  │  목적: 제어봉 위치 → 공간 응답 매핑 학습
  │  Mamba: 가중치 초기화 상태 (학습 안 함)
  │  기간: 수렴까지 (loss plateau)
  │
  ▼
Phase 2 — CRS 공동 미세조정 (전체 파이프라인)
  │  데이터: CRS 575-step 시계열 + Branch 보조 손실
  │  경로: encoder → Mamba → decoder (autoregressive)
  │  목적: temporal dynamics 학습
  │  기법: pushforward + curriculum + scheduled sampling (§5)
  │  Branch 보조 손실: λ_branch ≈ 0.1~0.3 (catastrophic forgetting 방지)
  │  기간: 수렴까지
  │
  ▼
(선택) Phase 3 — Neural PC 도입 (baseline 성능 평가 후)
     디코더 r회 반복 (가중치 공유) → MASTER Full PC 커플링 학습
     L_diff_rel과 시너지 가능
     상세: `archive/2026-03-30 Neural PC(IIET) 도입 고찰.md`
```

### 4.2. Phase 1 — Branch 사전학습

**목적**: encoder + decoder가 "rod 위치 → 공간 응답" 관계를 충분히 학습한 상태로 Phase 2 진입

**이점**:
- Branch 데이터가 CRS보다 **29배 풍부** → encoder/decoder 초기화 품질 향상
- 제어봉 민감도 학습이 CRS autoregressive 학습의 전제 조건 (rod 변화에 반응 못 하면 시계열 학습 무의미)
- Mamba SSM은 이 단계에서 학습하지 않으므로 빠른 수렴

**Loss (Phase 1)**:
```
L_phase1 = L_data(inner) + λ_halo · L_data_halo + λ_diff · L_diff_rel
```
- L_data: state 예측 MSE (inner 5×5)
- L_data_halo: halo cell MSE (λ=0.3)
- L_diff_rel: 선택 (Albedo BC 학습 필요 시)

**학계 근거**: MPP (McCabe et al., NeurIPS 2024) — 서로 다른 물리 시스템에서 사전학습한 backbone이 특정 시스템으로 전이 시 우수

### 4.3. Phase 2 — CRS 공동 미세조정

**Loss (Phase 2)**:
```
L_phase2 = L_crs + λ_branch · L_branch

L_crs = L_data + λ_halo · L_data_halo + λ_Bateman · L_Bateman
      + λ_σXe · L_σXe + λ_diff · L_diff_rel

L_branch = L_data_branch + λ_halo · L_data_halo_branch
```
- λ_branch ≈ 0.1~0.3: Branch 보조 손실 → catastrophic forgetting 방지
- L_crs: CRS 시계열의 모든 물리 손실 포함
- L_branch: Phase 1과 동일 형태, 가중치만 낮춤

**CRS와 Branch의 배치 구성**:
- 옵션 A: **Mixed batch** — 각 배치에 CRS 시퀀스 + Branch 스냅샷 혼합
- 옵션 B: **Alternating batch** — CRS 배치와 Branch 배치를 번갈아 학습
- **권장: 옵션 A** (gradient가 매 step 양쪽 경로에 동시 흐름 → 안정적)

## 5. Autoregressive 훈련 기법

### 5.1. 기법 적용 순서

Phase 2 내에서 아래 기법을 순차/병행 적용:

| 기법 | 적용 시점 | 핵심 | 우선순위 |
|---|---|---|:---:|
| **Pushforward trick** (k=4~8 unroll) | Phase 2 처음부터 | distribution shift 완화 | **최우선** |
| **Scheduled sampling** | Phase 2 중기 | teacher forcing 1.0 → 0.0 점진 전환 | 권장 |
| **Noise injection** (σ=0.01~0.1) | 보조 | 입력에 Gaussian noise 추가 | 선택 |

### 5.2. Pushforward Trick — 최우선 적용

**원리** (Brandstetter et al., ICLR 2023):

훈련 시 모델 자신의 출력을 다음 입력으로 사용하여 test-time distribution과 일치시킴:

```
일반 teacher forcing:
  Loss = ‖f_θ(state_GT(t)) − state_GT(t+1)‖²    ← 항상 GT 입력

Pushforward (k=2 예시):
  ŝ(t+1) = f_θ(state_GT(t))                       ← 1-step: GT 입력
  ŝ(t+2) = f_θ(ŝ(t+1))                            ← 2-step: 자기 예측 입력
  Loss = ‖ŝ(t+2) − state_GT(t+2)‖²               ← 2-step의 오차만 역전파
```

- k-step unrolling 후 마지막 step의 loss만 계산 → 모델이 자기 노이즈에 강건해짐
- **k=4~8 권장**: k가 너무 크면 메모리 부담 + gradient vanishing, 너무 작으면 효과 미미

**L_total**:
```
L_total = L_one-step + λ_push · L_pushforward
L_pushforward = ‖f_θ(ŝ(t+k-1)) − state_GT(t+k)‖²
```

### 5.3. Scheduled Sampling

```
epoch 1~10:   teacher_forcing_ratio = 1.0 (100% GT 입력)
epoch 10~30:  teacher_forcing_ratio = 1.0 → 0.5 선형 감소
epoch 30~50:  teacher_forcing_ratio = 0.5 → 0.0
epoch 50~:    teacher_forcing_ratio = 0.0 (100% 자기 예측)
```

- 각 timestep에서 확률적으로 GT 또는 자기 예측을 입력으로 선택
- pushforward trick과 병행 가능 (상호 보완적)

### 5.4. 시퀀스 처리

Mamba의 parallel scan은 575 step 전체를 한번에 처리할 수 있다. GPU 메모리가 부족한 경우에만 gradient checkpointing을 적용하여 메모리를 절약하되, gradient 흐름은 전체 시퀀스에 걸쳐 유지한다.

> **참고**: TBPTT(Truncated Backpropagation Through Time)는 GRU/LSTM 시대의 기법으로, parallel scan이 가능한 Mamba/SSM 계열에서는 사용하지 않는다. DreamerV3의 Block GRU는 TBPTT를 사용하지만, Drama의 Mamba-2는 parallel scan으로 이를 대체하였다.
