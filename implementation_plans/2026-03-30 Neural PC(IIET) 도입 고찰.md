## 1. 문제 정의: 왜 PC가 필요한가

### 1.1 현재 상황

우리 모델의 Bateman ODE 적분은 **시점 t의 GT 값만 사용**하여 1회 적분:

```
입력: φ(t), σ(t), T_f(t), N_Xe(t), N_I(t)   ← 전부 시점 t의 MASTER GT
  ↓ Bateman ODE (Euler 또는 해석해)
출력: N_Xe(t+1) 예측
  ↓ 비교
MASTER GT: N_Xe(t+1)                          ← Full PC로 계산된 값
  → 오차: mean ~0.9%, max ~4.4%
```

### 1.2 오차의 원인

MASTER는 **1회 적분 후 전체 물리량을 재계산**하는 Full PC를 수행.
우리는 이 재계산 없이 1회만 적분 → 중간 갱신이 빠져 있음.

이 ~0.9%는 적분 방법(Euler vs 해석해)의 차이가 아님 (둘 다 ~0.9%).
**구조적 차이** — MASTER가 중간에 수행하는 커플링 재계산이 없기 때문.

## 2. MASTER Full PC의 작동 원리

> 매뉴얼 확인: 항상 Full Predictor-Corrector 사용 (idepl 파라미터 Not Working).

### 2.1 단계별 흐름

```
시점 t의 상태: φ(t), T_f(t), ρ_m(t), σ(t), N_Xe(t), N_I(t)

┌─────────────────────────────────────────────────────────┐
│ Step 1. Predictor (Bateman ODE 적분)                     │
│                                                          │
│   φ(t), σ(t)로 Bateman 해석해 적분                       │
│   → N_Xe_임시(t+1), N_I_임시(t+1)                       │
│                                                          │
│   이 시점에서 N_Xe가 바뀌었지만, φ와 T_f는 아직 옛날 값   │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Step 2. 중성자 수송 재계산                                │
│                                                          │
│   N_Xe_임시(t+1)가 Σ_a에 영향 → 확산방정식 다시 풀기      │
│   → φ(t+1) 갱신                                         │
│                                                          │
│   핵심: N_Xe가 증가하면 흡수↑ → φ가 감소                  │
│         이 φ 변화가 인접 노드로 전파 (공간 커플링)         │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Step 3. 열수력 재계산                                    │
│                                                          │
│   φ(t+1) → 에너지 침적량 변화 → 열전달 방정식 풀기        │
│   → T_f(t+1), ρ_m(t+1) 갱신                             │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Step 4. 단면적 재계산                                    │
│                                                          │
│   T_f(t+1), ρ_m(t+1) → Taylor 전개로 σ(t+1) 갱신        │
│   (MASTER 내부에서 MAS_XSL 계수 사용)                     │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Step 5. Corrector (Bateman ODE 재적분)                   │
│                                                          │
│   갱신된 φ(t+1), σ(t+1)의 **가중평균**으로 Bateman 재적분 │
│   → N_Xe_최종(t+1), N_I_최종(t+1)                       │
│                                                          │
│   이제 모든 물리량이 서로 일관 (self-consistent)          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 핵심: 왜 이게 정확한가

각 물리량이 다른 물리량에 의존하는 **순환 관계**:

```
N_Xe ←→ φ: N_Xe 증가 → 흡수↑ → φ 감소 → N_Xe 생성↓
φ ←→ T_f: φ 변화 → 에너지 침적 변화 → T_f 변화
T_f ←→ σ: T_f 변화 → 스펙트럼 변화 → σ_a^Xe 변화
σ ←→ N_Xe: σ 변화 → 소멸항 변화 → N_Xe 변화
```

1회 적분으로는 이 순환이 반영되지 않음. Full PC는 이 순환을 한 바퀴 돌려서 self-consistent 상태를 찾음.

## 3. IIET의 작동 원리 (단계별 설명)

### 3.1 배경: Transformer의 residual connection = ODE

일반 Transformer 블록의 residual connection:
```
y_{n+1} = y_n + F(y_n, θ)
```

이것은 **Euler 전진법**과 수학적으로 동일:
```
ODE: dy/dt = F(y)
Euler: y_{n+1} = y_n + h · F(y_n)   (h = step size = 1)
```

즉 Transformer의 각 레이어를 통과하는 것 = ODE를 한 스텝 적분하는 것.

### 3.2 IIET: Implicit Euler의 반복 풀이

**Explicit Euler** (일반 Transformer):
```
y_{n+1} = y_n + F(y_n)       ← y_n만 사용 (현재 값)
```

**Implicit Euler**:
```
y_{n+1} = y_n + F(y_{n+1})   ← y_{n+1}이 양변에 있음!
```

y_{n+1}을 모르는데 y_{n+1}을 써야 함 → **fixed-point iteration**으로 풀기:

```
pred^0 = y_n + F(y_n)                    ← 1회차: 보통 Transformer
pred^1 = y_n + F(LayerNorm(pred^0))      ← 2회차: pred^0을 다시 넣음
pred^2 = y_n + F(LayerNorm(pred^1))      ← 3회차: pred^1을 다시 넣음
...
pred^r = 최종 출력
```

### 3.3 Forward 전파 (r=2 예시)

```
입력 x (예: Mamba hidden state h_t)

┌─ 1회차 (초기 예측) ──────────────────────────────┐
│ pred^0 = DecoderBlock(x)                          │
│                                                    │
│ x=[B,20,5,5,D] → Attention → FFN → pred^0        │
│                   (가중치 W)                       │
└────────────────────────┬───────────────────────────┘
                         ↓
┌─ 2회차 (1차 보정) ───────────────────────────────┐
│ input_1 = x + LayerNorm(pred^0)                   │
│ pred^1 = DecoderBlock(input_1)                    │
│                                                    │
│ 같은 DecoderBlock, 같은 가중치 W                   │
│ 하지만 입력이 다름: x가 아닌 x + LN(pred^0)       │
└────────────────────────┬───────────────────────────┘
                         ↓
┌─ 3회차 (2차 보정) ───────────────────────────────┐
│ input_2 = x + LayerNorm(pred^1)                   │
│ pred^2 = DecoderBlock(input_2)                    │
│                                                    │
│ 같은 DecoderBlock, 같은 가중치 W                   │
│ 입력: x + LN(pred^1)                              │
└────────────────────────┬───────────────────────────┘
                         ↓
출력: pred^2 (최종)
```

**핵심**: DecoderBlock은 **하나**. 가중치 W는 3회 모두 **동일**.
달라지는 것은 **입력**뿐 — 이전 iteration의 출력을 피드백으로 받음.

### 3.4 Backward 역전파

```
Loss = L_data(pred^2, GT) + L_Bateman(pred^2) + ...

∂L/∂pred^2
    ↓ DecoderBlock 역전파 (가중치 W에 대한 gradient 1)
∂L/∂input_2 = ∂L/∂(x + LN(pred^1))
    ↓ LayerNorm 역전파
∂L/∂pred^1
    ↓ DecoderBlock 역전파 (가중치 W에 대한 gradient 2)
∂L/∂input_1 = ∂L/∂(x + LN(pred^0))
    ↓ LayerNorm 역전파
∂L/∂pred^0
    ↓ DecoderBlock 역전파 (가중치 W에 대한 gradient 3)
∂L/∂x

가중치 W의 총 gradient = gradient_1 + gradient_2 + gradient_3
```

**효과**: 동일한 가중치 W가 3개의 서로 다른 gradient 신호를 받음.
- gradient_1: "최종 출력을 직접 개선해라"
- gradient_2: "중간 보정을 개선해라"
- gradient_3: "초기 예측을 개선해라"

→ 1회 통과보다 **풍부한 학습 신호** → 같은 파라미터 수로 더 나은 성능.

### 3.5 LayerNorm (RK-Norm)이 필수인 이유

LayerNorm 없이 반복하면:
```
pred^0 = F(x)
pred^1 = F(x + pred^0)        ← pred^0의 스케일이 x와 다를 수 있음
pred^2 = F(x + pred^1)        ← 스케일 불일치 누적 → 발산
```

LayerNorm이 각 iteration의 출력을 **정규화**하여 스케일을 맞춤 → 안정적 수렴.
논문: "RK-Norm 없으면 학습 완전 실패"

## 4. MASTER Full PC vs IIET 비교

### 4.1 근본적 차이

| | MASTER Full PC | IIET |
|---|---|---|
| **무엇을 반복하는가** | 물리 방정식 (확산, 열수력, Taylor, Bateman) | 신경망 블록 (Attention + FFN) |
| **보정 기준** | 물리 법칙 (명시적) | 학습된 표현 (암묵적) |
| **self-consistency** | 물리적으로 보장 | 학습에 의존 (보장 안됨) |
| **물리량 간 커플링** | 명시적: φ→T_f→σ→N_Xe→φ | 암묵적: 신경망이 채널 간 관계 학습 |
| **적용 도메인** | 원자로 물리 전용 | 범용 (NLP, CV 등) |

### 4.2 비유

**MASTER Full PC**:
"시험 답안 작성 → 채점관(물리 법칙)이 틀린 부분을 지적 → 물리 공식으로 해당 부분 재계산"
→ 채점 기준이 명확하고 정확함

**IIET**:
"시험 답안 작성 → 같은 시험을 다시 풀되, 이전 답안을 참고 → 반복할수록 답이 수렴"
→ 채점관 없이 스스로 개선. 물리 법칙을 직접 쓰지 않음

### 4.3 IIET가 MASTER Full PC를 대체할 수 있는가?

**직접 대체: 불가능.** IIET는 물리 방정식을 풀지 않으므로.

**간접 효과: 가능성 있음.** 이유:
1. Physical Loss(L_Bateman, L_diffusion 등)가 학습 신호로 작동
2. IIET 반복이 이 Physical Loss를 최소화하는 방향으로 feature를 개선
3. 결과적으로 **물리 일관성이 높은 출력**이 나올 수 있음

단, 이건 **검증 필요한 가설**이지 보장이 아님.

## 5. 우리 모델에의 도입 경로

### 5.1 경로 A: IIET (암묵적 보정)

디코더 블록을 r회 반복. 물리 방정식은 호출하지 않음.

```
[인코더] → latent       (1회)
[Mamba]  → h(t)         (1회)
[디코더] h(t) → pred^0                           (초기)
         h(t) + LN(pred^0) → [같은 디코더] → pred^1  (보정 1)
         h(t) + LN(pred^1) → [같은 디코더] → pred^2  (보정 2)
```

- 장점: 구현 단순, 파라미터 증가 없음, end-to-end 미분 가능
- 단점: 물리 일관성 보장 안됨, 효과는 학습에 의존

### 5.2 경로 B: 명시적 물리 루프

디코더 출력의 물리량을 물리 공식으로 보정 → 재입력.

```
[디코더] → pred^0 = {φ̂, T̂_f, ρ̂_m, N̂_Xe, ...}

[물리 보정]
  1. T̂_f, ρ̂_m → Taylor → σ̂(t+1)         ← 미분 가능
  2. φ̂, σ̂, N̂_Xe → Bateman → N_Xe_보정    ← 미분 가능
  3. 보정된 물리량을 디코더 입력에 반영

[디코더] h(t) + 보정된 물리량 → pred^1 (최종)
```

- 장점: 물리 커플링을 **명시적으로** 수행. MASTER Full PC와 구조적으로 유사.
- 단점: 구현 복잡, 수송방정식(φ 재계산)은 미포함 (확산방정식 FDM 필요)

### 5.3 경로 C: IIET + 물리 보정 혼합 (하이브리드)

IIET 반복 안에서 물리 보정을 삽입.

```
[디코더] → pred^0

[물리 보정] pred^0의 T̂_f, ρ̂_m → Taylor → σ̂_보정
           pred^0의 φ̂, σ̂_보정 → Bateman → N_Xe_보정

[IIET iteration]
  h(t) + LN(pred^0) + 물리보정잔차 → [같은 디코더] → pred^1
```

- 장점: 물리 신호를 IIET iteration에 주입 → 물리 일관성 유도 + 암묵적 개선
- 단점: 가장 복잡한 구현

### 5.4 경로 비교

| | 구현 난이도 | 물리 일관성 | 파라미터 | 추론 비용 | MASTER PC 유사도 |
|---|:---:|:---:|:---:|:---:|:---:|
| 경로 A (IIET) | 낮음 | 보장 안됨 | 동일 | ×(r+1) | 낮음 |
| 경로 B (물리 루프) | 중간 | **명시적** | 동일 | ×2 | 중간 (φ 재계산 미포함) |
| 경로 C (하이브리드) | 높음 | 부분 명시적 | 동일 | ×(r+1) | 가장 높음 |
| 미도입 (baseline) | — | 없음 | 동일 | ×1 | 없음 |

## 6. 결론 및 권고

### 6.1 현재 단계에서의 판단

1. **IIET는 MASTER Full PC를 직접 대체하지 않음** — 물리 방정식을 풀지 않는 범용 수학 기법
2. 그러나 **Physical Loss와 결합 시** 물리 일관성을 암묵적으로 개선할 가능성 있음
3. **경로 B(명시적 물리 루프)**가 MASTER Full PC와 가장 유사하며, Bateman + Taylor 보정은 전부 미분 가능
4. 수송방정식 재풀이(φ 재계산)는 어떤 경로에서도 직접 구현이 어려움 → L_diffusion이 이를 보조

### 6.2 구현 우선순위

| 순서 | 작업 | Phase | 비고 |
|:----:|------|:-----:|------|
| 1 | **baseline 먼저 구현** (PC 루프 없이) | SE/T/SD | 인코더 + Mamba + 디코더 1회 통과 |
| 2 | **baseline 성능 확인** | — | ~0.9% 오차가 실제 성능에 미치는 영향 측정 |
| 3 | 경로 B (물리 보정) 시도 | SD | Bateman + Taylor 보정 → 디코더 재통과 |
| 4 | 경로 A (IIET) 비교 실험 | SD | 물리 보정 없이 순수 반복 → 경로 B와 비교 |
| 5 | 경로 C (하이브리드) 선택적 | SD | 경로 B가 부족할 경우 |

### 6.3 SE-Phase에서의 영향

**없음**. PC 루프는 디코더(SD-Phase) 범위이며, 공간 인코더는 **입력 물리량을 latent로 압축하는 역할만** 담당. SE-Phase 구현은 PC 루프 도입 여부와 무관하게 진행 가능.

### 6.4 잠재적 문제

| # | 문제 | 대응 |
|---|------|------|
| 1 | IIET 효과 불확실 | baseline 먼저 구현 후 비교 실험으로 판단 |
| 2 | 물리 루프의 오차 전파 | α 보간: `phi_safe = α·pred^0.φ + (1-α)·GT.φ`, α는 학습 가능 |
| 3 | RK-Norm 누락 시 학습 실패 | 각 iteration 전 LayerNorm **필수** |
| 4 | 추론 오버헤드 | 초기 r=1, 성능 개선 확인 후 r=2 |
| 5 | 수송방정식 미포함 | L_diffusion이 공간 커플링을 gradient로 보조 |
