## 1. 파이프라인 전체 흐름

```
[데이터 생산] config.yaml
  lp_geometry: 'quarter_core'
  symmetry_type: 'rotation'
      │
      ▼ save_lps() 시 자동 생성
  lp_generation_record.json
  {lp_geometry, symmetry_type, seed, ...}
      │
      ▼
[전처리] config_preproc.yaml
  lp_geometry: 'quarter_core'          ← 동일 필드명
  symmetry_type: 'mirror'              ← 100LP 기준 (mirror)
      │
      ▼ dataset_builder._write_metadata()
  HDF5 metadata attrs
  {lp_geometry: 'quarter_core', symmetry_type: 'mirror'}
      │
      ▼  ★ 모델 구현 시 여기서 읽음 ★
[모델 dataloader]
  sym = h5f['metadata'].attrs['symmetry_type']    # 'mirror' or 'rotation'
  geo = h5f['metadata'].attrs['lp_geometry']      # 'quarter_core' etc.
      │
      ├──▶ halo_expand(quarter, sym=sym)           # 올바른 모드로 halo 생성
      ├──▶ Conditional LAPE(x, sym)                # 대칭 유형별 위치 임베딩 선택
      └──▶ L_diff_rel의 stencil ghost neighbor     # halo cell 값 사용
```
