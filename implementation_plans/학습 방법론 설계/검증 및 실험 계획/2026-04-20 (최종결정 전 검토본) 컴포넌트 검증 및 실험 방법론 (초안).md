### 0.2 향후 검토 예정 항목

- 컴포넌트별 수렴 판정 임계값 (loss/정확도 수치)
- 제거 실험 (Ablation Study) 조합 중 우선순위 (모든 조합 실행 불가)
- 구간별 오차 가중치 (정출력/감출력/저출력/상승)
- Phase 전환 자동 트리거 vs 수동 확인 여부
- 외부 자문 결과 반영 (UltraReview 이후)

### 0.3 핵심 원칙

본 문서의 모든 절차는 다음 원칙으로 수렴:

> **"한 번에 하나씩, 가장 단순한 것부터, 기준점(baseline) 대비 측정하라."**
> — Karpathy, "A Recipe for Training Neural Networks" (2019)

## 1. 용어 안내

| 영문 | 한국어 | 의미 |
|------|-------|------|
| Sanity Check | 기초 건전성 검사 | 학습 시작 전 기본 정상 동작 확인 |
| Single-batch Overfitting | 단일 배치 과적합 | 2~5개 샘플로 loss ≈ 0 수렴시켜 구조 정상성 확인 |
| Ablation Study | 제거 실험 | 컴포넌트를 하나씩 빼고 성능 비교 |
| Baseline | 기준점 | 비교 기준이 되는 가장 단순한 모델 |
| OFAT (One-Factor-At-a-Time) | 단일 요인 변경 실험 | 한 번에 한 가지 변수만 바꿔 인과 관계 분리 |
| Champion/Challenger | 챔피언/챌린저 | 현재 최선 모델에 신규 변형을 도전시켜 우세 시 교체 |
| Rollout | 전개 | 예측을 다음 step 입력으로 주입해 장기 시퀀스 생성 |
| Probing Classifier | 탐침 분류기 | 중간 표현 위에 얕은 선형 분류기 부착 → 표현의 유의미성 검사 |

### 1.1 테스트 ID 체계 (전 섹션 공통)

훈련 단계에서 수행되는 모든 테스트는 **TR-** 접두어로 시작. MLflow experiment name 및 artifact 이름에 직접 사용. 섹션 간 교차 참조 가능.

| 접두 | 카테고리 | 범위 | 소속 섹션 | 예시 |
|:----:|---------|------|:--------:|------|
| **TR-S-XX** | Sanity check | TR-S-01 ~ TR-S-06 | §2.1 | TR-S-01 (loss 초기값), TR-S-06 (dead activation) |
| **TR-V-XXX** | Verification (컴포넌트 단위) | 모듈별 접미어: SE(인코더), SP-M(Mamba), SP-A(Attention), DC(디코더), IN(통합) | §2.2 | TR-V-SE-A3-T8, TR-V-SP-M1, TR-V-DC-1, TR-V-IN-1 |
| **TR-R-XX** | Removal (제거 실험, Ablation) | TR-R-01 ~ TR-R-10 + TR-R-B0~B3 (Baseline), TR-R-Full | §3.2 | TR-R-03 (Mamba 제거), TR-R-B2 (Phase 1 only baseline) |
| **TR-X-XX** | eXchange (교체 실험, Swap) | TR-X-01 ~ TR-X-07 (Top-7 매핑) | §8.2 | TR-X-04 (p_load 경로), TR-X-06 (Mamba:Attn 비율) |
| **TR-M-XX** | Metric (평가 지표) | TR-M-01 ~ TR-M-06 | §5 | TR-M-02 (575-step rollout) |
| **TR-P-X** | Phase checkpoint | TR-P-1, TR-P-2a, TR-P-2b, TR-P-3 | §4 | TR-P-2b (Phase 2b 완료 시점) |

### MLflow experiment name 규칙

```
phase{N}_{TR-ID}_{short-desc}
```

예시:
- `phase1_TR-S-01_loss_init`
- `phase1_TR-V-SE-A3-T8_conditional_lape`
- `phase2a_TR-X-06_mamba_attn_ratio_1to1`
- `phase2b_TR-R-03_mamba_removal`
- `phase3_TR-V-IN-4_rollout_drift`

**필터 활용**:
- `phase2b_*` → Phase 2b의 모든 실험
- `*_TR-X-*` → 전 Phase의 교체 실험
- `*_TR-V-SE-*` → 전 Phase의 공간 인코더 검증

### 정적 V&V 와의 구분

기존 `v&v/03_encoder_components/` 의 SE-A1~B3 shape/parameter count 테스트는 **코드 작성 직후 정적 검증** 이므로 TR- 접두 미부여 (기존 ID 유지). 단, **학습 후 재확인** 항목 (SE-A3-T8~T10, SE-B3-T6, SE-B4~B6) 은 훈련 단계 테스트이므로 본 문서에서 **TR-V-SE-A3-T8** 형식으로 표기.

## 2. 1부. 컴포넌트 유효성 검증 방법론

### 2.1 Sanity Check 체크리스트 (구현 직후 필수)

코드 작성 직후 "학습이 가능한 상태"인지 확인하는 최소 점검 항목. 기존 `v&v/03_encoder_components/` SE-A/B 패턴 확장.

| ID | 항목 | 방법 | 판정 기준 |
|---|------|------|----------|
| TR-S-01 | Loss 초기값 | 무작위 초기화 직후 첫 loss 출력 | MSE는 데이터 분산과 같은 자릿수. 비정상(NaN, 0) 발생 시 초기화 버그 |
| TR-S-02 | 단일 배치 과적합 | 2~5개 샘플 고정 + epoch 100+ | loss → 초기값의 1/100 이하 |
| TR-S-03 | Gradient 존재 | `tape.gradient(loss, model.trainable_variables)` 각 변수 norm 측정 | 모든 학습 대상에서 norm > 0 |
| TR-S-04 | Gradient 범위 | gradient norm 분포 히스토그램 | 최대/최소 비율 ~10^3 이내 (소실/폭발 방지) |
| TR-S-05 | 데이터 순서 불변성 | 셔플 seed를 2종으로 바꿔 10 epoch 학습 | 최종 loss 차이가 학습 곡선 내 진동 수준 |
| TR-S-06 | Dead activation | GELU 출력 중 0 비율 | < 5% (SE-B3-T3 패턴) |

**기준 참고**: [cs231n.github.io/neural-networks-3/](https://cs231n.github.io/neural-networks-3/), [karpathy.github.io/2019/04/25/recipe/](http://karpathy.github.io/2019/04/25/recipe/)

### 2.2 컴포넌트별 검증 항목

본 프로젝트의 5개 주요 컴포넌트별 세부 검증 항목. V&V ID는 기존 SE-A/B 체계 뒤에 이어서 부여 (SP: Spatiotemporal Processor, DC: Decoder, IN: Integration).

#### 2.2.1 공간 인코더 (Spatial Encoder) — ✓ 일부 완료, 학습 후 재확인

기존 SE-A1~B3 완료. 학습 후 "미결" 표기된 항목들:

| ID | 항목 | 방법 | 판정 기준 | 시점 |
|---|------|------|---------|:----:|
| TR-V-SE-A3-T8 | Conditional LAPE 효과 | 같은 cell에서 mirror vs rotation 케이스 임베딩 비교 | 두 케이스 출력이 유의미하게 달라야 함 | TR-P-1 |
| TR-V-SE-A3-T9 | LAPE 스케일 지배 여부 | 학습 후 α norm 측정 | α가 폭주(>10)하지 않음 | TR-P-1 |
| TR-V-SE-A3-T10 | LAPE norm map 패턴 | boundary vs interior cell의 LAPE norm 플롯 | 물리적으로 해석 가능한 패턴 | TR-P-1 |
| TR-V-SE-B3-T6 | Gradient flow 재확인 | 학습 중/후 gradient norm 분포 | Phase 1 과정 중 분포 유지 | Phase 1 과정 |
| **TR-V-SE-B4** (신규) | 탐침 분류기 테스트 | 인코더 출력에 선형 layer 부착 → Xe/power 예측 | Linear probe RMSE가 trivial baseline보다 유의하게 낮음 | TR-P-1 |
| **TR-V-SE-B5** (신규) | Attention map 시각화 | 특정 cell에서 attention weight를 (6×6) 격자로 출력 | 인접 cell 집중 등 구조적 패턴 확인 | TR-P-1 |
| **TR-V-SE-B6** (신규) | sym_type 민감도 | rotation vs mirror 동일 입력 → 인코더 출력 차이 | 유의미한 차이 (수치 비교) | TR-P-1 |

**참고**: 탐침 분류기(Probing Classifier)는 [Alain & Bengio (2016)](https://arxiv.org/abs/1610.01644)가 제안. 중간 표현의 정보량을 측정하는 표준 방법.

#### 2.2.2 시공간 프로세서 — Mamba 블록

| ID | 항목 | 방법 | 판정 기준 |
|---|------|------|---------|
| TR-V-SP-M1 | h(t) 정보 전달 | h(t) 있을 때 vs 없을 때(h=0 주입) t+1 예측 오차 비교 | h(t) 있을 때 오차 현저히 낮아야 함 |
| TR-V-SP-M2 | Parallel scan gradient 분포 | 첫 step과 마지막 step의 gradient norm 비교 | 575 step 전체에 고른 분포 (비율 < 10^2) |
| TR-V-SP-M3 | p_load concat 반응성 | p_load=0.5 vs 1.0 출력 차이 | 유의미한 출력 변화 |
| TR-V-SP-M4 | 단일 CRS 시퀀스 과적합 | 1개 LP(575 step)에 대해 loss → 0 | Mamba 시간 표현 용량 확인 |
| TR-V-SP-M5 | h(t) 포화 확인 | 각 step h(t) norm 추이 플롯 | burn-in 후 안정화. 발산 시 정규화 추가 |

**참고**: Mamba의 h(t) 해석은 ["The Hidden Attention of Mamba Models" (2024)](https://arxiv.org/html/2403.01590v1) — hidden state를 implicit attention으로 해석하고 시각화.

#### 2.2.3 시공간 프로세서 — Attention 블록

| ID | 항목 | 방법 | 판정 기준 |
|---|------|------|---------|
| TR-V-SP-A1 | Attention map 동역학 패턴 | 정출력 step vs 감출력 step에서 attention map 비교 | 동역학 변화 구간에서 패턴이 달라져야 함 |
| TR-V-SP-A2 | p_load AdaLN-Zero 효과 | p_load 값별 γ/β 출력 플롯 | 초기 0 → 학습 후 유의미한 값. γ/β가 p_load와 연동 |
| TR-V-SP-A3 | 공간 결합 필요성 | Attention 제거 (Mamba만) vs 전체 모델 비교 | 공간 분포 예측에서 전체 모델이 우수 |

#### 2.2.4 디코더 (경량 Conv3D 1×1×1)

| ID | 항목 | 방법 | 판정 기준 |
|---|------|------|---------|
| TR-V-DC-1 | **역정규화 후** 출력 물리 범위 | 역정규화된 power, Xe 농도 등의 min/max/std 측정 | 역변환 후 물리 단위에서 학습 데이터 분포 내 유지 (정규화 공간에서는 음수 정상 — 통합 정리 §4.2 참조) |
| TR-V-DC-2 | Gradient 전달 | L_physics gradient가 프로세서까지 도달하는지 norm 측정 | 디코더 전후 norm 비율이 합리적 |
| TR-V-DC-3 | 디코더 표현력 한계 | 1층 vs 2층 비교 | 2층이 유의미한 이득 없으면 1층 채택 |

#### 2.2.5 전체 파이프라인 통합

| ID | 항목 | 방법 | 판정 기준 |
|---|------|------|---------|
| TR-V-IN-1 | Step별 오차 누적 곡선 | 1/10/100/575 step 예측 오차 플롯 | step에 따른 오차 증가율이 안정적 |
| TR-V-IN-2 | 구간별 오차 분리 | 정출력(144) / 감출력(36) / 저출력(72) / 상승(36) step 오차 분리 측정 | 급변 구간 오차 급증 여부 확인 |
| TR-V-IN-3 | Branch 예측 일관성 | CRS 시점 t에서 rod_map만 다른 29 Branch 예측의 물리적 일관성 확인 | rod 삽입 깊이에 대한 단조성 등 물리적 패턴 |

**참고**: Autoregressive rollout 안정성은 ["Towards Stability of Autoregressive Neural Operators" (2023)](https://arxiv.org/html/2306.10619v2) — 시공간 대리모델의 핵심 평가 축.

## 3. 2부. 실험 방법론 — 비교·선별 체계

### 3.1 3단계 실험 절차

**1단계 — 기준점 확립**

기준점 없이는 어떤 "개선"도 의미가 없음 (출처: [CMU ML Blog, 2020](https://blog.ml.cmu.edu/2020/08/31/3-baselines/)). 본 프로젝트의 기준점 후보:

| ID | 구조 | 목적 |
|----|------|------|
| TR-R-B0 (Trivial) | 직전 step 값 복사 (persistence) | 변화가 느린 정출력 구간 하한선 |
| TR-R-B1 (Linear 외삽) | 시간 방향 선형 외삽 | 저출력 안정 구간 하한선 |
| TR-R-B2 (Phase 1 only) | 공간 인코더 + Attention 학습된 상태 (Mamba 없음, step 독립 예측) | 시간 동역학 학습 이전 수준 |
| TR-R-B3 (Full no-physics) | 전체 아키텍처 + L_data만 (L_physics 제거) | 물리 제약의 효과 분리 |

**2단계 — 단일 요인 변경 실험 (OFAT)**

- 한 번에 하나의 컴포넌트/하이퍼파라미터만 변경 → 인과 관계 명확
- 한계: 상호작용 효과는 잡지 못함 (필요 시 2x2 격자 실험으로 보완)
- 출처: [Wikipedia: One-Factor-At-a-Time](https://en.wikipedia.org/wiki/One-factor-at-a-time_method)

**3단계 — 챔피언/챌린저 토너먼트**

- 각 단계에서 가장 우수한 구성 = 챔피언
- 신규 변형 = 챌린저. 챌린저가 우세 시 챔피언 교체
- MLOps 표준 관리 기법 (출처: [DataRobot Blog](https://www.datarobot.com/blog/introducing-mlops-champion-challenger-models/))

### 3.2 제거 실험 (Ablation Study) 후보 목록

물리 loss 항의 기여도 측정 (ablation) 에 한정. 아키텍처 / Phase 구조 / p_load 주입 등 의 대안 비교 는 §8 TR-X (A/B swap) 로 일임.

| ID | 제거/변경 대상 | 확인 목적 | 예상 소요 |
|:---:|---|---|:---:|
| TR-R-B0~B3 | (위 Baseline 표 참조) | 하한선 설정 | 소 |

## 7. 주의 사항

- **실행 시간 제약**: 전체 제거 실험 소요 시간 > 3개월 가능. 우선순위에 따라 "필수" 케이스부터 착수, "선택"은 시간 여유 시
- **하이퍼파라미터 재튜닝**: 제거 실험 시 하이퍼파라미터 자동 최적화 없이 "원본과 동일" 설정 유지. 이는 OFAT 원칙 준수 위함. 단, 수치 발산 시 최소한의 LR 조정 허용
- **통계적 유의성**: 동일 실험 최소 3 seed 반복. 단일 run 차이는 우연일 수 있음
- **Seed 관리**: seed 값을 config에 기록. 재현성 유지
- **"제거했더니 더 좋았다" 케이스**: 해당 컴포넌트가 불필요하거나, 하이퍼파라미터 미튜닝이거나, 버그일 가능성. 반드시 원인 분석 후 판단

## 8. 부품 카탈로그 및 교체 비교 프로토콜

> §3.2 **제거 실험 (Ablation)** 과 **교체 실험 (Swap)** 은 다른 관점:
> - 제거 실험: 부품을 뺐을 때 성능 비교 (있냐 없냐)
> - 교체 실험: 부품을 다른 후보로 바꿨을 때 성능 비교 (A냐 B냐)
>
> 본 장은 **교체 실험** 의 체계를 정의한다. 설계 과정에서 검토되었으나 2순위로 밀린 차선책들을 "끼워 넣기 가능한 부품" 으로 정리.

### 8.1 설계 철학 — 모듈 부품화 (Pluggable Module Pattern)

**원칙**: 각 부품을 **독립 클래스** 로 구현하고, 부품 조합은 **YAML config** 로 지정한다. 부품 교체 = config 한 줄 변경.

**TF2/Keras 구현 가이드**:

```python
# src/models/processor/stblock.py (예시)
class STBlock(tf.keras.layers.Layer):
    def __init__(self, temporal_block, spatial_block, p_load_injector, ...):
        super().__init__()
        self.temporal = temporal_block      # 주입 (Mamba-3, Mamba-2, GRU, ...)
        self.spatial = spatial_block        # 주입 (FullAttn, BlockGrid, ...)
        self.p_load_inject = p_load_injector  # 주입 (AdaLN-Zero, FiLM, concat-only)
        ...
```

**Config 예시**:

```yaml
processor:
  block_count: 3   # baseline N=3
  temporal:
    type: mamba3          # 기준 | 후보: mamba2, s4, gru
    config: {state_dim: 32, complex_state: true, trapezoidal: true}   # Drama 16 → 32 확장
  spatial:
    type: full_attention  # 기준 | 후보: block_grid, sparse
    config: {heads: 4, dim: 128}
  p_load_inject:
    path_mamba: concat            # 기준 | 후보: none
    path_attention: adaln_zero    # 기준 | 후보: none, film
  lape_skip:
    type: raw_alpha_wd    # 기준 | 후보: rezero, layernorm_lape, none
```

**효과**:
- 교체 실험 = config diff 한 줄 + 동일 학습 파이프라인 재실행
- 부품 클래스 재사용 → 코드 중복 제거
- Registry 패턴 (`TEMPORAL_BLOCKS = {"mamba3": Mamba3Block, "mamba2": Mamba2Block, ...}`) 으로 확장 용이

### 8.2 Top-7 토론 집중 결정 매트릭스

**선정 원칙**: 설계 과정에서 실제로 "고군분투한" 결정만 교체 실험 대상으로 선정. 판단 기준:

- **외부 자문** 을 구한 적이 있는가
- 고려된 **대안이 3개 이상** 이었는가
- **결정이 반전/재검토** 된 이력이 있는가
- **미해결 질문** 이 남아 있는가

위 4개 중 **2개 이상 해당** 하는 결정만 아래 표에 포함. `(최종결정 전 검토본)` 14개 문서 스캔 결과 7개 결정이 선정됨.

### 8.2.2 "관찰 대상" — 스왑 아닌 모니터링/경험적 튜닝

설계 결정이 아니라 **학습 중 관찰 + 사후 튜닝** 영역.

- Attention 크기 (D/H 조합) — 초기 학습 정체 시 확장 고려
- L_data_halo λ 가중치 (기본 0.3)
- Branch 샘플링 비율 (Phase별)
- 차등 학습률 수치 (Phase 2b의 E:1e-5, A:5e-5, M:1e-3)
- K 커리큘럼 세부 스케줄 (K=2→4→...→575 각 단계 epoch 수)

이들은 Top-7 교체 실험 외에 **MLflow 로깅 기반 모니터링 + 필요 시 스윕 실험** 으로 처리.

### 8.4 직렬 테스트 프로토콜

#### 8.4.1 의존성 그래프 (Top-7 기반)

```
[상류: 구조 결정]                    [하류: 세부 튜닝]
  TR-X-01 시간 아키텍처                TR-X-04 p_load 주입
```

- **상류 (TR-X-01, TR-X-02)**: 교체 시 전체 재학습. Phase 완료 후 "차기 버전 사이클" 로만 가능
- **하류 (TR-X-03~07)**: 기존 baseline checkpoint 에서 swap + 재학습. 실험 비용 낮음

#### 8.4.2 Tier 분류 (Top-7 재매핑)

| Tier | 포함 | 시점 | 비용 |
|:----:|------|------|:---:|
| **T1 (필수)** | TR-X-04 (p_load), TR-X-05 (디코더), TR-X-06 (Mamba:Attn 비율), TR-X-03 (h(t) 관리) | TR-P-2b 직후 | 중 |
| **T2 (권장)** | TR-X-07 (L_diffusion 잔차 학습 방식) | TR-P-2b 직후 | 중 |
| **T3 (차기)** | TR-X-01 (시간 아키텍처), TR-X-02 (Phase 구조) | 본 버전 완료 후 별도 사이클 | 대 |

**Top-7 외 항목**: §8.2.2 관찰 대상은 Tier 외, 학습 로그 기반 사후 튜닝.

#### 8.4.3 실행 순서 (Phase 완료 시점 기반)

```
TR-P-1 (Phase 1 완료) 후:
  준비: TR-P-1 baseline checkpoint 확보 (Full C안 구성)
  (Top-7 중 상류 항목은 이 시점에서 교체 불가 — 하류만 가능)

TR-P-2a (Phase 2a 완료) 후:
  T1 · TR-X-06: Mamba:Attention 비율 sweep (2:1 ↔ 1:1 ↔ 3:1)
    → Phase 2a에서 가장 먼저 (프로세서 구조 확정)

TR-P-2b (Phase 2b 완료) 후:
  T1 · TR-X-04: p_load 주입 경로 (이중 ↔ 단일 AdaLN ↔ 단일 concat)
  T1 · TR-X-05: 디코더 (경량 Conv3D ↔ 2층 MLP ↔ FullAttn+AdaLN 원형)
  T1 · TR-X-03: h(t) 관리 (detach ↔ 제한적 gradient)
  T2 · TR-X-07: L_diffusion 잔차 학습 방식 (명시 7-point ↔ Cross-Attention 암묵)

TR-P-3 (Phase 3 완료) 후:
  최종 Full 모델 평가 (TR-M-01 ~ TR-M-06 전 지표)
  Top-7 T1/T2 실험 종합 → 챔피언 확정

차기 버전 (별도 사이클):
  T3 · TR-X-01: 시간 아키텍처 패턴 (C안 ↔ A안) — 대규모 재학습
  T3 · TR-X-02: Phase 구조 (3-Phase ↔ 2-Phase 단순화)
```

### 8.5 교체 실험 결과 기록 형식

각 swap 실험은 다음 템플릿으로 기록:

```yaml
# configs/experiments/TR-X-04_p_load_path_single.yaml
experiment:
  id: TR-X-04
  seed_run: 001
  mlflow_name: "phase2b_TR-X-04_p_load_single_adaln"
  tier: T1
  name: "p_load 주입 경로: 이중 → 단일 (AdaLN만)"
  baseline_checkpoint: phase2b_champion_v1.ckpt
  changes:
    - component: processor.p_load_injector
      from: {path_mamba: concat, path_attention: adaln_zero}
      to:   {path_mamba: none,   path_attention: adaln_zero}
  hypothesis: "이중 경로의 Mamba concat 경로가 실제로 성능에 기여하는가 확인"
  expected_delta: "미확정 (이론상 concat 제거 시 p_load 시간 응답 둔화 가능)"

metrics:
  - TR-M-01 (1-step MAE)
  - TR-M-02 (575-step rollout)
  - TR-M-03 (구간별 오차) — 정출력/감출력 전환 구간 중점
  - TR-M-04 (Branch 예측 오차) — rod_map 변화 응답

seeds: [42, 137, 2024]
```

- 파일: `v&v/0X_swap_experiments/TR-X-04_report.md`
- MLflow run name: `phase2b_TR-X-04_p_load_single_adaln` (seed별 run 3개)
- 내용: 기준 vs 교체 지표 비교 + 학습 곡선 + 챔피언 변경 여부 판정
- 판정 기준: 모든 seed에서 일관된 개선 시 챔피언 교체. 단일 seed 개선은 우연 간주

### 8.6 부품화 설계 체크리스트 (코드 구현 시)

새 부품 클래스 작성 시 다음 항목 확인:

1. **Registry 등록**: `TEMPORAL_BLOCKS[name] = Class` 패턴으로 YAML에서 이름으로 인스턴스화 가능
2. **계약 준수**: §8.3 입출력 shape/dtype 일치
3. **Config 스키마**: `__init__` 파라미터가 YAML 설정으로 모두 주입 가능
4. **단위 테스트**: Shape 검증 + Gradient flow 검증 (기존 SE-A1~B3 패턴)
5. **Checkpoint 호환성**: 기존 체크포인트에서 해당 부품만 재초기화 가능하도록 네임스페이스 분리
