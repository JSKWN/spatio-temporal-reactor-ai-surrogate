## 1. 용어 안내

낯선 영문 용어의 한국어 병기:

| 영문 | 한국어 | 의미 |
|------|------|------|
| **Autoregressive Rollout** | 자기회귀 전개 | 자기 예측을 다음 step 입력으로 사용 |
| **K-step Pushforward** | K-step 자기회귀 오차 학습 | K step 자기회귀 후 GT와 비교 (누적 오차 학습) |
| **Co-adaptation** | 상호 적응 | 두 모듈이 서로에 맞춰 함께 미세조정 |
| **Differential Learning Rate** | 모듈별 차등 학습률 | 모듈마다 다른 학습률 적용 |
| **Curriculum Learning** | 커리큘럼 학습 | 쉬운 샘플 → 어려운 샘플 순서 학습 |
| **Teacher Forcing** | (정답 강제 입력) | 학습 시 매 시점 GT를 입력으로 사용 |
| **Catastrophic Forgetting** | (치명적 망각) | 이전 학습 결과를 잃어버리는 현상 |

## 2. 학습 전략 — LLM 3단계 대응 (1차 정리)

| LLM 학습 단계 | 우리 프로젝트 | 학습 대상 | 데이터 | 학습 신호 |
|------------|---------|--------|------|---------|
| **Pretraining** | **Phase 1** 공간 사전학습 | Encoder + Attention + Decoder | Branch (~167만) | 1-step (시점별 GT 입력) |
| **SFT (Instruction)** | **Phase 2** 시공간 학습 (2a + 2b) | Phase 2a: Mamba 단독, Phase 2b: 전체 (상호 적응) | CRS 시퀀스 + Branch sampling | 575-step parallel scan (Teacher Forcing) |
| **RLHF (Alignment)** | **Phase 3** 자기회귀 물리 정렬 | 전체 (낮은 lr) | CRS + Branch (커리큘럼) | **K-step 자기회귀 (자기 예측 입력)** + Physical Loss 강조 |

### 우리 도메인의 강점

- LLM RLHF는 인간 피드백 (노이즈 있음) 사용
- 우리는 **원자로 노심 해석 코드의 Ground Truth** + **물리 제약 loss** 사용
- → 결정론적 GT가 존재하므로 RLHF보다 강한 정렬 가능

### Phase 본질 — 모델 구조는 동일, 학습만 다름

3 Phase 모두:
- **모델 구조 동일** (Encoder + Mamba + Attention + Decoder)
- **입출력 동일** (state, rod_map, p_load → state(t+1))

차이점:
1. **학습 활성 모듈** (Phase 1=공간, Phase 2=시간(+상호 적응), Phase 3=전체)
2. **사용 데이터** (Branch 위주 → CRS 위주 + Branch 보조 → 혼합 + 커리큘럼)
3. **Mamba 학습 신호** (1-step → 575-step parallel scan → **K-step 자기회귀 전개**)
4. **Loss 가중치** (균형 → 균형 → **Physical Loss 강조**)

## 3. 데이터 구조 (round-robin LP-profile 매칭)

| 단위 | 수량 | 비고 |
|------|------|------|
| 시퀀스 (LP, profile 1개씩 round-robin) | **100** | LP 수와 동일 |
| 각 시퀀스 길이 (CRS step) | **575 step** | 5분 간격 × 48시간 |
| 각 CRS step의 Branch | **30개** (b=0=critical, b=1~29=분기 29개) | rod_offset 변경 + 5분 진화 |
| **CRS 시점 합계** | **57,500** (= 100 × 575) | |
| **Branch 시점 합계** | **~1,667,500** (= 57,500 × 29) | CRS의 ~29배 |

### Branch 데이터 특성 (검증됨)

- **Frozen Xenon 아님**: 5분간 Xe 진화 정확히 반영 (검증: [piecewise-test/2026-04-20_branch_xenon_evolution_결과.md](../../piecewise-test/2026-04-20_branch_xenon_evolution_결과.md))
- b=0은 critical 자체, b=1~29가 실제 rod_offset 분기
- L_Bateman 적용 가능 (단일 step, 5분 진화 잔차)

### 부하추종 시간 프로파일

```
시간(h)  0────────12──15─────────21──24
출력(%)  100%      100→저출력    저출력    저출력→100%
Steps    144 steps  36 steps    72 steps  36 steps  ...

5분 간격, 총 575 steps ≈ 48시간
```

| 구간 | 특성 | 학습 난이도 |
|------|------|----------|
| 정출력 (144 steps) | 변화 거의 없음, Xe 평형 | 쉬움 |
| 감출력 (36 steps) | p_load 급변, 제어봉 대응, Xe 축적 시작 | 어려움 |
| 저출력 유지 (72 steps) | Xe 피크 → 감소 | 어려움 |
| 상승 (36 steps) | p_load 복귀, Xe 소멸 | 어려움 |

## 4. Phase 1 — 공간 사전학습 (Pretraining)

### 4.1 목적

- Encoder + Attention + Decoder가 "rod_map 변화 → 노심 공간 응답"을 학습
- 풍부한 Branch 데이터로 공간 응답 연산자 안정화
- Mamba는 1-step만 사용 (h(0)=0, 누적 없음)

### 4.2 학습 설정 (1차 정리)

```
학습 대상: Encoder + Attention + Decoder (Mamba는 1-step만 사용)
데이터: Branch (~167만 시점, 단일 step)
입력: GT state(t), rod_map, p_load → 예측 state(t+1)
학습 신호: 1-step (Teacher Forcing)

Loss: L_data + L_data_halo + L_diff_rel + L_Bateman + L_sigma_a_Xe
       (모든 시간 비의존 항목 + L_Bateman 단일 step 잔차)
       (L_keff는 후순위/검토중)

Optimizer: tf.keras.optimizers.AdamW(learning_rate=1e-3, weight_decay=1e-4)
```

### 4.3 L_Bateman Phase 1 적용 가능 근거

- Branch는 frozen Xenon 아님 ([검증](../디코더 구현 및 Physical Loss 적용/2026-04-20 Branch Xenon 진화 검증 — L_Bateman Phase 1 적용 가능성.md))
- 5분 시간 진화 반영 → 단일 step Bateman 잔차 학습 가능
- `(N_xe(t+5min) - N_xe(t)) / 300s ≈ ODE 우변` 형태로 학습

### 4.4 종료 트리거 (잠정)

- 공간 응답 학습 안정 (Loss 곡선 평탄화)
- L_diff_rel 잔차 목표 도달
- Phase 1 완료 시 인코더/Attention/디코더 가중치 저장

## 5. Phase 2 — 시공간 학습 (SFT, 2단계 분리, 1차 정리)

### 5.1 단계 분리 근거 — 완전 freeze의 함정

**위험** (외부 자문 검토):
- Phase 1에서 Attention은 "Encoder의 깨끗한 직접 출력"에 최적화됨
- Phase 2에서 Mamba 활성화 → Mamba 통과 후 잠재 벡터 분포 변화
- **Attention 완전 freeze 시 Mamba가 "구식 분포"로 끼워맞춰야 함** → 학습 병목

**해결**: Phase 2a (Mamba 단독) + Phase 2b (상호 적응) 단계 분리

**우리 ConditionalLAPE skip의 부분 완화**:
- Attention 직전 LAPE skip 재주입 → 위치 정보는 Phase 1 분포 일부 복원
- 그러나 Mamba 출력의 동역학적 잠재 표현 변화는 보호 못 함
- → Phase 2b 상호 적응 여전히 필요

### 5.2 Phase 2a (~30% epoch) — Mamba 단독 학습

```
학습 대상: Mamba / h(t)만
Freeze: Encoder + Attention + Decoder (lr = 0)
데이터: CRS 시퀀스 + Branch sampling
목적: Mamba가 시간 동역학에 빠르게 수렴 (안정적 시작점 확보)

Mamba LR: 1e-3 (표준)
다른 모듈 LR: 0 (freeze)
```

### 5.3 Phase 2b (~70% epoch) — 상호 적응 (Co-adaptation)

```
학습 대상: 전체 (Mamba + Encoder + Attention + Decoder)
Freeze 해제: Encoder/Attention/Decoder의 freeze 해제
데이터: CRS 시퀀스 + Branch sampling
목적: Mamba 출력 분포에 공간 모듈이 맞춰 미세조정 (상호 적응)
```

#### 5.3.1 모듈별 차등 학습률 (Phase 2b)

**개념**: 네트워크의 각 부분 (모듈, 블록, 레이어 그룹) 마다 **서로 다른 학습률을 적용**.
- 이미 수렴한 모듈은 낮은 lr로 보호 (큰 변화 시 학습 결과 파괴)
- 새로 학습하는 모듈은 높은 lr로 빠르게 적응

| 모듈 | 학습률 (잠정) | 이유 |
|------|------|------|
| **인코더** (Phase 1 학습 완료) | 매우 낮음 (예: 1e-5) | 이미 수렴, 흔들리면 학습 결과 파괴 |
| **Attention** (Phase 1 학습 완료) | 낮음 (예: 5e-5) | 인코더와 같은 이유. Mamba 출력 분포 적응 위해 약간 학습 허용 |
| **Mamba** (Phase 2a부터 새로 학습) | 표준 (예: 1e-3) | 시간 동역학 학습 진행 중, 빠른 적응 필요 |
| **디코더** (Phase 1 학습 완료) | 매우 낮음 또는 freeze | 출력 head, 크게 바뀔 필요 없음 |

#### 5.3.2 TF2/Keras 구현 패턴

```python
# 가중치 그룹별 학습률 분리 (Optimizer 여러 개)
opt_mamba = tf.keras.optimizers.AdamW(learning_rate=1e-3, weight_decay=1e-4)
opt_attn  = tf.keras.optimizers.AdamW(learning_rate=5e-5, weight_decay=1e-4)
opt_enc   = tf.keras.optimizers.AdamW(learning_rate=1e-5, weight_decay=1e-4)
opt_dec   = tf.keras.optimizers.AdamW(learning_rate=1e-5, weight_decay=1e-4)

with tf.GradientTape(persistent=True) as tape:
    pred = model(inputs)
    loss = compute_total_loss(pred, gt)

grads_mamba = tape.gradient(loss, mamba.trainable_variables)
grads_attn  = tape.gradient(loss, attention.trainable_variables)
grads_enc   = tape.gradient(loss, encoder.trainable_variables)
grads_dec   = tape.gradient(loss, decoder.trainable_variables)

opt_mamba.apply_gradients(zip(grads_mamba, mamba.trainable_variables))
opt_attn.apply_gradients(zip(grads_attn, attention.trainable_variables))
opt_enc.apply_gradients(zip(grads_enc, encoder.trainable_variables))
opt_dec.apply_gradients(zip(grads_dec, decoder.trainable_variables))
```

### 5.4 공통 사항 (Phase 2a, 2b 모두)

```
Loss: L_data + L_data_halo + L_diff_rel + L_Bateman + L_sigma_a_Xe + λ_branch · L_branch
       (Phase 1과 동일 + L_branch 보조)
       (L_keff는 후순위/검토중)

Mamba 학습 단위: 575-step parallel scan (GT 입력, Teacher Forcing)
```

#### 5.4.1 Mamba parallel scan 흐름

```
시점 0: GT_state(0)   → predict → pred(1)    vs GT_state(1)    → loss
시점 1: GT_state(1)   → predict → pred(2)    vs GT_state(2)    → loss
시점 2: GT_state(2)   → predict → pred(3)    vs GT_state(3)    → loss
   ...
시점 574: GT_state(574) → predict → pred(575) vs GT_state(575) → loss

(575 step 병렬 처리, parallel scan)
```

- 각 시점의 입력 = GT (이전 step의 예측 사용 X)
- 누적 오차 학습 X (매 시점 GT에서 새로 시작)
- 목적: Mamba가 1-step transition 정확히 학습

### 5.5 Branch 데이터의 Phase 2 역할

- Phase 1에서 Encoder/Decoder/Attention 충분히 학습됨
- Phase 2 Branch 사용 시: **"동일 (state, p_load) 조건에서 다른 rod_map → 다른 시간 진화 응답"을 Mamba가 학습**
- h(t) 공급: **CRS parallel scan에서 추출 → detach 후 Branch에 주입**
  - detach 필수 (Branch gradient가 CRS 시퀀스 역방향 역전파 차단)

### 5.6 λ_branch — 작게 할 필요 없음 (Phase 분리 효과)

| Phase | λ_branch 의미 | 권고 값 |
|-------|------------|------|
| Phase 1 | Branch가 주 학습 → 균형 조절 | (Branch 단독, λ 무관) |
| Phase 2a | freeze로 영향 범위 제한 → "보호" 불필요 | 작게 할 필요 없음. **CRS:Branch sampling 비율로 통제** |
| Phase 2b | 차등 학습률로 보호 (lr 자체가 1/10) | 작게 할 필요 없음 |

→ 데이터 비율 통제는 **CRS batch : Branch batch sampling 비율**로 통제

### 5.7 데이터 비율 vs 학습 비율 — Mamba 학습 보호 전략

**우려**: 데이터 비율 CRS:Branch = 1:29 → 무대책 학습 시 Mamba underfit

**완화 전략**:
1. **CRS 시퀀스 단위 학습** (parallel scan 575 step 동시 학습 → 시퀀스 1개 = Mamba에게 575 transition 학습)
2. **Branch sampling**: 매 CRS 시퀀스의 일부 step에서만 branch 평가 (예: 50/575 step)
3. **Phase 분리**: Phase 1에서 Branch 충분히 학습 → Phase 2에서 보조 역할만

## 6. Phase 3 — 자기회귀 물리 정렬 (Alignment, 신설, 1차 정리)

### 6.1 핵심 변경: Parallel Scan OFF, K-step Autoregressive ON

```
Phase 1, 2: parallel scan (GT 입력) → 1-step transition 학습
Phase 3: 1-step recurrence × K (자기 예측 입력) → K-step 누적 오차 학습
```

### 6.2 학습 흐름 (자기회귀 전개)

```
시점 t: GT_state(t) → predict → pred(t+1)
                                  ↓ (자기 예측을 다음 step 입력으로)
                               predict → pred(t+2)
                                           ↓
                                          ...
                                       predict → pred(t+K)
                                                   ↓
                                               vs GT_state(t+K) → loss
                                               (K-step 누적)
```

- **첫 입력만 GT**, 이후 K-1 step은 **자기 예측 사용**
- K step 동안의 누적 오차 학습
- 모델이 "**자기 오차를 다음 step에서 damping**" 하는 방법 학습

### 6.3 커리큘럼 학습 (K 점진 확장, 잠정)

```
K=2 → K=4 → K=8 → K=16 → K=32 → K=64 → K=128 → K=256 → K=575
       (각 K에서 안정화 후 다음 K로 진행)
```

- **K가 작을수록**: 누적 오차 작음, Data Loss 의미 있음
- **K가 클수록**: 누적 오차 큼, Data Loss 약해짐 → **Physical Loss로 정렬**
- 최종 K=575 (전체 시퀀스 자기회귀)

### 6.4 학습 설정

```
학습 대상: 전체 unfreeze (Encoder + Mamba + Attention + Decoder)
Mamba 동작: 1-step recurrence + InferenceParams cache (parallel scan OFF)
데이터: CRS 시퀀스 + Branch (커리큘럼 K 적용)
목적:
  1. Phase 1, 2에서 분리 학습된 모듈 간 gap 조정 (상호 적응 마무리)
  2. 자기회귀 누적 오차 damping 학습 (장기 안정성)
  3. 어려운 구간 (감출력/저출력/상승) 물리 정확도 향상
```

### 6.5 모듈별 차등 학습률 (Phase 3, 잠정)

Phase 2와 동일 원칙. 단, **Phase 2b 대비 1/5 ~ 1/20** 추가 감소:

| 모듈 | Phase 3 학습률 (잠정) | Phase 2b 대비 |
|------|------------|------------|
| **Mamba** | 5e-5 ~ 1e-4 | × 1/10 ~ 1/20 |
| **Attention** | 1e-5 ~ 5e-5 | × 1/5 ~ 1/10 |
| **Encoder** | 1e-6 ~ 1e-5 | × 1/10 |
| **Decoder** | 1e-6 ~ 1e-5 | × 1/10 |

→ 전체적으로 작은 lr로 미세조정 (Phase 1, 2 학습 결과 보존)

### 6.6 Loss 가중치 변화 (K에 따라, 잠정)

| K | Data Loss 의미 | Physical Loss 가중치 |
|---|------------|------------------|
| K=2~4 | 의미 있음 (궤적 큰 차이 X) | 표준 (Phase 2와 동일) |
| K=8~16 | 약해짐 (궤적 어긋남 시작) | ↑ (예: × 1.5) |
| K=32~128 | 더 약해짐 (궤적 발산 가능) | ↑↑ (예: × 2~3) |
| K=575 | 거의 의미 없음 | ↑↑↑ (Physical Loss가 주, 예: × 5) |

### 6.7 Phase 3가 필요한 이유 (Phase 1과 중복 아님)

| 측면 | Phase 1 | Phase 3 |
|------|--------|--------|
| 학습 신호 | Teacher Forcing (1-step GT 매핑) | **자기회귀 전개 (K-step 자기 예측)** |
| 가중치 업데이트 방향 | "1-step 공간 응답 정확도" | "**누적 오차 damping**" (장기 안정성) |
| 모델이 본 입력 분포 | 항상 GT | 자기 오차 포함된 입력 (Phase 1, 2에서 못 봄) |

→ **동일 가중치 + 동일 Loss여도 학습 신호 자체가 다름**

### 6.8 메모리 부담

- K-step 동안 활성값 보존 (gradient용) → K가 커질수록 메모리 ↑
- **Gradient checkpointing** 적용으로 완화 (forward 활성값 재계산)
- K=575 시점에는 메모리 부담 큼 → checkpointing 필수

## 8. 추론 프로토콜

모델은 **단순한 1-step 예측 함수**:
```
f(state(t), rod_map_query, p_load_query, h(t)) → state(t+1)_pred, h(t+1)
```

### 8.1 4가지 운영 패턴

#### 패턴 A: 연속 자기회귀 (시뮬레이터 재현)

```
초기: state(0), h(0) = zeros

for t in 0..T-1:
    rod_map_query, p_load_query  ← 외부에서 주어짐
    state(t+1)_pred, h(t+1) = Model(state(t), rod_map_query, p_load_query, h(t))
    state(t) ← state(t+1)_pred
    h(t)     ← h(t+1)
```

#### 패턴 B: 동일 시점 다중 query (What-if 평가)

```
주어진 시점 t에서:
  h_backup = deepcopy(h(t))
  for rod_map_query in 다중_후보:
      state(t+1)_pred, _ = Model(state(t), rod_map_query, p_load_query, h_backup)
      results.append(state(t+1)_pred)
```

#### 패턴 C: 자기회귀 + 분기 평가 혼합

```
시점 t에서:
  h_backup = deepcopy(h(t))
  for rod_map_query in 다중_후보:
      branch_pred, _ = Model(state(t), rod_map_query, p_load_query, h_backup)
  state(t+1), h(t+1) = Model(state(t), rod_map_main, p_load_main, h_backup)
```

#### 패턴 D: Iterative Critical Search (수동 CRS 탐색)

```
for t in 0..T-1:
    h_backup = deepcopy(h(t))
    while True:
        rod_map_query  ← 후보 지정
        state(t+1)_pred, h(t+1)_candidate = Model(state(t), rod_map_query, ..., h_backup)
        if abs(keff_pred - 1.0) < tolerance:
            break
    state(t+1) ← state(t+1)_pred
    h(t+1)    ← h(t+1)_candidate
```
