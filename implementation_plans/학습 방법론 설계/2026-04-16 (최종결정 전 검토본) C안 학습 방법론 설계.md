## 1. C안 아키텍처 요약

```
[Spatial Encoder]     →  [Spatiotemporal Processor]     →  [경량 Decoder]  →  Loss
 FullAttn ×3 (기존)       Mamba Block ×M + Attn Block ×A      MLP 1~2층
 (B,720,21)→(B,720,128)  (시간 전이 + 공간 결합 통합)          (B,720,128)→(B,720,10)
```

### 프로세서 내부 구조 (1개 Layer 단위)

```
z(t) ∈ (720, 128)
  → [Mamba Block]        ← cell별 독립, 시간 축 처리
  → [Self-Attention]     ← 720 cell 간 공간 결합
  → out(t) ∈ (720, 128)
```

- Layer N개 적층 (N = 2~4, 실험으로 결정)
- Mamba:Attention 비율 = 1:1 또는 2:1 (검토 답변 A: 3~4:1, B: 1:1~2:1)
- Attention은 각 시점에서 독립적으로 720 cell을 처리 (시점 간 의존 없음 = stateless)
- p_load는 Mamba 입력에 concat + Attention에 AdaLN으로 주입

### A안 대비 변경점

| 항목 | A안 (기존) | C안 |
|------|-----------|-----|
| 시간 모델 | Mamba (cell-wise 독립) | Mamba + Attention (공간 결합 포함) |
| 디코더 | FullAttn + AdaLN-Zero | **MLP 1~2층** (경량 투영) |
| 공간 결합 위치 | 인코더/디코더에서만 | **프로세서 내부 매 시점** |
| 인코더 | FullAttn ×3 | FullAttn ×3 (변경 없음) |

## 2. 학습 데이터 구조

### 데이터 규모

- 100 LP × 20 profile × 575 CRS steps = 1,150,000 CRS 시점
- 각 CRS 시점에서 ~29 Branch = ~600,000+ Branch 시점
- 총 ~178만 시점

### 부하추종 시간 프로파일

```
시간(h)  0────────12──15─────────21──24
출력(%)  100%      100→저출력    저출력    저출력→100%
Steps    144 steps  36 steps    72 steps  36 steps  ...

5분 간격, 총 575 steps ≈ 48시간
```

- 정출력 구간 (144 steps): 변화 거의 없음, Xe 평형 상태
- 감출력 구간 (36 steps): p_load 급변, 제어봉 대응, Xe 축적 시작
- 저출력 유지 (72 steps): Xe 피크 → 감소, 출력 재분배
- 상승 구간 (36 steps): p_load 복귀, Xe 소멸, 제어봉 인출

## 3. 학습 전략 후보

### 후보 A: 2-Phase 학습

> **출처**: `2026-04-09 SSM 기반 시계열 모델 설계.md` §4.2~4.3에서 제안됨 (미확정). V-JEPA 2 (frozen encoder + predictor), π₀ (사전학습 + 미세조정), MPP (NeurIPS 2024, 이종 물리 사전학습)를 근거로 설계.

**Phase 1 (Branch Pretrain)**:
```
Encoder(FullAttn ×3) → Processor의 Attention만 → Decoder(MLP) → Loss
                        ↑ Mamba는 bypass (단일 step, 시간 전이 불필요)
```
- Branch 데이터로 인코더/Attention/디코더를 학습
- 공간 연산자 학습: "rod_map 변경 → 노심 1-step 응답"
- Loss: L_data + L_data_halo + L_diff_rel + L_Bateman

**Phase 2 (CRS Training)**:
```
Encoder → Processor 전체 [Mamba + Attention] → Decoder → Loss
           ↑ Mamba 새로 추가
           ↑ Attention은 Phase 1 가중치로 시작
```
- Mamba 새로 초기화, Attention/Encoder/Decoder는 Phase 1 가중치 유지
- CRS 시퀀스로 시간 동역학 학습
- Branch 보조 Loss(λ_branch = 0.1~0.3) 유지하여 Attention 공간 결합 보존

**장점**: Branch 데이터(단일 step)로 안정적으로 공간 표현을 먼저 확보. Mamba 학습 시 이미 좋은 인코더/디코더 위에서 시작.
**단점**: Phase 전환 시 Attention의 역할이 바뀜 (1-step 공간 결합 → 시간 맥락 내 공간 교정). 이 전환이 매끄럽지 않을 수 있음. 또한 2-Phase 경험이 검증된 선례는 V-JEPA 2/π₀ 정도이며, PDE 대리모델에서의 검증은 없음.

### 후보 B: 단일 Phase 동시 학습 (DreamerV3/Drama 방식)

```
CRS 시퀀스에서 Encoder + Processor(Mamba+Attention) + Decoder를 처음부터 동시 학습
Branch 데이터는 보조 Task로 CRS 학습에 혼합
```

- DreamerV3/Drama에서 사용하는 표준 방식
- 모든 컴포넌트가 처음부터 시간 맥락을 보면서 학습
- Phase 전환에 따른 역할 변화 문제가 없음

**장점**: Phase 전환 없이 단순. Mamba와 Attention이 처음부터 함께 학습하므로 역할 분담이 자연스럽게 형성됨.
**단점**: 학습 초기에 Mamba가 불안정한 예측을 하면서 인코더/디코더 학습을 방해할 수 있음. Branch 단독 학습 대비 공간 표현의 초기 품질이 낮을 수 있음.

### 후보 C: 혼합 방식

Phase 1에서 Branch + 짧은 CRS(window=4~8)를 함께 학습. Mamba를 완전히 배제하지 않되, 짧은 시퀀스로 시작하여 점진적으로 window를 확대.

**장점**: Mamba가 처음부터 관여하면서도, 짧은 시퀀스에서 시작하므로 안정적.
**단점**: 구현 복잡도 증가. 커리큘럼 설계가 필요.

## 4. 시퀀스 처리 및 Branch 보조 학습

### 기본 구조: Parallel Scan (전체 시퀀스)

Mamba의 parallel scan은 575 step 전체를 한번에 처리한다. C안의 Attention도 시점별 독립이므로 575개를 병렬 처리 가능. GPU 메모리가 부족한 경우에만 gradient checkpointing을 적용한다.

> **참고**: TBPTT(Truncated Backpropagation Through Time)는 GRU/LSTM 시대의 기법으로, parallel scan이 가능한 Mamba/SSM 계열에서는 사용하지 않는다. DreamerV3의 Block GRU는 TBPTT를 사용하지만, Drama의 Mamba-2는 parallel scan으로 이를 대체하였다.

```
CRS sequence(0:575) 전체를 한번에 처리:

  1. Encoder: GT state(0:575) → z(0:575)              [575 시점 병렬]
  2. Mamba Layer 1: parallel scan over 575 steps        [720 cell 병렬]
  3. Attention:     각 시점에서 720 cell mixing          [575 시점 병렬]
  4. Mamba Layer 2: parallel scan over 575 steps        [720 cell 병렬]
  5. Decoder(MLP):  → pred(1:576)                      [575 시점 병렬]
  6. Loss 계산
```

### Branch 보조 학습

```
CRS 학습 중 시점 t에서:
  1. Mamba hidden state h(t) 스냅샷
  2. state(t) + branch_rod_map → Encoder → z_branch(t)
     z_branch(t) → Processor(1 step, h(t)) → Decoder → branch_pred(t+1)
  3. Branch GT와 비교 → Branch Loss
  4. h(t) 복원 → CRS 계속
```

## 5. h(t) 관리 기법

### 출처 및 적용 범위

| 기법 | 출처 | 원래 맥락 | 적용 범위 | C안 적용 |
|------|------|----------|----------|---------|
| **InferenceParams snapshot/restore** | **Drama** (02_Drama.md §2.6) | Mamba-2의 SSM state + conv_state를 deepcopy → 평가 → 복원 | Mamba 전용 | 추론 시 Branch-in-CRS |
| **Detached h(t) 주입** | **Drama 변형** | CRS parallel scan → h(t) 추출 → .detach() → Branch 초기 상태 | GRU/Mamba 공통 | Branch 보조 학습 |
| **Prefix burn-in** | **Drama** (02_Drama.md §2.5) | 실제 데이터 K step → parallel scan → SSM state 워밍업 | GRU/Mamba 공통 | CRS 학습/추론 |

### C안에서 Attention은 stateless

- Attention은 과거를 기억하는 hidden state가 없음
- h(t) snapshot 시 **Mamba state만 저장하면 됨**
- A안과 동일한 snapshot 복잡도

### 추론 시 자기회귀 루프

```
for t in 0..574:
  h(t+1), d(t) = Mamba_step(z(t), h(t))       ← 1-step recurrence
  d'(t) = Attention(d(t), all 720 cells)        ← 공간 결합
  pred(t+1) = Decoder(d'(t))

  if t in branch_timepoints:
    h_backup = deepcopy(h(t+1))
    for rod_offset in 29_offsets:
      z_branch = Encoder(pred(t+1), rod_map_offset)
      _, d_branch = Mamba_step(z_branch, h(t+1))
      d_branch' = Attention(d_branch)
      branch_pred = Decoder(d_branch')
    h(t+1) = h_backup

  z(t+1) = Encoder(pred(t+1))                   ← 자기회귀 재입력
```

## 6. Loss 체계

### 기본 Loss (CRS + Branch 공통)

```
L = L_data + α₁·L_data_halo + α₂·L_diff_rel + α₃·L_Bateman + α₄·L_σXe
```

### 보조 Loss (선택)

- **L_cdp (잠재 예측, Dreamer-CDP)**: Processor 출력 d(t)가 다음 시점 잠재 상태 z(t+1)을 직접 예측. 디코더 경유 없이 프로세서에 직접 gradient.
- **L_branch (Branch 보조)**: CRS 학습 중 Branch 예측 Loss. λ_branch = 0.1~0.3.

### 잔차 예측 (GraphCast 패턴, 선택)

```
d(t) = Processor(z(t), h(t))
z_next(t) = z(t) + d(t)            ← 잔차 연결
pred(t+1) = Decoder(z_next(t))
```

변화량만 예측. 정출력 구간에서 delta ≈ 0 → 안정적 유지.

## 7. 롤아웃 안정화

| 기법 | 출처 | 내용 |
|------|------|------|
| **Pushforward trick** | MNO, SSM 문헌 | k-step 자기회귀 unroll gradient. k=1→4→8 점진 증가 |
| **Scheduled sampling** | 표준 기법 | teacher_ratio 1.0→0.0 선형 감쇠 |
| **Prefix burn-in** | Drama | 첫 K step GT로 SSM state 워밍업 → 자기회귀 전환 |
| **Gradient checkpointing** | 표준 기법 | GPU 메모리 부족 시 forward 활성값 재계산으로 메모리 절약 (gradient 흐름 유지) |

## 8. 검토 요청 사항

### Q1. 학습 전략 선택: 2-Phase vs 단일 Phase vs 혼합

2-Phase(§3 후보 A)는 `2026-04-09 SSM 기반 시계열 모델 설계.md`에서 제안된 전략이나 미확정 상태이다. DreamerV3/Drama는 단일 Phase로 모든 컴포넌트를 동시 학습한다. C안(Mamba+Attention)에서 어느 방식이 적합한가?

구체적 우려:
- 2-Phase에서 Phase 1의 Attention이 "단일 step 공간 결합"을 학습한 후, Phase 2에서 "Mamba 출력의 공간 교정"으로 역할이 바뀌는 전환이 매끄러운가?
- 단일 Phase에서 초기 Mamba 불안정이 인코더/디코더 학습을 오염시키는 정도는?

### Q1-1. 2-Phase의 출처와 학습 순서

2-Phase는 V-JEPA 2의 동결 인코더 + predictor 학습 패턴에서 차용. Branch 데이터가 단일 step 예측이므로 시간 모델 개입 없이 공간 인코더/디코더만 먼저 학습 가능하고, CRS 데이터는 575 step 시퀀스이므로 사전학습된 공간 모델 위에 시간 모델을 추가하는 방식.

그런데 **어떤 것을 먼저 학습하는지의 순서**가 결과에 미치는 영향은?

- **(a) 공간 먼저**: 공간 인코더/디코더를 N epoch 학습 → 시계열 모델(Mamba) 추가하여 CRS 학습. 공간 표현이 안정적으로 확보된 상태에서 시간 모델이 올라가지만, Phase 전환 시 Attention의 역할이 바뀌는 문제.
- **(b) 시계열 먼저**: Mamba로 h(t)를 먼저 조정해놓고, 이후에 공간 인코더/디코더를 학습. h(t)의 시간 맥락이 공간 표현 학습을 안내하지만, 초기 공간 표현이 없는 상태에서 Mamba가 의미 있는 h(t)를 만들 수 있는지 의문.
- **(c) 한 epoch 내 번갈아 학습**: CRS batch와 Branch batch를 한 epoch 내에서 교대로 학습. Phase 분리 없이 모든 컴포넌트가 처음부터 함께 적응. DreamerV3/Drama의 단일 Phase 방식과 유사.

(a)가 현재 제안이나, (b)와 (c)도 검토가 필요. 특히 (c)는 Phase 전환 문제를 원천적으로 회피하므로 매력적이나, 초기 학습 불안정이 우려됨. 어느 순서가 적합한가?

### Q2. Mamba와 Attention의 순서

Mamba(시간) → Attention(공간) vs Attention(공간) → Mamba(시간)?

실제 시뮬레이터는 매 step "공간 미분(∇²φ) 계산 → 시간 전진"을 수행하므로, "Attention → Mamba" 순서가 물리적으로 더 자연스러울 수 있다. 반면 "Mamba → Attention"은 "먼저 시간 전진 후 공간 교정"에 해당한다. 어느 쪽이 적합한가?

### Q3. 디코더 경량화 수준
