## 0. 문서 목적

- **Phase epoch 비중 확정**: Phase 1 : Phase 2 : Phase 3 = **1 : 1 : 1**
- **각 Phase 내 a/b 전환 비율**: **1 : 1** (고정 ratio 기본)
- 단순 `loss_plateau` 트리거의 한계 분석 + **복합 트리거 (AND 조건)** 를 **대안** 으로 제시
- 본 문서는 **설계 제안** — 기존 문서 반영은 §8 참조

## 1. Phase 구조 재정립 (축 기반)

기존 문서가 "모듈 + Loss + 데이터" 를 혼재 기술하여 Phase 축이 모호했음. **학습 축 (축: 학습 대상) 기준으로 재정리**:

| Phase | 학습 축 | 주 학습 신호 | 수렴 판단의 "무엇" |
|:----:|---------|----------|-----------|
| **1** | **공간 응답 연산자** | 제어 조건 → 공간 물리량 | 공간 backbone 안정화 (Mamba 학습 전 출발점) |
| **2a** | **시간 동역학 (CRS main route)** | 575-step parallel scan, CRS 경로 | Mamba 시간 동역학 안정점 |
| **2b** | **CRS + Branch 상호 적응** | Main route + 분기 29개 전 시나리오 | 공간 + 시간 co-adaptation |
| **3** | **자기회귀 누적 오차** | K-step rollout error | 장기 rollout 안정성 |

**핵심 함의**: 세 Phase 가 **서로 다른 축** 을 학습. 같은 `val_loss plateau` 로 모든 전환 판정 시 오판정 가능.

## 2. Phase Epoch 비중 및 전환 비율 (기본 확정)

### 2.1 Phase 간 비중: 1 : 1 : 1 (확정, 대안 없음)

총 epoch 을 3등분:

- **Phase 1** (공간 사전학습) = 총 epoch 의 1/3
- **Phase 2** (시공간 학습) = 총 epoch 의 1/3
- **Phase 3** (자기회귀 rollout) = 총 epoch 의 1/3

**근거**:
- **"모듈별 Phase 분리 학습" 자체가 표준화된 방법론이 아님** → 전환 기준의 표준도 없음. 따라서 Phase 간 비중 **고정** 이 가장 단순·robust 한 선택
- Phase 별 파라미터 비중도 대략 유사 (공간 인코더 ~52% / 시공간 프로세서 ~47%) 로 1:1:1 근거 보강
- 1:1:1 은 **예측 가능한** baseline. 실험 결과 기반 조정 여지는 남겨둠

총 epoch 수 자체는 별도 결정 사항 (잠정, 실험 기반 조정).

### 2.2 Phase 간 전환 (1→2, 2→3): 자연 전환 (고정)

- **Phase 1 끝 = Phase 2 시작** (전체 1/3 지점)
- **Phase 2 끝 = Phase 3 시작** (전체 2/3 지점)
- **별도 트리거 없음** — 해당 Phase epoch 수 도달 시 자동 전환
- **대안 없음** (비표준 영역이라 복합 트리거 도입 근거 약함)

### 2.3 Phase 내부 a/b 전환: 기본 1:1 + 대안 복합 트리거

**해당 대상**: **Phase 1a → 1b**, **Phase 2a → 2b** 두 전환에만 적용

| 전환 | 기본 (고정) | 대안 (복합 트리거) |
|------|---|---|
| Phase 1a → 1b | Phase 1 epoch 의 **50% 지점** | 동적 탐지 (§4.1) |
| Phase 2a → 2b | Phase 2 epoch 의 **50% 지점** | 동적 탐지 (§4.2) |

**근거**:
- 균등 분배가 **단순 + robust** → 기본값
- 하지만 데이터 / 하이퍼파라미터 편차에 따라 최적 전환 시점 다를 수 있음 → 복합 트리거가 유의미한 개선 보이는지 **대안으로 실험**

### 2.4 Phase 3 내부 K 커리큘럼 (GraphCast 기준점 + JAWS 대안)

**기준점 (GraphCast 스타일)**:
- **K_schedule**: [2, 5, 10, 20, 50, 100, 150] — K_max=150 (이전 초안 K=575 축소)
- **근거**: GraphCast 학습 K=12, 추론 K=40 (학습이 추론의 30%) → 본 프로젝트 추론 K=575 × 30% ≈ 170 의 보수치 150
- **학습/추론 gap 허용**: 학습 K_max=150, 추론 K=575 extrapolation
- **Epoch 배분 — 짧은 K 에 집중** (`weight(K) ∝ 1/√K`):

```
epochs_at_K (Phase 3 내부, 1/√K 비례, 대략 비율):
  K=2    : 많이
  K=5    : 중간-많이
  K=10   : 중간
  K=20   : 중간-적게
  K=50   : 적게
  K=100  : 적게
  K=150  : 매우 적게
```

> **주의**: K=1 (Branch single-step) 은 **Phase 1 에서 이미 학습** 됨. Phase 3 K 커리큘럼은 **K=2 부터 시작**. GraphCast 비교 시 K=1 도 함께 표기되는 이유는 GraphCast 가 K=1 까지 단일 모델로 처리하기 때문이며, 본 프로젝트는 K=1 을 Phase 1 에서 분리 학습한 후 Phase 3 에서 K≥2 만 다룸.

근거:
- 짧은 K 의 학습 품질이 긴 K 의 기반 (GraphCast 관행)
- GraphCast 실제 비율 (K=1 : K=2 : K=4 = 8 : 2 : 1) 은 1/K 에 가까우나, 1/K 는 K=100+ 에서 거의 0 → 본 프로젝트는 1/√K 로 균형 (긴 K 도 long-term damping 학습 신호 최소 확보)
- 정확 수치는 실험 후 조정 (잠정)

**대안 (JAWS 2026 스타일)**:
- K=[2, 5, 10] 까지만 강하게 학습, K≥20 은 skip 또는 매우 짧게
- 대신 **Jacobian regularization** (JAWS 2026) 으로 long-term stability 확보
- 짧은 time window (K=5) 학습만으로 전통적 long-window (K=10) baseline 의 long-term rollout 정확도 match/surpass 가능

**K-step rollout 구성 (각 K 값 공통)**:

- **시작점** (step 0): 항상 CRS state(t) — 임계 운전 궤적에서 출발
- **중간 step** (step 1 ~ K-1): 모델 자기 예측, CRS rod_map 사용, **매 step L_data + L_physics** (CRS GT 비교)
- **마지막 step** (step K): 70:30 비율로 sampling (K 무관 고정)
  - **CRS ending 70%**: step K-1 의 rod_map 과 GT 모두 CRS 시나리오 — 정상 운전 궤적 추적 연습
  - **Branch ending 30%**: 해당 Branch 레코드의 (rod_map, GT state) 그대로 사용 — **제어 응답 학습**
- **Prefix burnin** (rollout 전 K_burnin step): GT 입력 teacher forcing, **h(t) 초기화 (warm-up)**, Loss 없음

상세는 §4.4.

## 3. 단순 Plateau 트리거의 실패 모드 (대안 필요 근거)

### 3.1 Saddle point 중간 정체

- Mamba 의 `Δt / A` **exponential parameterization** 특성상 "감도 급증 → 재수렴" 흔함
- 조기 plateau 감지 → 전환 → Mamba 미학습 상태에서 공간 모듈 풀림 ("Mamba 조기 unfreeze 함정")

### 3.2 Physics loss 의 구조적 정체

- ~~L_Bateman / L_diffusion (물리 단위) vs L_data (정규화 공간) → **scale 차이**~~ — **2026-04-23 해소**: 방법 1 정규화 공간 loss 일괄 적용 으로 모든 L_physics (L_Bateman, L_σXe, L_diff_rel) 가 정규화 공간 에서 계산 → scale 차이 1차 해소 (Physical Loss 레퍼런스 §1.9 / §2.5 / §3.6 참조)
- 단 잔존 우려: L_Bateman A+B Hybrid 의 A 항 vs B 항 의 균형 (λ_A=0.3, λ_B=0.1 + ReLoBRaLo Phase 2b 중반 adaptive)
- `val_total_loss` plateau = "L_data 수렴, L_physics 진행 중" 오판 가능 — 각 loss 항 별도 모니터링 필수 (Physical Loss §1.9.4 실무 체크리스트 참조)

### 3.3 Drama/TokenWM 계열 "축적 학습" 패턴

- SSM / world model 초반 20~40% epoch 거의 flat → 이후 급수렴
- Plateau 로 오판하면 조기 전환 위험

### 3.4 결론

- 고정 비율 1:1 은 **예측 가능성 확보** — 기본값
- 하지만 데이터 / 하이퍼파라미터 편차에 따라 최적 전환 시점이 달라질 수 있음
- **복합 트리거를 대안으로** 실험 후 고정 비율 대비 성능 비교 가능

## 4. 전환 트리거 구조

**범위 정리**:
- 복합 트리거 대안은 **Phase 1a → 1b**, **Phase 2a → 2b** 두 전환에만 적용
- **Phase 간 전환** (1→2, 2→3) 은 자연 전환 (epoch 도달 시 자동), 대안 없음
- **Phase 3 K 커리큘럼** 은 별도 메커니즘 (GraphCast 기준점 + JAWS 대안), 복합 트리거 대안 없음

### 4.1 Phase 1a → 1b 전환

**기본 (고정)**: Phase 1 epoch 의 **50% 지점** 에서 전환

**대안 (복합 트리거, AND 조건)**:
```
min_epoch_ratio ≥ 30% of Phase 1 epoch
L_data_branch  5 epoch 상대 개선 < 0.5%
L_diffusion    5 epoch 상대 개선 < 1%
L_sigma_a_Xe   5 epoch 상대 개선 < 1%
[상한] epoch_ratio ≥ 70% of Phase 1 → 강제 전환
```

**판단 내용**: "공간 응답 연산자가 충분히 수렴했는가"

**보조 지표**:
- ConditionalLAPE α 값 수렴
- 대칭 group (mirror / rotation) 오차 격차 ≤ 임계값
- 디코더 출력 역정규화 후 물리 범위 분포 안정 (TR-V-DC-1 연장, min/max/std 통계)

### 4.2 Phase 2a → 2b 전환

**기본 (고정)**: Phase 2 epoch 의 **50% 지점** 에서 전환

**대안 (복합 트리거, AND 조건)**:
```
min_epoch_ratio ≥ 30% of Phase 2 epoch
L_data_CRS       5 epoch 상대 개선 < 0.5%
L_Bateman        5 epoch 상대 개선 < 1%
mamba_h_norm_CV  5 epoch window < 2%
[상한] epoch_ratio ≥ 70% of Phase 2 → 강제 전환
```

**판단 내용**: "Mamba 가 CRS main route 시간 동역학에 수렴했는가" (Branch sampling 합류 전)

**주 지표 — Mamba 내부 상태 안정성**:
- Hidden state h(t) norm EMA 의 변동계수 (CV)
- A 행렬 eigenvalue 분포 안정성
- Δt 평균 / 분산 안정성

### 4.3 Phase 1 → 2, Phase 2 → 3 전환 (자연 전환, 대안 없음)

- 해당 Phase 의 할당 epoch 완료 시 자동으로 다음 Phase 진입
- **별도 트리거 / 대안 없음** (모듈별 Phase 분리 학습은 비표준 영역 → 단순 고정 비율 우선)

**참고 (별도 검토 항목, §10)**:

Phase 2 → 3 전환은 **학습 패러다임 자체의 전환** (parallel scan → autoregressive) 이므로 다음의 **rollout 사전 체크** 가 안전성 검증 용도로 유용할 수 있음 (전환 트리거 대안이 아닌 별도 안전성 검증):

- 매 N epoch 마다 K=10, K=50 test rollout 실행
- `rollout_error / teacher_forcing_error` 비율 관찰
- 비율이 임계값 (예: K=10 ≤ 3, K=50 ≤ 10) 초과 시 경고

### 4.4 Phase 3 내부 K 커리큘럼 (GraphCast 기준점 + JAWS 대안)

**기준점 (GraphCast 스타일, 권장)**:

- **K_schedule**: `[2, 5, 10, 20, 50, 100, 150]` — K_max=150
- **근거**: GraphCast 학습 K=12 / 추론 K=40 (학습이 추론의 30%) → 본 프로젝트 추론 K=575 × 30% ≈ 170 의 보수치
- **학습/추론 gap 허용**: 학습 K_max=150, 추론 K=575 extrapolation
- **Epoch 배분**: `weight(K) ∝ 1/√K` (대략 비율)

```
epochs_at_K (Phase 3 내부, 1/√K 비례, 대략):
  K=2    : 많이
  K=5    : 중간-많이
  K=10   : 중간
  K=20   : 중간-적게
  K=50   : 적게
  K=100  : 적게
  K=150  : 매우 적게
```

> **주의**: K=1 (Branch single-step) 은 **Phase 1 에서 이미 학습** 됨. Phase 3 K 커리큘럼은 **K=2 부터 시작**.

**근거 — 1/√K 선택**:
- GraphCast 실제 (K=1 : K=2 : K=4 = 8 : 2 : 1) 은 1/K 에 가까우나, 1/K 는 K=100+ 에서 거의 0 → long-term damping 학습 신호 부족
- 1/√K 는 짧은 K 에 충분히 집중 + 긴 K 도 최소 학습 신호 보장
- 정확 수치는 실험 후 조정 (잠정)

**Trade-off (GraphCast 관찰)**: 긴 K 학습 시 짧은 K 정확도가 떨어질 수 있음. 추론이 주로 짧은 범위 (K=1~10) 면 K_max 더 축소 가능

**대안 (JAWS 2026 스타일)**:

- **K_schedule**: `[2, 5, 10]` 까지만 (K_max=10)
- K≥20 은 skip 또는 매우 짧게 (각 K 에 5 epoch 정도)
- 대신 **Jacobian regularization** (JAWS 2026) 으로 long-term stability 확보
- **근거**: JAWS 는 짧은 time window (K=5) 학습만으로 전통적 long-window (K=10) baseline 의 long-term rollout 정확도 match/surpass 입증
- **장점**: memory cost 매우 낮음, 짧은 K 정확도 유지에 유리
- **단점**: Jacobian regularization 구현 복잡도 (별도 모듈 필요), 본 프로젝트 N=575 step 같은 극단 long-horizon 정확도 미보장

**K-step rollout 구성 (각 K 공통)**:

| Step | 입력 | Loss 적용 | 비고 |
|:----:|------|---------|------|
| **Burnin** (전, K_burnin step) | CRS GT teacher forcing | **없음** | h(t) 초기화 (warm-up) 전용 |
| **step 0** (rollout 시작) | CRS state(t) | — | 시작점 고정 |
| **step 1 ~ K-1** (중간) | 모델 자기 예측 (Scheduled Sampling 가능), CRS rod_map | **L_data + L_physics** (CRS GT 비교) | 매 step 표준 학습 |
| **step K** (마지막) | 70:30 sampling | L_data + L_physics | 아래 표 참조 |

**마지막 step (step K) 70:30 sampling** (K 무관 고정):

| 분류 | 비중 | Step K-1 rod_map | 비교 GT |
|------|:---:|---|---|
| **CRS ending** | **70%** | CRS rod_map(t+K-1) | CRS state(t+K) |
| **Branch ending** | **30%** | Branch 레코드의 rod_map (rod 변경) | Branch 레코드의 GT state |

**복합 트리거 대안 없음** (Phase 내부 세부 전환 대안 범위는 1a→1b, 2a→2b 두 건에 한정)

## 5. A/B 테스트 구조 (로드맵 §3 config 대상)

```yaml
training.phase_transition_triggers:

  # Phase 간 전환 (자연 전환, 대안 없음)
  phase_inter_transition:
    mode: "natural"             # Phase N epoch 도달 시 자동 전환
    # 1→2, 2→3 동일 적용

  # Phase 1a → 1b (대안 복합 트리거 적용 대상)
  phase1a_to_1b:
    mode: "fixed_ratio"         # 기본 (고정)
    fixed_ratio: 0.50           # Phase 1 의 50% 지점
    alternative_composite:      # 대안 (실험 비교)
      min_epoch_ratio: 0.30
      max_epoch_ratio: 0.70
      metrics:
        - {name: L_data_branch, rel_improvement_threshold: 0.005, window: 5}
        - {name: L_diffusion,    rel_improvement_threshold: 0.01,  window: 5}
        - {name: L_sigma_a_Xe,   rel_improvement_threshold: 0.01,  window: 5}

  # Phase 2a → 2b (대안 복합 트리거 적용 대상)
  phase2a_to_2b:
    mode: "fixed_ratio"
    fixed_ratio: 0.50           # Phase 2 의 50% 지점
    alternative_composite:
      min_epoch_ratio: 0.30
      max_epoch_ratio: 0.70
      metrics:
        - {name: L_data_CRS,        rel_improvement_threshold: 0.005, window: 5}
        - {name: L_Bateman,         rel_improvement_threshold: 0.01,  window: 5}
        - {name: mamba_h_norm_CV,   threshold: 0.02, window: 5}

  # Phase 3 K 커리큘럼 (GraphCast 기준 + JAWS 대안, 복합 트리거 대안 없음)
  phase3_K_curriculum:
    mode: "graphcast_inverse_sqrt_K"   # 기본 (1/√K 비례 epoch 배분)
    K_schedule: [2, 5, 10, 20, 50, 100, 150]
    epoch_distribution: "1/sqrt(K)"    # 짧은 K 에 더 많이
    alternative_jaws:                  # 대안: JAWS 2026 스타일
      K_schedule: [2, 5, 10]
      jacobian_regularization: true
      long_K_skip: true
```

**대안의 역할**:
- **Phase 1a→1b, 2a→2b**: 고정 비율 (1:1) 대비 복합 트리거의 성능 비교
- **Phase 3 K 커리큘럼**: GraphCast 스타일 (긴 K 학습) 대비 JAWS 스타일 (짧은 K + Jacobian) 비교

## 6. Mamba Detach 구현 선택 (Phase 1)

Phase 1 에서 Mamba 를 "학습 배제" 하는 방식:

### 옵션 (a): Mamba forward skip

- 공간 인코더 → **Mamba 생략** → 디코더
- 디코더 입력 차원이 Phase 1 vs 2 에서 달라짐 → **파이프라인 불일치** (단점)

### 옵션 (b): Mamba forward 유지 + gradient detach ✓ 권장

- 공간 인코더 → Mamba → `tf.stop_gradient` → 디코더
- **동일 파이프라인**, gradient flow 만 제어

### (b) 의 하위 선택

| 하위 | 설명 | 권장 |
|:---:|------|:---:|
| **(b-1)** Random init 유지 | Mamba 가 noise 출력 | △ |
| **(b-2)** Identity 초기화 | Mamba output ≈ input (복소 상태 A ≈ 0), skip connection 효과 | **✓** |

**(b-2) 권장 근거**:
- "Skip connection + 작은 선택적 보정" 자연스러운 초기 상태
- Phase 2 co-adaptation 전환 시 Mamba 가 identity 근처에서 학습 시작 → **충격 완화**
- 통합 정리 §3 의 "Mamba → Attention → MLP residual 구조" 와 정합

**구현 세부** (추후 검토):
- Mamba-3 의 Trapezoidal 이산화 특성과 A ≈ 0 초기화의 호환성 검증
- 복소 상태 파라미터에서 "identity 근사" 구현 방식

## 7. LLM 동적 Epoch 관행과의 비교

**핵심 결론**: LLM pretraining 은 **동적 전환을 사실상 쓰지 않음**. 그러나 본 프로젝트 특성 상 **복합 트리거는 대안으로 유용**.

### 7.1 LLM 관행 (GPT / Llama / Gemma / Qwen / AFM)

| 이유 | 내용 |
|------|------|
| Cosine schedule + 고정 step | 전체 학습 step 사전 고정 → 동적 전환과 충돌 |
| 대규모 분산 학습 비용 | 수천 GPU × 수 개월 → 동적 로직이 장애점 |
| Abrupt learning / emergence | Plateau 처럼 보여도 내부 표상 재조직 중 |
| Perplexity ↔ downstream gap | Loss plateau 가 학습 한계 신뢰성 있게 지시 못 함 |

**예시**: Qwen 2 (2단계 pretraining), Apple AFM (3단계 pretraining) 모두 **"데이터셋 완료" / "token 수 달성"** 기반 전환 — plateau 감지 없음.

### 7.2 본 프로젝트와의 차이

| 항목 | LLM | 본 프로젝트 |
|------|-----|---------|
| 학습 규모 | 수천 GPU, 수 개월 | 단일 머신, 수 주 |
| Phase 축 | 데이터 domain 전환 | **학습 대상 모듈 전환** |
| 수렴 지표 | Perplexity (불신) | **물리 잔차 + rollout 오차** (objective 명확) |

### 7.3 본 프로젝트 설계 (LLM 관행 × 도메인 특수성 교집합)

**고정 비율 (1:1:1, 1:1) = LLM 관행의 예측 가능성** 흡수 → **기본값**

**복합 트리거 = 본 프로젝트 특수성** (Mamba saddle, physics scale 차이, rollout 위험) → **대안**

```
기본 = 고정 비율 [LLM 관행 장점]
  ↓ 실험 결과 복합 트리거 우위 시
대안 = 복합 트리거 [본 프로젝트 특수성 반영]
```
