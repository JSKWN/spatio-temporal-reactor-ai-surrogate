## 1.2 학습 전략 평가

| 후보 | 평가 |
|------|------|
| **A** (단일 Phase + detach 주입) | Drama 패턴과 부합하나, Branch GT가 존재하는 우리 문제에서 Branch loss가 CRS 본류 동역학을 충분히 교정하지 못하는 한계 |
| **B** (gradient 통과) | 원리적으로 가장 많은 감독 신호를 활용하나, Branch loss가 h(t)를 통해 CRS 전체 prefix로 역전파되면 575 step + 29 분기 결합으로 매우 긴 gradient graph가 생겨 불안정성 큼 |
| **C** (2-Phase 공간 → 시간) | **권장**. Branch 1-step GT가 많고 rod_map 변화에 대한 공간 응답을 먼저 안정적으로 학습 가능. Phase 1에서 Encoder+Attention+Decoder만으로 공간 반응 연산자 학습, Phase 2에서 Mamba 추가 |
| **D** (epoch 내 교대) | 후보 A와 유사한 한계 |
| **E** (시간 → 공간) | 시간 모델을 공간 결합 부족 상태에서 먼저 학습하므로 representation mismatch 위험. **가장 비권장** |

V-JEPA식 완전 동결보다는 Phase 2에서 encoder/attention을 작은 lr로 미세조정 권장. 원자로는 비디오 예측보다 물리 coupling이 강해 시간 모델 추가 후 표현 재정렬 필요.

## 1.3 h(t) 전달 (Detach 주입 권장)

기본값은 **detach 주입**. Drama도 imagination 시작점에 informative한 hidden state를 real transition으로 얻고 분기 rollout에 사용하지만, 분기 과정이 본류 sequence model 전체를 다시 학습하도록 연결하지 않음.

Branch GT가 있다는 점이 gradient 통과를 완전히 배제하진 않지만, 정당화하지도 못함. **감독 신호가 더 많다는 사실과 gradient 경로가 안정적이라는 사실은 다름**.

3단계 적용 권장:
1. Detach 주입을 기본으로 사용
2. Branch head가 충분히 수렴한 뒤, 일부 시점에 한해 짧은 window만 gradient 통과 시도
3. 이때도 global norm clipping과 작은 Branch loss weight 함께 적용

**h=0 초기화는 가장 비추천**. Branch는 "같은 시점의 노심 상태에서 rod_map만 바뀐 1-step 반응"을 묻는 문제이므로, Xe/I-135 축적 이력과 직전 flux/temperature 문맥이 빠지면 물리적으로 의미 있는 분기점이 사라짐. Branch는 독립 표본이 아니라 CRS trajectory 위의 conditional query.

## 1.4 안정화 조합

575 step 전체를 Mamba parallel scan으로 학습한다고 해서 gradient 문제가 자동으로 사라지지 않음. Drama도 Mamba가 긴 시퀀스를 더 잘 다루지만 긴 의존성과 학습 초기 모델 불완전성이 핵심 난점이라고 설명.

**최소 필수 조합 (4개)**:

| 기법 | 역할 |
|------|------|
| Delta prediction | 절대 상태보다 변화량 예측. 정출력 구간의 trivial identity 학습과 스케일 문제 감소 |
| Prefix burn-in | 처음 8~16 step GT로 hidden state 안정화 후 rollout 시작 (단, 추가 질문 3 참조) |
| Short-horizon pushforward | 1-step → 4-step → 8-step으로 unroll 확대 |
| Physical loss | 확산 잔차와 Bateman ODE로 장기 rollout에서 상태 정확도와 물리 정합성 모두 유지 |

**추가 권장**: CDP형 latent prediction loss. 경량 decoder 뒤에서만 supervised signal을 받는 대신, processor 출력이 다음 latent를 직접 맞추도록 하면 Mamba+Attention 블록이 더 직접적으로 교정.

## 1.5 상충 관계

- **Scheduled sampling vs Pushforward**: 목적이 겹침. 둘 다 train-test mismatch 감소. 처음부터 둘을 강하게 같이 쓰면 최적화 불안정. 권고: 초기에는 burn-in + teacher forcing 위주 → 중기부터 pushforward로 전환 → scheduled sampling은 필요할 때만 약하게
- **Branch 보조 loss**: 항상 강하게 두면 CRS 주과제와 gradient conflict. Phase 2 초반에 약하게, CRS rollout 안정화 후 점진적으로 증가

## 1.6 권장 프로토콜

```
Phase 1: Branch 데이터로 Encoder + Attention + Decoder 학습
  Loss = state delta MSE + physical loss

Phase 2 초기: Mamba 추가, CRS 학습 시작
  prefix burn-in 8~16, detach된 h(t)로 Branch 보조 loss

Phase 2 중기: pushforward 1→4→8 step 증가, CDP latent loss 추가

Phase 2 후기: 선택된 branch subset에 대해서만 truncated gradient-through-h(t) 시도, clipping 필수
```

평가 기준: one-step MSE 외에 64/128/575-step rollout drift, axial Xe mode 보존, physical residual 폭주 시점, 29개 Branch ranking 정확도 모두 확인.

## 1.7 답변 A의 추가 질문

### 추가 질문 1: 2-Phase와 Mamba 학습 방식 결합?

순차적 결합 의미. **2-Phase를 큰 틀로 두고, Phase 2의 Branch 학습 시 h(t) snapshot/detach 전략 사용**.

- Phase 1: 공간 학습
- Phase 2 본학습: Mamba 붙여 CRS 575-step dynamics 학습
- Phase 2 내부의 Branch 보조학습: CRS 진행 중 시점 t의 h(t) 추출 → snapshot + detach → Branch 예측

핵심 해석: "공간 학습 후 별도의 세 번째 단계로 h(t) detach 학습을 새로 한다"가 아니라, Phase 2 안에서 CRS 본학습과 함께 Branch를 보조과제로 넣는 구조. **"2-Phase는 학습 순서, detach는 Phase 2에서 Branch를 연결하는 방식"**.

### 추가 질문 2: Prefix burn-in은 추론 시 적용?

**주로 학습 시 적용 기법**. 추론 시에는 GT prefix가 실제로 주어지는 경우에만 적용 가능.

| 시점 | 적용 가능성 |
|------|----------|
| 학습 시 | 초기 hidden state가 비어있거나 부정확해서 전체 rollout이 초반부터 흔들리는 문제 감소 |
| 추론 시 - warm-start | 직전 몇 step 실제 상태가 있으면 적용 가능 |
| 추론 시 - cold-start | 초기 상태만 있으면 적용 불가 |

원자로 운용 시나리오는 online predictor에 가까우면 추론에도 적용. Pure simulation rollout benchmark면 별도 비교 실험으로 분리.

### 추가 질문 3: 2-Phase로 공간 먼저 학습 후 Mamba parallel 학습 시, prefix burn-in이 개입할 단계가 있나?

**현재 설정에서는 prefix burn-in이 사실상 필요 없음**. Mamba를 575 step 전체에 parallel scan으로 학습하면, 각 시점의 hidden state는 시퀀스 prefix 전체의 함수로 계산됨. 시점 t의 상태는 이미 0~t의 GT 입력을 모두 본 결과이므로 별도 warm-up 불필요.

**전체 시퀀스 supervised parallel training 자체가 이미 최대한의 burn-in 역할을 함**.

Prefix burn-in이 필요한 경우:
- 잘린 subsequence로 학습할 때 (window 시작점의 hidden state가 비어있음)
- 자기회귀 rollout loss를 걸 때 (pushforward, branch rollout 등)

우리 2-phase 설정에서 단계별 적용 여부:

| 단계 | burn-in 필요? | 이유 |
|------|------------|------|
| Phase 1 공간 사전학습 | 불필요 | 시간 hidden state 자체를 사용 안 함 |
| Phase 2 full-sequence parallel Mamba 학습 | 불필요 | 575 step 전체 GT 제공 |
| Phase 2의 pushforward / AR rollout / branch snapshot | 필요할 수 있음 | 잘린 시점부터 rollout 시작 |

기본 학습 루프가 "575-step GT 시퀀스 전체 입력 → Mamba parallel scan → next-state supervision"이라면 prefix burn-in은 기본 recipe에서 제외 가능. 더 중요한 것은 **delta prediction, physical loss, 필요 시 short-horizon rollout loss**.

### 추가 질문 4: 추론 시 1-step rollout 가능한가?

**가능**. 권장 학습방법을 사용하더라도 추론은 1-step autoregressive rollout 가능. "window"는 학습 안정화용 절단 구간 또는 평가용 부분 롤아웃 구간을 뜻하며, 추론이 반드시 window 기반일 필요 없음.

| 구분 | 학습 | 추론 |
|------|------|------|
| 기본 형태 | parallel scan | 1-step autoregressive rollout |
| window 사용 | 필요 시 사용 (안정화/효율 목적) | 필수 아님 |
| hidden state | 전체 시퀀스에서 자동 계산 | cache/snapshot로 유지 |

window의 세 가지 의미:
- 학습용 truncated rollout window: pushforward나 gradient 안정화용 4/8/16-step unroll
- 데이터 샘플링 window: 575 step 대신 일부 구간만 잘라 mini-batch
- warm-start context window: 추론 직전 실제 관측 몇 step으로 hidden state 초기화

Branch도 동일: CRS rollout 중 시점 t에서 h(t) snapshot → rod_map만 29개로 변경 → 각각 1-step 또는 short-horizon rollout. Branch는 "window 모델"이 아니라 **공유된 state에서 fan-out하는 분기 rollout**.

# 2부. 답변 B 정리

## 2.1 핵심 권고

| 항목 | 권고 |
|------|------|
| 학습 전략 | 후보 C (2-Phase) 강력 권장 + Phase 2에서 후보 B(gradient 통과) 결합 |
| h(t) 전달 | Detach 없이 gradient 통과 권장 (단, stochastic branching으로 메모리 관리) |
| 안정화 필수 조합 | 잔차 예측 + Pushforward + Prefix burn-in + Physical Loss |
| 사용 금지 | 잠재 예측 Loss (CDP), Scheduled Sampling |

## 2.2 학습 전략 평가

| 후보 | 평가 |
|------|------|
| **A & D** (단일 Phase 직진) | Mamba(시간)와 Attention(공간) 동시 학습 시 "이웃 cell을 봐서 예측할지, 내 과거를 보고 예측할지" 혼란. 720개 공간 결합이 안정화되기 전 Mamba가 엉뚱한 시계열 패턴을 외워버리면 Local Minima 위험 |
| **E** (시간 → 공간) | 절대 피해야 함. **타원형 PDE 특성상 공간 정보 없는 시간 전이는 원자로 물리에서 성립하지 않음** |
| **C** (수정된 2-Phase Curriculum) | **권장** |

**권장 2-Phase Curriculum**:

- **Phase 1: "트램펄린(Spatial Laplacian) 마스터하기"**
  - 데이터: Branch (60만 시점)
  - 구조: Encoder → Attention Block (Mamba 비활성/Bypass) → Decoder
  - 목표: rod_map 29가지 변화에 대해 "1 step 뒤 노심이 공간적으로 어떻게 즉각 반응하는가"를 Attention이 학습. 타원형 확산 방정식의 Spatial Operator 학습

- **Phase 2: "시간 지연(ODE) 및 시공간 통합"**
  - 데이터: CRS 시퀀스 + Branch
  - 구조: Mamba 활성화, [Mamba → Attention] 전체 파이프라인
  - 목표: 공간 반응성이 이미 확보된 상태에서 Mamba는 Xe/I-135 축적, 온도 지연 등 시간 동역학에 집중

## 2.3 h(t) 전달 (Gradient 통과 권장)

**Detach 없이 Gradient 통과가 원리적으로 압도적 우위**.

| 측면 | Drama (detach) | 우리 문제 (gradient 통과 권장) |
|------|---------------|--------------------------|
| 환경 | RL, 미래 보상 예측이 매우 불안정 | 결정론적 시뮬레이터 GT |
| Detach 이유 | Value function 노이즈가 시간 모델 망가뜨림 | RL 노이즈 없음 |
| Gradient 통과 효과 | (사용 안 함) | "h(t)에 다양한 What-if 대비 정보를 누적하라"는 학습 신호 |

**메모리 폭발 방지 (Stochastic Branching)**: 575 step 중 무작위로 4~8개 시점 샘플링 + 각 시점에서 29개 중 3~5개 Branch만 샘플링. 충분한 정칙화 효과.

## 2.4 안정화 조합

Mamba의 Parallel Scan 학습은 매 시점 GT를 입력받는 Teacher Forcing을 빠르게 수행하는 것과 같음. 추론 시 자기 예측을 입력하면 Exposure Bias로 575 step을 버티지 못함.

**필수 조합**:

| 기법 | 등급 | 내용 |
|------|------|------|
| 잔차 예측 | 필수 | ΔState = state(t+1) - state(t) 예측. 5분 간격 부드러운 변화에 적합. 누적 오차 드라마틱 감소. GraphCast 등 물리 AI의 1원칙 |
| Pushforward Trick | 필수 | Phase 2 후반 적용. K=2 → K=5 → K=10 점진 증가. 모델이 자기 예측 오차를 인지하고 발산하지 않게 자가 교정 학습. MNO 성공 핵심 |
| Prefix Burn-in | 권장 | 추론/롤아웃 시 처음 1~3 step GT 주입으로 h(t) 워밍업 |
| Physical Loss | 필수 유지 | 잔차에 더해 복원된 물리 공간에서 Bateman ODE/확산 잔차 계산. 경량 디코더에서 완벽 작동 |

**상충/불필요한 기법**:
- **잠재 예측 Loss (CDP)**: **사용 금지**. 우리 모델은 디코더가 MLP로 극도로 가볍고 Physical Loss 직결. 물리 공간 제약이 훨씬 강력하고 정확하므로, 부정확한 잠재 공간 Loss 추가는 최적화 방향 교란
- **Scheduled Sampling**: Pushforward와 목적 중복. 구현이 직관적이고 물리 모델에 잘 맞는 Pushforward 사용

## 2.5 실행 로드맵

| 단계 | 내용 |
|------|------|
| Architecture | C안 (Encoder → Mamba+Attention → MLP Decoder), 잔차(Δ) 예측 적용 |
| Phase 1 (Spatial Master) | Branch 데이터로 Mamba 끄고 Attention과 분기 정합성만 완벽 학습 |
| Phase 2 (Spatiotemporal) | CRS 데이터로 Mamba 활성화. Parallel Scan 학습하되, 간헐적으로 Branch GT에 gradient 통과시켜 h(t) 일반화 극대화 |
| Phase 3 (Rollout Stabilization) | Pushforward Trick으로 K-step AR 롤아웃 환경에서 모델이 스스로 오차 억제하도록 미세조정. Physical Loss 적극 활용 |

## 2.6 답변 B의 추가 질문

### 추가 질문 1: 잔차 예측 데이터가 절대값으로 구성되어 있는데 가능한가?

**완벽하게 가능**. 데이터를 미리 전처리하여 Δ로 변환할 필요 없음. 모델 아키텍처 끝단과 Loss 함수만 수정.

```python
delta_pred = model(state_t, ...)
state_t_next_pred = state_t + delta_pred
loss_data = MSE(state_t_next_pred, state_t_next_gt)
loss_physics = PhysicsLoss(state_t_next_pred)
```

**왜 잔차?**:
- 정출력 구간에서 5분 뒤 상태가 99.9% 동일. 절대값 예측 시 신경망이 y=x 항등함수를 매번 완벽히 모사해야 함 (어려움)
- 잔차 예측 시 정출력 구간 정답은 0. 신경망은 0 출력을 매우 잘함

**구현 주의**:
1. 정규화 위치: 데이터를 [-1,1] 또는 mean=0/std=1로 정규화. 모델은 정규화된 공간의 잔차 예측. Physical Loss 계산 시 역정규화 필수
2. 음수 방어: 물리량(중성자속, 농도)은 0 미만 불가. `state_t_next_pred = ReLU(state_t + delta_pred)` 또는 음수 패널티 Loss

### 추가 질문 2: 잔차 예측 시 디코더를 파트1(잔차) + 파트2(절대값 종합)로 나누고 Physical Loss는 파트2에?

**정확한 직관**. 다만 신경망을 굳이 둘로 나눌 필요는 없음:
- **파트 1 (Neural Decoder)**: 가벼운 MLP, 잔차(Δ)만 예측
- **파트 2 (Deterministic Reconstructor)**: 학습 가능한 신경망 아니라 **텐서 덧셈 + 역정규화 수행하는 수학적 연산 블록**

**데이터셋 분리 불필요**. 원본 절대값 데이터 유지하고 GPU 상에서 동적으로 잔차 계산.

**학습 파이프라인**:
```
[Step 1] 데이터 준비: X_t, X_next_gt (절대값, 정규화됨)
[Step 2] 파트 1 (Neural): delta_pred = Model(X_t)
[Step 3] 파트 2 (Deterministic): X_next_pred = X_t + delta_pred
[Step 4] 역정규화: X_next_pred_physical = Denormalize(X_next_pred)
[Step 5] Loss 종합:
   - Loss_data = MSE(X_next_pred, X_next_gt)
   - Loss_physics = BatemanODE(X_next_pred_physical, X_t_physical)
                  + DiffusionResidual(X_next_pred_physical)
   - Total = Loss_data + λ·Loss_physics
```

이렇게 하면 **신경망은 학습하기 쉬운 잔차만 예측(안정성)** + **Physical Loss는 완벽한 물리 단위의 절대값으로 정확히 계산(정합성)** 동시 확보.

# 3부. Gradient 통과의 실제 의미와 한계 (논의로 명확해진 사항)

답변 B의 "gradient 통과" 권고를 깊이 분석한 결과, 그 효과가 처음 받는 인상보다 훨씬 제한적임이 확인되었다.

## 3.1 Forward는 detach 여부와 무관하게 동일

```
parallel scan 결과: [h(0), h(1), ..., h(99), h(100), h(101), ..., h(574)]
                                              ↑
                                  Branch는 이 값 하나만 입력으로 사용

Branch_t=100 forward:
  z_branch(100) = Encoder(state(100), branch_rod_map)
  d_branch = Mamba_step(z_branch(100), 초기 state = h(100))
  branch_pred = Decoder(d_branch)
```

h(100)의 **값**은 detach 여부와 상관없이 같은 텐서를 그대로 사용. 차이는 backward(역전파)에서만 발생.

## 3.2 차이는 backward에서 발생

Mamba의 recurrence:
```
h(t) = A_t · h(t-1) + B_t · x(t)
```

h(100)의 계산 그래프는 h(0)부터 h(99)까지 모두를 포함.

**Detach 주입 시 backward**:
```
Branch Loss → Branch Decoder → Mamba_step (h(t) 받음) → ... → h(t)에서 STOP
```

**Gradient 통과 시 backward**:
```
Branch Loss → Branch Decoder → Mamba_step → h(t) →
              → CRS Mamba forward 전체를 거슬러 올라감 →
              → t=0까지 모든 step 통과 → Encoder까지 도달
```

## 3.3 영향 받는 가중치 — 구체적인 Mamba 파라미터

"Mamba 가중치"는 모호한 표현이다. Mamba 블록 내부의 학습 가능한 파라미터는 다음과 같다:

| 가중치 | 역할 |
|--------|------|
| W_in | 입력 projection (x → 내부 표현) |
| W_B | 입력으로부터 B_t 생성 (시점별 입력 의존 파라미터) |
| W_C | 입력으로부터 C_t 생성 |
| W_Δ | 입력으로부터 Δ_t 생성 (시간 step 크기) |
| A_log | A 파라미터화 (Mamba-2 SSD에서는 head별 스칼라) |
| D | Skip connection / direct path |
| W_conv | 1D 깊이별 convolution kernel (SSM 직전) |
| W_gate | Gating 가중치 |
| W_out | 출력 projection |

이 모든 가중치는 **모든 시점에서 공유**된다. 매 시점 t에서 입력 z(t)로부터 B_t, C_t, Δ_t가 생성될 때 사용되는 것이 W_B, W_C, W_Δ 동일 가중치.

**Detach vs Gradient 통과 시 영향 비교**:

| 컴포넌트 | Detach | Gradient 통과 |
|----------|--------|--------------|
| Branch Decoder | 업데이트 받음 | 업데이트 받음 |
| Branch의 Mamba_step (W_in, W_B, W_C, W_Δ, A_log, D, W_conv, W_gate, W_out — 공유 가중치) | 업데이트 받음 | 업데이트 받음 |
| Branch의 Encoder (z_branch 생성 가중치) | 업데이트 받음 | 업데이트 받음 |
| **CRS의 Mamba 0~t step (위와 동일한 W_B, W_C, W_Δ 등)** | **업데이트 안 받음** | **업데이트 받음** |
| **CRS의 Encoder 0~t step (z_crs 생성)** | **업데이트 안 받음** | **업데이트 받음** |
| **CRS의 Attention 0~t step (C안의 공간 결합)** | **업데이트 안 받음** | **업데이트 받음** |

## 3.4 Gradient 통과가 정보 공유는 아님

임의의 시점 t에서 CRS의 hidden state h(t)를 살펴보자. h(t)는 Mamba recurrence에 의해 다음과 같이 정의된다:

```
h(t) = A_t · h(t-1) + B_t · z_crs(t)
     = f(z_crs(0), z_crs(1), ..., z_crs(t), Mamba 가중치)
```

즉 h(t)는 **CRS 경로의 입력 z_crs(0~t)와 Mamba 가중치만의 함수**이다.

**핵심 통찰**: 이전 시점들(t' < t)에서 수행된 Branch 예측의 결과는 h(t) 값에 **전혀 반영되지 않는다**. Branch는 CRS와 분리된 side computation이며, 그 결과가 다음 시점의 CRS 입력으로 들어가지도 않는다. 

구체적으로 t=100인 경우:
- h(100)은 z_crs(0), z_crs(1), ..., z_crs(100)만 의존
- t=10, 50, 99 등에서 수행된 Branch 예측들은 h(100)의 forward 계산에 어떤 영향도 주지 않음
- 각 Branch는 독립적인 가지(side branch)로 빠져나가고 사라짐

이는 detach 여부와 **무관한 forward 구조의 사실**이다:
- Detach 주입: Branch loss → 멈춤
- Gradient 통과: Branch loss → h(t)를 거슬러 CRS path 따라 올라감

두 경우 모두 forward에서는 Branch끼리 서로 정보를 공유할 수 없다. **여러 Branch는 CRS 경로 위의 서로 다른 시점에서 분기되어 나가는 독립 측정일 뿐이며, 분기들 사이에 정보 전달 통로가 존재하지 않는다.**

## 3.5 그렇다면 Gradient 통과의 실제 효과는?

**h(t) 값을 바꾸는 게 아니라, Mamba projection 가중치(W_B, W_C, W_Δ 등)가 받는 학습 신호의 종류와 양을 늘리는 것**이다.

CRS Mamba는 같은 가중치 W를 모든 시점에서 공유한다. 학습 중:

**Detach만 사용**:
```
∂(CRS Loss)/∂W → 가중치 업데이트
∂(Branch Loss)/∂W → Branch의 1-step Mamba_step만 통해 업데이트 (짧음)
```

**Gradient 통과**:
```
∂(CRS Loss)/∂W → 가중치 업데이트
∂(Branch Loss at t=20)/∂W → 0~20 step의 Mamba forward 통해 업데이트
∂(Branch Loss at t=50)/∂W → 0~50 step의 Mamba forward 통해 업데이트
∂(Branch Loss at t=100)/∂W → 0~100 step의 Mamba forward 통해 업데이트
... (여러 시점의 Branch loss가 같은 W에 누적)
```

차이는 **W 업데이트가 받는 신호의 종류**. Gradient 통과 시 W는 "CRS 예측 정확도" 외에 "어떤 시점 t에서든 Branch 예측이 잘 될 수 있도록 h(t)를 만들어라"라는 추가 신호를 받는다.

## 3.6 한계의 정확한 표현

답변 B가 주장한 "h(t) 일반화 극대화"는 두 가지를 혼동하면 오해를 부른다:

| 주장 | 진위 |
|------|------|
| "h(100)이 이전 Branch들의 정보를 누적하여 더 풍부해진다" | **틀림** |
| "여러 시점의 Branch loss가 같은 W에 gradient를 누적시키면, W가 학습되어 다음 epoch부터는 더 'Branch에 강건한 h(t)'를 생성하게 된다" | **맞음** |

후자가 실제 효과지만, 이는 **여러 epoch에 걸친 가중치 학습의 누적 효과**이지, 한 forward pass 내에서 Branch 정보가 공유되는 것이 아니다.

# 4부. 1-step Branch의 물리적 분석

Branch가 단일 step (5분) 예측이라는 점이 학습 방법론 선택에 결정적 영향을 준다.

## 4.1 시간 스케일 분석

5분 (1 step) 동안의 Branch 응답을 물리적 시간 스케일로 분해:

| 물리량 | 시간 스케일 | 5분 동안 변화 |
|--------|-----------|-------------|
| 중성자속 분포 | ~마이크로초 (정상상태로 즉시 평형) | rod 위치에 의해 즉각 결정 |
| 제어봉 반응도 효과 | 즉각 | 새 rod_map이 dominant |
| p_load 변화 효과 | 즉각 (출력 setpoint) | 현재 p_load에 의해 결정 |
| Xe-135 (τ ≈ 9.2h) | 시간 단위 | 약 0.5~1% 변화 (작음) |
| I-135 (τ ≈ 6.6h) | 시간 단위 | 약 0.7~1.3% 변화 |
| 온도 | 분~십분 단위 | 부분적 영향 |

## 4.2 Dominant Factor

5분 1-step Branch 예측의 dominant factor:
1. **현재 노심 상태 state(t)** ← Xe 농도, 온도, 출력 등이 이미 모두 포함됨
2. **새로운 rod_map** ← Branch에서 변경되는 변수
3. **현재 p_load**

이 셋은 모두 **현재 시점의 입력**으로 인코더에 들어간다. h(t)가 추가로 제공할 수 있는 것은 "최근 추세" (예: Xe가 올라가고 있는 중/내려가는 중) 정도이며, 1-step 예측에서는 영향이 작다.

## 4.3 학습 방법론 선택에의 함의

**h(t)의 기여가 작다 → gradient 통과의 이득도 작다**

답변 B가 주장한 "h(t)에 What-if 정보를 담도록 학습시킨다"는 효과는, h(t)가 Branch 예측에 결정적이라는 가정 위에서만 유효. 사용자의 물리적 관찰대로 Branch가 주로 **현재 입력에 의해 결정**된다면:

- Branch loss → CRS Mamba로 흘러도 학습 신호가 약함
- Mamba는 "현재 상태가 잘 인코딩되도록" 정도만 학습하면 됨 — 이것은 어차피 CRS loss로도 학습됨
- 100 step backward의 불안정성과 메모리 비용을 감수할 가치가 없음

**잠정 결론**: 답변 A의 detach 방식이 물리적으로 더 자연스럽다.

## 4.4 2-Phase 자체의 정당화 강화

이 관찰은 **2-Phase 학습 자체가 적합한 이유**를 강화한다:

- Branch가 주로 공간적 응답 문제 (현재 상태 + rod 변화 → 다음 상태) → Phase 1에서 Encoder + Attention + Decoder만으로 거의 모든 학습 신호를 흡수 가능
- Phase 2에서 Mamba가 추가로 학습하는 것은 **장기 시간 동역학** (Xe 진동, 온도 추세 등 575 step 누적 효과) → 이것은 CRS loss가 담당
- Branch의 보조 역할: Phase 2에서 detach로 가볍게 유지 → catastrophic forgetting 방지 정도

## 4.5 다중 step Branch로 확장 시 재검토

다만 **다중 step branch** (예: rod 변경 후 1시간 = 12 step 예측)나 **Xe 진동이 진행 중인 시점에서의 분기 평가**에서는 h(t)의 시간 맥락이 더 중요해진다. 우리는 1-step branch만 고려 중이므로 이 우려는 적용되지 않는다.

만약 향후 다중 step branch로 확장한다면 gradient 통과 또는 더 풍부한 h(t) 메커니즘 (예: explicit global token)을 재검토해야 한다.

# 5부. 종합 정리

## 5.1 두 답변의 공통 의견

| 주제 | 공통 의견 |
|------|----------|
| 학습 전략 | **후보 C (2-Phase: 공간 먼저, 시간 나중에) 권장**. Phase 1에서 Encoder+Attention+Decoder만으로 공간 반응성 학습, Phase 2에서 Mamba 추가 |
| 후보 E 평가 | 시간 먼저 → 공간 나중에는 비추천. 공간 결합 부족 상태에서 시간 모델 학습 시 representation mismatch / 타원형 PDE 특성상 부적합 |
| Phase 2 미세조정 | V-JEPA식 완전 동결보다 Phase 2에서 Encoder/Attention을 작은 lr로 미세조정 권장 |
| 잔차 예측 (Delta) | **필수**. 정출력 구간 항등함수 학습 회피, 누적 오차 감소. GraphCast 패턴 |
| Pushforward Trick | **필수**. Phase 2 후반 적용, K=1→4→8 점진 증가 |
| Physical Loss | **필수**. 디코더 출력에서 물리 절대값으로 계산, 경량 디코더에서도 충분히 작동 |
| Mamba 추론 방식 | parallel scan은 학습용. 추론은 1-step autoregressive rollout 가능 (InferenceParams 캐시) |
| Branch의 본질 | 독립 표본이 아니라 CRS trajectory 위의 conditional query |

## 5.2 두 답변의 차이점

| 관점 | 답변 A | 답변 B |
|------|--------|--------|
| h(t) 전달 방식 | 기본 **detach 주입**. Branch GT 존재가 gradient 통과를 정당화하지 못함 | **Gradient 통과 적극 권장**. Branch GT가 결정론적이므로 Drama의 detach 이유(RL 노이즈)가 우리에게 없음 |
| 메모리 관리 | (gradient 통과 시) global norm clipping + Branch loss weight 작게 | Stochastic Branching: 575 step 중 4~8 시점 + 29 중 3~5 Branch만 샘플링 |
| 잠재 예측 Loss (CDP) | **권장**. processor 출력이 다음 latent 직접 예측, decoder 경유 없이 직접 gradient | **사용 금지**. 부정확한 잠재 공간 Loss가 정확한 Physical Loss와 충돌 |
| Scheduled Sampling | 약하게 사용 가능 (초기 burn-in/teacher forcing → 중기 pushforward 전환) | **사용 안 함**. Pushforward와 목적 중복 |
| Prefix burn-in | 학습 안정화 기법. 단, full-sequence parallel teacher-forced 학습에는 불필요 (subsequence/AR rollout 학습 시에만 의미) | 권장 (추론/롤아웃 시 1~3 step GT 주입) |
| Branch 보조 Loss 가중치 | Phase 2 초반 약하게 → CRS 안정화 후 점진적으로 증가 | (명시적 언급 없음) |

## 5.3 핵심 통찰 — 명확해진 사항

- **2-Phase의 의미**: "학습 순서"이지 "별도 단계"가 아님. Phase 2 내부에서 CRS 본학습과 Branch 보조학습이 함께 수행됨
- **Mamba parallel scan vs 추론**: 학습은 전체 시퀀스 한번에, 추론은 1-step autoregressive로 hidden state 캐시 활용
- **Window 개념**: 추론 프로토콜의 본질이 아니라 학습 안정화/평가용 보조 개념
- **Prefix burn-in의 위치**: 575 step 전체 parallel teacher-forced 학습에서는 불필요. subsequence 학습이나 자기회귀 rollout loss 적용 시에만 의미
- **잔차 예측 + Physical Loss 양립**: 디코더 파트1(잔차 예측, Neural) + 파트2(절대값 복원, Deterministic 덧셈/역정규화) 구조로 모두 활용 가능
- **Gradient 통과의 정확한 의미**: h(t) 값 변경이 아니라 Mamba projection 가중치(W_B, W_C, W_Δ 등)에 추가 학습 신호 누적. Branch끼리 forward 정보 공유는 일어나지 않음
- **1-step Branch는 현재 입력이 dominant**: Xe/I 시간 스케일(시간 단위)이 step 간격(5분)보다 훨씬 크므로, h(t)의 시간 맥락 기여는 상대적으로 작음

## 5.4 두 답변이 충돌하는 결정 사항

| 결정 사항 | 답변 A | 답변 B | 잠정 판단 |
|----------|--------|--------|----------|
| h(t) detach vs gradient 통과 | detach가 안전 | gradient 통과가 일반화 우위 | 1-step Branch 물리 분석상 detach가 자연스러움 (4부 참조) |
| CDP latent loss 사용 여부 | 권장 | 금지 | 추가 검토 필요 |
| Scheduled sampling | 약하게 사용 | 사용 안 함 | 큰 차이 없음, 둘 다 pushforward 우선 |

## 5.5 합의된 권장 학습 흐름

```
[Phase 1] 공간 사전학습
  - 데이터: Branch

  - Loss: Data MSE (잔차) + Physical Loss (절대값 복원 후)

[Phase 2 초기] CRS 본학습 시작

  - Branch 보조: CRS 진행 중 h(t) → (detach 또는 gradient 통과, 미결정) → Branch 1-step 예측
```
