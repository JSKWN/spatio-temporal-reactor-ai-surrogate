**ReLoBRaLo 도입 시점 근거 — Phase 2b 중반이 타당한 이유**:

**ReLoBRaLo 핵심 원리** (Bischof 2021): 각 loss 항 L_i 의 **상대 진행도** (relative progress) 를 비교해 가중치 λ_i 를 자동 조정. 핵심 수식:

$$\text{progress}_i = \frac{L_i(t)}{L_i(t_0)}$$

**변수 의미**:
- **L_i(t)**: 현재 epoch t 에서 i 번째 loss 항 값 (예: L_Bateman 의 현재 값)
- **L_i(t_0)**: 기준 시점 t_0 에서의 L_i 값 (보통 학습 시작 시점 또는 ReLoBRaLo 활성화 시점)
- **progress_i**: "i 번째 loss 가 초기 대비 얼마나 감소했는가" 를 나타내는 비율. 0 에 가까울수록 많이 학습된 loss, 1 에 가까우면 거의 학습 안 된 loss. 상대적으로 덜 학습된 loss 에 더 큰 λ 를 주는 방식으로 균형 유지

→ ReLoBRaLo 는 "여러 loss 항이 **동시에, 안정된 기준점을 공유하며, 실제로 경합**" 하는 상태에서만 의미 있음.

**Phase 별 도입 가능성 분석**:

| Phase | 활성화된 Loss | 모듈 상태 | 경합 상태 | ReLoBRaLo 도입? |
|-------|------------|---------|---------|--------------|
| **Phase 1a** (DG-PINN pretraining) | L_data_branch 만 | Mamba freeze | 경합 없음 (단일 항) | **불가능** (의미 없음 — 균형 조정할 대상 자체가 없음) |
| **Phase 1b** (data + physics) | L_data_branch (state 12ch 포함: power/T/ρ/N/φ/σ_a^Xe/**Σ_f**) + L_data_halo + **L_data_Sigma_f** + L_diffusion + L_sigma_a_Xe + **L_Bateman (예측 물리량 입력)** + L_pload + L_AO (Δt=5min 1-step, 3D→scalar 일관성 포함) | Mamba freeze | 경합 시작 (모든 physics 활성, 단 1-step) | **불필요** (수동 λ + annealing 0 → 0.1 로 충분) |
| **Phase 2a** | Phase 1b + **L_data_CRS** (CRS 시계열 처음 활성) | **Mamba 활성화** (Encoder/Decoder/Attn freeze) | Mamba 를 통한 시간축 latent 분포가 학습 중 급변 → 기준점 L_i(t_0) 미확립 | **위험** (기준점 오염) |
| **Phase 2b 초반** | Phase 2a 와 동일 (전 loss) | **전 모듈 unfreeze** (co-adaptation 시작) | 2중 충격 — Mamba 계속 적응 + Encoder/Decoder/Attn 재배치로 gradient flow 요동 | **위험** (loss magnitude 급변 구간, EMA 가 λ 업데이트 못 따라감) |
| **Phase 2b 중반** | 전 loss | 전 모듈 학습 안정화 | **경합 본격화 + 안정기** | **OK** (도입 시점) |
| **Phase 3** | Phase 2b 동일 + K-step rollout 누적 오차 | 전 모듈 학습 + rollout | 경합 + K 증가에 따른 재균형 필요 | **유지** (유효) |

**요약 — Phase 2b 중반 도입 이유 3줄**:
1. **경합할 loss + 모듈이 모두 활성·안정해야 함** — Phase 1a 는 loss 항 단일, Phase 1b / 2a 는 일부 모듈만 학습 중 (Mamba freeze 또는 Encoder/Decoder/Attn freeze) → 균형 대상이 불완전
2. **기준점 L_i(t_0) 이 안정되어야 함** — Phase 2a 는 Mamba 가 막 활성화되어 시간축 latent 분포 급변 중 → 기준점 무의미
3. **급변 구간을 벗어나야 함** — Phase 2b 초반 전 모듈 unfreeze 직후 gradient flow 재배치로 loss magnitude 요동, EMA 업데이트가 못 쫓아감

→ 세 조건 모두 **Phase 2b 중반** 에 처음 동시 충족 → 이 시점에 ReLoBRaLo 활성화.

**Phase 전환 trigger 연계**: Phase 2a → 2b 전환 복합 trigger (§3 `training.phase2a_to_2b_trigger`) 가 충족된 후, Phase 2b 내부에서 추가 안정화 구간 (예: 2b epoch 의 50% 지점) 경과 시 ReLoBRaLo 활성화. 상세 trigger 는 구현 시 별도 정의 (`losses.balancing.relobralo.activation_trigger`).

**MLflow 추적 항목** (모니터링 hook 으로 기록): γ/β 분포, LAPE α 추이, Albedo trainable drift, p_load_encoder 출력 분산, **Mamba h_norm_cv** — 검증 방법론 §3.3 표의 "검증" 분류 항목과 연계.

**Mamba h_norm_cv 계산 공식** (2026-04-22 재설계):

**요약**: 720 cell 의 h(t) norm 평균 → 시간축 평균 → epoch 간 EMA (선택) → 5-epoch window CV → < 2% 시 Mamba 안정

```
[Step 1] 각 cell 별 L2 norm (복소수 포함, 64개 real elements):
    h_norm(e, t, cell) = || h(e, t, cell) ||_2
    - e: epoch, t: 시간 step (1 ~ 575 parallel scan 중 하나), cell: 720 공간 cell

[Step 2] 720 cell 평균:
    h_norm_cell_avg(e, t) = (1/720) · Σ_cell h_norm(e, t, cell)

[Step 3] 시간축 평균 (epoch 당 scalar):
    h_norm_epoch(e) = (1/T) · Σ_t h_norm_cell_avg(e, t), T=575 (또는 현재 K-step window)

[Step 4] Epoch 간 EMA smoothing (선택, β=0.1):
    h_norm_ema(e) = β · h_norm_epoch(e) + (1-β) · h_norm_ema(e-1)

[Step 5] 5-epoch window CV (변동계수):
    window = { h_norm_ema(e*-4), ..., h_norm_ema(e*) }
    CV = std(window) / mean(window)

[전환 조건] CV < 0.02 (2%) → Mamba 안정 → Phase 2a → 2b 복합 트리거의 주 지표 충족
```

**정규화 안 하는 이유**: h(t) 를 정규화하면 norm 이 상수 (보통 1) 로 수렴해 CV 자체가 무의미. Raw norm 의 epoch 간 안정성이 Mamba 시간 동역학 수렴의 직접 지표.
