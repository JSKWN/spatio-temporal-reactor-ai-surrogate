## Q1. 시공간 프로세서의 Mamba + Attention 하이브리드 구조

### (a) 이론적 타당성

**결정 요지**: N=2 blocks × [Mamba-3 ×2 → Self-Attention → 소형 MLP], Mamba:Attention = 2:1, Mamba 먼저.

일반론으로는 탄탄한 결정이다. Jamba (AI21, 2024, arXiv 2403.19887) / Nemotron-H (NVIDIA, 2025) / Hymba (NVIDIA, ICLR 2025) 등 모든 주요 Mamba-Attention 하이브리드가 "Mamba 비중 높음 + Attention 드문드문" 패턴 (LLM 도메인에서 3:1~7:1) 을 채택한다. 720 토큰이라는 매우 짧은 공간 시퀀스 규모에서 Attention 비용이 저렴하므로 (O(720²) ≈ 518K), 2:1 로 Attention 을 상대적으로 높인 것은 **공간 결합 우위** 라는 본 도메인 특성을 반영한 정당화 가능한 선택이다.

순서 (Mamba → Attention) 근거도 합당하다: 인코더가 이미 3 stages × FullAttention3D 로 "첫 공간 결합" 을 수행하여 z(t) 에 공간 정보가 녹아 있으므로, 프로세서에 진입 직후에는 "시간 전이" 를 먼저 수행하고 Attention 으로 시점별 재정리하는 흐름이 자연스럽다.

### (b) 원자로 물리 특화 적합성

결정적으로 짚어야 할 물리 사실:

- **Mamba 는 cell-wise 독립 시간 전이** — 720 cell 각자 자기 cell 의 h(t) 만 본다
- **Attention 은 stateless 시점 공간 결합** — 시간 기억 없음

즉 "공간과 시간의 상호작용" 은 오직 Mamba ↔ Attention **교대 통과 횟수** 를 통해서만 일어난다. **N=2 blocks 는 Mamba 4회, Attention 2회 통과**, 즉 "공간-시간 교차 통합" 이 2 라운드만 발생한다.

본 도메인의 Xenon oscillation 은 본질적으로 **축방향 반대칭 공간 모드 + 주기 ~25–35h 의 시간 진동** 이 결합된 현상이다 (Duderstadt & Hamilton 1976, §10-4). 5분 step 기준 주기가 300–420 step, 우리 시퀀스 (575 step) 내에 약 1.5~1.9 주기가 들어간다. 이 스케일의 공간-시간 결합 모드를 **단 2라운드의 교차 통합** 으로 학습시키는 것은 **tight** 하다. 최소 N=3 이 안전한 시작점이다.

또한 Mamba-3 의 state_dim=16 (복소) = 32 real parameters 는 Xe 진동 단일 모드 추적에는 충분하나, **제어봉 이동 과도기 + Xe 진동 + 축방향 출력 분포 skew** 세 개의 독립 동역학을 **하나의 cell-독립 h 에 동시 압축** 해야 한다. state_dim=16 으로 부족할 가능성이 있다. Drama 원논문은 state_dim=16 에서 단일 world model 동역학을 학습했지만, 본 문제는 다중 동역학의 중첩이다.

### (c) 잠재 허점 / 개선안

**허점 1 — N=2 공간-시간 교차 라운드 부족**.
- 실험 요청: N=2 vs N=3 vs N=4 ablation 을 Phase 2a 초기에 빠르게 체크. 파라미터 수는 block 당 ~100K 수준으로 추정되므로 N 확장 비용이 작다.
- Parallel scan 비용은 단일 Mamba 기준 O(T log T) 이고 N 에 linear 하게 스케일 (wall-clock) → N=4 도 부담 가능.

**허점 2 — state_dim=16 의 용량**.
- Mamba-3 의 복소 상태는 "단일 진동 모드" 추적에 자연스러우나, 여러 채널의 서로 다른 τ 가 섞인 상태를 32 real params 로 압축해야 한다. state_dim=32 (64 real) 를 차선책으로 유지.

**허점 3 — 순서 역전 (Attention → Mamba) 의 재검토 근거**.
- Drama 는 "공간 단계에서 (3D CNN) 이미 충분히 공간 결합" 을 가정. 우리도 인코더에서 FullAttention ×3 으로 동일 상황.
- 하지만 Mamba 가 시간 누적한 **이후** 각 시점의 공간 불일치 (rod 이동으로 유발된 비대칭 flux 분포) 를 Attention 이 치료하는 흐름은 자연스럽다. **현 순서 유지 권장**.
- 반대로 Attention 우선이 유리한 시나리오: "시공간 프로세서가 공간 결합을 처음부터 다시 해야 하는" 경우인데, 인코더가 이미 했으므로 해당 없음.

**권장**: N=3 을 baseline 으로 변경, state_dim=32 를 대안으로 Phase 2a 에서 비교. Mamba:Attention 비율 2:1 자체는 유지.

**참고 문헌**:
- Jamba: A Hybrid Transformer-Mamba Language Model (Lieber et al., 2024, arXiv 2403.19887)
- Hymba: A Hybrid-head Architecture for Small LLMs (Dong et al., ICLR 2025)
- DeepOMamba for parametric PDEs (J. Comp. Phys. 2025) — Mamba 기반 물리 대리모델에서 공간 결합 부족 개선을 위해 공간 attention을 병렬 추가한 선례
- Xenon oscillation spatial mode: Duderstadt & Hamilton, *Nuclear Reactor Analysis* (Wiley 1976) §10-4

## Q2. p_load 이중 경로 주입 (Mamba concat + Attention AdaLN-Zero)

### (a) 이론적 타당성

DiT (Peebles & Xie, ICCV 2023) 가 AdaLN-Zero 의 정본이고, γ, β 0 초기화로 residual 경로가 "처음에 항등 함수" 로 시작해서 점진적으로 조건 변조를 학습하는 안정성 패턴은 잘 확립돼 있다. Mamba concat 은 DreamerV3 (Hafner et al., 2023) 의 action → GRU 입력 concat 과 동형이다.

**이중 경로 사용의 선례**: Stable Diffusion 3 (Esser et al., 2024) 가 텍스트 조건을 동시에 (1) tokens concat + (2) AdaLN-Zero 양쪽으로 주입하여 학습 안정성 확보. PixArt-Σ (Chen et al., 2024) 도 유사. 이중 경로 자체는 SOTA 검증 패턴.

### (b) 원자로 물리 특화 적합성

p_load 의 물리적 역할:
1. **Fuel Doppler + moderator feedback** → Σ_a 변조 (온도 의존). 전역 스칼라 → 720 cell broadcast 는 타당.
2. **Xe 생성/소멸률** (φ 의존) → 정상출력 대비 급감출력 시 Xe peak → 축방향/반경방향 분포 왜곡.

양쪽 효과 모두 "전역 스칼라 입력 → cell 별 상이한 반응" 이다. **입력 단계에서 broadcast 는 올바르나, 반응은 공간 의존적** — 이것이 AdaLN 경로가 가치 있는 이유다:
- Mamba concat 은 **시간 축으로 매 step 새 p_load 주입** (시간 동역학 속도 제어)
- AdaLN-Zero 는 **시점 t 에서 Attention 입력 분포를 조건에 맞게 shift/scale** (공간 상호작용 세기 조절)

즉 두 경로는 **시간 차원 vs 공간 차원** 의 직교 변조이며 redundancy 우려는 과대평가된 것이다.

### (c) 잠재 허점 / 개선안

**허점 1 — γ, β 0 초기화 편향 우려 (요청서 제기)**.
- "초기에 concat 만 유효 → 학습 경로 편향" — **이 우려는 근거 약함**.
- AdaLN-Zero 의 0 초기화는 **모델이 조건을 필요로 할 때 gradient 가 유도** 하는 구조 (residual gate). DiT 에서 수천 epoch 에 걸쳐 γ, β 가 자연스럽게 nonzero 로 성장함이 실험으로 확립됨.
- 오히려 Phase 1 에서 Mamba 가 bypass (1-step) 이고 p_load 가 Phase 1 에도 주입된다면, Phase 1 에서는 AdaLN 만이 유효하게 작동할 수도 있다 (concat 이 Mamba bypass 로 무효화되므로).
- → **Phase 1 에서의 p_load 경로가 어느 것인지 명시 필요**. 3-Phase 문서 §4.2 에서 Phase 1 학습 설정에 p_load 처리를 분명히 언급하지 않음. 이것은 중요한 누락.

**허점 2 — 720 cell 균일 broadcast 의 물리 정합성**.
- p_load 자체는 전역 스칼라이지만, p_load 변화에 대한 **반응** 은 cell 별 차등 (제어봉 삽입 위치 근방 vs 원거리).
- 균일 concat 은 "반응의 공간 차등" 을 모두 모델 본체 (Mamba + Attention) 가 학습하도록 떠넘긴다. 이는 모델이 해낼 수 있지만, **힌트** 로 `p_load · f(rod_map)` 같은 "rod-modulated p_load broadcast" 가 학습 속도를 크게 높일 가능성이 있다 (FiLM + rod_map 근사).
- 그러나 이는 optimization 이며 필수는 아님. 후순위.

**허점 3 — 공유 임베더 양방향 gradient 의 학습 경로 간 경쟁**.
- MLP(1→8) 가 **한 개** 이고 gradient 가 양 경로에서 합쳐짐. 양 경로의 학습 동역학이 다르면 (e.g., concat 은 early loss-sensitive, AdaLN 은 mid/late loss-sensitive) 임베더가 두 목표 사이에서 "평균" 을 택하여 **양 경로 모두 최적점에서 이탈** 가능성.
- 현재 Fallback 계획 (AdaLN 제거 → 별도 임베더) 이 마련돼 있어 실제 문제 발생 시 대응 가능.
- **모니터링 권장**: p_load_encoder 출력 분포의 분산과 γ, β 출력 분포를 같이 추적. γ, β 가 Phase 2b 말기까지 |γ|, |β| < 0.01 에 머무르면 AdaLN 경로 **죽음** 으로 판정 → 차선책 2 (concat 단독) 진입.

**권장**: 이중 경로 유지. Phase 1 의 p_load 경로를 명시 문서화. γ/β 모니터링 threshold 사전 정의.

**참고 문헌**:
- Peebles & Xie, "Scalable Diffusion Models with Transformers" (DiT), ICCV 2023, arXiv 2212.09748 — AdaLN-Zero 원전
- Esser et al., "Scaling Rectified Flow Transformers for High-Resolution Image Synthesis" (SD3), arXiv 2403.03206 — 이중 경로 조건 주입 사례
- Hafner et al., DreamerV3, arXiv 2301.04104 — action → GRU concat 패턴

## Q3. 3-Phase 학습 방법론 (Phase 1 → 2a/2b → 3)

### (a) 이론적 타당성

3-Phase 구조의 이론적 근거는 강하다:
- Phase 1 ← PIMRL (arXiv 2503.10253) 의 "공간 사전학습 + 시간 미세조정" 패턴
- Phase 2 split (2a Mamba-only → 2b 상호 적응) ← LoRA warmup 및 V-JEPA 2 의 "stage-wise freeze → unfreeze" 패턴
- Phase 3 K-curriculum ← Pushforward trick (Brandstetter et al., 2022), MNO 계열의 rollout 안정화 표준

**3 Phase 가 모두 필요한가?** 요약 답: 필요하다.
- Phase 1 ≠ Phase 3 **동일 가중치, 다른 학습 신호** (Teacher Forcing vs 자기회귀) → 가중치 업데이트 방향이 다르다. 이는 요청서가 잘 정리한 것이고 정확.

### (b) 원자로 물리 특화 적합성

데이터 비대칭:
- Branch: 1.67M 시점, 단일 5min 전이 (풍부하지만 **시간 연속성 없음**)
- CRS: 57.5K 시점, 575-step 연속성 (**시간 연속성 풍부** 하지만 양 적음)

본 설계는 Branch 를 Phase 1 에, CRS 를 Phase 2/3 에 배치하여 **"데이터 특성 ↔ 학습 목적 정합"** 을 달성한다. 올바른 분할.

L_Bateman 을 Phase 1 에 적용 가능하다는 검증 (Branch 가 frozen Xe 가 아님) 은 Phase 1 가치를 크게 높인다.

### (c) 잠재 허점 / 개선안

**허점 1 — 30% Phase 2a → 2b 전환의 임의성**.
- 현 권고 "30% epoch" 는 pre-scheduled 이며 실제 Mamba 수렴 여부와 무관.
- **대안**: `Mamba loss plateau` (최근 N epoch validation loss 상대 변화 < 1%) 를 자동 전환 트리거로. 또는 gradient norm 의 running average 가 안정 범위 진입.
- 또 다른 관점: Mamba 가 Phase 2a 에서 "충분히" 학습됐는지 판정은 **Encoder/Attn/Decoder 의 residual error distribution** 을 관찰하는 것이 더 직접적이다 (Mamba 가 학습할수록 다른 모듈의 pre-Mamba 출력과 post-Mamba 출력의 차이가 정상 범위로 수렴).

**허점 2 — 지수 K 커리큘럼의 물리적 근거 재확인**.
- 요청서의 전제: "K 가 클수록 누적 오차 큼 → Physical Loss 가 주".
- 실제 **Xe/I 동역학은 linear ODE at fixed flux** (chaotic 아님) → **stable**. 오차 누적은 ε_per_step · K (linear), 지수 증가 아님.
- 제어봉 이동 과도기는 비선형이지만 국지적 (수 step). 575-step 전체로 평균하면 linear 근사 타당.
- → 지수 K 커리큘럼 (K=2, 4, 8, 16, ...) 은 **학습 난이도 증가가 너무 급격** 할 수 있다. Linear (K=4, 8, 12, 16, 20, 24, ...) 또는 제곱 중간 (K=2, 4, 7, 12, 20, 32, ...) 이 더 자연스러울 수 있다.
- **실험 제안**: K=8, 16, 32 에서 학습 실패율을 비교. 지수 유지 여부 결정.

**허점 3 — 차등 학습률 100× 편차의 co-adaptation 방해 가능성 (요청서 제기)**.
- Phase 2b: Mamba 1e-3, Attention 5e-5, Encoder 1e-5 → **Mamba 가 Attention 대비 20×, Encoder 대비 100×** 빠르게 움직임.
- 이는 **Transfer learning 의 표준 비율 (10× differential)** 대비 상당히 크다. BitFit, LoRA 등의 기법에서도 10× 가 일반적.
- Co-adaptation 의 핵심은 "두 모듈이 서로 적응" 인데, 한 모듈 (Mamba) 만 빨리 움직이면 co-adapt 가 아니라 "다른 모듈이 Mamba 를 따라가는 one-way adapt" 이 된다.
- **권장 수정**: Mamba 5e-4, Attention 1e-4, Encoder/Decoder 2e-5 (20× 비율). 더 보수적 co-adapt.

**허점 4 — Teacher Forcing OFF → Scheduled Sampling 누락 (요청서 제기)**.
- 현 Phase 3 는 "첫 입력만 GT, 이후 K-1 step 자기 예측" (hard autoregressive).
- 이는 **급격한 distribution shift** 이며 학습 불안정 위험.
- **Scheduled Sampling** (Bengio et al., 2015, NIPS): epoch 초기 GT 사용 확률 ~1, epoch 진행 시 선형/exponential 감소. 이 기법 없이 hard AR 전환은 위험.
- **권장**: 각 K 단계 내에서 scheduled sampling 사용. 초기 ε=0.9 → 말기 ε=0.1 (ε = GT 사용 확률).

**허점 5 — Phase 1 Branch 편향 고착 (요청서 제기)**.
- 이 우려는 **실질적** 이다. Branch 는 "현재 시점의 state + rod_map 변화" 의 5min 응답이다. "정상 운전 48h 의 장기 구조" 는 학습하지 않는다.
- Phase 1 에서 Encoder/Attn/Decoder 가 "small-time Green 함수" 에 과적합되면 Phase 2 에서 장기 구조 복구에 시간이 더 걸린다.
- **완화**: Phase 1 데이터 혼합에 CRS 의 단일 step 샘플링 (5-10% 비중) 을 포함. Branch 가 절대 다수지만 CRS 샘플이 "48h 전체 분포" 의 작은 signal 을 지속 제공. 이것 없이 Branch 단독으로 Phase 1 진행은 편향 위험.

**권장 우선순위**:
1. Phase 2b 차등 LR 을 **20× 비율** 로 축소 (Mamba 5e-4, Attn 1e-4, Enc/Dec 2e-5)
2. Phase 3 에 **Scheduled Sampling** 명시 추가
3. Phase 2a → 2b 전환 트리거를 **loss plateau 기반 자동** 으로 변경
4. Phase 1 에 **CRS 샘플 5-10% 혼합**
5. K 커리큘럼은 일단 유지하되, 지수 vs 선형 조기 ablation

## Q4. h(t) 관리 — Detach vs Gradient-through

### (a) 이론적 타당성

Drama (Wang et al., ICLR 2025) 가 detach 패턴의 정본이며, Mamba world model 학습에서 h(t) gradient 를 sequence length 전체로 역전파하지 않는다. BPTT 변형인 TBPTT 의 자연스러운 SSM 확장이라고 볼 수 있다.

Gradient-through 는 RNN/LSTM 시대의 full-sequence BPTT 에 해당. Mamba parallel scan 은 **CRS 내에서는 본질적으로 full gradient through time** 을 이미 수행 (575 step 전체). 요청서가 고민하는 것은 **CRS → Branch cross-trajectory** 의 gradient 전파이다.

### (b) 원자로 물리 특화 적합성

Xe τ ≈ 9.2h >> 5min step 의 의미:
- 단일 step 예측에서 h(t) 는 "누적 Xe 진화의 현재 snapshot". 단일 step 내 gradient 는 h(t) 의 값에 의존하지만 **h(t) 자체의 과거 이력에 덜 민감**.
- 즉 detach 하더라도 단일 step 정확도는 크게 해치지 않는다. **Drama 의 가정이 본 도메인에서도 성립** 한다.

반면 Branch 의 풍부한 데이터 (29× CRS) 가 CRS 의 Mamba 가중치에 닿지 못하는 것은 **데이터 효율성 관점에서 큰 손실**:
- Branch gradient 는 detach 경계에서 멈춰 **Encoder/Attn/Decoder** 에는 전달되지만 Mamba weight 에는 전달 안 됨.
- 그러나 이것이 **Mamba 학습 저하** 로 직결되지는 않는다 — Mamba 는 CRS parallel scan 으로 이미 학습됨.
- Branch 가 Mamba 에 주는 signal 은 "서로 다른 rod_map 에서의 5min 전이" 이며, 이 signal 의 핵심은 **Encoder/Attn/Decoder 의 rod_map 감도** 이지 Mamba 의 시간 동역학이 아님. Branch 효과가 Mamba 에 직접 닿지 않아도 **간접적으로 Mamba 가 받는 Encoder 출력의 질이 개선** 됨.

### (c) 잠재 허점 / 개선안

**허점 1 — "외부 자문 답변 A vs B 충돌" 은 맥락 오해**.
- 내가 볼 때 두 답변은 **다른 질문에 대한 답**.
- Answer A (detach): "CRS sequence 내 gradient 관리". Drama 패턴 표준.
- Answer B (gradient-through): "CRS → Branch cross-gradient 필요성".
- 둘은 배타적이지 않다. CRS 내는 parallel scan 으로 full gradient, Branch cross-injection 에서 detach. 이것이 현 결정.
- → **현 결정 유지 권장**.

**허점 2 — Branch 의 손실 보상 부재**.
- Branch 가 Mamba 에 닿지 않는 손실은 있지만, Branch 학습 자체가 Phase 1 의 메인 데이터이므로 Encoder/Attn/Decoder 는 Branch 로 충분히 학습됨.
- Phase 2 Branch 샘플링은 **"Encoder/Attn/Decoder 가 다양한 rod_map 에 지속 노출되도록" 유지** 하는 regularization 역할이지, Mamba 학습 용도가 아님.
- **Phase 2 Branch 역할을 "regularization" 으로 명시** 하면 설계 의도가 명확해진다.

**허점 3 — Phase 2 말기 제한적 gradient-through 실험의 위험**.
- "Phase 2 말기 제한적 gradient 허용" 은 **신중하게** 진행해야 함:
  - 575 step × 29 branch 의 메모리 폭발 문제는 실제.
  - 완화: **window K=8~16 limited gradient** (최근 K step 만 gradient 전파). Truncated BPTT 와 유사.
  - 또는 **branch 당 gradient weight 0.1** (full gradient 지만 scaled down).
- 그러나 이것은 **Phase 3 완료 후** 의 실험적 옵션으로 유보 권장. Phase 2 내 도입은 학습 불안정 위험이 너무 크다.

**권장**: Detach 현행 유지. Phase 2 말기 gradient-through 는 **Phase 3 완료 후 실험** 으로 후순위화. Branch 의 Phase 2 역할을 regularization 으로 문서 명시.

**참고 문헌**:
- Wang et al., "Drama: Mamba-based Model-Based Reinforcement Learning", ICLR 2025
- Williams & Peng, "An Efficient Gradient-Based Algorithm for On-Line Training of Recurrent Network Trajectories" (TBPTT), 1990

## Q5. Physical Loss 조합 (L_Bateman / L_sigma_a_Xe / L_diffusion / L_keff)

### (a) 이론적 타당성

PINN (Raissi et al., 2019) 이래 multi-term physical loss 의 가장 큰 이슈는 **gradient scale mismatch**. ReLoBRaLo (Bischof & Kraus, 2021), NTK-weighting (Wang et al., 2022), Uncertainty Weighting (Kendall et al., 2018) 등 자동 balancing 기법이 개발되어 왔다.

L_Bateman 의 해석해 (matrix exponential) 사용은 Euler forward / RK4 대비 **stiff ODE (Xe τ vs I τ ≈ 9.2h vs 6.6h)** 에서 안정적이며 올바른 선택이다.

### (b) 원자로 물리 특화 적합성

**L_Bateman**:
- Xe-135 (λ_Xe = 2.09e-5 s⁻¹) 와 I-135 (λ_I = 2.87e-5 s⁻¹) 는 5min = 300s step 에서도 비약적 변화 없음 (λ Δt ~ 6e-3 ~ 9e-3).
- 매트릭스 지수 2×2 정확히 계산 가능. Autograd 친화.
- Branch 에 적용 가능함이 검증됨 ([2026-04-20 Branch Xenon 진화 검증]) — 이는 Phase 1 을 크게 강화한다.

**L_sigma_a_Xe (Taylor expansion)**:
- σ_a,Xe(P, Tf, M) = σ_0 + ∂σ/∂P · ΔP + ∂σ/∂Tf · ΔTf + ∂σ/∂M · ΔM.
- 시간 비의존, 전 Phase 적용 가능. 계산 저렴.
- Phase 1 적용 정당.

**L_diffusion (L_diff_rel)**:
- 2군 확산 방정식의 CMFD 7-point 잔차.
- **상대 잔차** 형식 (`|residual| / |flux|`) 은 **필수**: flux 가 중심/경계에서 3+ order of magnitude 차이 남. 절대 잔차는 중심 cell 이 gradient 를 독점.
- 상대 형식은 본 도메인에 정확히 맞는 선택.

**L_keff (Rayleigh 몫)**:
- φᵀMφ / φᵀNφ. 분모 φᵀNφ = `Σ Σ_f · φ` 는 정상 운전 중 0 에 가까워지지 않음 (flux 가 전역적으로 양수). **수치 불안정성 우려는 과대평가**.
- 진짜 문제: **keff 의 요구 정밀도 (~10 pcm = 1e-4) 가 flux 의 학습 정밀도 대비 한참 높음**. L_keff 의 gradient signal 이 L_data 의 gradient 에 묻힌다.
- → **후순위 판단은 타당**. Phase 3 말기 또는 후튜닝에서 재검토.

### (c) 잠재 허점 / 개선안

**허점 1 — L_Bateman + L_sigma_a_Xe + L_diffusion 의 gradient 스케일 경합**.
- L_Bateman: Xe 농도 ~1e15 #/cm³, dN/dt ~ 1e11, gradient norm 크다.
- L_diffusion: flux ~ 1e14 #/cm²/s, Σ_a · φ ~ 1e14 · 1e-2 = 1e12, 유사 스케일.
- L_sigma_a_Xe: σ ~ 1e-24 cm² 에 Taylor 기울기 수준, 스케일 매우 작음. **gradient 무시 위험**.
- → L_sigma_a_Xe 는 **별도 λ scaling (×1e20 level)** 으로 올리지 않으면 억압될 것. 수동 λ 튜닝 필수.
- **권장**: Phase 2b 부터 **uncertainty weighting (Kendall 2018)** 도입. 각 loss 의 log-variance 를 learnable parameter 로. ReLoBRaLo 보다 구현 단순하고 성능 유사.

**허점 2 — log(N_Xe) 안전성**.
- log-z-score normalization + exp 역변환 → N_Xe = exp(z + log(μ)) 는 수학적으로 양수 보장.
- **단 float32 precision** 에서 exp(-large) → 0 가능. 반사체 영역 (flux 매우 낮음) → N_Xe 기계 epsilon 근처 → `log(N_Xe)` 발산.
- **권장**: `log(tf.maximum(N_Xe, 1e-20))` 또는 `log1p(N_Xe / 1e-20)` 형태의 ε-clamp 명시. 이미 고려됐을 가능성 있으나 문서화 권장.

**허점 3 — L_diffusion 경계 조건의 Albedo 캘리브레이션 검증 범위**.
- R0~R3 영역별 Albedo 초기값이 캘리브레이션됨 (문서 내 R0 보고서 확인). 그러나 trainable Albedo 의 학습 중 drift 가 초기값 대비 얼마나 벗어나는지는 **Phase 2 이후 모니터링 필요**.
- Albedo 가 무한대 drift 하면 경계 조건 물리성이 깨진다. 클립 또는 정규화 제약 고려.

**허점 4 — L_keff 후순위의 실질적 의미**.
- L_keff 없이 flux 만 학습하면 **keff 는 post-hoc** (학습된 flux 에서 Rayleigh 로 추출) 으로만 존재.
- Phase 3 말기에 L_keff 도입 시, 이미 flux 가 수렴한 상태에서 keff 에 맞춰 flux 를 미세 조정하는 것은 gradient signal 이 약해 효과가 제한적일 수 있음.
- → L_keff 를 **Phase 2b 중반부터** 매우 작은 λ (e.g., λ=0.001) 로 도입하여 gradient 경로만 확보하는 전략 고려.

**권장 우선순위**:
1. Phase 2b 부터 **Uncertainty Weighting** 도입 (Kendall et al., 2018)
2. log 연산에 **ε-clamp 명시**
3. Albedo trainable 의 drift 모니터링 + 클립 정책
4. L_keff 를 Phase 2b 중반부터 소규모 λ 로 조기 도입 검토

**참고 문헌**:
- Raissi, Perdikaris, Karniadakis, "Physics-Informed Neural Networks", J. Comp. Phys. 2019
- Kendall, Gal, Cipolla, "Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics", CVPR 2018
- Bischof & Kraus, "Multi-Objective Loss Balancing for Physics-Informed Deep Learning" (ReLoBRaLo), arXiv 2110.09813
- Wang, Wang, Perdikaris, "When and Why PINNs Fail to Train: A Neural Tangent Kernel Perspective", J. Comp. Phys. 2022

## Q6. 외부 제어 입력 (rod_map, xs_fuel) 처리

### (a) 이론적 타당성

Channel concat 방식은 CNN-기반 PDE 대리모델의 가장 보편적 패턴 (GraphCast, FourCastNet, Pangu-Weather 모두 정적 메타데이터를 채널로 concat). FiLM / cross-attention 은 "전역 스칼라 조건" 에 적합하고, **공간 분포 조건** 에는 채널이 자연스럽다.

### (b) 원자로 물리 특화 적합성

**xs_fuel**:
- 10 channel, 시나리오 단위 상수. 각 cell 의 연료 거시단면적.
- 이는 **공간 분포** 조건이므로 채널 표현이 **올바름**. FiLM 으로 전환할 수 없다 (FiLM 은 전역 스칼라 모듈레이터).

**rod_map**:
- 1 channel, 매 step 변동, 공간 분포.
- 역시 **공간 분포** 조건이므로 채널이 올바름.
- Mamba 의 "급변 신호 약점" 우려는 SSM 일반론의 clean-text 연구에서 나온 것이며, **본 도메인에 직접 적용되지 않음**:
  - 본 도메인에서 rod_map 은 **시간 축으로** 일반적으로 smooth 이동 (제어봉은 물리적으로 연속). 급변은 "새로운 rod 위치" 의 삽입/추출 이벤트 정도.
  - Mamba-3 의 **복소 상태 + Trapezoidal 이산화** 는 Euler 대비 sharp transient 를 훨씬 잘 처리 (O(Δ³) vs O(Δ)).
  - rod_map 의 공간 급변은 Attention (stateless spatial coupling) 이 처리.
- → **Mamba 의 rod_map 처리 능력 우려는 과도**. 현 설계에서 해결되어 있다.

### (c) 잠재 허점 / 개선안

**허점 1 — xs_fuel 의 시간 낭비 (요청서 제기)**.
- 시나리오당 1회 계산 후 cache 가 효율적. 그러나 본 프로젝트는:
  - 입력 CellEmbedder Conv3D 1×1×1 (21→128) 이 매우 저렴 (~2.7K params, 720 cell).
  - 전체 pipeline 비용 대비 <0.1%. 최적화 이득 미미.
  - Condition embedding 경로 분리 시 코드 복잡도 증가.
- **권장**: **현 설계 유지**. Optimization 이며 correctness 아님. 향후 wall-time bottleneck 으로 드러나면 재검토.

**허점 2 — rod_map 의 FiLM/cross-attention 대안 주장 (요청서 제기)**.
- rod_map 은 공간 분포. FiLM (전역 스칼라 modulator) 로 표현하려면 rod_map 전체를 하나의 스칼라로 압축해야 하는데, 이는 **정보 손실**.
- Cross-attention: Q = core state, K/V = rod_map tokens. 이는 **기술적으로 가능** 하지만 720 core cell × 720 rod cell × d = 720² d 계산 (~=518K d), 이는 FullAttention 과 같은 비용. 현 설계의 채널 concat + Attention 이 이미 동일한 표현력을 더 저렴하게 달성한다.
- → **현 설계 (채널 concat) 유지. FiLM / cross-attention 로 변경 불필요**.

**허점 3 — rod_map 의 학습 signal 이 희석될 우려 (실제 허점)**.
- 입력 21ch 중 rod_map 은 **1 ch**. Conv3D 1×1×1 의 첫 가중치 (21→128) 가 rod_map channel 에 부여하는 gradient 영향은 평균 1/21 ≈ 4.8%.
- rod_map 은 본 문제의 가장 중요한 "제어 입력" 인데 입력 비중이 4.8% 밖에 안 됨.
- **완화**: rod_map 을 **별도 작은 임베딩 (1→16 Conv3D 1×1×1)** 을 거친 후 state embedding (21-1=20ch → 112D) 과 concat (16+112=128). 이렇게 하면 rod_map 이 128D 중 16D (12.5%) 를 차지하여 gradient 영향 균형.
- 이는 "별도 조건 경로" 가 아닌 **입력 표현의 rebalancing** 이며 구현 비용 작음.
- **향후 검토 권장** 이나 우선순위 중간.

**권장 우선순위**:
1. 현 설계 (채널 concat) 유지
2. rod_map 입력 임베딩 rebalancing 검토 (Phase 2 성능 미달 시)
3. xs_fuel 효율화는 언급만 하고 미실행

**참고 문헌**:
- GraphCast (Lam et al., Science 2023)
- FourCastNet (Pathak et al., NeurIPS 2022)
- Perez et al., "FiLM: Visual Reasoning with a General Conditioning Layer", AAAI 2018

## 종합 권고 사항 (priority-ranked)

설계 전반은 동시대 SOTA 흐름과 일관되며 구조적 결함은 없다. 그러나 실험 착수 전 다음 4개 항목의 정리를 권한다:

### 🔴 P0 (실험 전 반드시 정리)

1. **Phase 2b 차등 학습률의 100× 비율을 20× 비율로 축소**. Mamba 5e-4, Attention 1e-4, Encoder/Decoder 2e-5. 현 100× 는 co-adaptation 이 아닌 one-way adapt 위험. ULMFiT/LoRA 표준은 10~20×.

2. **Phase 3 에 Scheduled Sampling 명시**. Hard Teacher-Forcing-OFF 전환은 distribution shift 로 학습 불안정 위험. 각 K 단계 내 ε 선형 감소 (ε=0.9 → 0.1) 권장.

### 🟡 P1 (실험 중 관찰 후 결정)

3. **N=2 blocks 를 N=3 으로 baseline 변경**. Xenon oscillation 의 공간-시간 결합 모드는 2 라운드 cross-coupling 으로는 tight. Phase 2a 초기 ablation (N=2 vs N=3) 필수.

4. **Phase 2a → 2b 전환을 epoch 비율 기반에서 Loss plateau 기반 자동 트리거로 변경**.

5. **Uncertainty Weighting (Kendall et al., 2018) 을 Phase 2b 부터 도입**. 수동 λ 는 L_sigma_a_Xe 가 gradient 스케일에서 밀려 학습 안 될 위험.

### 🟢 P2 (모니터링 + 후순위)

6. **Phase 1 학습에 CRS 단일 step 5-10% 혼합** — Branch 편향 완화.

7. **γ/β (AdaLN) 및 α (LAPE ReZero) 크기 모니터링 threshold 사전 정의**. Phase 2b 말기 여전히 |γ| < 0.01 이면 AdaLN 경로 죽음 → concat 단독 전환.

8. **rod_map 입력 rebalancing** (16D 별도 embed) — Phase 2 성능 미달 시 검토.

9. **log() 연산의 ε-clamp 명시 문서화** (수치 안정성).

10. **L_keff 를 Phase 2b 중반부터 매우 작은 λ 로 조기 도입** 검토 (gradient path 유지).
