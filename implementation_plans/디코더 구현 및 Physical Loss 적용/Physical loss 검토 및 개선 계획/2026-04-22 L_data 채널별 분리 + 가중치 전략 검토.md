- `result_ao`: derived via compute_AO(power_3D_pred) → L_AO 만
- 공통 근거: 직접 예측 헤드 부재, L_data direct fit 시 (pred, GT) 와 (formula, GT) 두 경로 경합

→ **정규화 전 채널 간 스케일 최대 ~10²¹ 배 차이** (branch_i135 ~10⁻⁸ vs branch_flux ~10¹³)

### 1.3 단일 MSE 의 잠재 문제

1. **정규화 (Welford / per-channel z-score) 적용 후에도 남는 차이**:
   - **학습 난이도**: power peaking 은 sharp / 국소적, k_eff 는 smooth — 같은 epoch 에 같은 진도로 학습되지 않음
   - **물리적 중요도**: k_eff 100 pcm 이내 vs tcool 1% 이내 — 허용 오차가 다름
   - **수렴 속도**: smooth 양 빠름, peaked 양 느림

2. **단일 MSE 의 결과**:
   - 지배적 채널 (예: power 의 분산이 커서 정규화 후에도 큰 loss) 이 덜 중요한 채널 (예: ao) 의 학습을 방해 가능
   - 어려운 채널 (예: branch_xe 진동) 이 충분히 학습 안 됨

## 2. 자료 탐색 결과 (2024-2026)

### 2.1 GraphCast (Lam et al., Science 2023) — 표준 관행

NVIDIA PhysicsNeMo 의 **공식 GraphCast 구현** (`GraphCastLossFunction`):

```python
loss = (invar - outvar) ** 2                                # [T, C, H, W]
# (a) 채널별 inverse variance (시간 차분 std 의 역수)
loss = loss * 1.0 / torch.square(self.time_diff_std)
# (b) 변수별 명시 가중치
variable_weights = self.variable_weights.view(1, -1, 1, 1)
loss = loss * variable_weights
# (c) 공간 area 가중치 (grid cell 면적)
loss = loss.mean(dim=(0, 1))
loss = torch.mul(loss, self.area)
loss = loss.mean()
```

**3단 가중**:
1. 채널별 inverse variance: `1 / std²` — 각 변수의 시간 차분 표준편차로 정규화
2. 변수별 명시 가중치 `variable_weights`: 물리적 중요도 반영
3. 공간 area 가중치 (본 프로젝트는 quarter-core 1/4 대칭 → sym_type-aware fullcore 환산 필요, L_pload 와 동일 주의)

**GraphCast 의 variable_weights 값** (원문 인용):
> "ω was 1 for the three-dimensional atmospheric variables and 2-meter temperature, and 0.1 for the 10-meter wind components, mean sea level pressure, and accumulated precipitation."

→ 변수군 간 **10배 차등** 가중치 사용. 표면 풍속 / 기압은 의도적으로 낮음.

### 2.2 FourCastNet, Pangu-Weather

- 변수별 정규화 통계 강제 사용 — 채널별 z-score normalization 후 채널 간 가중치 부여 표준
- per-variable weighting 없이는 SciML surrogate 가 안정적 다중 변수 학습 불가

### 2.3 MTL (Multi-Task Learning) 일반 문헌

**전략 A — 고정 명시 가중치** (GraphCast 방식): 수동 튜닝, 해석 가능, 디버깅 쉬움
**전략 B — Homoscedastic Uncertainty Weighting** (Kendall & Gal 2018): 각 task 의 σ_i 학습 가능 파라미터, 자동 보정. 단 본 프로젝트의 ~10⁻⁷ 신호에서 σ 학습 불안정 위험
**전략 C — GradNorm / ReLoBRaLo** (gradient 기반 동적): 이미 본 프로젝트 Phase 2b 중반 이후 도입 결정 (로드맵 §2)

## 3. 본 프로젝트 적용 방안 (검토 출발점)

### 3.1 채널 그룹화 제안 (변수명 기반, 2026-04-22~23 갱신)

**그룹화 원칙** (사용자 결정):
- 그룹명 = **변수명 기반** (power, tcool, tfuel 등) — "Primary / Thermal-hydraulic" 같은 추상 명 금지
- **옵션 A 채택** (2026-04-23): 그룹 단위 λ + 그룹 내 채널 별 inverse variance
- **2-group 변수 (flux, Sigma_f) 는 g1 / g2 별도 그룹** — 군 간 물리 / 스케일 차이 큼, 개별 중요도 조절 필요. N_Xe_I 만 그룹 유지 (둘 다 동일 핵종 계열)
- Option D 배제 채널 (sigma_a_xe, ao) 은 별도 (제외) 행 표기

| 그룹명 | 구성 채널 | 제안 λ 초기값 | 근거 |
|------|---------|:------------:|------|
| **power** | result_power | **1.0** | 원자로 핵심 출력 (MW peaking) |
| **tcool** | result_tcool | 0.5 | 냉각재 온도, 열수력 coupling |
| **tfuel** | result_tfuel | 0.5 | 연료 온도, Doppler 효과 (L_σXe 입력) |
| **rhocool** | result_rhocool | 0.5 | 냉각재 밀도, 감속 효과 (L_σXe 입력) |
| **keff** | result_keff | **1.0** | 임계도 (scalar) |
| **N_Xe_I** | branch_xe + branch_i135 | 0.3 | 핵종 수밀도, L_Bateman 가 주요 제약 (A+B Hybrid), L_data 는 보조. 동일 Xe/I-135 계열 → 그룹 유지 |
| **flux_g1** (fast) | branch_flux[g1] | 0.5 | Fast 중성자속, 핵분열 생산 주 경로 (νΣ_f1·φ_1), L_diff_rel R_g1 과 정합 |
| **flux_g2** (thermal) | branch_flux[g2] | 0.5 | Thermal 중성자속, Xe 흡수 주 경로 (σ_a^Xe·φ_2), L_diff_rel R_g2 와 정합 |
| **Sigma_f_g1** (신설) | branch_Sigma_f[g1] | 0.3 | **L_data_Sigma_f g1 (2026-04-22 신설 state ch 10)**, L_Bateman 예측값 입력. Fast 핵분열 단면적 |
| **Sigma_f_g2** (신설) | branch_Sigma_f[g2] | 0.3 | **L_data_Sigma_f g2 (2026-04-22 신설 state ch 11)**, L_Bateman 예측값 입력. Thermal 핵분열 단면적 |
| **(제외) sigma_a_xe** | branch_sigma_a_xe | — | **Option D 배제** — L_Taylor 가 σ̂ 미사용 (T̂_f, ρ̂_m 경로) + L_Bateman direct gradient 보유 → 경합 회피 |
| **(제외) ao** | result_ao | — | **Option D 확장 배제 (2026-04-23)** — AO 는 power 3D 의 상부/하부 절반 비율 로 유도 (derived), L_AO 만 |
| **(제외) Yield** | branch_yield_xe, branch_yield_i, critical_yield_xe, critical_yield_i | — | L_Bateman 입력 (보조 데이터), L_data 학습 대상 아님 (Physical Loss 레퍼런스 §1.3) |
| **(제외) Sm** | branch_sm | — | state 제외 (§0.2 정의), L_data 대상 아님 |

→ **출발점** 이며, ablation 으로 조정. GraphCast 관행 = **10배 이내 범위** 권장.

**2-group 변수 (flux / Sigma_f) g1·g2 분리 근거**:
- 스케일 차이: flux g1 ~10¹³ vs g2 ~10¹² n/cm²/s, Σ_f 도 군 별 약 10배 차이 — inverse variance 로 일부 흡수 가능 하나 λ 로 추가 제어 가능
- 물리 역할 차이: g1 = 핵분열 생산 주 경로 (νΣ_f1·φ_1), g2 = Xe 흡수 주 경로 (σ_a^Xe·φ_2) — 중요도 별도 조정 여지
- L_diff_rel 의 군 별 R_g1/R_g2 별도 통계 와 정합 — 정규화 일관성

**N_Xe_I 그룹 유지 근거**:
- N_Xe, N_I 는 동일 Xe/I-135 붕괴 사슬 (I → Xe → 흡수)
- 두 채널 모두 L_Bateman 의 N̂ 비교 대상 (A+B Hybrid 의 A 항)
- 스케일 유사 (~10⁻⁸ /barn-cm) 및 물리적 결합 강함 → 동일 λ 자연스러움

### 3.2 정규화 전략 (2단 가중, 2026-04-23 갱신)

```
L_data_total = Σ_group λ_group × Σ_channel ∈ group MSE(pred_norm_c, GT_norm_c)
```

- **Step 1**: 채널별 z-score 정규화 (Welford 누적, 기존 normalizer 사용)
- **Step 2**: 그룹별 명시 가중치 λ_group (변수명 기반, §3.1 표 참조)
- ~~Step 3 (GraphCast inverse variance, `/time_diff_std[c]²`)~~ — **미도입 (2026-04-23 결정)**: z-score 단일 정규화 + 그룹별 명시 λ (옵션 A) 의 2단 가중 만 으로 채널 균형 충분 판단. 학습 후 monitoring (gnorm 비율, 채널별 수렴 속도) 결과 따라 추후 도입 검토 가능.

### 3.3 ReLoBRaLo 와의 관계 (2026-04-23 확정)

- **Phase 1a / 1b / 2a / 2b 전반 (중반 이전)**: 위 그룹별 고정 가중치 (manual)
- **Phase 2b 중반 이후**: ReLoBRaLo 활성화 (3 사전조건 충족 시, [Loss 설계 운용 가이드 §5.4.2](2026-04-23 Loss 설계 운용 가이드 (계층 분류 + 가중치 + 모니터링).md) 참조)
  - **활성 단위 (확정)**: **그룹 단위 = 7 개 항** — L_data, L_data_halo, L_Bateman, L_σXe, L_diff_rel, L_pload, L_AO 의 outer weight 자동 조정
  - L_data 의 inner_groups (변수명 기반 10 그룹), L_Bateman 의 inner A/B (0.75/0.25) 는 **manual 고정 유지** (ReLoBRaLo 외부 layer)
  - 개별 채널 단위 (15-17 개 항) 는 항 수 폭증 → 학습 불안정 우려 → 미채택

### 4.3 위험 / 우려 사항

1. **복잡도 증가**: 단일 MSE → 5+ 그룹 × inverse variance × 그룹 가중치 = 디버깅 비용 증가
2. **그룹 분류 자체의 임의성**: 본 프로젝트의 어느 채널이 어느 그룹에 속해야 하는지 명확한 물리 기준 없음 — 임의 분류 → ablation 으로 검증 필요
3. **ReLoBRaLo 와의 상호작용 미검증**: 채널별 분리 + ReLoBRaLo 의 조합이 GraphCast / FourCastNet 에서 실증된 사례 없음 (그쪽은 manual 고정 가중치 위주)
4. **Inverse variance 사전계산 의 정합성**:
   - time_diff_std 가 Phase 별로 변화 가능 (예: rod 변화 시 vs 안정 운전 시) — 단일 std 가 합리적인가
   - LP 별로 다른 분포 — LP 평균 vs LP 별 별도 std 결정 필요
