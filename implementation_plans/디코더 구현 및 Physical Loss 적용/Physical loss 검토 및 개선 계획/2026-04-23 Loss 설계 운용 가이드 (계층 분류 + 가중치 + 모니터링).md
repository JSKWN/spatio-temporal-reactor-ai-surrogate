## 1. Loss 계층 분류 (3-tier)

본 프로젝트 의 Loss 들 은 **성격 이 다른 3 계층** 으로 분류. **같은 계층 내 비교 / 평가** 원칙 적용 — 계층 다른 loss 에 동일 가중치 부여 / 비교 금지.

### 1.1 계층 정의

| 계층 | 명칭 | 역할 | Loss 종류 |
|:---:|------|------|----------|
| **1** | **Data supervision** | 직접 GT 매칭 (주력 학습 신호) | L_data (그룹 단위, σ_a^Xe / AO 배제), L_data_halo, **L_data_Sigma_f (L_data 의 inner group 으로 통합, §3.1)** |
| **2** | **Physics residuals** | 물리 잔차 (공식 제약) | L_Bateman (A+B Hybrid), L_σXe (Taylor), L_diff_rel (CMFD) |
| **3** | **Aggregation consistency** | 집계 정합성 (보조 제약) | L_pload, L_AO |

### 1.2 계층 별 특성

| 계층 | 학습 신호 강도 | 주된 역할 | 가중치 운용 원칙 |
|:---:|:----------:|----------|----------------|
| 1 | **강** (주력) | 모든 state 채널 의 직접 GT fit | outer 1.0 / 그룹별 inner λ 차등 (옵션 A) |
| 2 | **중** | Phase 1a 후 데이터 anchor 위에 물리 제약 추가 (DG-PINN) | 5종 동일 outer 0.1 (Phase 1b annealing 종료 후) |
| 3 | **약** (보조) | L_data 가 놓친 global 편향 정정 (셀별 power → 합산 / 비율 scalar) | outer 0.1 — 작아도 충분 (이미 L_data power 그룹 λ=1.0 가 주 학습) |

### 1.3 계층 별 비교 / 평가 원칙

- **계층 내 비교**: 같은 계층 내 loss 들 의 값 / gradient norm 비교 → 가중치 조정
- **계층 간 비교 금지**: 계층 1 (data) 과 계층 3 (aggregation) 의 절대값 / gradient 직접 비교 시 잘못된 결론
- **계층 별 모니터링** (§6): 계층 별 gradient norm 합산 비교

## 2. L_Bateman A+B Hybrid 가중치 구조 (#3 정정)

### 2.1 수식

```
L_Bateman = outer_weight × (inner_A · L_A + inner_B · L_B)   (정규화 공간)

L_A = MSE(Ñ̂, Ñ_phys)               ← N̂ 에 physics constraint
L_B = MSE(Ñ_phys, Ñ_GT(t+1))        ← 예측 계수 에 GT anchor

inner_split: 합 = 1.0 (정규화 명시)
  inner_A: 0.75
  inner_B: 0.25

outer_weight: 0.1 (Phase 1b annealing 종료 후)

Effective:
  L_A: outer × inner_A = 0.1 × 0.75 = 0.075
  L_B: outer × inner_B = 0.1 × 0.25 = 0.025
```

### 2.2 옵션 A 채택 근거 (Inner 합=1.0 정규화)

**문제**: 이전 표기 (`λ_A=0.3 + λ_B=0.1`) 는 inner 합 = 0.4 로 정규화 안 됨 → outer (0.1) 곱 시 effective L_A=0.03, L_B=0.01 로 다른 계층 2 loss (각 effective 0.1) 대비 3-10배 약화 발생.

**옵션 A 정정**: inner 합 = 1.0 으로 명시 정규화 → outer 0.1 곱 시 effective 합 = 0.1 (다른 계층 2 loss 와 동일 스케일).

**비율 유지**: A:B = 0.75:0.25 = 3:1 (이전 0.3:0.1 = 3:1 비율 일관)

### 2.3 A:B 비율 근거

- **A 항** (N̂ ↔ N_phys): self-consistency, 외삽 / OOD dynamics stability — rollout 에서 핵심
- **B 항** (N_phys ↔ N_GT): GT anchor, model misspecification 보정
- A > B: rollout 시 A 항 의 self-consistency 가 dynamics 안정 핵심
- 상세 근거: [Physical Loss §1.9.2](2026-03-30 Physical Loss 통합 레퍼런스.md)

### 2.4 Config 매핑

```yaml
losses:
  L_Bateman:
    enabled: true
    outer_weight: 0.1     # annealing target (Phase 1b 종료 후)
    inner_split:
      A: 0.75            # MSE(N̂, N_phys)
      B: 0.25            # MSE(N_phys, N_GT)
    # Effective: L_A = 0.075, L_B = 0.025
```

## 3. L_data 가중치 구조 (#6 정정)

### 3.1 단일 outer + inner_groups 통합

**구조**: L_data 는 단일 outer weight (1.0) + inner_groups (변수명 기반 옵션 A). **L_data_Sigma_f 별도 outer 폐기** → L_data 의 inner group 으로 통합.

```yaml
losses:
  L_data:
    enabled: true
    outer_weight: 1.0
    inner_groups:        # 옵션 A: 그룹 단위 λ + 그룹 내 inverse variance
      power:      1.0
      keff:       1.0
      tcool:      0.5
      tfuel:      0.5
      rhocool:    0.5
      flux_g1:    0.5
      flux_g2:    0.5
      Sigma_f_g1: 0.3   # (구) L_data_Sigma_f 통합
      Sigma_f_g2: 0.3   # (구) L_data_Sigma_f 통합
      N_Xe_I:     0.3   # branch_xe + branch_i135 동일 λ
    excluded:           # Option D + 확장
      - sigma_a_Xe      # L_σXe + L_Bateman 만 (derived)
      - AO              # L_AO 만 (derived from power 3D)

  L_data_halo:
    enabled: true
    outer_weight: 0.3   # halo (6,6) 영역
```

### 3.2 그룹화 단위 (옵션 A 채택)

- **그룹 단위 λ** (그룹 내 채널 별 inverse variance 추가 가중 **미도입**, 2026-04-23 결정)
- 채널 정규화 = **z-score 단일** (Welford 누적, 채널별 μ/σ 만 적용)
- **2-group 변수 (flux, Sigma_f) 는 g1 / g2 별도 그룹** — 군 간 물리 / 스케일 차이 큼
- **N_Xe_I 만 그룹 유지** (Xe/I-135 동일 붕괴 사슬)
- 상세 근거: [L_data 채널 분리 검토 §3.1](2026-04-22 L_data 채널별 분리 + 가중치 전략 검토.md)

### 3.3 정규화 전략 (z-score 단일, GraphCast inverse variance 미도입)

```
L_data_total = Σ_group λ_group × Σ_channel∈group MSE(pred_norm_c, GT_norm_c)
```

- **Step 1**: 채널별 z-score 정규화 (Welford 누적, 기존 normalizer)
- **Step 2**: 그룹별 명시 가중치 λ_group (변수명 기반, §3.1 표 참조)
- ~~Step 3 (GraphCast inverse variance)~~ — **미도입 (2026-04-23 결정)**: z-score 단일 정규화 만 으로 충분 판단. 추가 `1/time_diff_std[c]²` 가중 안 함.

> **미도입 근거**: GraphCast 의 inverse variance 는 정적 분산 큰 변수 (T_fuel) 가 학습 dominant 하는 것 방지 목적. 본 프로젝트 는 z-score 정규화 + 그룹별 명시 λ (옵션 A) 의 2단 가중 만 으로 채널 균형 충분 판단. 학습 후 monitoring (§6.2 gnorm 비율) 결과 따라 추후 도입 검토 가능.

## 4. Annealing 정책 (#7 정정)

### 4.1 ratio-based + 하한

```yaml
training:
  phase1b:
    physics_weight_annealing:
      mode: linear
      ratio_of_phase1: 0.10               # Phase 1 epoch 의 10%
      min_epochs: 5                        # 하한
      max_epochs: null                     # 또는 20 (선택적 상한)
      # 실제: max(min_epochs, int(phase1_total_epochs * ratio_of_phase1))
      start: 0.0
      end_L_Bateman: 0.1                   # 5종 모두 동일 target
      end_L_sigma_a_Xe: 0.1
      end_L_diff_rel: 0.1
      end_L_pload: 0.1
      end_L_AO: 0.1
      # L_data 는 1.0 유지 (ramp 대상 아님)
```

**구현 의사 코드**:
```python
annealing_epochs = max(5, int(phase1_total_epochs * 0.10))
# 선택적 상한: max(5, min(20, int(phase1_total_epochs * 0.10)))
```

### 4.2 5종 동시 ramp-up

- Phase 1b 시작 시 L_physics 5종 (계층 2 + 계층 3) 모두 동시에 0 → 0.1
- L_Bateman 의 inner_split 은 정해진 비율 유지 (annealing 동안 outer 만 변화, inner_A=0.75 / inner_B=0.25 고정)
- catastrophic forgetting (Phase 1a 학습 망각) 방지 목적

#### 5.4.1 K-step 1/K 평균화

**문제**: K-step rollout 에서 L_Bateman 을 K-step 매 step 단순 합산 시, K 배 스케일 증가 → L_data 와 균형 깨짐 (K=150 시 ~150배).

**채택 안: 1/K 평균화**

```
L_Bateman_total = (1/K) · Σ_{k=1}^{K} L_Bateman(step_k)
                = (1/K) · Σ_{k=1}^{K} [outer × (inner_A · L_A_k + inner_B · L_B_k)]

→ K 와 무관 하게 L_data 와 동일 스케일 유지
→ outer_weight = 0.1 (Phase 1b/2 와 동일) 그대로 사용
```

**적용 대상** (계층 2 모두 동일 원칙):
- L_Bateman: 1/K 평균
- L_σXe: 1/K 평균 (K-step 매 step σ_a^Xe 비교)
- L_diff_rel: 1/K 평균 (K-step 매 step PDE 잔차)

**계층 3 (L_pload, L_AO)**: K-step 매 step 적용 시 동일 1/K 평균 (스케일 일관성)

**계층 1 (L_data)**: K-step 매 step 1/K 평균 (이미 K-curriculum 표준 — GraphCast 패턴)

**Config 매핑**:

```yaml
training:
  phase3:
    K_aggregation:
      mode: "average"          # 1/K 평균화
      apply_to: [L_data, L_data_halo, L_Bateman, L_sigma_a_Xe, L_diff_rel, L_pload, L_AO]
      # 모든 시간 의존 loss 에 일괄 적용
```

#### 5.4.2 ReLoBRaLo 활성 (Phase 2b 중반 부터 Phase 3 까지)


## 6. 모니터링 가이드 (#8)

### 6.1 기록 항목 (매 epoch)

```python
metrics = {
    # === Loss 값 ===
    'L_data':                L_data.item(),
    'L_data_halo':           L_data_halo.item(),
    'L_physics_total':       sum([L_Bateman, L_σXe, L_diff_rel, L_pload, L_AO]).item(),
    'L_Bateman_combined':    L_Bateman.item(),
    'L_Bateman_A':           L_A.item(),     # sub-loss 별도 (필수)
    'L_Bateman_B':           L_B.item(),
    'L_σXe':                 L_σXe.item(),
    'L_diff_rel':            L_diff_rel.item(),
    'L_pload':               L_pload.item(),
    'L_AO':                  L_AO.item(),

    # === Gradient norm (계층별) ===
    'gnorm_layer1_data':     compute_gnorm_subset([L_data, L_data_halo]),
    'gnorm_layer2_physics':  compute_gnorm_subset([L_Bateman, L_σXe, L_diff_rel]),
    'gnorm_layer3_aggr':     compute_gnorm_subset([L_pload, L_AO]),

    # === 물리 해석 지표 (원 공간) ===
    'bateman_residual_%':    (L_Bateman_orig_scale / source_scale).mean() * 100,
    'diff_residual_g1_%':    (L_diff_rel_g1_orig / source_g1_scale).mean() * 100,
    'diff_residual_g2_%':    (L_diff_rel_g2_orig / source_g2_scale).mean() * 100,
    'pload_error_%':         abs(pload_pred - pload_GT) / pload_GT * 100,
    'AO_error_%':            abs(AO_pred - AO_GT) / abs(AO_GT) * 100,
}
```

### 6.2 판단 기준

#### 계층별 gradient norm 비율

```
gnorm_layer2_physics / gnorm_layer1_data
  < 0.1 :  physics 무시되는 신호 → physics 가중치 ↑ 검토
  > 2.0 :  physics 과도해 data 학습 방해 → physics 가중치 ↓ 검토
  적정 범위: 0.3 ~ 1.0
```

#### 물리 해석 지표 (Phase 1b annealing 완료 후)

| 지표 | 목표 | floor (구조적 한계) | 해석 |
|------|------|------------------|------|
| `bateman_residual_%` | ≤ 1% | ~0.9% (Full PC 부재) | floor 보다 훨씬 작으면 overfit 의심 |
| `diff_residual_g1_%` | ~2.45% (MAS_NXS) 근처 | 2.25% (MAS_XSL) | floor 보다 훨씬 크면 학습 부족 |
| `diff_residual_g2_%` | ~2.19% (MAS_NXS) 근처 | 6.39% (MAS_XSL) | 동상 |
| `pload_error_%` | ≤ 0.5% | — | 누설 적분 정확도 |
| `AO_error_%` | ≤ 1% | — | 상부/하부 비율 정확도 |

상세 floor 근거: [Physical Loss §3.4.6](2026-03-30 Physical Loss 통합 레퍼런스.md)

### 6.3 sub-loss 별 모니터링 (L_Bateman A+B)

- `L_Bateman_A` 와 `L_Bateman_B` 를 **별도 스칼라** 로 기록
- 두 항 의 비율 변화 추적 → A 가 압도적 (B → 0) 또는 그 반대 시 학습 불균형 신호
- Phase 2b ReLoBRaLo 활성 시 A, B 별도 항 으로 등록 → adaptive balancing

#### 신규 단계 요약

| Step | 산출물 | 입력 | 비고 |
