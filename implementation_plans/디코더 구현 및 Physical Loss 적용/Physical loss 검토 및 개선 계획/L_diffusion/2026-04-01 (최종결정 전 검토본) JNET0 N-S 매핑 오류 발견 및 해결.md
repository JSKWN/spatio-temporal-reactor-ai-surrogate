# JNET0 N/S 매핑 오류 발견 및 해결

## 발견 일자: 2026-04-01

---

## 1. 문제 현상

Albedo 캘리브레이션(R0~R3) 과정에서 다음 이상 현상 발생:

- **50:50 양수/음수 분할**: radial 경계면에서 정확히 절반이 α > 0, 절반이 α < 0
- **N/S 전류 연속 불일치**: 인접 연료-반사체 노드 쌍에서
  - E/W 면: `fuel.E + neighbor.W = 0` (완벽 연속 ✓)
  - N/S 면: `fuel.N + neighbor.S ≠ 0` (연속 불성립 ✗)
- **내부 연료-연료 면에서도 N/S 불일치** → 반사체 문제가 아닌 근본적 매핑 오류

## 2. 원인: J 인덱스 방향 매핑 오류

### MAS_NXS의 I, J 인덱싱 규약

MAS_NXS에서 I = 열(X방향), J = 행(Y방향), 1-based.

**핵심**: MASTER에서 **J가 증가하면 남쪽(South) 방향**으로 이동.
이는 행렬 출력이 위에서 아래로 내려가는 규약과 일치:

```
J=1  → 노심 최상단 (North)
J=2  → ...
...
J=ny → 노심 최하단 (South)
```

### 올바른 JNET0 방향 매핑

```
JNET0-E = +I 방향 (동쪽, I 증가)
JNET0-W = -I 방향 (서쪽, I 감소)
JNET0-S = +J 방향 (남쪽, J 증가)  ← 핵심!
JNET0-N = -J 방향 (북쪽, J 감소)  ← 핵심!
```

### 이전 코드의 잘못된 가정

```python
# ❌ 이전 코드: j 증가 = 북쪽(N)으로 가정
FACE_DIR = {'N': (1, 0), 'S': (-1, 0), 'E': (0, 1), 'W': (0, -1)}
# j+1 = N방향, j-1 = S방향으로 처리
```

```python
# ✅ 올바른 매핑: J 증가 = 남쪽(S)
# N면 = J-1 방향, S면 = J+1 방향
# (I,J).S면은 (I,J+1).N면과 맞닿음
# (I,J).N면은 (I,J-1).S면과 맞닿음
```

## 3. 검증: 전류 연속 테스트

내부 연료-연료 면에서 `outward + outward = 0` 검증 (K=11, g=1):

### ✅ 올바른 매핑 (확정)

```
(I,J).E + (I+1,J).W = 0.000e+00  ✓  (E = +I)
(I,J).S + (I,J+1).N = 0.000e+00  ✓  (S = +J = 남쪽)
(I,J).N + (I,J-1).S = 0.000e+00  ✓  (N = -J = 북쪽)
```

모든 조합에서 **정확히 0** → 전류 연속 완벽 성립.

### ✗ 이전의 잘못된 매핑

```
(I,J).N + (I,J+1).S ≠ 0  ✗  (N을 +J로 가정 → 틀림)
```

## 4. 노심 형상 예시로 이해

MAS_INP 어셈블리 배치 (`ibat2d(i,j), i=1,nx`), j=1,ny):

```
        ← I 증가 (동쪽, East) →

 J=1  ↑  o   o   o   R4  R2  R1  R2  R4  o   o   o     ← 북쪽(North)
 J=2  |  o   o   R6  R3  A3  B3  A3  R3  R6  o   o
 J=3  |  o   R6  R5  A3  A2  A2  A2  A3  R5  R6  o
 J=4  |  R4  R3  A3  B5  A3  A2  A3  B5  A3  R3  R4
 J=5  |  R2  A3  A2  A3  A2  A3  A2  A3  A2  A3  R2
 J=6  J  R1  B3  A2  A2  A3  A2  A3  A2  A2  B3  R1
 J=7  증  R2  A3  A2  A3  A2  A3  A2  A3  A2  A3  R2
 J=8  가  R4  R3  A3  B5  A3  A2  A3  B5  A3  R3  R4
 J=9  |  o   R6  R5  A3  A2  A2  A2  A3  R5  R6  o
 J=10 |  o   o   R6  R3  A3  B3  A3  R3  R6  o   o
 J=11 ↓  o   o   o   R4  R2  R1  R2  R4  o   o   o     ← 남쪽(South)
```

**예시: 연료 A3 (I=5, J=2) 의 JNET0 방향**

```
                    (I=5, J=1) = R3 반사체
                         ↑
                    JNET0-N (J 감소 = 북쪽)
                         |
  (I=4, J=2) ← JNET0-W ─┼─ JNET0-E → (I=6, J=2)
  = R6 반사체              |              = A3 연료
                    JNET0-S (J 증가 = 남쪽)
                         ↓
                    (I=5, J=3) = A3 연료
```

- **JNET0-N**: 이 노드의 **북쪽 면** (J 감소 방향, J=1 반사체와 인접)
- **JNET0-S**: 이 노드의 **남쪽 면** (J 증가 방향, J=3 연료와 인접)
- **JNET0-E**: 이 노드의 **동쪽 면** (I 증가 방향, I=6 연료와 인접)
- **JNET0-W**: 이 노드의 **서쪽 면** (I 감소 방향, I=4 반사체와 인접)

## 5. 이 오류의 영향 범위

### 영향 받은 분석
- Albedo R0~R3의 radial 경계면 분류: N/S면의 이웃 노드를 잘못 식별
- 50:50 양수/음수 분할: N/S 방향 이웃이 뒤바뀌어 반사체가 아닌 연료를 반사체로 식별 (또는 그 반대)
- Fick's law 검증: N/S 면의 이웃 flux 비교 대상이 잘못됨

### 영향 받지 않은 분석
- **밸런스 테스트 (SUM × 2 = 0%)**: 6면 합산이므로 방향 무관 → 여전히 유효
- **JNET0 = half net current (α=2)**: 밸런스 기반이므로 영향 없음
- **ABS, SCA, NFS, FLX 등 스칼라 물리량**: 방향과 무관

### 해결 방안
이전 코드에서 `FACE_DIR`의 N/S 매핑을 수정:
```python
# ❌ 이전
FACE_DIR = {'N': (+1, 0), 'S': (-1, 0), 'E': (0, +1), 'W': (0, -1)}

# ✅ 수정 (J 증가 = 남쪽)
FACE_DIR = {'N': (-1, 0), 'S': (+1, 0), 'E': (0, +1), 'W': (0, -1)}
# N면 이웃 = J-1 (j_nd 기준), S면 이웃 = J+1 (j_nd 기준)
```

Albedo 캘리브레이션을 수정된 매핑으로 재수행하면 50:50 문제 해소 기대.
