## 2. AdaLN-Zero (DiT) — p_load 조건 주입

> **출처**: Peebles & Xie, "Scalable Diffusion Models with Transformers", ICCV 2023

### 메커니즘

기존 ViT 블록의 LayerNorm을 조건부 변조로 교체:

```
기존:       x = x + Attention(LayerNorm(x))
            x = x + FFN(LayerNorm(x))

AdaLN-Zero: (γ1, β1, α1, γ2, β2, α2) = MLP(y)
            x = x + α1 ⊙ Attention(γ1 ⊙ LN(x) + β1)
            x = x + α2 ⊙ FFN(γ2 ⊙ LN(x) + β2)
```

- **y**: 조건 신호 (우리의 경우 p_load 임베딩)
- 블록당 **6개 벡터** (γ, β, α) × (attention, FFN) = 단일 MLP에서 생성
- **α 초기값 = 0** → 학습 초기 각 블록이 identity → 안정적 학습 시작

### DiT 논문 조건 주입 비교 실험

| 방법 | FID-50K | 메커니즘 |
|------|:-------:|----------|
| In-context token | 최하 | 조건을 추가 토큰으로 append |
| Cross-attention | 중간 | 조건을 KV, hidden을 Q |
| AdaLN | 양호 | LN 후 (γ,β) 변조 |
| **AdaLN-Zero** | **최우수** | AdaLN + zero-init α 게이팅 |

모델 스케일이 커질수록 격차 확대 (DiT-XL/2: FID 2.27).

### 우리 모델에의 적용

- 기존 Block/Grid Attention3D + FFN 구조 → ViT 블록과 동형 → **drop-in 교체**
- p_load(스칼라) → MLP → y → 블록별 (γ,β,α) 생성
- 모든 공간 토큰에 동일하게 적용 (전역 운전 조건이므로 적합)
- TF 2.14: LayerNorm(center=True, scale=False) + Dense + element-wise ops

### FiLM과의 비교

| 특성 | FiLM | AdaLN-Zero |
|------|------|-----------|
| 변조 위치 | feature map 직접 | **LayerNorm 후** (정규화된 분포에 적용 → 안정적) |
| 잔차 제어 | 없음 | **α 게이팅** (블록 기여도 학습) |
| 초기화 | 랜덤 | **α=0** (identity 시작 → 학습 안정성) |
| 적용 대상 | CNN 범용 | **Transformer/Attention 구조 특화** |

→ 우리 인코더가 Attention 기반이므로 **AdaLN-Zero가 FiLM보다 적합**.

## 3. Calibration-FiLM — xs_fuel 고정 물성 주입 (대안)

> **출처**: arXiv 2601.16123, 2026

### 메커니즘

타임스케일 분리를 활용:
1. **느린 경로**: GCN이 캘리브레이션 데이터 → 256차원 하드웨어 임베딩 h
2. **FiLM 생성**: MLP가 h → 레이어별 (γ, β) 
3. **빠른 경로**: CNN이 실시간 데이터 처리, FiLM으로 변조
4. **Weight folding**: (γ,β)를 CNN 가중치에 사전 반영 → 추론 시 추가 비용 0

### 우리 모델에의 적용 (04-03 초안 대안)

```
현재 초안:  xs_fuel(10ch) → 채널 concat → 인코더(21ch 입력)
대안:       xs_fuel(10ch) → 3D CNN 인코더 → (γ,β) → 메인 인코더의 각 Conv/Attention 변조
```

| 비교 | 채널 concat (현재 초안) | Calibration-FiLM (대안) |
|------|:---:|:---:|
| 구현 복잡도 | 단순 | 중간 (별도 인코더 + FiLM 연결) |
| 정보 보존 | 10ch 전부 전달 | bottleneck 가능성 |
| 추론 비용 | 매 스텝 21ch 처리 | weight folding 시 11ch와 동일 |
| 공간 특이성 | 노드별 다른 물성 직접 반영 | 전역 벡터로 압축 시 공간 정보 손실 |

**현재 판단: 미적용**. xs_fuel은 (20,5,5) [데이터 입력] / (20,6,6) [인코더 처리] 위치마다 다른 어셈블리 조성(A2, B5 등)이므로 공간 비균일.
- FiLM의 (γ,β)는 **채널별 스칼라** → 모든 공간 위치에 동일 변조 적용 → 노드별 물성 차이 표현 불가
- 공간 해상도까지 유지한 (γ,β)를 생성하면 (20,5,5,C) [입력] 또는 (20,6,6,C) [인코더 처리] 크기가 되어 concat과 연산량 동일 → FiLM 분리의 이점 소멸
- Calibration-FiLM은 **공간 균일 물성**(예: 단일 디바이스 노이즈 특성)에만 유효한 기법
- → 초안의 **채널 concat이 정보 손실 없이 최적**

## 4. PeRCNN (Π-block) — 물리 구조 하드코딩

> **출처**: Rao et al., 2021 (arXiv 2105.00557)

### 메커니즘

```
Π-block: output = Σ_k (Π_j Conv_j(U)) + bias
```

- 여러 Conv 분기의 출력을 **element-wise 곱셈** 후 합산
- 곱셈 구조가 PDE 비선형항을 자연스럽게 표현: `u × ∂²u/∂x²` → Conv(u) × Conv(u)
- 물리 계수는 **Conv 필터 가중치로 학습** (하드코딩 아님)
- Euler 전진: `U(t+1) = U(t) + Δt × Π-block(U(t))`

### 우리 모델에의 적용 가능성

xs_fuel 채널과 state 채널의 물리적 곱셈 관계를 인코딩:
- 반응률: `Σ_f × φ` (xs_fuel ch[1] × state ch[6,7])
- 흡수률: `Σ_a × φ` (xs_fuel ch[2]+ch[1] × state ch[6,7])

Conv 분기 A(xs_fuel) × Conv 분기 B(state) → 물리적 반응률 구조를 학습

**현재 판단**: 아키텍처 전체 교체가 필요하여 현 단계에서는 미적용. 다만 인코더 내부의 **선택적 곱셈 경로**로 부분 도입 검토 가능.

## 5. Perceiver IO — 이종 입력 통합

> **출처**: Jaegle et al., DeepMind, ICML 2021

### 메커니즘

```
Read:    cross-attention(latent_queries, input_tokens) → latent
Process: self-attention(latent) × N layers → latent
Write:   cross-attention(output_queries, latent) → output
```

- 입력 크기와 무관한 고정 latent 배열 (예: 256개)로 압축
- 스칼라, 3D 텐서, 시퀀스 등 이종 입력을 단일 토큰 시퀀스로 통합
- p_load는 1개 토큰, 3D state는 500개 토큰으로 자연스럽게 혼합

### 우리 모델에의 적용

500토큰(20×5×5 quarter) 또는 720토큰(20×6×6 halo expand 후)은 충분히 작아서 Perceiver IO의 핵심 장점(입력 크기 축소)이 불필요.
오히려 spatial inductive bias(Conv3D)를 잃는 단점이 큼.
→ **현 단계 미적용**. 공간 격자가 커질 경우(full-core 18×18) 재검토.

## 6. PI-CRNN — 공간/시간 분리 학습

> **출처**: Menicali et al., arXiv 2505.10919, 2026

### 메커니즘

1. **CAE** (Convolutional Autoencoder): stride=2 Conv로 공간 압축 (~16배)
2. **ConvLSTM**: 압축된 latent 공간에서 시계열 모델링
3. **분리 학습**: CAE 먼저 학습(재구성 loss) → 동결 → ConvLSTM 학습(PDE loss)

### 우리 모델과의 유사점/차이

| 항목 | PI-CRNN | 우리 모델 |
|------|---------|----------|
| 공간 인코더 | CAE (stride Conv) | MaxViT3D 기반 (Attention + Conv) |
| 시간 모델 | ConvLSTM | Mamba SSM |
| 분리 학습 | ○ (CAE 동결 후 시간 학습) | 미정 (end-to-end vs 분리) |
| PDE loss | 질량/운동량/에너지 보존 | L_diffusion, L_Bateman |
| static/dynamic 분리 | 미다룸 | xs_fuel 분리 설계 |

**참고할 점**: 분리 학습 전략 (공간 인코더를 먼저 학습 → 동결 → 시간 모델 학습)이 학습 안정성에 기여할 수 있음.

## 7. 기타 기법

### MoNODE (NeurIPS 2023)
- 정적 모듈레이터 s를 ODE 상태에 concat: `dz/dt = f([z; s], t)`
- s = 전체 시퀀스의 시간 평균으로 생성
- **한계**: 1D latent 공간 전제. 3D 공간 구조 직접 처리 불가 → 미적용

### ContextAR (arXiv 2505.12274, 2025)
- CCA (Conditional Context-aware Attention): 조건 유형별 마스킹으로 교차 어텐션 차단
- 각 조건(canny, depth 등)을 별도 토큰 서브시퀀스로 append
- **한계**: 2D 이미지 생성용. 마스킹 아이디어는 xs_fuel ↔ state 어텐션 분리에 차용 가능하나, 우리 격자(500토큰)에서는 불필요한 복잡성

### Temporal FiLM (Birnbaum et al., NeurIPS 2019)
- RNN hidden state → (γ,β) → Conv 변조
- p_load 같은 시간 의존 스칼라 주입에 적합
- **위치**: AdaLN-Zero에 흡수됨 (동일 목적, 더 우수한 성능)
