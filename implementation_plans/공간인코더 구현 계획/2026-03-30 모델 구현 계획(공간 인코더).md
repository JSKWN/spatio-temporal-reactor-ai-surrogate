## 입출력 텐서 설계

### 자기회귀 제약

추론 시 디코더 출력(t) → 인코더 입력(t+1) 피드백. 따라서:
- **시간 가변 상태**: 입출력 채널이 동일해야 함
- **고정 물성 (xs_fuel)**: 자기회귀 루프 밖 — 별도 경로로 1회 또는 매 스텝 주입
- **쿼리 (rod_map, p_load)**: 외부에서 매 스텝 주입 — 모델 예측 대상이 아님

### 전체 HDF5 물리량 입출력 비교

> ○ = 포함, × = 미포함, △ = 검토 필요
> **입력(t)** = 인코더가 받는 시점 t의 상태, **출력(t+1)** = 디코더가 예측하는 시점 t+1의 상태

#### E. 스칼라 출력 — 별도 헤드 + 3D→스칼라 일관성 loss

| 물리량 | HDF5 필드 | 입력 | 출력 | Physical Loss | 비고 |
|--------|-----------|:---:|:---:|------|------|
| 유효증배계수 | `result_keff` | × | ○ | L_keff (Rayleigh 몫) | GAP→Dense 또는 Mamba hidden→Dense |
| 축방향 출력 편차 | `result_ao` | × | ○ | **L_AO**: AO(P̂_3d) vs AO_GT | AO = (P_top−P_bot)/(P_top+P_bot) |
| 목표 출력 비율 | `query_pload` | ○ (쿼리) | — | **L_pload**: Σ(P̂_3d)/P_design vs p_load_input | 추가 GT 불필요 — 입력 쿼리만으로 계산 |

**3D→스칼라 일관성 loss 계열**: 3D 출력에서 유도한 스칼라와 GT/입력 쿼리의 일치를 강제

| Loss | 공식 | 의미 |
|------|------|------|
| **L_pload** | (Σ(P̂_3d) / P_design − p_load_input)² | 총 출력 보존 (입력 쿼리 기반) |
| **L_AO** | (AO(P̂_3d) − AO_GT)² | 축방향 분포 보존 (GT 기반) |
| **L_keff** | (k_pred − k_GT)² | 임계도 보존 (Rayleigh 몫 또는 별도 헤드) |

### 채널 정리

**자기회귀 상태 (C_state = 10ch)**: Sm 제외, φ(2g)·σ_a^Xe(2g) 포함

| ch | 물리량 | 단위 | 정규화 |
|:---:|--------|------|:------:|
| 0 | power | MW/node | zscore 또는 minmax |
| 1 | T_cool | °C | zscore |
| 2 | T_fuel | °C | zscore |
| 3 | ρ_cool | g/cc | zscore |
| 4 | N_Xe | #/barn-cm | log_zscore |
| 5 | N_I | #/barn-cm | log_zscore |
| 6 | φ_g1 (fast) | n/cm²/s | log_zscore |
| 7 | φ_g2 (thermal) | n/cm²/s | log_zscore |
| 8 | σ_a^Xe_g1 | barn | log_zscore |
| 9 | σ_a^Xe_g2 | barn | log_zscore |

**별도 입력**:
- rod_map: 1ch, 매 스텝 쿼리 (인코더 채널 concat)
- p_load: scalar, **FiLM 확정** (상세 기법은 미정). 적용 위치는 아래 고찰 참조
- xs_fuel: 10ch, LP 고정 (인코더 채널 concat)

> **채널 순서**: Conv3D/Attention은 가중치가 채널별 독립 학습되므로 순서 자체는 성능에 영향 없음. 위 순서는 가독성 기준.

### p_load FiLM 적용 위치 고찰

시점 t의 예측 구조:

```
인코더: state(t) + rod_map(t+1) + xs_fuel → z(t)   "이 상태에서 이 제어 입력이면?"
Mamba:  z(1..t) → h(t)                               "지금까지의 이력 맥락"
디코더: h(t) → state(t+1)                            "다음 상태 예측" ← p_load(t+1)은 여기에 해당
```

**인코더의 역할 정의**: state(t)는 **현재 상태**, rod_map(t+1)은 **t+1을 유도하는 제어 입력(쿼리)**. 따라서 인코더는 "현재 상태 압축"이 아니라 **"현재 상태 + 제어 입력 → 전이에 필요한 공간 정보 추출"**이 역할.
이는 MASTER의 동작과 대응: MASTER도 현재 상태 + 다음 제어봉 위치를 받아 다음 상태를 계산함.

p_load(t+1)은 rod_map(t+1)과 동일한 성격 — **"다음 스텝의 목표 조건"**. 다만 p_load는 스칼라(전역 운전 체제)이므로, 3D 공간 처리보다 출력 분포를 직접 조건화하는 **디코더가 1차 적용 위치**로 더 자연스러움.

| 적용 위치 | 역할 | 구현 | 판단 |
|----------|------|------|:---:|
| **인코더** | 현재 상태 압축 | — | **미적용** (p_load 정보가 rod_map+power에 이미 존재) |
| **Mamba** | 시간 이력 맥락 | z(t)에 concat | △ 보조 (p_load 이력이 Xe 동역학에 영향) |
| **디코더** | 다음 상태 복원 | FiLM | **1차 적용** (출력 조건을 직접 제약) |

#### p_load FiLM은 전 채널에 적용 (채널 선택 적용 아님)

p_load가 power에 직접 관련된 것은 맞지만, 물리 결합이 **모든 채널을 연결**하고 있습니다:

```
p_load 낮아짐
    │
    ▼
power ↓ ──→ T_fuel ↓ ──→ σ_a^Xe 변화
            T_cool ↓
            ρ_cool ↑ (냉각재 밀도 상승)
    │
    ▼
핵분열율 ↓ ──→ N_I 생성율 ↓ ──→ N_Xe 동역학 변화
            φ ↓
```

즉 p_load 조건 하에서 일관된(physically consistent) state(t+1)을 복원하려면 **10개 채널 전체의 분포가 동시에 조정**되어야 합니다. power 채널의 FiLM만 건드리면 나머지 채널들이 물리적으로 불일치한 상태를 예측할 가능성이 높아집니다.

WaveNet에서도 같은 원리가 적용됩니다 — 화자 정체성(speaker identity)은 특정 주파수 채널에만 적용되지 않고 **전체 오디오 생성 분포를 전역 컨디셔닝**합니다.

### xs_fuel concat 근거

| 관점 | concat 적합 이유 |
|------|-----------------|
| 공간 구조 | xs_fuel은 (Z,qH,qW,10) — state와 동일 격자. 각 노드의 물성이 해당 노드 상태와 직접 대응 |

## 구현 전 고찰 사안

> 구현 착수 전 결정 필요: ★, 정리 완료(참고용): ☆

### ★ D_latent 크기 — Mamba 인터페이스

인코더 출력 (B,20,6,6,D_latent) → reshape → Mamba 입력 (B×720,T,D_latent). [개정 2026-04-09: halo expand 후 720 = 20·6·6]
D_latent = **각 셀이 Mamba에 전달하는 정보량**.

| D_latent | 장점 | 단점 |
|:---:|------|------|
| 작음 (~32) | Mamba 경량 | 21ch 입력 정보 손실. 10ch 상태 + 10ch 물성 + 1ch 제어의 상호작용 병목 |
| 기존 (64) | 기존 모델 검증됨 | 기존은 입력 10ch(xs_fuel만) → 출력 1ch(power만). 지금은 21ch → 10ch |
| 큰 값 (128) | 정보 보존 충분 | Mamba 연산량 증가 (720셀 × 576스텝 × 128) |

기존 64는 xs_fuel 10ch 전용이었으므로, 21ch 입력에는 **64~128 범위**로 확대 검토 필요.

### ★ Full Attention vs Block/Grid Attention

> **개정 2026-04-09**: halo expand로 N = 500 → 720. 본 절의 N=500 표기는 quarter (5,5) 가정 시점이며, 실제 인코더는 N=720 처리. (720/500)² = 2.07× FLOPs 증가. 절대치 trivial.

| 방식 | 수용 영역 | 연산량 (N=720, halo) | 장점 | 단점 |
|------|----------|:---:|------|------|
| Conv3D only | 커널 한정 (3³=27) | O(N) | 로컬 강력 | 원거리 불가 |
| Block Attn | 윈도우 내 (4×2×2=16) | O(N×16) | 로컬 세밀 | 윈도우 밖 미참조 |
| Grid Attn | stride (2×2×2=8) | O(N×8) | 전역 sparse | 인접 누락 가능 |
| Block+Grid | 교대 적용 | O(N×24) | 양쪽 커버 | 구현 복잡 |
| **Full Attn** | 720 전체 | O(N²)=518.4K | 모든 쌍 참조 | N=720이면 부담 미미 |

**물리적 고려**: 제어봉 삽입/인출은 Z방향 전체(Z=20)에 영향 → 먼 셀 간 상호작용 중요. N=720이면 Full Attention 연산도 현실적.

**기존 경험**: xs_fuel→power 예측에서 Voxel Attention(Block+Grid)이 CNN보다 우수 → 공간 위치 간 비균일 상호작용이 중요하다는 근거.

Block/Grid 분할은 (20,5,5) 크기에서 나눗셈 호환 문제(Prob-7)가 있었으나, Full Attention이면 이 문제도 해소. halo (6,6)에서는 (2,2) 또는 (3,3) 분할이 자연스러우나 Full Attention 채택으로 무관.

### ★ Conv3D 유지 여부 (Stem, MBConv3D)

| 구성 | 경계 처리 | physical loss 일관성 | 기존 경험 |
|------|----------|:---:|:---:|
| **Attention only** (Conv 제거) | 경계 문제 없음 (토큰 기반) | ○ | Voxel Attn > CNN |
| **Conv + Attention** (현재 구조) | Conv3D zero padding → 경계 artifact | BoundaryPad 필요 | 기존 코드 구조 |
| **Conv only** | zero padding 전 경계 | BoundaryPad 필수 | CNN < Voxel Attn |

Conv3D `padding='same'`은 모든 경계(y=0, y=4, x=0, x=4, z=0, z=19)에서 0을 이웃으로 주입 → 비물리적. Attention은 존재하는 토큰만 참조하므로 이 문제가 원천적으로 없음.

Conv를 유지하려면 BoundaryPad(reflect/zero 조합) 부활 필요. Attention only로 가면 BoundaryPad 불필요 + 기존 성능 우위 경험과 부합.

> Stem3D(입력 채널 → hidden 채널 변환)는 Conv가 아닌 Linear projection(Dense)으로 대체 가능. ViT의 patch embedding과 동일 패턴.

### ☆ Skip Connection — 인코더 → 디코더

인코더와 디코더 사이에 Mamba가 있어 skip 경로가 단순하지 않음:
```
인코더 feature ──skip──→ 디코더?
     ↓
   Mamba (시간 전이)
     ↓
   디코더
```

- **skip 없음**: D_latent가 모든 정보를 담아야 함. 단순하지만 병목 위험
- **시점 내 skip**: 같은 t의 인코더 feature를 디코더에 전달 (Mamba 우회). xs_fuel 등 시간 불변 공간 정보에 적합
- → **SD-Phase에서 결정**. 인코더는 skip 가능하도록 중간 feature를 보존하는 구조로 설계하면 충분

### ☆ 정규화 — Pre-LN 통일

기존 BN 주석 처리 → LN 통일 확정. Attention/FFN은 Pre-LN(LN → 연산 → residual add)이 표준.
디코더에서 AdaLN-Zero 도입 시 LN 자체가 조건부 변조되므로, 인코더는 일반 Pre-LN, 디코더는 AdaLN-Zero Pre-LN으로 분리.

### ☆ Mirror symmetry — halo expand로 인코더에 직접 전달 (개정 2026-04-09)

> **개정**: 본 절의 원래 결정 ("인코더는 quarter 내부 정보만 압축, mirror는 디코더 전용") 은 `05_symmetry_mode.md` (2026-04-08) 결정으로 폐기. 현재는 **halo expand로 quarter 너머 mirror/rotational 매핑 cell을 인코더 입력 단계에서 직접 부여**.

원래 (DEPRECATED): 인코더는 입력된 (20,5,5) 텐서 내부의 정보만 압축. Quarter-core 바깥의 mirror ghost 정보는 불필요. 대칭면 처리는 디코더의 L_diffusion 계산 전용.

개정 후: 입력 단계에서 `halo_expand(sym)` 호출 → (20, 5, 5) → (20, 6, 6) 변환. 인코더 본체는 (6,6) feature space에서 처리하며, halo cell이 boundary cell의 attention input으로 직접 들어감 → 대칭 inductive bias 자동 흡수. 자세한 근거: `05_symmetry_mode.md` §3

### ☆ 인코더 역할 범위

"state(t) + 제어입력 rod_map(t+1) + 물성 xs_fuel → 전이에 필요한 공간 정보 추출".
p_load는 인코더 미주입 (디코더 FiLM). 상세: §인코더 입력 경로 설계 참조.

## 도입 검토 항목

### 3D RoPE / STRING (위치 인코딩 대체)

| 특성 | relative_bias_table | RoPE / STRING |
|------|-------------------|---------------|
| gradient | `build()` 캐시 → 단절 (수정 가능) | `call()` 내 Q·K 직접 적용 → 단절 없음 |
| 파라미터 수 | num_heads × (2D-1)(2H-1)(2W-1) | 0 (결정론적 회전 행렬) |
| 해상도 일반화 | D,H,W 고정 | 임의 좌표 적용 가능 |
| 이동 불변성 | 테이블 범위 내 | 수학적으로 엄격 보장 |

**물리적 관점**: attention은 절대 위치가 아닌 상대 거리 Δz,Δy,Δx에 의존해야 함 — 동일 거리 어셈블리 쌍은 위치 무관하게 유사한 중성자 결합 특성.

**현재 판단**: 단기 gradient 수정으로 충분. RoPE 도입은 이점 명확 시 진행.

**조사 레퍼런스**:
- **STRING (2025)**: Separable Translationally InvaRiaNt Position EncodinGS. Lie 지수 맵 기반 분리 가능 3D PE
- **GeoPE (2025)**: 쿼터니언 log-exp 평균 등방성 3D 회전
- **LieRE (2024)**: Lie 군 회전 고차원 결합 인코딩
- **Swin3D**: cRSE — 위치·색상·법선 다채널 차이를 연속값 처리

### p_load 스칼라 처리 (FiLM / AdaLN-Zero)

**p_load의 성격**: MASTER `EXE_STD` 카드에 입력되는 **독립적 계산 조건(쿼리)**.

| 입력 | p_load 관계 | 설명 |
|------|:-----------:|------|
| xs_fuel | 무관 | 순수 핵연료 물성 (BOC 고정) |
| rod_map | 간접 | p_load에 의해 결정되지만 별개의 3D 입력 |
| p_load | — | 스칼라. XS에도 rod_map에도 포함되지 않은 독립 조건 |

**p_load가 물리 변수에 미치는 영향 경로**:

| 시간 스케일 | 물리 과정 | 영향 변수 | 비고 |
|:-----------:|----------|-----------|------|
| 즉각 (초~분) | 제어봉 이동 → 중성자속 → 온도 피드백 | phi, T_f, rho_m, rod_map | p_load 직접 구동 |
| 과도 (수 시간) | I-135 → Xe-135 비평형 | N_Xe, N_I, sigma_a_Xe | p_load 변화의 시간 지연 결과. Mamba 핵심 대상 |
| 출력 | 공간 분포 + 스칼라 | power, keff, AO | p_load + Xe 과도 중첩 |

**적용 위치 결론**:
- **인코더(SE-Phase)**: p_load와 무관한 3D 물리량 압축만 담당 → **주입하지 않음**
- **Mamba(T-Phase)** 또는 **디코더(SD-Phase)**: p_load 주입 방식(FiLM/AdaLN-Zero/concat) 및 위치를 T-Phase 설계 시 결정

**FiLM/AdaLN-Zero 변수별 적용**:
AdaLN-Zero의 gamma, beta는 채널별 독립 생성 → 변수별 다른 조건화 강도가 **자동 학습**됨 (예: power 채널은 강하게, Xe 채널은 약하게).

**스칼라 출력 (keff, AO)**:
디코더 3D 출력에서 global average pooling → Dense head, 또는 Mamba hidden state에서 직접 Dense head.

### Iterative Decoder — IIET 기반 Predictor-Corrector 루프

> 참고 논문: IIET (2509.22463v2, EMNLP 2025). 적용 Phase: **SD-Phase** (SE-Phase 범위 밖).

#### 배경: ~0.9% 바닥 오차와 MASTER Full PC

MASTER Full PC는 출력 물리량(φ, T_f, ρ_m, σ, N_Xe)을 **커플링 루프로 재계산**. 우리 모델은 1회 예측만 수행하여 이 커플링이 없음 → ~0.9% 바닥 오차. L_data가 이 간극을 담당하지만, **디코더 내부에서 self-consistent refinement**를 수행하면 모델 자체가 커플링을 학습할 수 있음.

#### IIET 방식

| | PCformer (Explicit PC) | IIET (Implicit Iterative Euler) |
|---|---|---|
| 반복 구조 | Predictor ≠ Corrector (비대칭) | **동일 블록 r회 반복** (대칭, 가중치 공유) |
| 오차 전파 | pred_1 오류 → corrector 오염 | iteration이 self-correct |
| 파라미터 | 증가 가능 | **증가 없음** |
| 필수 조건 | — | 각 iteration 전 **LayerNorm (RK-Norm)** 필수 |

#### 우리 모델에의 적용

```
[인코더] state(t) → latent           (1회)
[Mamba]  latent sequence → h(t)       (1회)
[디코더] h(t) → pred^0               (초기 예측)
         h(t) + LayerNorm(pred^0) → pred^1    (iteration 1)
         h(t) + LayerNorm(pred^1) → pred^2    (iteration 2)
         pred^r = 최종 출력
```

인코더/Mamba는 1회만 실행, **디코더 블록만 r+1회 반복** (가중치 공유).

#### Physical Loss와의 관계

모든 Loss는 **최종 pred^r에 적용** — 변경 없음. iteration이 물리량 간 self-consistency를 개선하므로 L_Bateman, L_sigma_a_Xe, L_diffusion 잔차가 감소할 것으로 기대.

## (보관) L_diffusion 관련 — 인코더 범위 밖

> 인코더에서 직접 다루지 않으나, 입력 채널 구성 및 모델 출력 설계에 참고.

### xs_fuel 10채널 → 확산방정식 항 유도

| xs_fuel 채널 | 물리량 | 유도 가능 항 | L_diffusion 용도 |
|:------------:|--------|------------|:----------------:|
| [3] Σ_tr,g1 | 수송 단면적 (fast) | D_g1 = 1/(3·Σ_tr,g1) | 확산계수 |
| [8] Σ_tr,g2 | 수송 단면적 (thermal) | D_g2 = 1/(3·Σ_tr,g2) | 확산계수 |
| [1]+[2] Σ_f,g1 + Σ_c,g1 | 핵분열 + 포획 | Σ_a,g1 = Σ_f + Σ_c | 흡수 단면적 |
| [6]+[7] Σ_f,g2 + Σ_c,g2 | 핵분열 + 포획 | Σ_a,g2 = Σ_f + Σ_c | 흡수 단면적 |
