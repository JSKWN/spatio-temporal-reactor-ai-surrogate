### 우리 격자에 가장 적합한 기법: STRING / LieRE

**적합 이유**:

1. **물리 좌표 직접 입력 + 추후 비균일 메시 확장**: 현재 연료 영역(K=2~21)은 전부 10cm 균일 메시이므로 정수 인덱스와 물리 좌표의 차이가 없음. 그러나 추후 반사체(K=1: 30cm, K=22: 30cm)를 포함하면 비균일 메시가 됨. STRING/LieRE는 **물리 좌표(cm)를 연속 실수값으로 직접 입력** 가능하므로, 코드 변경 없이 반사체 포함으로 확장 가능.

   ```
   현재 (연료만, 균일):     z = [5, 15, 25, ..., 195] cm  (10cm 간격)
   추후 (반사체 포함, 비균일): z = [15, 35, 45, ..., 205, 225] cm  (30+10+...+10+30)
   → STRING/LieRE는 동일 코드로 비균일 간격 자연 반영
   ```

2. **축 간 상호작용**: 원자로 노심에서 각 노드는 상하좌우앞뒤 6방향 인접 노드로부터 중성자 확산 누설의 영향을 받음. 이 영향은 Z 방향과 XY 방향이 독립이 아님 — 예를 들어 제어봉 삽입은 Z 방향 변화이지만 동일 Z 평면의 XY 출력 분포를 변화시키고, 반대로 XY 위치에 따라 축방향 flux 분포도 달라짐. 3D RoPE(축별 분리)는 이 **Z-XY 교차 결합**을 PE 수준에서 표현 불가하나, STRING/LieRE는 **생성자 {L_k}가 전체 d×d feature 차원을 공유하는 dense matrix**이므로, 하나의 생성자 학습 과정에서 축 간 결합이 암묵적으로 포함됨. (3D RoPE는 feature를 d/3씩 물리적으로 분리하여 축 간 정보 교환 불가.)

3. **이동 불변성**: 동일한 상대 거리에 있는 노드 쌍은 위치 무관하게 유사한 중성자 결합 특성을 가짐 — 이것이 확산방정식의 본질적 성질. STRING은 R(rᵢ)ᵀR(rⱼ) = R(rⱼ−rᵢ)를 수학적으로 보장하여 이 물리적 성질을 정확히 인코딩.

4. **gradient 안정성**: 기존 RPE의 Prob-6(gradient 단절) 문제가 구조적으로 발생하지 않음. 모든 연산이 forward pass 내에서 수행되므로 gradient가 자연스럽게 흐름.

**STRING vs LieRE 차이**: STRING은 LieRE에 **교환 조건(commutativity)**을 추가한 개선판. 교환 조건이 R(a)·R(b) = R(a+b) 성질을 보장하여 이동 불변성이 엄밀해짐. LieRE는 이 조건이 없어 더 자유로운 회전을 학습할 수 있으나, 이동 불변성은 근사적.

## 2. 배경: 위치 인코딩의 필요성

합성곱 신경망(Conv3D)은 커널의 공간 구조 자체가 "인접 노드가 가깝다"는 위치 정보를 내포. Attention은 토큰 간 순서나 거리 개념이 없어, 위치 인코딩 없이는 (z=0,y=0,x=0)과 (z=19,y=4,x=4)을 구별할 수 없음. 공간 인코더를 Attention-only로 구성하므로 위치 인코딩이 필수.

## 3. 기법별 상세

### 3.1 기존 RPE (Relative Position Bias Table)

> **출처**: Swin Transformer (Liu et al., 2021), 기존 CustomMaxViT3D 코드

**원리**: (num_heads, 2D-1, 2H-1, 2W-1) 크기의 학습 가능 테이블. 토큰 쌍 (i,j)의 상대 좌표 (Δz, Δy, Δx)로 인덱싱하여 attention logit에 bias를 더함.

```
attention_logits = Q·Kᵀ/√d + relative_bias[Δz, Δy, Δx]
```

- **장점**: 이미 구현 + 검증됨 (Global MAPE 0.694%). 모든 (Δz,Δy,Δx) 조합을 독립 학습하므로 축 간 결합 표현 가능.
- **단점**: 정수 인덱스 기반 → 추후 반사체 포함(비균일 메시) 시 대응 불가. Prob-6 gradient 단절 수정 필요 (`tf.gather_nd`를 `call()`로 이동). 해상도 변경 시 재학습.

### 3.2 3D RoPE (축별 분리)

> **출처**: Su et al. (2021) RoFormer의 3D 확장

**원리**: feature 차원을 3등분하여 Z/Y/X 좌표별로 독립적인 sin/cos 회전을 적용. Q, K 모두에 적용하면 Q·Kᵀ에서 절대 위치가 상쇄되어 상대 거리만 남음.

```
features[0:d/3]     → RoPE(z_coord)
features[d/3:2d/3]  → RoPE(y_coord)
features[2d/3:d]    → RoPE(x_coord)
```

- **장점**: 구현 가장 간단. 파라미터 0개. gradient 안전.
- **단점**: **축 간 결합 완전 부재** — Z 정보를 인코딩하는 feature가 Y/X 정보를 전혀 모름. 물리 좌표 입력은 가능하나 교차 영향을 표현 못 함.

### 3.3 STRING (Separable Translationally Invariant Position Encodings)

> **출처**: arXiv 2502.02562, ICML 2025 Spotlight
> "STRING is the most general of all translationally invariant position encoding mechanisms using matrix multiplication."

#### 수학적 구조

**핵심 공식**:

```
R(r) = exp(L₁·r₁ + L₂·r₂ + L₃·r₃)
```

각 변수의 의미:

| 기호 | 크기 | 개수 | 의미 | 우리 모델에서 |
|------|:---:|:---:|------|-------------|
| **r = (r₁, r₂, r₃)** | 스칼라 3개 | 토큰마다 1조 | 토큰의 공간 좌표 (연속 실수값) | (z_cm, y_cm, x_cm) — 노드의 물리 위치 |
| **L_k** | d×d | **3개** (축당 1개) | k번째 **공간 축**의 회전 생성자 (학습 가능) | L_z, L_y, L_x — "이 축 방향 거리가 attention에 미치는 영향 패턴" |
| **exp(·)** | — | — | 행렬 지수 함수 | 좌표를 feature 공간의 회전으로 변환 |
| **R(r)** | d×d | 토큰마다 1개 | 위치 r에서의 회전 행렬 (직교) | Q, K 벡터에 곱하여 위치 정보 주입 |

여기서 d = head_size (예: 32). k는 embed_dim이 아니라 **공간 축 번호** (k=1,2,3 = Z,Y,X). 즉 학습해야 할 생성자 행렬은 **3개**이고, 각각 d×d 크기.

**반대칭 행렬 (skew-symmetric)이란**: `Lᵀ = -L`. 대각선이 0이고 상삼각과 하삼각이 부호만 반대인 행렬. d×d에서 자유 파라미터는 d(d-1)/2개.

```
예시 (d=4):     L = [  0   a   b   c ]
                    [ -a   0   d   e ]
                    [ -b  -d   0   f ]
                    [ -c  -e  -f   0 ]   ← 학습해야 할 값: a,b,c,d,e,f (6개)
```

d=32이면 축당 32×31/2 = 496개 파라미터. 3축이면 총 1,488개 — 매우 소량.

#### 회전 생성자란 무엇인가

2D에서 벡터를 θ만큼 회전시키는 행렬을 떠올리면:

```
R(θ) = [cos θ  -sin θ]
       [sin θ   cos θ]
```

이걸 θ가 아주 작을 때(무한소 회전) 테일러 전개하면 `R(θ) ≈ I + θ·L`:

```
L = [ 0  -1]
    [ 1   0]    ← 이것이 "생성자"
```

**생성자 L은 "이만큼의 좌표값을 넣으면, 그에 비례하여 벡터를 회전시켜주는 레시피"**. 좌표값이 "각도" 역할을 함. 정확한 회전은 `R(θ) = exp(θ·L)` — 행렬 지수를 취하면 테일러 근사가 아닌 정확한 회전 행렬이 나옴.

STRING에서는:
- 이 L이 2×2가 아닌 **d×d** (고차원 feature 공간에서의 회전)
- 값이 `[0,-1; 1,0]` 고정이 아닌 **학습** (데이터에서 최적 회전 패턴을 찾음)
- 좌표값 r_k(cm)가 "각도" 역할: 좌표가 크면 더 많이 회전

#### Attention에 적용하는 방법

두 노드 i(위치 rᵢ)와 j(위치 rⱼ)가 있을 때, 각각의 Q, K 벡터에 해당 위치의 회전을 적용:

```
위치 rᵢ의 토큰 i:  Q'ᵢ = R(rᵢ) · Qᵢ     ← Q 벡터(d차원)를 위치에 따라 회전
위치 rⱼ의 토큰 j:  K'ⱼ = R(rⱼ) · Kⱼ     ← K 벡터(d차원)를 위치에 따라 회전
```

각 노드의 좌표가 다르므로 R(rᵢ) ≠ R(rⱼ) — 서로 다른 양만큼 회전. Attention score를 구하면:

```
Attention score (스칼라 1개):
  Q'ᵢ · K'ⱼᵀ = Qᵢᵀ · R(rᵢ)ᵀ · R(rⱼ) · Kⱼ
              = Qᵢᵀ · R(rⱼ - rᵢ) · Kⱼ        ← ★ 상대 위치만 남음
```

**"상대 위치만 남는다"의 의미**: 최종 score는 **스칼라 1개**(i→j의 attention 가중치). 이 값을 결정하는 것은 Q, K의 내용물 + **R(rⱼ - rᵢ)**. 여기서 rⱼ - rᵢ = (Δz, Δy, Δx)는 두 노드의 좌표 **차이 벡터**.

즉, 절대 위치 rᵢ, rⱼ 각각은 사라지고 **"두 노드가 얼마나 떨어져 있는가"**만 score에 영향.

**예시**: 다음 두 노드 쌍은 동일한 Δ = (10cm, 0, 0)이므로 동일한 score를 받음:
- 노드(z=5, y=0, x=0) ↔ 노드(z=15, y=0, x=0)
- 노드(z=105, y=2, x=2) ↔ 노드(z=115, y=2, x=2)

이것이 "같은 거리의 어셈블리 쌍은 위치 무관하게 유사한 중성자 결합 특성"이라는 물리와 정확히 대응.

#### 교환 조건과 이동 불변성

위 등식에서 핵심 단계: `R(rᵢ)ᵀ · R(rⱼ) = R(rⱼ - rᵢ)`.
이것이 성립하려면 회전 행렬이 `R(a) · R(b) = R(a+b)` 성질을 가져야 함.

**문제**: 행렬 지수의 곱셈은 일반적으로 `exp(A) · exp(B) ≠ exp(A+B)`.

이유를 설명하는 것이 **Baker-Campbell-Hausdorff (BCH) 정리**:
```
exp(A) · exp(B) = exp(A + B + ½[A,B] + ...)
```
여기서 `[A,B] = A·B - B·A` (교환자, commutator). A와 B를 곱하는 순서에 따라 결과가 달라지면 [A,B] ≠ 0 → exp(A)·exp(B) ≠ exp(A+B).

**해결**: 만약 **A·B = B·A** (교환, commute)하면 `[A,B] = 0`이므로:
```
exp(A) · exp(B) = exp(A + B)     ← 교환 시에만 성립!
```

STRING은 생성자에 **교환 조건 L_i · L_j = L_j · L_i**를 부과. 이러면:
```
R(a) · R(b) = exp(Σ L_k·a_k) · exp(Σ L_k·b_k)
            = exp(Σ L_k·(a_k + b_k))
            = R(a + b)                              ← 보장됨
```

따라서 `R(rᵢ)ᵀ · R(rⱼ) = R(-rᵢ) · R(rⱼ) = R(rⱼ - rᵢ)` → 이동 불변성 엄밀 성립.

이것이 **"Separable"**의 의미 — 교환 조건 덕분에 각 축의 기여가 독립적으로 곱해짐(분리 가능).

#### 축 간 결합이란, 그리고 왜 가능한가

**축 간 결합**: "Z 방향의 위치 차이가 Y/X 방향 정보에도 영향을 준다"는 것. 원자로에서 제어봉 삽입(Z 방향 변화)이 동일 Z 평면의 XY 출력 분포를 바꾸는 것처럼, 하나의 축 좌표가 다른 축의 feature 표현에도 영향을 미쳐야 함.

**교환 조건은 이동 불변성을 위한 것이지, 축 간 결합과는 무관.**

축 간 결합이 가능한 실제 이유: **생성자 {L_k}가 전체 d×d feature 차원을 공유하는 dense matrix**이기 때문.

```
3D RoPE:   features를 d/3씩 물리적으로 분리
           [z 전용 feature | y 전용 feature | x 전용 feature]
           → z 회전이 y/x 전용 feature에 아무 영향 없음 → 축 간 정보 교환 불가

STRING:    L_z, L_y, L_x 모두 동일한 d×d 전체 feature 공간에서 작용
           → L_z의 (i,j) 성분이 feature i와 j를 결합하는데,
             L_y도 같은 feature i와 j에 영향
           → z 좌표 변화가 y 관련 feature에도 영향을 줌 = 축 간 결합
```

직관적으로: 3D RoPE는 "Z 담당 직원, Y 담당 직원, X 담당 직원"이 서로 대화 안 하는 것이고, STRING은 "한 팀이 Z/Y/X 전부 같이 처리"하는 것.

#### RoPE와의 포함 관계

> "RoPE is a simple special case of STRING, taking a particular choice of generators." (STRING 논문)

```
1D RoPE:  L = [  0  θ₁  0   0  ]     ← 2×2 블록 대각 행렬
              [ -θ₁  0  0   0  ]       (각 블록이 독립 2D 회전)
              [  0   0  0  θ₂  ]
              [  0   0 -θ₂  0  ]

STRING:   L = [  0   a   b   c ]     ← dense 반대칭 행렬
              [ -a   0   d   e ]       (모든 feature 쌍이 결합)
              [ -b  -d   0   f ]
              [ -c  -e  -f   0 ]
```

RoPE는 L을 2×2 블록 대각으로 제한한 것. STRING은 이 제한을 풀어 dense로 일반화.

#### 물리 좌표 입력

좌표 r_k가 연속 실수값이므로 정수 인덱스가 아닌 **물리적 좌표(cm)**를 직접 입력 가능:

```
현재 (연료만, 균일): r = (z_cm, y_cm, x_cm) = (5, 15, 25, ..., 195, ...) cm
추후 (반사체 포함):  반사체 30cm + 연료 10cm 혼합 → 비균일 간격도 자연 반영
```

#### 변형: Circulant-STRING

순환 행렬(circulant matrix)로 생성자를 특수화하면 FFT로 O(d log d) 계산 가능. 메모리/연산 효율적.

#### 정리

- **장점**: 물리 좌표 직접 입력(추후 비균일 메시 확장 대비) + 축 간 결합 (dense 생성자) + 이동 불변성 (교환 조건) 모두 충족. RoPE의 수학적 일반화. ICML 2025 Spotlight.
- **단점**: 행렬 지수(`expm`) 또는 Cayley 변환 구현 필요. 중간 난이도. **원자로 물리 도메인 검증 사례 없음** — RPE 대비 ablation 필수.

#### 수치 안정화 기법 — expm의 gradient 폭발 방지

행렬 지수 `exp(Σ L_k·r_k)`에서 L_k·r_k의 크기가 크면 exp가 발산할 수 있음. 이를 방지하는 기법들:

| 기법 | 원리 | 장단점 |
|------|------|--------|
| **Cayley 변환** | `R = (I+A)(I-A)⁻¹` — expm 대신 역행렬 1회로 직교 행렬 생성 | expm 없이 직교 보장. 단, 고유값 -1 근처에서 불안정 |
| **생성자 노름 제약** | `L_k ← L_k · min(1, c/‖L_k‖_F)` — Frobenius 노름 clamp | exp 입력 크기 직접 제한. 단순 |
| **좌표 스케일링** | r_k를 정규화 (예: cm → 0~1) → L_k·r_k 크기 제한 | 물리 좌표의 스케일 차이 흡수 |
| **QK-Norm** | Q, K를 dot product 전에 L2 정규화: `Q ← Q/‖Q‖, K ← K/‖K‖` | R(r)·Q 변환 후 벡터 크기 변동 방지. ViT-22B, SD3 등에서 채택 |
| **α 게이팅 (zero-init)** | AdaLN-Zero의 α=0 초기화 — 블록 출력이 identity에서 시작 | 학습 초기 불안정 방지. STRING과 직접 관련은 아니나 전체 블록 안정성 기여 |

우리 모델에서의 적용:
- **좌표 스케일링**: 현재 연료 Z축 5~195cm, XY 0~4피치 → 정규화하여 L_k·r_k 크기 제어
- **QK-Norm**: STRING PE 도입 시 R(r)·Q 변환으로 벡터 크기 변동 가능 → 병행 권장
- **생성자 노름 제약**: 학습 중 L_k가 과대해지는 것을 방지하는 안전장치

### 3.4 LieRE (Lie group Rotary Encoding)

> **출처**: arXiv 2406.10322, Stanford AIMI, 2024

**원리**: STRING과 동일한 수식 `R(p) = exp(Σ p_k·A_k)`이나, 생성자 A_k에 **교환 조건 미부과**. 더 자유로운 회전을 학습 가능하나 R(a)·R(b) = R(a+b) 보장 안 됨.

- 추가 파라미터 0.68%, 추가 FLOPs <1.4% (경량)
- STRING과 LieRE는 **독립 연구 그룹의 병렬 연구**. LieRE(Stanford AIMI, 2024.06) → STRING(독립 저자, 2025.02). STRING은 LieRE를 직접 인용하지 않음.

| | LieRE | STRING |
|---|---|---|
| 설계 목표 | 표현력 극대화 (비가환) | 이동 불변성 보장 (가환) |
| 검증 도메인 | 의료 영상 (CT, MRI), 자연 이미지 | 로보틱스 정책 학습 |
| 비균일 메시 | 수식상 가능하나 **실검증은 균일 격자** | 수식상 가능, 로보틱스에서 연속 좌표 사용 |

**판단**: 이동 불변성이 물리적으로 중요한 우리 격자에서는 STRING이 우선이나, 비가환 생성자가 더 자유로운 축 간 교차 항을 학습할 수 있으므로 비교 실험 가치 있음. **두 기법 모두 원자로 물리 도메인 검증 사례 없음** — RPE 대비 ablation 필수.

### 3.5 GeoPE (Quaternion-based 3D PE)

> **출처**: arXiv 2512.04963, 2025

**원리**: 쿼터니언 sandwich product로 feature triplet을 3D 회전. 축별 회전을 Lie 대수에서 평균하여 결합.

- **단점**: 물리 좌표 입력 미검증. feature가 3의 배수여야 함. 구현 복잡. STRING 대비 이점 없음.

### 3.6 LAPE / STRING 적용 위치 — 학계 표준 패턴 (2026-04-09 추가)

> **목적**: LAPE (절대 PE) 와 STRING (상대 PE, RoPE 계열) 이 트랜스포머 어디에 어떻게 들어가는지를 학계 원문 근거와 함께 명시. 본 프로젝트 코드 (`src/models/spatial/layers3d.py`, `attention3d.py`) 와 문서 (`00_README.md` 아키텍처 다이어그램) 의 정합성 근거.

#### 3.6.1 LAPE — Stem 출력 직후, 어텐션 블록 입력단, 1회만 add

**적용 위치**:
```
[CellEmbedder]  Conv3D(1,1,1), 21 → 128         ← Stem (patch embedding 등가)
       │
       ▼ (B, 20, 6, 6, 128)
[LearnedAbsolutePE3D]  +(20, 6, 6, 128) embedding   ← LAPE 1회 add (여기!)
       │
       ▼ (B, 20, 6, 6, 128)
[reshape] flatten → (B, 720, 128)
       │
       ▼
[Attention Stage 1, 2, 3 ...]                     ← LAPE 더 이상 안 더함
```

**적용 방식**:
```python
# src/models/spatial/layers3d.py:215-218 (LearnedAbsolutePE3D.call)
return x + self.embedding[tf.newaxis, ...]   # broadcast add
```

**원문 근거**:

| 모델 | 절대 PE 종류 | 적용 위치 | 적용 빈도 |
|---|---|---|:---:|
| **Vaswani et al. 2017** ("Attention Is All You Need") | sinusoidal | input embedding 직후 | **1회** |
| **Devlin et al. 2018** (BERT) | learnable absolute | token embedding 직후 | **1회** |
| **Dosovitskiy et al. 2020** (ViT) | learnable absolute | patch embedding 직후 | **1회** |
| **본 프로젝트 (LAPE)** | learnable absolute | CellEmbedder 직후 | **1회** |

ViT 원문 인용 (Dosovitskiy et al. 2020, §3.1):
> "Standard learnable 1D position embeddings ... are added to the patch embeddings to retain positional information."

**왜 1회만 add 해도 충분한가 — Residual Stream 관점**:

Pre-LN 트랜스포머의 각 블록은 residual connection 구조:
```
x_{l+1} = x_l + Attention(LN(x_l))    ← residual stream에 sublayer 결과 누적
x_{l+1} = x_l + FFN(LN(x_l))
```

LAPE가 입력 단계에 1회 add되면, residual stream `x_0 = embedding + LAPE` 에서 시작. 이후 모든 layer에서:
```
x_l = x_0 + Σ (Attention/FFN sublayers)
    = embedding + LAPE + Σ (sublayers)
                ↑
        영원히 살아남아 마지막 layer까지 전달
```
- LAPE가 가산 형태로 residual stream에 새겨지므로, 매 layer마다 다시 더할 필요 없음
- 매 layer add는 BERT/ViT 어디에서도 채택하지 않는 패턴 (불필요한 중복)

**LAPE를 attention 안에 넣지 않는 이유**:
- LAPE는 token feature에 위치별 정체성을 새기는 것이 본질
- attention QKV projection 후에 더하면 회전 변환과 충돌할 수 있고 (STRING과 간섭), 무엇보다 학계 어디에서도 그렇게 하지 않음
- LAPE를 flatten 전 (5D 텐서) vs flatten 후 (3D 텐서) 어디에 더할지는 수학적으로 동일 (broadcast). 본 프로젝트는 5D 단계에서 add (좌표 인덱싱 자연스러움)

#### 3.6.2 STRING — 매 어텐션 블록 내부, W_Q/W_K projection 직후, dot product 직전

**적용 위치**:
```
[FullAttention3D] (Stage k, k = 1, 2, 3)
       │
       ▼ x (B, 720, 128)
   ┌── Q = W_Q · x          (linear projection)
   │   K = W_K · x
   │   V = W_V · x
   │   │
   │   ▼ split heads → Q, K, V (B, H, 720, head_dim)
   │   │
   │   ▼
   │   Q, K = STRING(Q, K, coords)        ← 여기! Q/K 회전 적용
   │   │   (V는 적용 안 함)
   │   │
   │   ▼
   │   attention_score = Q · K^T / √d_k
   │   attention_weight = softmax(attention_score)
   │   output = attention_weight · V
   └── ▼ (B, 720, 128)
```

**적용 방식**:
```python
# src/models/spatial/attention3d.py:285-294 (FullAttention3D.call)
Q = self.q_proj(x); K = self.k_proj(x); V = self.v_proj(x)
Q = split_heads(Q); K = split_heads(K); V = split_heads(V)
Q, K = self.string_pe(Q, K, coords)         # ← STRING (Q/K만)
attention_output = scaled_dot_product(Q, K, V)
```

**원문 근거**:

| 모델 | 상대 PE 종류 | 적용 위치 | 적용 빈도 | V에 적용? |
|---|---|---|:---:|:---:|
| **Su et al. 2021** (RoFormer/RoPE) | rotation matrix | Q/K projection 직후 | **매 attention layer** | ✗ |
| **STRING (arXiv 2502.02562, ICML 2025)** | dense rotation (Lie algebra) | Q/K projection 직후 | **매 attention layer** | ✗ |
| **본 프로젝트 (STRINGRelativePE3D)** | block-diag 2×2 + 축별 freq | Q/K projection 직후 | 3 stages × 1 attention | ✗ |

RoPE 원문 인용 (Su et al. 2021, §3.4):
> "We multiply the keys and queries of every attention layer by sinusoidal embeddings via a rotation matrix."

STRING 원문 인용 (arXiv 2502.02562, §2):
> "RoPE is a simple special case of STRING, taking a particular choice of generators."

**왜 W_Q/W_K projection *직후* 여야 하는가 — 수학적 필연성**:

STRING (RoPE 포함) 의 핵심은 다음 trick:
```
Q'_i = R(r_i) · Q_i              ← Q를 좌표에 따라 회전
K'_j = R(r_j) · K_j              ← K를 좌표에 따라 회전

Attention score = Q'_i^T · K'_j
                = (R(r_i) Q_i)^T · (R(r_j) K_j)
                = Q_i^T · R(r_i)^T · R(r_j) · K_j
                = Q_i^T · R(r_i)^{-1} · R(r_j) · K_j     ← R 직교행렬, R^T = R^{-1}
                = Q_i^T · R(-r_i + r_j) · K_j           ← 교환 조건 R(a)·R(b) = R(a+b)
                = Q_i^T · R(r_j - r_i) · K_j
```

**핵심 등식**: `R(r_i)^T · R(r_j) = R(r_j - r_i)` — 절대 위치 r_i, r_j는 사라지고 **상대 거리 (r_j - r_i)** 만 남음.

이 trick이 작동하려면:
1. **회전 R 적용 시점이 W_Q/W_K projection *후* 여야 함**:
   - projection 전 (token feature x 자체) 에 R을 곱하면 → W_Q · (R · x) 에서 R과 W_Q가 섞여서 직교성 깨짐
   - projection 후 (Q, K 벡터) 에 R을 곱해야 → R^T · R = I 구조가 깔끔히 보존
2. **dot product 직전 이어야 함**:
   - Q, K 회전 후 즉시 dot product를 해야 위 식 전개가 valid
   - 회전 후 추가 변환 (예: 또 다른 linear layer) 이 있으면 등식 깨짐
3. **매 attention layer 마다 새로 적용**:
   - 매 layer마다 새로운 W_Q, W_K 가 있고 새로운 Q, K가 생성
   - 각 layer의 새 Q, K에 대해 회전을 다시 적용해야 함
   - LAPE처럼 1회 add로는 불가능 (가산이 아닌 곱셈/회전이므로 residual stream으로 전달 안 됨)

**왜 V에는 적용하지 않는가**:
- attention 출력: `output = softmax(QK^T) · V`
- softmax(QK^T) 가 이미 위치 정보를 완전히 인코딩 (Q/K 회전 trick 으로)
- V는 **content (값 내용)** 을 집계하는 역할이지 위치 정보의 운반체가 아님
- V를 회전시키면 출력이 좌표계에 종속되어 translation invariance 깨짐
- RoPE 원문, STRING 원문, 모든 RoPE 계열 구현체에서 V는 회전하지 않음 — **만장일치 표준**

#### 3.6.3 두 PE의 동시 사용 정당화 — Dufter et al. 2022

**원문**: Dufter, Schmitt & Schütze (2022) "Position Information in Transformers: An Overview", Computational Linguistics 48(3):733

> "They provide theoretical arguments that their absolute and relative position embeddings are complementary. Indeed, in their experiments the combination outperforms either alone."

**상보성의 이유**:

| 항목 | LAPE (절대) | STRING (상대) |
|---|---|---|
| 인코딩 대상 | 각 위치의 **고유 정체성** | 두 위치 사이의 **상대 거리** |
| 적용 방식 | feature add | Q/K rotation |
| 적용 위치 | 입력 단계 1회 | 매 attention layer 내부 |
| 번역 불변성 | ✗ 깨뜨림 | ✓ 유지 |
| 학습 파라미터 | (Z·H·W·D) 개 위치별 변수 | 회전 주파수만 (위치별 변수 없음) |
| 본 프로젝트 역할 | **boundary cell 식별** (반사체 인접 셀의 고유 정체성 학습) | **상대 거리 기반 attention** (등거리 셀 쌍은 동일 score) |

두 PE는 **서로 다른 자리 + 서로 다른 메커니즘**으로 들어가므로 간섭 없음. ViT (LAPE) 와 RoFormer (RoPE) 의 결합이 본 프로젝트의 직접적 선례.

#### 3.6.4 본 프로젝트 설계 정합성 검증

| 항목 | 학계 표준 | 본 프로젝트 | 정합 |

## 5. 정규화 기법 해설

### LayerNorm (Layer Normalization)

**하는 일**: 하나의 샘플, 하나의 토큰 내에서 feature 차원 전체의 평균을 0, 분산을 1로 정규화한 뒤, 학습 가능한 γ(스케일)와 β(시프트)를 적용.

```
입력: x = [x₁, x₂, ..., x_d]   (하나의 토큰의 d개 feature)

1. 평균 계산:  μ = (1/d) Σᵢ xᵢ
2. 분산 계산:  σ² = (1/d) Σᵢ (xᵢ - μ)²
3. 정규화:     x̂ᵢ = (xᵢ - μ) / √(σ² + ε)
4. 스케일+시프트: yᵢ = γᵢ · x̂ᵢ + βᵢ   (γ, β는 학습 파라미터, 채널별)
```

**핵심**: 정규화 범위가 **하나의 토큰 내부** (feature 차원). 다른 토큰, 다른 배치 샘플과 무관하게 독립적으로 정규화.

### BatchNorm이 퇴출된 이유

BatchNorm은 **배치 내 같은 채널의 통계**(평균, 분산)로 정규화:

```
BatchNorm: 배치 내 N개 샘플의 동일 채널 → 평균/분산 계산
LayerNorm: 1개 샘플의 d개 채널 → 평균/분산 계산
```

| 문제 | 설명 | 우리 모델에서 |
|------|------|-------------|
| **소배치 불안정** | B < 16이면 배치 통계가 신뢰 불가 → 정규화 노이즈 | 3D 텐서 (B, 20, 5, 5, 21) [데이터 입력] 또는 (B, 20, 6, 6, 21) [인코더 처리] 모두 B가 크기 어려움 |
| **train/infer 불일치** | 학습: 배치 통계, 추론: 이동 평균 → 분포 차이 | 자기회귀 추론 시 배치=1 → moving average 의존 |
| **시퀀스 비균질** | 위치별 통계가 다른 시퀀스 데이터에서 같은 위치끼리 묶어 평균하는 것이 부자연스러움 | 노심 z=0과 z=19의 분포가 근본적으로 다름 |
| **분산 학습** | GPU 간 배치 통계 동기화 오버헤드 | — |

**결론**: Transformer/Attention 모델에서는 LayerNorm이 표준. 기존 코드에서 BN 주석 처리한 것도 이 문제 때문.

### Pre-Norm vs Post-Norm

Attention/FFN 블록에서 LayerNorm의 위치:

```
Post-Norm: x = Norm(x + Attention(x))     ← 원래 Transformer (2017)
Pre-Norm:  x = x + Attention(Norm(x))     ← 현대 표준 (GPT, ViT, LLaMA)
```

| | Post-Norm | Pre-Norm |
|---|---|---|
| 학습 안정성 | ❌ 깊은 네트워크에서 불안정 | ✅ 안정 (잔차가 정규화 전에 합산) |
| 최종 성능 | △ 충분히 학습하면 약간 우수 | ○ 안정적으로 수렴 |
| 채택 현황 | 초기 Transformer | **현대 모델 대부분** |

우리 인코더는 2~4블록으로 얕으므로 Post-Norm의 불안정은 발생하지 않으나, 표준을 따라 **Pre-Norm 채택**.

### RMSNorm (Root Mean Square Normalization)

LayerNorm에서 **평균 중심화(mean subtraction)를 생략**한 간소화 버전:

```
LayerNorm: x̂ = (x - μ) / √(σ² + ε)     ← 평균 빼기 + 분산 나누기
RMSNorm:   x̂ = x / √((1/d)Σxᵢ² + ε)    ← RMS로 나누기만
```

- ~15% 연산 절약, 성능은 LayerNorm과 동등
- LLaMA, Gemma 등 최신 LLM에서 채택
- 단, TF 2.14에 내장되어 있지 않아 직접 구현 필요 (5줄 정도)

### Gradient Clipping

다중 loss 합산 (L_data + L_Bateman + L_σXe + L_diffusion) 시 각 loss의 gradient 스케일이 크게 다를 수 있음:
- N_Xe ~ 10⁻⁹, φ ~ 10¹³ → 정규화 후에도 loss 간 gradient 비율 불균형 가능

```
Norm clipping: g ← g · min(1, c / ‖g‖)    (방향 보존, 크기만 제한)
```

- c (max norm) = 1.0이 보편적 시작점
- 다중 loss 합산 시 필수에 가까움
